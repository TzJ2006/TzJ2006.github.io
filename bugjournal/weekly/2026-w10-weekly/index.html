Issues and Solutions#

1. STAIG’s two-stage approach actually completely ignores gene encoder output, using the original HVG as GCN input, resulting in scGPT+UNI2+STAIG≡PCA+UNI2+STAIG [MIHD] (2026-03-02)#

Solution: Three-layer code tracing (Fusion.py→runner.py→STAIGTrainer.py) confirmed the architectural fact: gene encoder was decoupled from GNN training. Improvements should start with the GNN structure rather than replacing the gene encoder.

2. Pi0.5 evaluation client reported keepalive ping timeouts, with all tasks having SR=0%; multi-client concurrency was 24% slower than serial processing [Pi0.5/VLA Evaluation] (2026-03-02)#

Solution: Add ping_timeout=None to connect() to disable the 20-second timeout (JAX JIT compilation took 30-60 seconds, exceeding the default value). Confirmed that VLA inference was strictly serial, and multi-client concurrency was ineffective. The correct solution is BatchedVLAServer batch inference.

3. STAIG’s fusion embeddings failed in some sections during training, resulting in a NaN Spearman r in cross-sample evaluation [MIHD] (2026-03-03)#

Solution: Add zero embedding detection in load_fused_embeddings() (norm<1e-6), automatically triggering –recompute_fusion to retrain STAIG for about 50 seconds.

4. Pi0.5 LoRA training configuration names contain ‘pi05_libero’, and users suspect that the initialization weights come from libero rather than pi05_base [Pi0.5/VLA Training] (2026-03-03)#

Solution: Three-layer evidence (config.py weight_loader field + training log Restoring checkpoint path + comparison group configuration) confirmed that both sides used pi05_base correctly. The configuration name describes the data loading configuration; weight_loader is the only basis.

5. BC-RNN had SR=0% on all normal tasks, which did not match the training rollout [error_recovery_benchmark] (2026-03-04)Solution: The code is exploring ways to fix the bug in the baseline_accuracy evaluation for object observation keys. Strategies that accept empty observations are being implemented. In M14, having an Error SR close to 0% is a core argument of the benchmark (the current strategy lacks error recovery capabilities). Comparing it with Normal SR can help clarify the issue.#

6. In the training feature norm_stats.json for Openpi-moe, there is a missing prev_actions key without any errors. In VAE, directly concatenating hist_actions (unnormalized) and actions (normalized) causes scale mismatch [zhaoganlong/openpi] (2026-03-05)#

Solution: Modify compute_norm_stats.py to dynamically detect prevActions and write them into the statistics. The fundamental mechanism is to silently skip missing keys when apply_tree(strict=False). New training features must force a synchronized normalization script.

7. All background services of CalendarPro have never been run in production environments. Despite passing all 321 tests, this integration flaw has not been discovered [CalendarPro] (2026-03-05)#

Solution: Add await coordinator.start_all() in the on_ready() method of discord_bot.py. The root cause is that the registration mode and lifecycle management are decoupled. Unit tests cannot detect silent failures at the integration level.

8. The headless Docker container SAPIEN reports a Render Error. The bare except hides the real error. The apt driver version does not match the kernel module [VLA-RoboTwin] (2026-03-06)#

Solution: Expose the real error. Confirm that there is a missing NVIDIA Vulkan ICD. Extract the user-mode GL library that matches the kernel version from the official 535.104.12 .run package. Inject it through the VK_ICD_FILENAMES/LD_LIBRARY_PATH environment variables (no root required).

9. The initial version of Research Scout performs a single-stage deep evaluation on all 50 papers. There is a significant waste of tokens, and low-relevance papers consume a lot of analysis resources [gadget/Research Scout] (2026-03-07)#

Solution: Reconstruct it into a two-stage pipeline: Stage 1 for lightweight filtering; Stage 2 for in-depth analysis (up to 20 high-relevance papers). This is based on the reading habits of real researchers.

10. In error_recovery_benchmark, 65 symlinks cause an indirect layer during code migration. After removing the symlinks, the depth of the sys.path parent chain needs to be recalculated [error_recovery_benchmark] (2026-03-08)#

Solution: The four-way parallel Agents clean up all symlinks according to a nine-stage plan without overlap. The main thread handles Makefile/documents. Fix two unexpected sys.path depth errors (Path.resolve() no longer folds symlinks).

11. In VLA-RoboTwin, the data collected for critical_region is always zero in key areas. The τ=0.10m threshold is purely based on estimates, and it does not match the actual working space scale [VLA-RoboTwin] (2026-03-08)#

Solution: Do not adjust parameters directly. Instead, record the original ee_target_distances (4-8 distance variables per task). Analyze the real data distribution before calibrating the threshold based on the data.

Learning Outcomes#

Architecture#

  • Three criteria for large-scale implementation: 1) Whether new services are connected to the startup link; 2) Whether cross-layers (IntentType→routing→Handler→service) are updated consistently; 3) Whether silent exceptions (except: pass) mask runtime errors. Passing the test is a necessary condition, not a sufficient condition. Integration layer defects can only be discovered through end-to-end verification.
  • Before designing an Agent-based system, actively study mature projects in the same field. The EventBus/CronScheduler/Plugin registration patterns of OpenClaw and GSD’s STATE.md persistent memory/multi-Agent context isolation are highly reusable architecture patterns. Related fixes should be merged into a single change rather than implemented stage by stage to avoid code rework.
  • apply_tree(strict=False) is an implicit risk in ML training pipelines. If the norm_stats calculation script that needs normalization is forgotten, it will cause a silent scale mismatch that affects training quality. The mismatch records of AI systems are free annotation data. Automatically回流为utterance增广 can achieve low-cost continuous self-improvement.
  • Two-stage information processing (crude screening + fine evaluation) is better than single-stage full-processing in terms of token efficiency and analysis depth. Stage 1 focuses on problem relevance/innovation/relevance to the source (judged within 30 seconds). Stage 2 focuses on key insights that drive results rather than engineering details, and critically examines the baseline and metrics chosen by the author.

Debugging#

  • Analyze the root cause of low GPU utilization in VLA inference: action chunking makes the inference calls sparse (about 8 times per trial × 2.5s/200s total time = 10%). Multi-client concurrency cannot improve the throughput of strict serial inference. The correct optimization direction is BatchedVLAServer batch inference + multi-GPU independent deployment.
  • Identify the diagnostic path for BC-RNN SR=0%. First, check whether the observation key name/dimension matches the training data. Then, distinguish between “evaluation bug” and “training fails due to defects” based on historical training logs. Avoid misdiagnosing it as a model capability problem. The bare except is the most common debugging obstacle in rendering/simulation frameworks. It should be changed to except Exception: traceback.print_exc().
  • Parameter settings should be driven by experiments rather than intuition. First, collect a small amount of data to verify the actual distribution before setting thresholds. Incorrect estimates of key parameters (such as distance threshold, driver version) can result in invalid entire batches of data or installation failures.

Tools#

  • Four-way parallel sub-Agents + the main thread handles non-paralleling parts. The large-scale code migration of 65 symlinks can be completed within a single session. The key is to divide the agent tasks without overlap according to file collections. BatchedVLAServer parallel evaluation (44 minutes/9 tasks) saves about 80% of time compared to serial evaluation.
  • Fix the standard solution for headless HPC nodes with Vulkan/EGL rendering: Extract the user-mode GL library that matches the kernel version from the official NVIDIA .run package. Inject it through VK_ICD_FILENAMES/__EGL_VENDOR_LIBRARY_FILENAMES/LD_LIBRARY_PATH without rooting. EGL/Vulkan device configuration is independent of CUDA. If CUDA works, Vulkan can also work.
  • The arXiv comment field is actually an area for conference acceptance announcements. A special search for conference papers can be achieved by searching the full text with all:“CVPR 2025” and filtering after finding comments. au:“Author Name” AND (kw1 OR kw2) is an effective way to track a specific researcher’s latest work.

Domain Knowledge#

  • Learned fusion (QFormer) is significantly better than simple concatenation: scGPT 512d representation itself is valuable, but it needs non-linear projection to be fully activated (+117% vs scGPT-only). The gene encoder of STAIG and the GNN training are decoupled. Differences between documents and code can only be discovered through code tracing. Zero-shot ability is a core competitive point in differentiation with methods that rely on training.
  • Pi0.5 LoRA fine-tuning can achieve 95 - 100% for Stack-type tasks (total 58.9%). BC-RNN fails completely (0%). The complexity of the task (multiple steps, delicate operations) is the main factor determining the success rate difference between VLA and traditional sequence modeling. The Pi0.5 LoRA model also requires at least 2 80GB GPUs (FSDP is a prerequisite).
