Learnings#

Architecture (architecture)#

  • Foundation model embeddings (scGPT 512d, UNI2 1536d) are NOT drop-in replacements for dimensionality reduction (PCA 50d) in clustering tasks. scGPT optimizes for cross-dataset transfer, not within-dataset variance. Zero-shot foundation model embedding fundamentally underperforms PCA: FM optimizes for cross-tissue generalization (pretraining objective is language modeling/contrastive learning), not variance maximization. PCA does task-aligned dimensionality reduction, maximizing variance≈maximizing cluster separability. Validated by Microsoft and 3+ papers + MIHD own data (PCA ARI=0.261 vs scGPT=0.170, 54% gap). STAIG graph-based fusion completely ignores gene encoder outputs, using raw HVG matrix instead. QFormer achieved 2× ARI improvement by learning gene-vision alignment

  • Evaluation metric distortion is often not a single bug but cascading chain of multiple bugs. User’s intuition more accurate, ultimately discovered 4 cascading bugs
  • Training-vs-evaluation mismatch debugging checklist: When policy performs well in training but fails in evaluation, systematically check: (1) observation dictionary keys, (2) environment initialization parameters, (3) action space bounds, (4) data normalization, (5) camera rendering setup. BC-RNN 0% SR on normal tasks + 0.28% on error recovery suggests env mismatch, not policy failure
  • Data preprocessing pipelines need explicit validation that all expected features have normalization statistics. Silent skipping (strict=False) can propagate scale mismatches through entire training runs without errors. apply_tree’s lenient behavior allows scale mismatches in concatenated features (unnormalized prev_actions + normalized actions). Always validate expected keys have norm stats, especially for concatenated features
  • Context loss requires tracing complete call chain: data collection (conversation_history.json storage) → data passing (provider call signatures) → data usage (prompt construction). Missing any layer breaks context. Effective code audits require multi-dimensional analysis: syntactic (TODO/FIXME/NotImplementedError), behavioral (exception handling patterns, empty loops, stub methods), structural (missing test coverage, unused exports), operational (service registration, startup hooks)

Should always log exceptions (logger.error/warning), distinguish critical functionality (error) and secondary functionality (warning), preserve diagnostic information. Python circular import solution: package-level lazy import (only all), module-level function-internal import, test patch source module not call site
  • Log archaeology (reading full log files) can reveal patterns invisible in code: Desktop Video’s ‘saved non-scoped bookmark’ repeated 20× across sessions exposed missing entitlements. Concurrent write bugs (errorLog FileHandle race) silently destroy diagnostic information. High-frequency timer logs (879 updateClockLabels) obscure real errors → need state-based logging, not tick-based
  • Documentation synchronization pattern: Use parallel exploration agents (one per major directory), verify all numbers from actual files (summary.json, filesystem stats), then systematically update document section by section. Don’t trust document claims - always check code/data ground truth. Tools (tools)#

  • macOS window layer controls not only visual Z-order but event routing permissions: .desktopWindow layer (below desktop icons) fundamentally doesn’t receive user input events at system level, must use .normal or higher layer to implement interactive ‘browse mode’. This is system-level constraint not API configuration issue. ignoresMouseEvents=false ineffective for .desktopWindow layer. .normal(level=0) is lowest interactive layer that doesn’t cover other apps
  • macOS sandbox security-scoped bookmarks require both compile-time entitlement declaration (in .entitlements file) AND runtime lifecycle management (startAccessingSecurityScopedResource/stop). Empty entitlements → silent fallback to non-scoped bookmarks → 100% restore failure. Bookmark system has no visible error channel when entitlements are missing
  • NSVisualEffectView supports two blending modes: .behindWindow (samples desktop/wallpaper) vs .withinWindow (samples same window’s content); SwiftUI .material uses .behindWindow and cannot sample video content. CALayer properties have implicit ~0.25s animations by default; use CATransaction.setDisableActions(true) for instant updates
  • In GPU cluster environment, prioritize searching existing model copies and performing format conversion rather than downloading through proxy. LLaVA base model conversion from HF format to original format (including vocab truncation fix 32064→32000) saved weeks. HF model format conversion isn’t just renaming keys, involves semantic differences like vocab size. Container had CUDA compute libraries but not rendering stack - Docker containers with NVIDIA GPUs need both CUDA runtime AND Vulkan/OpenGL libraries for graphics-based simulation, driver version must exactly match kernel module
  • h5py file handles cannot be shared across fork child processes (known PyTorch DataLoader limitation). Only safe solution is main process preload to numpy then fork. NFS random reads far slower than sequential reads, one-time loading 10× faster than on-demand. Preload all HDF5 to memory (uint8 saves 4× storage), close file handles, slice from numpy arrays in getitem. Change DataLoader to num_workers=8, pin_memory=True. Resolved 5-6% GPU utilization bottleneck to 80%+
  • Slurm’s srun with –jobid flag can bypass SSH PAM restrictions (like slurm_adopt) by executing commands directly in allocated job context. When GPU nodes run 100% utilization workloads, administrative commands via srun experience significant latency (60+ seconds) → design monitoring scripts to handle timeouts gracefully with lightweight fallback commands (nvidia-smi over nvitop)
  • PyTorch CUDA compatibility trap: torch.cuda.is_available() only checks if CUDA driver/library exist, not GPU compute capability compatibility (e.g., P100 sm_60 vs PyTorch requires sm_70+). CUDA operations fail asynchronously, Python try/except cannot catch errors. Most reliable fallback: CUDA_VISIBLE_DEVICES="" environment variable or switch compatible GPU
  • Language configuration for LLM pipelines requires injection at prompt level, not post-processing. Different parts need different rules (JSON keys stay English, content in target language). Config resolution chain: CLI flag > project.json > config.json > hardcoded default. Parallel agent execution dramatically accelerates independent module development.   • Multi-user shared conda environment import priority conflict minimal invasive solution: set PYTHONPATH=/path/to/user/src:$PYTHONPATH at script execution, prioritize user’s own source copy, doesn’t affect other users. Cross-platform script porting: Windows use Get-Item .LastWriteTime instead of find -newer, Start-Job instead of & parallel, PowerShell and bash differ in flow control and error handling. WKWebView transparency: drawsBackground=false makes entire rendering layer transparent, CSS on transparent layer still transparent, fix must be at rendering layer not CSS. App Sandbox blocks all outbound network by default

Domain Knowledge (domain)#

  • RM-IDEAL cross-sample evaluation complete workflow: (1) load string labels aligned by cache barcode, (2) load fusion embeddings from Phase 2 outputs or recompute, (3) compute RM-IDEAL scores using WWL graph kernel + Wasserstein distance, (4) compute embedding cosine similarity (niche centroid vs all spots), (5) evaluate correlation with Spearman r / Precision@K / Same-label rate, (6) generate spatial heatmap visualization. WWL kernel: K rounds of WL message passing, shared hash tables for label encoding, POT library ot.emd2() solves Wasserstein distance (10x speedup)
  • RM-IDEAL Spearman correlation exhibits significant directional asymmetry: 151508→151673 Layer_4 ρ=0.657, but reverse only 0.275. Reveals fusion embedding space quality differences in structural representation of same niche label across different samples, possibly related to spot density, noise level or annotation quality. Middle layers (Layer_3/6) show negative correlation (ρ -0.21 to -0.36) possibly confused with adjacent layers in embedding space, causing embedding-based similarity ranking opposite to topology-based RM-IDEAL ranking. Boundary/central layers easier to distinguish than middle transition layers. Cross-sample niche query evaluation metric interpretation: Spearman r≈0.44 (p<1e-150) indicates moderate positive correlation between fusion embedding cosine similarity and RM-IDEAL topology scores. P@K close to 0 indicates large head differences (embedding similarity and topology measure different dimensions). Low same-label rate suggests embedding spaces across sections not fully aligned
  • QueST evaluation framework: when no ground truth niche labels exist, construct synthetic ground truth using Wasserstein Weisfeiler-Lehman Graph Kernel over cell type annotations, then validate model’s similarity rankings with Pearson correlation. Metrics: Best Match Accuracy (top-1 correctness) and Overall Similarity Accuracy (correlation with WWL ground truth)
  • VLA systematic failures at task-critical phases (imprecise pressing, failed grasping/placing) can be addressed by training binary classifier to predict ‘critical regions’ using distance-based heuristics with gripper-state filtering. Enables cls token classifier training for high-stakes regions requiring extra caution. Must verify actual dataset size, per-step time and total steps before launching large-scale training to avoid resource waste. Phoenix MPM case: dataset scale estimation error (160K→1M) caused 13.5-day training vs expected hours
  • Multi-task ML pipelines built via incremental prototyping (start with 1 task) leave scattered single-task configurations (commented-out task lists, hardcoded paths). Scaling to production requires systematic audit of: data prep scripts, dataset mapping JSONs, checkpoint directories, and evaluation configs
  • Project description narrative strategy: external communication should start from application scenario target audience cares about (clinical patch query knowledge transfer), then explain technical path and key differentiators (zero-shot vs end-to-end training), finally elevate to domain value. Technical details serve narrative logic, not vice versa. Control each sentence within dual short clauses to reduce cognitive load. In spatial transcriptomics, adata.X can be either normalized values (scanpy pipeline) or raw counts (STHD pipeline) - must check code rather than assume from data type
