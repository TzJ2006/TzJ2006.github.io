Issues and Solutions#

1. STAIG’s two-stage pipeline completely ignores gene encoder output in practice, using raw HVG as GCN input — making scGPT+UNI2+STAIG ≡ PCA+UNI2+STAIG [MIHD] (2026-03-02)#

Solution: Three-level code tracing (Fusion.py → runner.py → STAIGTrainer.py) confirmed the architectural fact: gene encoder and GNN training are decoupled; improvements should target the GNN architecture rather than replacing the gene encoder

2. Pi0.5 eval client reports keepalive ping timeout, SR all 0%; multi-client concurrency is paradoxically 24% slower than serial [Pi0.5/VLA Evaluation] (2026-03-02)#

Solution: Added ping_timeout=None in connect() to disable the 20s timeout (JAX JIT compilation takes 30–60s, exceeding the default); confirmed VLA inference is strictly serial, making multi-client ineffective; the correct solution is BatchedVLAServer batch inference

3. STAIG fusion embeddings collapse to zero on certain sections, causing NaN Spearman r in cross-sample evaluation [MIHD] (2026-03-03)#

Solution: Added zero-embedding detection in load_fused_embeddings() (norm < 1e-6) that automatically triggers –recompute_fusion to retrain STAIG (~50 seconds)

4. Pi0.5 LoRA training config name contains ‘pi05_libero’, raising suspicion that initialization weights come from libero rather than pi05_base [Pi0.5/VLA Training] (2026-03-03)#

Solution: Three-layer evidence (config.py weight_loader field + training log checkpoint restoration path + control group configuration) confirmed that pi05_base is correctly used on both sides; the config name describes the data loading configuration, and weight_loader is the sole authoritative source

5. BC-RNN achieves SR=0% on all normal tasks, completely inconsistent with training rollout behavior [error_recovery_benchmark] (2026-03-04)#

Solution: Code investigation traced the bug to an object observation key error in baseline_accuracy evaluation; the policy was receiving empty observations. Near-0% Error SR in M14 is the benchmark’s core claim (existing policies lack error recovery ability) — the contrast with Normal SR is what makes the argument

6. Openpi-moe training feature norm_stats.json missing prev_actions key with no error raised; direct concatenation of hist_actions (unnormalized) and actions (normalized) in the VAE causes scale mismatch [zhaoganlong/openpi] (2026-03-05)#

Solution: Modified compute_norm_stats.py to dynamically detect prev_actions and write its statistics; the underlying mechanism is apply_tree(strict=False) silently skipping missing keys — any newly added training feature must mandatorily sync with the normalization script

7. All CalendarPro background services had never run in production; 321 passing tests failed to catch this integration bug [CalendarPro] (2026-03-05)#

Solution: Added await coordinator.start_all() in discord_bot.py’s on_ready(); the root cause is that the registration pattern was decoupled from lifecycle management — unit tests cannot detect silent failures at the integration layer

8. Headless Docker container SAPIEN reports Render Error; bare except hides the real exception; apt driver version mismatches the kernel module [VLA-RoboTwin] (2026-03-06)#

Solution: Exposed the real exception → confirmed missing NVIDIA Vulkan ICD → extracted user-space GL libraries strictly matching the kernel version from the official 535.104.12 .run package → injected via VK_ICD_FILENAMES/LD_LIBRARY_PATH environment variables (no root required)

9. Research Scout’s initial version performed single-stage deep evaluation on all 50 papers, severely wasting tokens and allowing low-relevance papers to consume most analysis resources [gadget/Research Scout] (2026-03-07)#

Solution: Refactored into a two-stage pipeline: Stage 1 lightweight screening → Stage 2 deep analysis (up to 20 high-relevance papers), modeled after real researcher reading habits

10. 65 symlinks in error_recovery_benchmark created indirection layers during migration; after symlink removal, sys.path parent chain depths needed recalculation [error_recovery_benchmark] (2026-03-08)#

Solution: 4-way parallel agents cleaned all symlinks without overlap following a 9-stage plan; main thread handled Makefile/documentation; proactively fixed 2 unplanned sys.path depth errors (Path.resolve() no longer folded through symlinks)

11. VLA-RoboTwin critical_region annotation data all zero; τ=0.10m threshold was purely estimated and does not match actual workspace scale [VLA-RoboTwin] (2026-03-08)#

Solution: Rather than directly tuning the parameter, switched to recording raw ee_target_distances (4–8 distance variables per task); analyze the actual data distribution first, then calibrate the threshold in a data-driven manner

Lessons Learned#

Architecture#

  • Three standard checks after large-scale implementation: (1) Are new services wired into the startup chain? (2) Are cross-layer components (IntentType → routing → Handler → service) consistently updated? (3) Are silent exceptions (except: pass) masking runtime errors? Passing tests is necessary but not sufficient — integration-layer defects can only be discovered through end-to-end verification
  • Before designing an agentic system, proactively research mature comparable projects — OpenClaw’s EventBus/CronScheduler/Plugin registration patterns and GSD’s STATE.md persistent memory/multi-agent context isolation are highly reusable architectural patterns. Related fixes should be consolidated into a single change rather than staged, to avoid code rework
  • apply_tree(strict=False) is a hidden danger in ML training pipelines: adding a new training feature that requires normalization without updating the norm_stats computation script leads to silent scale mismatch that degrades training quality. Mismatch logs in AI systems are free annotation data — automatically feeding them back as utterance augmentation enables low-cost continuous self-improvement
  • Two-stage information processing (coarse screening + deep evaluation) is superior to single-stage full processing in both token efficiency and analysis depth: Stage 1 focuses on relevance/novelty/source authority (30-second judgment); Stage 2 focuses on the key insights driving impact rather than engineering details, and critically examines the authors’ choice of baselines and metrics

Debugging#

  • Root cause analysis of low GPU utilization in VLA inference: action chunking sparsifies inference calls (~8 calls × 2.5s / 200s total = 10%); multi-client concurrency cannot improve strictly serial inference throughput; the correct optimization is BatchedVLAServer batch inference + independent multi-GPU deployment
  • Diagnostic path for BC-RNN SR=0%: first check whether observation key names/dimensions match training; then use historical training logs to distinguish “evaluation bug” vs “training itself failed”, avoiding misdiagnosis as a model capability issue. bare except is the most common debugging obstacle in rendering/simulation frameworks — replace it immediately with except Exception: traceback.print_exc()
  • Parameter values should be empirically driven, not intuitively estimated: collect a small amount of data to verify the actual distribution before setting thresholds. Incorrectly estimated critical parameters (e.g., distance thresholds, driver versions) can invalidate an entire batch of data or cause installation failures

Tools#

  • 4-way parallel sub-agents + main thread handling non-parallelizable parts can complete a large-scale 65-symlink code migration within a single session; the key is dividing agent task boundaries by disjoint file sets. BatchedVLAServer parallel evaluation (44 minutes / 9 tasks) saves ~80% time compared to serial execution
  • Standard solution for fixing Vulkan/EGL rendering on headless HPC nodes: extract user-space GL libraries strictly matching the kernel version from the NVIDIA official .run package (no kernel module installation required); inject via VK_ICD_FILENAMES/__EGL_VENDOR_LIBRARY_FILENAMES/LD_LIBRARY_PATH; no root required. EGL/Vulkan device configuration is independent of CUDA — CUDA working does not imply Vulkan is available
  • The arXiv comment field is the de facto venue acceptance announcement area; full-text search with all:“CVPR 2025” followed by comment filtering enables venue-specific paper search. au:“Author Name” AND (kw1 OR kw2) is an effective method for systematically tracking a specific researcher’s latest work

Domain Knowledge#

  • Learning-based fusion (QFormer) significantly outperforms simple concatenation: the scGPT 512d representation is valuable on its own but requires nonlinear projection to be fully activated (+117% vs scGPT-only). STAIG’s gene encoder and GNN training are decoupled — discrepancies between documentation and code can only be found through code tracing. Zero-shot capability is the core differentiating competitive advantage against training-dependent methods
  • Pi0.5 LoRA fine-tuning achieves 95–100% on Stack-class tasks (58.9% overall); BC-RNN completely fails (0%). Task complexity (multi-step, fine-grained manipulation) is the primary determinant of success rate gaps between VLA and traditional sequence modeling. Pi0.5 LoRA mode also requires at least two 80GB GPUs (FSDP is a necessary prerequisite)
