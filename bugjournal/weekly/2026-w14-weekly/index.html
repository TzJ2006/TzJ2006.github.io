Problems & Solutions#

1. Linux FloatBall window permanently stuck at 400×400, Tauri set_size() completely ineffective [TokenMonitor] (2026-04-04)#

Solution: Root cause: GTK bottom-up size negotiation takes max(window_requested, widget_natural_size), WebKitGTK natural size ≈200×200 locked the window. Fix: Call widget.set_size_request() in with_webview() on GTK main thread, remove set_resizable toggling (async callback would be locked by premature set_resizable(false)), keep Linux permanently resizable.

2. Linux WM causes systematic +305px/+113px position offsets when dynamically resizing transparent borderless windows; probe/corrective mechanisms cause visible flicker [TokenMonitor] (2026-04-05)#

Solution: Architectural refactoring: Fixed window size at expanded dimensions (152×56), never resize; use GDK input_shape_combine_region to control click-through areas (collapsed: ball only, expanded/dragging: full window). Eliminated ~150 lines of patch code.

3. Life-Copilot Discord Bot reports ’need OAuth authentication’ for calendar queries; root cause was two layered issues: (1) claude -p doesn’t load project-level MCP config; (2) non-interactive mode silently denies MCP tool permissions [Life-Copilot] (2026-04-05)#

Solution: Added –mcp-config .claude/mcp_servers.json to load 19 MCP tools, then added –permission-mode bypassPermissions to bypass non-interactive permission confirmation. Reproduced permission_denials through direct claude -p command execution to pinpoint root cause.

4. TEDDY gene-only ARI (0.186) underperforms PCA (0.255); foundation model fails to exceed simple baseline in clustering task [MIHD] (2026-03-30)#

Solution: Root cause analysis: TEDDY optimizes for disease classification rather than variance maximization; 1024d high-dimensional space degrades KMeans performance; training data is dissociated scRNA-seq, distribution differs from spatial transcriptomics.

5. Discord Bot dialogue system’s 3170-line intent router (semantic/, 24 handle* methods, conversation_history) creates maintenance burden and lacks multi-backend flexibility [Life-Copilot] (2026-04-04)#

Solution: Complete architectural refactoring: Replace with CLIAdapter + SessionManager + MCP Server (19 tools) + Skills (4) architecture supporting Claude/Codex/Gemini CLI fallback chain. Net deletion of 2823 lines, achieved decoupling through Skills JSON protocol triggering Discord UI.

6. Linux Screen Time Tracker initial solution used xdotool, incompatible with Wayland (user machine runs GNOME 50 Wayland) [Life-Copilot] (2026-04-04)#

Solution: Feasibility probing confirmed Wayland environment, switched to GNOME Introspect D-Bus (org.gnome.Shell.Introspect.GetWindows()) as primary path, xprop as XWayland fallback, PID→cmdline as final degradation. All subprocess calls add 5s timeout + proc.kill(). Issue proactively discovered by Critic subagent adversarial review.

7. MIHD paper narrative logic flaw: single modality can also do retrieval, ‘fusion = necessary’ assumption doesn’t hold [MIHD] (2026-04-03)#

Solution: Reconstructed narrative as three-stage argument: simple fusion is trap → spatial graph learning is bridge → correct fusion can exceed single modality. Incorporated user’s rebuttal into paper’s core findings.

8. ClaudeCLIAdapter.is_available() missing async keyword caused Claude to silently degrade to Codex and timeout [Life-Copilot] (2026-04-04)#

Solution: Await on sync method doesn’t raise compile error; runtime TypeError was swallowed by except Exception, causing Claude to be judged unavailable. Added async keyword, Claude available: True restored.

Learnings#

Architecture (architecture)#

  • STAIG fusion framework exhibits high robustness to gene encoder choice: Teddy 1024d (ARI=0.495) and PCA 50d (ARI=0.497) achieve nearly identical performance under STAIG framework, indicating spatial graph contrastive learning mechanism dominates clustering performance while gene embedding quality impact is significantly diluted. Core conclusion: embedding quality > clustering method selection (STAIG+mclust 0.546 > STAIG+Leiden 0.513).
  • Foundation models exhibit systematic transfer loss in cross-domain downstream tasks: TEDDY achieves SOTA in disease classification (ACC=0.72) but underperforms PCA in spatial transcriptomics clustering (0.186 vs 0.255). Root cause is optimization objective mismatch (discriminative vs variance maximization).
  • Linux X11 WM lacks atomicity guarantee for transparent borderless window resize requests; different WMs interpret same rect request with different semantics, causing systematic offsets. Complete elimination of resize (fixed window + GDK input shape) is the only reliable solution; local patches only shift symptoms between manifestations.
  • GTK window size control mechanism: GTK bottom-up negotiation takes max(window_requested, widget_natural_size); WebKitGTK natural size ≈200×200 logical; breakthrough requires calling widget.set_size_request(). Tauri’s set_size/set_min_size/set_max_size are ineffective for WebKitGTK windows; with_webview() is async dispatch, subsequent Tauri API calls enqueue before callback executes, potentially locking old size in FIFO order.
  • Radical simplification (replacing engine) often more effective than incremental patching: Thin Adapter + MCP Server + Skills three-layer pattern replaces proprietary routing logic, gaining cross-CLI generality. Large refactorings should undergo adversarial review (Critic subagent) before coding; this week’s Critic phase discovered high-value issues across multiple projects including Wayland compatibility, lark-oapi thread blocking, ThoughtStore non-singleton, 4 hidden code dependencies - extremely high ROI.
  • Multi-CLI system design best practices: Different CLI tools (Claude Code/Codex/Gemini) each have convention instruction files (CLAUDE.md/AGENTS.md/GEMINI.md); utilizing convention files is simpler and more reliable than runtime injection. Skills are essentially Markdown prompts, cross-platform migration only needs tool name mapping. App state verification and CLI tool context injection are two independent layers; token verification passing doesn’t mean CLI tool received correct context. Unified history ring buffer is more reliable than adapting each CLI’s session mechanism.

Debugging (debugging)#

  • Debugging nested process CLI tools (main process → Claude CLI → MCP server): first log actually passed command-line arguments, then isolate and verify parameter effects one by one, more efficient than directly guessing parameter behavior.
  • Runtime logs are fundamental means for underlying system debugging: when GTK, WM, OS and other underlying system behaviors occur, should prioritize confirming assumptions through logs/probes rather than inferring from code. Cross-platform animation strategies need overall consistency; disabling CSS transitions must synchronously adjust corresponding JS/Rust timing logic. X11 WM forces undecorated windows to stay within visible area (edge_offset needs platform differentiation); macOS/Windows allow half-off-screen.

Tools (tools)#

  • Claude CLI -p mode complete MCP configuration requires two steps: (1) explicitly pass –mcp-config to load project-level tools; (2) add –permission-mode bypassPermissions to bypass permission confirmation that cannot be completed in non-interactive mode. –allowedTools wildcard is unreliable in this mode due to race condition with MCP server initialization; bypassPermissions is the correct mode for backend service invocation.
  • structlog wrapping stdlib logging requires explicitly adding structlog.stdlib.ExtraAdder() to merge extra fields into structured output; otherwise extra parameters won’t auto-propagate, fields silently lost without error.

Domain Knowledge (domain)#

  • NeurIPS D&B Track submission strategy: ‘domain correctors’ (by quantifying specific flaws in existing benchmarks, e.g., 57% accuracy, triggering evaluation paradigm reconstruction) are more easily accepted and have more lasting impact than ’new method proposers’. Reviewers prioritize multi-dimensional evaluation, fair baseline comparison, human verification mechanisms (κ≥0.68), and counter-intuitive negative findings.
