Problems & Solutions#

1. Linux FloatBall window constant 400x400, Tauri set_size() completely ineffective [TokenMonitor] (2026-04-04)#

Solution: GTK bottom-up size negotiation takes max(window_requested, widget_natural_size), WebKitGTK natural size ≈200x200 causing window lockup. Fix: call widget.set_size_request() via with_webview() in GTK main thread, remove set_resizable toggling (with_webview is async dispatch, set_resizable(false) queued before callback execution locks old size), Linux permanently keeps resizable

2. Linux X11 WM cannot atomically handle transparent borderless window dynamic resize, causing systematic +305px/+113px position offset, probe/corrective mechanism flicker, WebView vs native window size inconsistency clipping [TokenMonitor] (2026-04-05)#

Solution: Fixed window size at expanded (152×56) never resize; use GDK input_shape_combine_region to control click-through: collapsed only ball area (56×56), expanded and dragging full window accepts events

3. Life-Copilot Discord Bot persistent ‘OAuth authentication required’ reply to calendar queries, root cause two overlapping issues: (1) claude -p doesn’t load project-level MCP config; (2) non-interactive mode MCP tool invocation permissions silently denied [Life-Copilot] (2026-04-05)#

Solution: Added –mcp-config .claude/mcp_servers.json and –permission-mode bypassPermissions sequentially in claude_adapter.py command; located root cause via direct claude -p command reproducing permission_denials field

4. adaln_attention ARI≈0.003 on Visium HD, nearly complete failure; simple fusion (concat ARI=0.191) worse than single-modality PCA-only (ARI=0.351) [MIHD] (2026-04-03)#

Solution: Confirmed results genuine, converted both negative results to core findings: adaln collapse marked as large-scale data failure case, concat<PCA-only reconstructed as ‘simple fusion is trap’ core paper argument

5. Paper narrative logic flaw: single modality can also do retrieval, ‘fusion=necessary’ assumption not necessarily true [MIHD] (2026-04-03)#

Solution: Reconstructed narrative as three-part argument: simple fusion is trap → spatial graph learning is bridge → correct fusion can surpass single modality, incorporating user’s rebuttal into paper core finding

6. FloatBall Linux: GTK WM async behavior causes set_size/set_position non-atomic, window won’t shrink back after collapse after expansion [TokenMonitor] (2026-04-03)#

Solution: Via set_resizable(true) unlock GTK WM hint → set_position → set_size → set_resizable(false) re-lock, ensure operation timing; adopt last_rect state tracking to replace direct WM read avoiding stale values

7. Leiden clustering needs fair comparison with KMeans, but Leiden defaults to resolution parameter controlling cluster count, cannot precisely match n_clusters [MIHD] (2026-03-31)#

Solution: Implemented binary search in cluster_with_leiden(): iteratively adjust resolution in [0.01, 10] range until cluster count matches, max 20 iterations

8. HD pipeline Vision Extraction failure: VisionExtractor uses data_root/section_id path to find images, but HD data is in crop_dir, causing 11/13 experiments to fail [MIHD] (2026-03-31)#

Solution: VisionExtractor.init added crop_dir parameter; extract() passes image_dir_override=crop_dir; _load_image_and_coords() prioritizes image_dir_override; runner.py passes crop_dir when creating VisionExtractor

9. Linux Screen Time Tracker initial approach using xdotool, doesn’t support Wayland (user machine running GNOME 50 Wayland) [Life-Copilot] (2026-04-04)#

10. ssh_remote.rs host_cache_dir() uses assert!() causing panic crash on path validation failure [TokenMonitor] (2026-04-02)#

Solution: Changed function signature to Result<PathBuf, String>, all call chains synchronized propagate Result, call sites use unwrap_or_default() to handle

11. Linux GTK window set resizable(false), programmatic set_size() rejected by WM, causing float ball unable to collapse [TokenMonitor] (2026-04-02)#

Solution: In Linux branch, set_resizable(true) temporarily unlock before set_size, set_resizable(false) re-lock after operation complete

Learnings#

Architecture (architecture)#

  • STAIG fusion framework exhibits high robustness to gene encoder: Teddy 1024d (ARI=0.495) vs PCA 50d (ARI=0.497) nearly identical performance under STAIG framework, indicating spatial graph contrastive learning mechanism dominates clustering performance, gene embedding quality influence greatly diluted. High-quality embedding itself matters more than clustering algorithm choice: STAIG+mclust (0.546) > STAIG+Leiden (0.513), Leiden helps weak embeddings (+29%) but diminishing returns for strong embeddings (+6%)
  • Linux X11 WM provides no atomicity guarantee for transparent borderless window resize requests, different WMs interpret same coordinates with different semantics causing systematic offset; complete elimination of resize (fixed window + GDK input shape) is only reliable solution, local patches just transfer between symptoms. Linux transparent borderless Tauri/WebKitGTK float window architectural constraints: WM, GTK, WebView content area three layers’ size/position don’t auto-sync, this is platform constraint not bug. Correct architecture should be fixed large native window + internal pure CSS expand/collapse, not relying on frequent outer window resize
  • Life-Copilot radical simplification (replacing engine) often more effective than incremental patching: Thin Adapter + MCP Server + Skills three-layer pattern replacing proprietary routing logic, gained cross-CLI generality. Large refactors should have adversarial review (Critic subagent) before coding, can discover hidden dependencies and architectural risks in advance. This week Critic stage discovered Wayland compatibility, lark-oapi thread blocking, ThoughtStore non-singleton, 4 hidden code dependencies etc. high-value issues, extremely high ROI

Debugging (debugging)#

  • GTK window size control mechanism full picture: GTK bottom-up negotiation takes max(window_requested, widget_natural_size); WebKitGTK natural size ≈200x200 logical, breakthrough must call widget.set_size_request(), Tauri’s set_size/set_min_size/set_max_size ineffective for WebKitGTK windows; with_webview() is async dispatch, subsequent Tauri API calls queue before callback execution, under FIFO may lock old size; WebKitGTK scrollbars belong to GTK widget layer, must use set_policy(Never, Never) at GTK code layer to disable
  • Runtime logs are fundamental means of low-level system debugging: when GTK, WM, OS level system behaviors occur, should prioritize confirming assumptions via logs/probes empirically rather than inferring from code; cross-platform animation strategies need overall consistency, when disabling CSS transitions must synchronously adjust corresponding JS/Rust timing logic; X11 WM forcibly clamps undecorated windows within visible area (edge_offset needs platform differentiation), macOS/Windows allow half off-screen
  • Claude CLI -p mode complete MCP configuration needs two steps: (1) explicitly pass –mcp-config to load project-level tools; (2) add –permission-mode bypassPermissions to bypass permission confirmation impossible in non-interactive mode; –allowedTools wildcard unreliable in this mode due to race condition with MCP server initialization, bypassPermissions is correct mode for backend service calls
  • When debugging nested process CLI tools (main process → Claude CLI → MCP server), first log actual passed command-line parameters then isolate and verify parameter effects one by one, more efficient than directly guessing parameter behaviors

Domain Knowledge (domain)#

  • Foundation Model systematic transfer loss on cross-domain downstream tasks: Teddy SOTA on disease classification (ACC=0.72), but worse than PCA on spatial transcriptomics clustering (0.186 vs 0.255), root cause is optimization objective (distinctiveness vs variance maximization) mismatch. Data scale is key variable for multimodal fusion method selection: Visium HD (17502 spots) vs DLPFC (~3900 spots) scale difference can make AdaLN change from effective to completely ineffective
  • NeurIPS D&B Track submission strategy: ‘domain corrector’ (quantifying specific defects in existing benchmarks, e.g., 57% accuracy, triggering evaluation paradigm reconstruction) more likely accepted and lasting impact than ’new method proposer’; reviewers most value multi-dimensional evaluation, fair baseline comparison, human validation mechanism (κ≥0.68), and counter-intuitive negative findings

Tools (tools)#

  • GDK input shape intercepts events at X11 layer, setPointerCapture can’t bypass it; drag scenario needs set_float_ball_dragging command at Rust side to temporarily expand input shape to full window; cairo-rs 0.18 + gdk 0.18 can directly use input_shape_combine_region in Tauri with_webview callback, no need for complex dependencies
  • ccplan skill’s divergent exploration (6 solutions) + adversarial review + feasibility probe process can effectively transform complex platform architecture problems into executable plans, ECL documents provide cross-session context persistence, reducing repeated analysis cost
  • Modern Linux desktop (Ubuntu 22.04+) defaults to Wayland, cross-platform window tracking solutions must use GNOME Introspect D-Bus as main path, xdotool/xprop can only serve as XWayland fallback; background sampling tasks shouldn’t fully rely on CronScheduler (minimum tick 60s), tasks needing non-minute periods should use independent asyncio.Task loops
  • Multi-CLI system design best practices: different CLI tools (Claude Code/Codex/Gemini) each have convention instruction files (CLAUDE.md/AGENTS.md/GEMINI.md), using convention files simpler and more reliable than runtime injection; Skills essentially Markdown prompts, cross-platform migration only needs tool name mapping; unified history ring buffer more reliable than adapting each CLI’s session mechanism
  • ‘Code implemented parameter interface’ doesn’t equal ‘parameters being passed’: integration layer is most common wiring breakage point, when verifying should check call side not just implementation side; Python await sync method’s TypeError if swallowed by except Exception causes function silent downgrade instead of error reporting
