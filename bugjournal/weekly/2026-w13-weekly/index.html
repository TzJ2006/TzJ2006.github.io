Learnings#

Architecture (architecture)#

  • ECL (Evolving Constraint Language) methodology successfully catches critical issues before implementation through hypothesis interrogation, adversarial filtering, and feasibility probing. Examples: tray icon size constraints, missing ccusage features, performance implications of subprocess spawning. ccplan’s Critic agent identified 12+ design issues including CRITICAL contradictions across projects, validating ‘requirements are hypotheses’ premise.
  • Hub+spoke design pattern is optimal for multi-variant systems (multi-language, multi-platform): shared general framework (hub) + on-demand specific configurations (spokes), avoiding both duplication and dilution. Single-language focus directive prevents prompt dilution when processing one language.
  • Backward compatibility encompasses more than CLI entry points: (1) module import paths (2) config file formats (3) data file structures. Shim re-export is an effective pattern for maintaining import compatibility during large refactorings.
  • Cross-platform applications should use conditional compilation (#[cfg(target_os)]) to isolate platform-specific code rather than removing features—preserve macOS-native implementations (Keychain, NSVisualEffectView) while adding fallbacks for other platforms. Static configuration beats dynamic detection for known window types; runtime detection adds complexity and failure modes without benefit.
  • Feature Guard Protocol prevents regression across sessions: memory and context are ephemeral, but disk is persistent. Guards written to ECL files survive across sessions and compactions. Portability (traveling with skill directory) is superior to project-specific solutions.

Debugging (debugging)#

  • Hypothesis-driven debugging with parallel investigation scales better than sequential trial-and-error—3 subagents investigating H1/H2/H3 concurrently saves time and accumulates evidence. Dual-track investigation (static + instrumented probing) preserves speed while adding depth only when needed.
  • Cascading feedback loops require simultaneous multi-layer intervention; fixing one layer alone only weakens but doesn’t eliminate the cycle. Example: TokenMonitor dashboard jitter needed fixes at window layer (throttle), data layer (equality checks), and computation layer (grace period).
  • Observable success criteria (like fixed reference points) are better than abstract metrics for debugging UI issues. User-visible behavior focus cuts through complexity faster than debugging abstractions.

Tools (tools)#

  • Migration smoke tests are critical for large refactorings: write import contract tests capturing all external consumers, run before and after. This creates a safety net for confident changes (47 tests × 2 runs = 94 validations). Large-scale refactoring requires parallel agent execution for independent tasks to reduce sequential bottleneck.
  • Auto-execution phases in workflows require quantitative depth requirements (minimum N findings, M layers of analysis, at least 1 probe) to prevent ‘fast but shallow’ behavior; qualitative instructions like ‘be thorough’ are insufficient against efficiency incentives. Complex multi-phase workflows need explicit state preservation protocols when invoking external tools that return metadata-heavy results.
  • Skills’ context cost is real and quantifiable (~15K-20K tokens for 250+ skill indices); project-specific allocation superior to global installation. Research-first approach with quality constraints (e.g., >1k GitHub stars) produces better designs than isolated invention.
  • Wave-based execution with verification checkpoints (cargo check, cargo test, vitest after each wave) prevents cascading failures in large refactors by catching issues early. Hybrid tooling strategy (parallel agents for complex logic, sed/bash for bulk string replacements, direct Edit for targeted fixes) optimizes for both correctness and speed.

Domain Knowledge (domain)#

  • Win32+WebView2 architecture has inherent 1-frame delay between SetWindowPos (native) and CSS re-layout (web); only position:fixed bypasses this timing dependency. CSS transitions that affect layout dimensions trigger ResizeObserver on every frame; delaying layout change until after visual transition completes prevents IPC call storms.
  • Windows GUI application silent subprocess execution best practice: use std::os::windows::process::CommandExt creation_flags with CREATE_NO_WINDOW (0x0800_0000) to avoid console window popup. Windows system tray icons have fundamentally different constraints (16-32px DPI-aware) than macOS menu bar icons (22px template rendering), requiring separate scaling logic.
  • Tauri v2’s tray.set_title() is macOS-only; Windows/Linux require set_tooltip() as fallback. Platform differences (macOS has native blur, Windows/Linux don’t) cause same config to produce different visual results—requires application-layer platform detection and compensation. Cross-platform window positioning in Tauri: plugin abstractions sacrifice accuracy for portability; pixel-perfect positioning requires platform-specific native APIs.
  • Spatial transcriptomics cross-slice tasks: embedding space comparability (shared coordinate system) is more critical than embedding quality. Foundation Model (scGPT) outputs fixed representation space, naturally suitable for cross-slice retrieval, while PCA/STAIG independent processing creates non-comparable coordinate systems. Single-slice clustering optimal method (STAIG) ≠ cross-slice retrieval optimal method (scGPT)—need different encoders for different task objectives.
  • TEDDY-G (rank-value encoding) outperforms TEDDY-X (binned expression); linear probe far superior to fine-tuning (RA: 0.56→0.93) suggests high embedding quality but fine-tuning prone to overfitting. TEDDY vocab uses Ensembl ID (43808 tokens); HuggingFace single-cell foundation models have non-standard forward interfaces—cannot assume standard HuggingFace API, must read source code to confirm gene_ids parameter names and output dict structures.
