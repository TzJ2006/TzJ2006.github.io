Learnings#

Architecture (architecture)#

  • ECL methodology successfully catches critical issues before implementation through hypothesis interrogation, adversarial filtering, and feasibility probing. Examples: tray icon size constraints, missing ccusage features, performance implications of subprocess spawning. Validates ‘requirements are hypotheses’ premise—every stated requirement should be challenged
  • Embedding space comparability (shared coordinate system) is more critical than embedding quality for cross-slice tasks. Foundation models (scGPT) naturally suitable for cross-slice retrieval due to fixed representation space, while PCA/STAIG independent processing creates incomparable coordinates
  • Hub+spoke design pattern is best practice for multi-variant systems (multi-language, multi-platform): shared general framework (hub) + on-demand specific configurations (spokes), avoiding both duplication and dilution
  • Cross-platform doesn’t mean removing platform-specific features—it means properly isolating them behind feature flags or conditional compilation. Preserve macOS-native implementations (Keychain, NSVisualEffectView) while adding fallbacks for other platforms
  • Win32+WebView2 architecture has inherent 1-frame delay between SetWindowPos (native) and CSS re-layout (web); only position:fixed bypasses this timing dependency

Debugging (debugging)#

  • Migration smoke tests are critical for large refactorings: write import contract tests capturing all external consumers, run before and after. Creates safety net for confident changes (47 tests × 2 runs = 94 validations)
  • Hypothesis-driven debugging with parallel investigation scales better than sequential trial-and-error; 3 subagents investigating H1/H2/H3 concurrently saves time and accumulates evidence
  • Cascading feedback loops require simultaneous multi-layer intervention: fixing window resize alone wasn’t enough until data layer (payload equality) and computation layer (grace period) were also addressed
  • Observable success criteria (like ‘Xs ago’ text position) are better than abstract metrics (window coordinates) for debugging UI issues

Tools (tools)#

  • Auto-execution phases in workflows require quantitative depth requirements (minimum N findings, M layers of analysis, at least 1 probe) to prevent ‘fast but shallow’ behavior; qualitative instructions like ‘be thorough’ are insufficient
  • Persistent guards (ECL feature_guard) + automated checking (guard-check.py) prevent feature regression across sessions. Memory and context are ephemeral, but disk is persistent. Portability (traveling with skill directory) is superior to project-specific solutions
  • Skills’ context cost is real and quantifiable (~15K-20K tokens for 250+ skill indices); project-specific allocation superior to global installation
  • Parallel agent execution accelerates independent tasks: 5 module extraction agents ran concurrently, reducing sequential bottleneck. Requires clear task boundaries and no shared state
  • Wave-based execution with verification checkpoints (cargo check, cargo test, vitest after each wave) prevents cascading failures in large refactors by catching issues early
  • CSS transitions that affect layout dimensions trigger ResizeObserver on every frame; delaying layout change until after visual transition completes prevents IPC call storms and preserves atomic window repositioning
  • Windows GUI applications should use CREATE_NO_WINDOW (0x0800_0000) flag for silent subprocess execution to avoid console window popup

Domain Knowledge (domain)#

  • TEDDY-G (rank-value encoding) outperforms TEDDY-X (binned expression), achieving 8% higher accuracy than Nicheformer on held-out donor tasks. Linear probe far superior to fine-tuning (RA: 0.56→0.93), indicating high embedding quality but fine-tuning overfitting risk
  • HuggingFace single-cell foundation models (TEDDY, scGPT) have non-standard forward interfaces—cannot assume standard HuggingFace API. Must read source code to confirm parameter names (gene_ids vs input_ids) and output dict structure
