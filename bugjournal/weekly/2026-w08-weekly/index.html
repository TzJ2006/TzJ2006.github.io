Learnings#

Architecture (architecture)#

  • VLA end-to-end integration requires: (1) policy server pattern for cross-conda-env execution, (2) camera observation support in environment wrapper, (3) observation preprocessing (quaternion→axis-angle, BGR→RGB, action chunking), (4) dual rollout modes (synthetic injection + natural capture). Fundamentally different from demo-replay synthetic perturbation approach.
  • Two-stage fusion (self-supervised refinement → multimodal integration) creates new experimental dimension bridging image-only and multimodal methods, but refined embeddings not universally superior - optimization objective alignment critical. Self-supervised clustering optimizes for cluster separability, which may not align with multimodal fusion’s information complementarity objectives.
  • Resume logic for long-running data collection must rebuild summary state from JSONL, not just skip episodes. Correct approach: (1) read JSONL on startup, (2) rebuild all_episodes dict, (3) add completed tuples to skip set, (4) continue collection. Ensures idempotency.
  • Cache variants (standard/freq/staig_strict) prevent collisions when same encoder produces different embeddings based on preprocessing flags - more robust than separate encoder names.

Debugging (debugging)#

  • Spatial coordinate systems in medical imaging require explicit semantic naming (pxl_row=Y, pxl_col=X) and validation via physical units (pixels × scalefactor = microns). Generic pixel_x/pixel_y naming is error-prone. Vision foundation model integration critically depends on spatial alignment - incorrect patch extraction can halve performance even with correct architecture.
  • Embedding dimension bottlenecks cause catastrophic model collapse: STAIG’s 64-dim output compresses 1536-dim vision features by 96%, producing embeddings with std=0.12 that are nearly indistinguishable. Spatial refinement then eliminates minority clusters via majority voting. Other fusion methods (256-512 dim) don’t exhibit this pathology.
  • Pure vision encoders cluster by morphological similarity not biological domains: UNI2 groups blood vessels across different cortical layers because they look similar. Low ARI reflects structural grouping (what looks similar) not functional grouping (what is biologically similar). General foundation models learn coarse semantics; domain-specific fine-tuning or feature refinement necessary for fine-grained tasks.
  • ‘Tests passing’ ≠ ‘system operational’ - integration testing with real dependencies (model checkpoints, external APIs) essential before claiming completion. Synthetic unit tests validate code correctness but not end-to-end functionality.
  • Observation pipeline bugs in multi-path codebases (injection vs natural capture) require exhaustive analysis - fix in one path doesn’t guarantee fix in all paths. Grep for all uses of the pattern.

Tools (tools)#

  • When integrating third-party ML libraries (openpi, robomimic), always check actual API signatures via inspect.signature() and read example code in source repo. Documentation is often incomplete or outdated. High-level APIs provide encapsulation; using low-level APIs manually is error-prone.
  • Cross-domain literature search is extremely valuable for identifying solution patterns. Image-only clustering problems are nearly identical across fine-grained bird classification, remote sensing, microscopy, and geology. All domains converged on the same 4-tier solution hierarchy: domain-specific pretraining > test-time adaptation > feature refinement > task-specific fine-tuning.
  • HPC cluster GPU access via Slurm job attachment (srun –jobid) more stable than direct SSH because leverages cluster’s resource management layer. Job IDs persist across sessions enabling reattachment. GPU availability should be judged by memory headroom (50%+ free), not idle status.
  • Explore agents excel in large project organization tasks: can recursively traverse directories, read file headers, count code lines, compare documents for redundancy - order of magnitude more efficient than manual exploration.

Domain Knowledge (domain)#

  • Error recovery benchmarks require 4-pillar validation: (1) large-scale diverse data (2000+ scenes), (2) theoretically-grounded taxonomy (FMEA/ISO standards), (3) baseline performance characterization (4+ policies), (4) demonstration of training improvement on foundation models (15-30% SR gain). This distinguishes publishable benchmarks from toy datasets.
  • VLA cross-task generalization fails when task domains differ (LIBERO vs PickPlace). Finetuning requires 500+ demos expandable via MimicGen from 10 source demos. Baseline model selection critically affects experimental validity - domain-mismatched checkpoints may underperform due to distribution shift rather than dataset quality.
  • Natural error capture (natural_capture mode) more efficient than injection mode: 50 rollouts generated 150 scenes (~3 per rollout) vs injection mode 10 rollouts generating only 3 scenes (~0.3 per rollout). Natural failure modes are more diverse and realistic.
  • Phase imbalance in error scene generation stems from success rate distribution creating ‘reachability problem’ - need policies with 60-70% SR (BC-RNN sweet spot) to reach late phases while still producing failures. Random policy fails too early, Pi0 has 0% SR.
