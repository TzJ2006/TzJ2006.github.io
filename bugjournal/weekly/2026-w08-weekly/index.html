Learnings#

Architecture (architecture)#

  • VLA end-to-end integration requires explicit camera observation handling: need include_images=True in state_extractor.extract() and camera_names specification for all camera-dependent policies. Policy observation format correctness is critical.
  • Embedding dimension bottlenecks cause catastrophic model collapse: STAIG’s 64-dim compresses 1536-dim vision features by 96%, producing near-indistinguishable embeddings (std=0.12). Spatial refinement then eliminates minority clusters. 256-512 dim necessary to avoid this pathology.
  • Spatial coordinate systems in medical imaging require explicit semantic naming (pxl_row=Y, pxl_col=X) and validation via physical units (pixels × scalefactor = microns). Generic pixel_x/pixel_y naming is error-prone and can halve performance.
  • Phase imbalance in error generation stems from success rate distribution creating reachability problem: need policies with 60-70% SR (BC-RNN sweet spot) to reach late phases while producing failures. Random fails too early, Pi0 has 0% SR - only BC-RNN reaches transport/place with failures.
  • Two-stage fusion (self-supervised refinement → multimodal integration) creates new experimental dimension, but refined embeddings not universally superior to raw embeddings. Optimization objective alignment critical: self-supervised clustering optimizes for separability, which may not align with multimodal fusion’s complementarity objectives.
  • Cache variants (standard/freq/staig_strict) prevent collisions when same encoder produces different embeddings based on preprocessing flags. More robust than separate encoder names. Embedding cache infrastructure eliminates redundant encoder instantiation, saving hours per run.
  • Resume logic for long-running data collection must rebuild summary state from JSONL, not just skip episodes. Correct approach: read JSONL on startup, rebuild all_episodes dict, add completed tuples to skip set, continue collection. Ensures idempotency.
  • Baseline model selection critically affects experimental validity. Using domain-mismatched checkpoints weakens usefulness claims due to distribution shift rather than dataset quality. Must explicitly validate or fine-tune baselines to ensure fair evaluation.
  • Observation pipeline bugs in multi-path codebases (injection vs natural capture) require exhaustive analysis - fix in one path doesn’t guarantee fix in all paths. Grep for all uses of the pattern.
  • Dependency topology diagrams should precede execution plans. Clear visualization of M14 as critical path blocker shaped resource allocation decisions and timeline estimates, enabling strategic scheduling.
  • Symlink-based checkpoint management enables version control of path references while preserving original data locations. Critical for shared filesystem collaboration.

Domain Knowledge (domain)#

  • Pure vision encoders cluster by morphological similarity not biological domains: UNI2 groups blood vessels across different cortical layers because they look similar. Need gene expression to distinguish functional regions with similar visual appearance.
  • Error recovery benchmarks require 4-pillar validation: (1) large-scale diverse data (2000+ scenes), (2) theoretically-grounded taxonomy (FMEA/ISO standards), (3) baseline performance characterization, (4) demonstration of training improvement (15-30% SR gain). This proves usefulness beyond infrastructure.
  • Natural error capture more efficient than injection mode: 50 rollouts generated 150 scenes (~3 per rollout), while injection mode 10 rollouts generated only 3 scenes (~0.3 per rollout). Natural failure modes more diverse and realistic.
  • VLA cross-task generalization fails when task domains differ (LIBERO vs PickPlace): Pi0_libero achieves 96-100% on Coffee/Stack but 0% on PickPlace. Domain-specific finetuning essential - requires 500+ demos expandable via MimicGen from 10 source demos.
  • Cross-domain literature search reveals universal solution patterns: image-only clustering problem in spatial transcriptomics identical to fine-grained bird classification, remote sensing, microscopy. All converged on same hierarchy: domain-specific pretraining > test-time adaptation > feature refinement > task-specific fine-tuning.
  • FMEA (Failure Mode and Effects Analysis) from industrial safety standards (ISO 15066/10218, IEC 61508) provides rigorous theoretical grounding for robot error taxonomies. Academic taxonomy validation requires both theoretical grounding and empirical evidence (Cohen’s/Fleiss’ kappa: within-category κ>0.6, cross-category κ<0.3).
  • Evaluation paradigm fundamentally changes what is measured: same task (tissue type identification) evaluated as supervised classification (tests linear separability) vs unsupervised clustering (tests natural cluster structure) measures different aspects of embedding quality. High KNN accuracy doesn’t guarantee good clustering ARI.
  • VLA model landscape for error recovery: 8 models claim recovery capabilities but only 2 open-source (RoboFAC, VLAC). FailSafe and VLA-SCT most practical for integration (training-free, plug-in architecture, validated on π0/OpenVLA). Recovery methods fall into 3 categories: dual-system (fast/slow), proactive (subtask backtracking), reactive (supervisor-guided correction).

Tools (tools)#

  • When integrating third-party ML libraries (openpi, robomimic), always check actual API signatures via inspect.signature() and read example code in source repo. Documentation often incomplete or outdated. Use high-level encapsulation functions instead of manual low-level APIs.
  • HPC cluster GPU access via Slurm job attachment (srun –jobid) more stable than direct SSH because leverages cluster’s resource management layer designed for long-running workloads. Job IDs persist across sessions enabling reattachment.
  • In HPC clusters, GPU availability should be judged by memory headroom (50%+ free), not idle status. Multiple jobs can coexist on same node if memory allows, maximizing cluster utilization.
  • Systematic documentation audit via /init skill highly effective at catching long-standing gaps accumulated during iterative development, especially for multi-tool projects where each tool documented independently.
  • Explore agents excel in large project organization tasks: recursively traverse directories, read file headers for quick content identification, count code lines, compare documents for redundancy - order of magnitude more efficient than manual exploration.
  • Windows OpenSSH does not support Unix domain sockets (ControlMaster/ControlPath), causing silent failures with cryptic errors (‘Bad file descriptor’, ’nonexistent pipe’). Solution: disable connection multiplexing options. Cross-platform compatibility issues often manifest as low-level errors that look like bugs.

Debugging (debugging)#

  • ‘Tests passing’ ≠ ‘system operational’ - integration testing with real dependencies (model checkpoints, external APIs) essential before claiming completion. Synthetic unit tests validate code correctness but not end-to-end functionality.
