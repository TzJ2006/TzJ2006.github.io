Problems and Solutions#

1. per-section PCA/STAIG produces non-comparable embedding spaces; SL@50=0 across 10 of 14 cross-section retrieval combinations; normalization cannot fix this [MIHD Spatial Transcriptomics] (2026-03-16)#

Solution: Switch to a pretrained Foundation Model (scGPT): all sections share the same model weights, so outputs naturally exist in the same coordinate space. SL@50 improved from 0.013 to 0.416, with 100% hit rate. The root cause was coordinate space incompatibility (mathematically unsolvable by normalization), not insufficient gene feature information.

2. CompositeBodyObject falls apart on env.reset() (lid floating, base offset), but replays normally in HDF5 (set_sim_state_flat overwrites body positions and masks the issue) [Error Recovery Benchmark] (2026-03-16)#

Solution: Added locations_relative_to_corner instance attribute storage in generated_objects.py init; restored the corner-to-center coordinate transform logic (aligned with commit 398af01b); all 139 unit tests passing.

3. Semantic Scholar incorrectly merges papers from researchers with the same name; prominent professors like Pieter Abbeel are matched to a namesake with h=4, severely distorting h-index statistics [Research Profiler] (2026-03-16)#

Solution: Refactored disambiguation weights (quantitative metrics override string similarity) + added three-tier disambiguation chain (exact ID → paper reverse lookup → name search) + –author-id CLI parameter; on rate limiting, use WebSearch to find the correct authorId and specify it directly.

4. vla_infer.py does not clear current_prompt after task completion; in the next loop, chunk_count is still ≥5 and model score is still high, immediately re-triggering the done message, causing infinite loop reporting completion [robobrain_pi] (2026-03-17)#

Solution: Added current_prompt=’’ and _publish_state(‘idle’) at the end of the task completion handler block. State machine completion must simultaneously reset the trigger condition and broadcast the state change.

5. π₀.₅ Pi0WithCompletionHead using composition causes all parent module parameter paths to gain a pi0/ prefix; pytree merge throws ‘2 children vs 1 child’ ValueError, making checkpoints completely unloadable [π₀.₅] (2026-03-18)#

Solution: Switch to inheritance (class Pi0WithCompletionHead(Pi0)); Pi0 parameters align directly with checkpoint paths at the top level. In Flax NNX, inheritance is the only approach that maintains checkpoint path compatibility.

6. RobotwinOutputs.call only returns {actions} field, silently discarding manip_progress; even when the model correctly predicts the progress field, it is completely inaccessible externally, and eval txt files are always empty [π₀.₅] (2026-03-18)#

Solution: Modified output transform’s return dict to include the manip_progress field, restoring data flow. When diagnosing silent-filter-type bugs, trace from the end of the data flow back upstream.

7. CalendarPro returns ‘I’m not sure’ instead of processing requests when users batch-report task statuses [CalendarPro] (2026-03-18)#

Solution: Analyzed the complete root cause chain: low embedding similarity → LLM classifier has no such intent → GENERAL handler discards AI reply. Added BATCH_UPDATE support across all three layers (routing / classification / handling); keyword count rule compensates for embedding blind spots.

8. MIHD all_aligned mode: –alignment parameter not injected into evaluate phase; EvaluationJob.alignment is always None; Harmony alignment results completely ignored; alignment functionality actually non-functional [MIHD Spatial Transcriptomics] (2026-03-18)#

Solution: Added 3 lines of code before the evaluate phase in run_pipeline.py: when phase==‘all_aligned’ and args.alignment has a value, inject alignment into each experiment’s extra_config.

9. drop_in_transit_D0 generates very few valid scenes; min_hold_height=0.85 is too low, causing objects on the table (z≈0.88) to be misclassified as held in mid-air, with 80% false positives [Error Recovery Benchmark] (2026-03-20)#

Solution: Raised threshold to 0.93 (table height 0.80 + Milk object height 0.08 + margin 0.05); false positives eliminated; D0 success rate improved from 10% to 50%.

10. python summarize/daily_summary.py hangs indefinitely at the @ccusage/codex step with no output and no errors [gadget] (2026-03-21)#

Solution: subprocess.run(capture_output=True) redirects stdin to DEVNULL, causing npx’s first-time install confirmation prompt to wait indefinitely. Added –yes flag to all 3 npx call sites to skip interactive confirmation.

11. In multi-object tasks like threading/pick_place, phase detection fails systematically; all frames in threading are labeled pre_reach, causing 12 of 13 skills to find no opportunities [Error Recovery Benchmark] (2026-03-21)#

Solution: Added InteractionSegmenter to segment trajectories by object interaction (EEF proximity + gripper state + co-motion detection), explicitly assigning target_object and phase per segment. Completely bypasses single-object phase detection — circumvents the defective abstraction rather than patching it. threading subtypes 3→25.

12. Panda robot gripper action polarity is opposite to code assumptions: Panda requires action=+1 to close; the hardcoded -1 actually opens the gripper, causing all injections for coffee/stack and other tasks to fail [Error Recovery Benchmark] (2026-03-22)#

Solution: Detect polarity in EnvWrapper.init based on robot arm type; set close_action=+1.0 for PandaGripper and -1.0 for RethinkGripper. Added a helper and replaced all hardcoded action[-1] instances.

13. drop_with_interaction generates 0 scenes across all tasks: the original implementation only applies a 1–3cm offset before releasing, but objects are 10–20cm apart and will never contact neighbors [Error Recovery Benchmark] (2026-03-22)#

Solution: Rewrote inject logic: actively carry the object to directly above the non-target (+0.15m), then release the gripper to let the object free-fall, tracking object-object contact throughout the settle process.

Key Learnings#

Architecture#

  • Pipeline contract alignment must trace the complete data flow: it is not enough to modify processing logic — data must also be persisted in the storage layer (e.g., writing segment_interactions() results to NPZ). The output transform is an implicit filter in the inference chain; any field not in the return dict is silently discarded. CLI parameters in multi-stage pipelines must be explicitly passed to each downstream stage — implicit sharing cannot be relied upon. “Unit tests passing ≠ E2E correctness.”
  • In multi-object robot manipulation tasks, target_object must be a first-class citizen propagated through all three stages: detector / injector / validator. objects[0] is the first object by dictionary insertion order — its semantics are completely wrong in multi-object scenarios and will not raise errors, causing all object-state-based judgments to fail systematically. Graspable objects and fixtures must be distinguished via grasp_geoms, not by pure distance/position.
  • Per-section independent processing (PCA/training) produces non-comparable embedding spaces — this is an architecture-level fundamental limitation that cannot be fixed by normalization or post-processing. The only correct zero-shot solution is a pretrained Foundation Model (shared weights). Gene FMs (scGPT 100%) significantly outperform Vision FMs (UNI2 71%) because H&E images have cross-sample batch effects while gene expression has stronger cross-sample consistency.
  • Extending trained models in Flax NNX: inheritance (class Child(Parent)) is the only approach that maintains checkpoint path compatibility. Composition (self.pi0 = Pi0(…)) prepends an extra prefix to all parent module parameter paths, causing pytree merge to fail completely. Existing framework freezing mechanisms (freeze_filter + nnx.DiffState) typically account for extensibility — prefer reusing them over building new ones.
  • When facing an architecture-level design flaw, “bypassing” is more fundamental than “patching”: phase detection’s single-object assumption is a design flaw that cannot be fixed by tuning parameters. InteractionSegmenter directly bypasses it rather than applying patches — this is the more elegant solution. Don’t patch the wrong abstraction; bypass it.

Debugging#

  • State machine design principle: completion events must do two things simultaneously — reset the trigger condition (clear current_prompt) and broadcast the state change (_publish_state(‘idle’)). Doing only one leaves hidden bugs: either an infinite loop or invisible upstream state. None checks must come before any attribute access (defensive programming).
  • When diagnosing unknown third-party library behavior, writing a minimal empirical test script is more reliable and efficient than static code analysis (the robosuite gripper polarity case). Log Counter statistics are more effective than reading logs line by line for locating systemic bugs (1,698 occurrences of ‘gripper not closed’ directly pointed to the target_object ambiguity root cause). When diagnosing implicit-filter-type bugs, trace from the end of the data flow back upstream.
  • Experiment naming conventions must strictly correspond to actual implementations. The naming inconsistency where ‘pca_uni2_staig_fusion’ actually used UNI caused long-term misunderstanding — actual configuration must be confirmed by reading logs, not just experiment names. Data quality issues (unbalanced distribution, all-pre_reach phase labels) should have monitoring mechanisms built in at the pipeline design stage.

Tools#

  • subprocess.run(capture_output=True) redirects stdin to DEVNULL; CLI tools that may have interactive prompts (npx, etc.) must receive –yes/-y, or they will hang indefinitely rather than timing out. capture_output=True is a highly stealthy hang root cause.
  • Slurm HPC cluster’s pam_slurm_adopt policy forcibly kills all associated processes when a job ends; SSH nohup cannot circumvent this. Long-running training jobs must be submitted as formal jobs via sbatch. MuJoCo physics simulation (enable_camera=False) is a pure CPU task; MUJOCO_GL=disabled can fully bypass OpenGL, and worker count should precisely match –cpus-per-task.
  • JAX has_aux mechanism: nnx.value_and_grad supports has_aux=True; compute_loss returns (loss_array, aux_dict), gradients are computed only for loss_array, and aux_dict transparently carries monitoring metrics. stop_gradient strictly constrains aux loss to MLP parameters; a large numerical disparity between action/aux loss values does not indicate training imbalance.
  • Hugo best practices: .Title is the most stable field for identifying specific sections (.Section and .RelPermalink have inconsistent casing across platforms). Static directory names identical to content page base names cause path conflicts. Auto-generated frontmatter should not use the current system time as the date. Theme partials can be safely overridden via project-level layouts/partials/ without forking the theme.
  • Spatial data nearest-neighbor queries: large-scale point clouds (>10K points) should prefer KD-trees (O(N log N) preprocessing) over cdist (O(N²) memory). In HD spatial transcriptomics (17K+ cells), the difference can be OOM vs. normal execution. Single Source of Truth (SSOT) principle: constants referenced across multiple files must be imported from a single canonical definition.

Domain Knowledge#

  • Recovery Behavior Group (RBG) grouping strategy: cluster 29 error subtypes by motor primitive into 5 groups; within-group demos can be cross-augmented — MimicGen generates 1,000+ samples from 10 source demos. This compresses 1,740 naive requirements to 329 (81% reduction). This is a paradigm broadly applicable to robot recovery data engineering.
  • π₀.₅ computation graph characteristics: inference is naturally two-step (VLM prefix forward → action expert denoising); during training, Exp1–4 can use teacher forcing to merge into a single joint forward pass. The from_hidden mode introduces a circular dependency (conditioning depends on model output), requiring an extra prefix-only forward. Mean pooling of prefix_output is more suitable for classification tasks than action expert’s suffix_output (encodes global state understanding).
  • LLM’s capacity for deep understanding of academic trajectories exceeded expectations: it can spontaneously identify high-level features like ‘infrastructure-minded researcher’ and ’technical prerequisites for paradigm shifts.’ The why_not_before field (attributing the historical prerequisites of each breakthrough across three dimensions: data / compute / insight) is the highest-value field in researcher profiles. Conference award recognition has extremely low ROI under current LLM capabilities and should be replaced with an external data source solution (exact title-matching JSON database).

AI Usage Notes#

Effective patterns:

  • ✓ Parallel multi-agent code review (three dimensions — reuse / quality / efficiency — independently analyzing the same diff): each dimension finds completely non-overlapping issue sets; parallel execution saves time
  • ✓ Systematic code tracing to locate root causes: tracing from the end of the data flow back upstream successfully located the output transform silent-discard bug, S2 rate-limit recursive infinite loop, and others
  • ✓ Proactively exceeding task boundaries for quality checks: in researcher profile analysis, proactively identified name-collision contamination (abnormal h-index to paper count ratios, impossible domain spans, etc.) without being explicitly prompted
  • ✓ Academic trajectory analysis reached academic commentary level: spontaneously identified high-level features like ‘infrastructure-minded researcher’; the why_not_before field delivered unexpectedly high value
  • ✓ Log aggregation statistics to locate systemic bugs: Counter statistics of 1,698 ‘gripper not closed’ occurrences directly pointed to the target_object ambiguity root cause, far more efficient than line-by-line analysis

Limitations:

  • ✗ When facing complex system problems, tends to locate isolated bugs rather than analyzing contract inconsistencies from a full data-flow perspective; requires humans to guide from the architectural level
  • ✗ Lacks proactive global data-flow verification after completing local fix plans; passing unit tests creates a false sense of “done”; multiple omissions (e05/e06/e09, target_pos propagation, collect script not integrated) required systematic human review to discover
  • ✗ Failure to pre-validate environment configuration leads to repeated full-batch failures (agent subprocesses lacking Bash permissions, conda activation failures, missing PYTHONPATH, wrong API key type); should validate with a single task before scaling to batch
  • ✗ Failed to proactively identify efficiency bottlenecks: when serial scene generation was slow, only waited rather than suggesting parallelization; when launching via SSH nohup, did not account for Slurm job expiration resource isolation
  • ✗ Insufficient prediction of framework-specific behavior: did not anticipate the impact of Flax NNX inheritance vs. composition on checkpoint paths; did not anticipate that JAX/NNX TrainConfig mechanism could be reused directly
  • ✗ Unaware of latest tool features (was told about effortLevel: max rather than knowing it); knowledge lag exists for rapidly-evolving AI tooling ecosystems
