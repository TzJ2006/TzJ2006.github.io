Weekly Report — 2026-W12 (2026-03-16 to 2026-03-22)#

This week, we conducted extensive research and development on two major directions: robot operations and spatial transcriptomics. The Error Recovery Benchmark was completed from design of the collection scheme (RBG grouping with 329 demo budgets) to architectural-level trajectory segmentation reconstruction using InteractionSegmenter. This week’s key breakthroughs focused on “finding and solving architectural-level root causes”: issues such as non-comparability in per-section PCA coordinate systems, ambiguity in multi-object target_object, and Flax NNX inheritance vs combination were fundamentally resolved this week.

Issues and Solutions#

1. per-section PCA/STAIG generated non-compatable embedding spaces, resulting in 10 out of 14 combinations having SL@50=0, and normalize failed to fix [MIHD Space Transcriptomics] (2026-03-16)#

Solution: Switch to a pre-trained Foundation Model (scGPT): all slices share the same model weights, so the output is naturally in the same coordinate system. SL@50 increased from 0.013 to 0.416, with a 100% hit rate. The root cause was inconsistent coordinate systems (mathematically, normalize could not be resolved), rather than insufficient gene feature information.

2. CompositeBodyObject scattered during env.reset() (lid floating, base offset), but normal when replayed via HDF5 (set_sim_state_flat covered the body position to mask the issue) [Error Recovery Benchmark] (2026-03-16)#

Solution: Added a locations_relative_to_corner instance attribute in init of generated_objects.py to restore the coordinate transformation logic from the corner to the center (aligned with commit 398af01b). All 139 unit tests passed.

3. Semantic Scholar mistakenly combined papers by multiple researchers with the same name, and famous professors like Pieter Abbeel were matched to the same person with h=4, causing serious distortion in h-index statistics [Research Profiler] (2026-03-16)#

Solution: Reconstructed the disambiguation weights (quantitative indicators overpower string similarity), added a three-level parsing chain (exact ID→paper reverse search→name search), and –author-id CLI parameters. During speed limits, the correct authorId was found through WebSearch and directly specified.

4. vla_infer.py did not clear the current_prompt after the task was completed. If the chunk_count in the next loop was still ≥5 and the model score was high, it would immediately trigger a done message, causing an infinite loop reporting completion [robobrain_pi] (2026-03-17)#

Solution: Added current_prompt=’’ and publish_state(‘idle’) at the end of the task completion processing block. The state machine must reset both the triggering conditions and broadcast the state change upon completion.

5. In π₀.₅ Pi0WithCompletionHead, using a combination method caused all parent module parameter paths to have a pi0 prefix. pytree merge threw a ‘2 children vs 1 child’ ValueError, and checkpoints could not be loaded at all [π₀.₅] (2026-03-18)#

Solution: Changed to inheritance (class Pi0WithCompletionHead(Pi0)), and Pi0 parameters were directly aligned with the checkpoint path at the top level. Inheritance in Flax NNX was the only way to maintain compatibility with checkpoints.

6. RobotwinOutputs.call only returned the {actions} field, silently discarding manip_progress. Even if the model correctly predicted the progress field, it could never be obtained externally, and the eval txt file was always empty [π₀.₅] (2026-03-18)#

Solution: Modified the output transform’s return dict to include the manip_progress field. The data flow was reconnected. Such bugs should be traced back from the end of the data flow.

7. When CalendarPro users reported task status, the system returned ‘I’m not sure’ instead of processing the request [CalendarPro] (2026-03-18)#

Solution: Analyzed the entire root cause chain: low embedding similarity → LLM classifier failed to classify this intent → GENERAL handler discarded an AI response. Added BATCH_UPDATE support in the routing/classification/processing triple-chain, and keyword counting rules compensated for the embedding blind spot.

Learning Outcomes#

Architecture#

  • Pipeline contracts must track the entire data flow: not only should the processing logic be modified, but the data must also be persisted at the storage level (e.g., results of segment_interactions() are written to NPZ). The output transform is an implicit filter in the reasoning chain, and any fields that don’t appear in return dict are silently discarded. In a multi-stage pipeline, CLI parameters must be explicitly passed to each downstream stage, not relying on implicit sharing. “Passing unit tests doesn’t mean E2E correctness”
  • In multi-object robot operations tasks, target_object must be transmitted through the detector/injector/validator stages consistently. Objects[0] is the first object in dictionary insertion order. This approach leads to systematic errors when the semantics of multi-object scenarios are completely incorrect and no errors are reported. Graspable objects and fixtures must be distinguished by the grasp_geoms field, and targets cannot be selected based solely on distance/position.
  • Independent processing per section (PCA/training) results in non-comparable embedding spaces, which is a fundamental limitation at the architecture level. This cannot be fixed through normalization or post-processing. The only correct zero-shot solution is to pre-train a Foundation Model (shared weights). Gene FM (scGPT 100%) outperforms Vision FM (UNI2 71%) because gene expression has stronger cross-sample consistency due to the cross-sample effect in H&E images.
  • Flax NNX extended models: Inheriting (class Child(Parent)) is the only way to maintain compatibility with checkpoint paths. Combining (self.pi0 = Pi0(…) adds a prefix before all parent module parameter paths, causing pytree merge to fail completely. The existing framework’s freezing mechanism (freeze_filter+nnx.DiffState) usually takes scalability into account and should be reused rather than creating new ones.
  • When facing architectural design flaws, “bypassing” is more fundamental than “patching”: The assumption of phase detection for single objects is a design flaw that cannot be fixed through parameter tuning. InteractionSegmenter bypasses it directly instead of patching it, which is a more elegant solution. Instead of patching, we bypass the wrong abstraction layer.

Debugging#

  • Principles of state machine design: Completing an event must involve two actions simultaneously—resetting the trigger conditions (clearing current_prompt) and broadcasting state changes (_publish_state(‘idle’)). Doing either one alone will lead to infinite loops or implicit bugs where upstream perception fails. The None check must occur before any property access (defensive programming).
  • When diagnosing behavior of unknown third-party libraries, writing a minimal test script is more reliable and efficient than static code analysis (e.g., in the case of npx gripper polarity). Logging Counter statistics is more effective at identifying systematic bugs than analyzing each bug individually (1698 occurrences of ‘gripper not closed’ directly pointing to target_object ambiguity). To troubleshoot implicit filtering class bugs, track back from the end of the data flow.
  • Experimental naming conventions must strictly correspond to the actual implementation. Using inconsistent names like “pca_uni2_staig_fusion” can lead to long-term misunderstandings. It’s necessary to confirm the actual configuration by reading logs rather than just looking at the experimental name. Quality issues with benchmark data (uneven distribution, all phase annotations are pre_reach) should be monitored during pipeline design.

Tools#

  • subprocess.run(capture_output=True) redirects stdin to DEVNULL. When calling CLI tools that may have interactive prompts (npx), it’s necessary to specify –yes/-y, otherwise the process will hang indefinitely instead of timing out. capture_output=True is the root cause of high-secrets hanging.
  • The pam_slurm_adopt policy in Slurm HPC clusters forces killing all related processes when a job ends. SSH nohup cannot bypass this. Long-training tasks must be submitted as official jobs using sbatch. MuJoCo physics simulation (enable_camera=False) is a pure CPU task. MUJOCO_GL=disabled can completely bypass OpenGL. The number of workers should exactly match –cpus-per-task.
  • JAX has_aux mechanism: nnx.value_and_grad supports has_aux=True. After compute_loss returns(loss_array, aux_dict), the gradient is calculated only for loss_array. Aux_dict transparently carries monitoring indicators. stop_gradient restricts aux loss to MLP parameters strictly. The ratio of action/aux loss values doesn’t indicate training imbalance.
  • Hugo best practices: .Title is the most stable field for identifying specific sections (the case sensitivity of .Section and .RelPermalink differs across platforms). Using the same static directory name as the basic name of the content page can cause path conflicts. Automatic frontmatter generation shouldn’t use the current system time as the date. The theme partial can be safely overridden through project-level layouts/partials without having to fork the theme.
  • Spatial data nearest neighbor query: For large point clouds (>10K points), KD trees should be used first (O(N log N) preprocessing), rather than cdist (O(N²) memory). In HD spatial transcriptomics scenarios with 17K+ cells, differences can reach OOM instead of normal operation. The principle of single truth source (SSOT): Constant sets that need to be referenced across multiple files must be imported from the only definable place.

Domain Knowledge#

  • Recovery Behavior Group (RBG) grouping strategy: Cluster 29 error subtypes into 5 groups based on motor primitives. Demos within the same group can be cross-enhanced. MimicGen generates over 1000 demos from 10 source demos. Compress 1740 simple requirements into 329 requirements (saving 81%). This is a paradigm that can be widely applied to robot recovery data engineering.
  • π₀.₅ calculation graph characteristics: Reasoning naturally involves two steps (VLM prefix forward → action expert denoising). During training, Exp1-4 can be combined into one joint forward using teacher forcing. The from_hidden mode introduces circular dependencies due to conditioning dependent on model output. An additional prefix-only forward is necessary. The mean pooling of prefix_output is more suitable for classification tasks than the suffix_output of action expert (encoding global state understanding).
  • LLMs have a deeper understanding of academic trajectory analysis than expected: They can automatically identify higher-order features such as “infrastructure thinking researchers” and “technical prerequisites for paradigm shifts”. The why_not_before field (the historical prerequisites for each breakthrough in terms of data/computing power/insight) is the most valuable field in researcher profiling. Identifying conference awards has very low ROI under current LLM capabilities. It should be replaced by an external data source solution (exact title matching JSON library).

Notes on AI Usage#

Effective Modes:

  • ✓ Parallel multi-agent code review: Analyze the same diff from three dimensions of reuse/quality/efficiency independently. This reveals completely overlapping problems across three dimensions, and parallel execution saves time.
  • ✓ Systematically track the root cause of code: Track back from the end of the data flow to identify bugs in the output transform that silently discard bugs, S2 rate-limiting recursive loops, etc.
  • ✓ Proactive quality checks beyond task boundaries: Actively identify confusion due to same names in researcher profiling (abnormal ratios between h-index and number of papers, impossible domain spans), which are not explicitly required in prompts.
  • ✓ Academic trajectory analysis reaches an academic review level: Automatically identifies higher-order features such as “infrastructure thinking researchers”. The why_not_before field has more value than expected.
  • ✓ Log aggregation statistics to identify systematic bugs: Counter statistics show that 1698 occurrences of ‘gripper not closed’ directly point to target_object ambiguity root causes. This is more efficient than analyzing each bug individually.

Limitations:

  • ✗ When dealing with complex system problems, we tend to focus on identifying isolated bugs. We lack the ability to analyze contract inconsistencies from a full data flow perspective and need human guidance at the architectural level.
  • ✗ After completing local repair plans, we lack proactive global data flow verification. Passing unit tests gives us a false sense of completion. Many omissions (e05/e06/e09, target_pos transmission, collect script not integrated) need to be identified through systematic reviews by humans.
  • ✗ Environmental configurations have not been pre-verified, resulting in multiple full failures for batch tasks (Agent sub-proxies lack Bash permissions, conda activation fails, PYTHONPATH is missing, API key type). We should verify them individually before expanding into batches.
