Problems & Solutions#

1. Per-section independent PCA/STAIG produces incomparable embedding spaces, causing cross-section retrieval to fail (14-29% hit rate) [MIHD Spatial Transcriptomics] (2026-03-16)#

Solution: Use Foundation Models (scGPT Gene FM, UNI2 Vision FM) that apply same pretrained weights to all sections, producing naturally comparable embeddings. scGPT achieves 100% hit rate with avg SL@50=0.416.

2. Post-fusion Harmony alignment improved batch mixing metrics but degraded cross-sample niche retrieval quality [MIHD Spatial Transcriptomics] (2026-03-19)#

Solution: Analyzed results and concluded Harmony’s linear correction cannot handle non-linear dimension meaning misalignment from per-section independent training. Recommended Joint STAIG multi-section training for next iteration.

3. Scholar database name disambiguation failure affects ~30% of analyzed researchers, merging multiple people into single profiles [Research Profiler System] (2026-03-16)#

Solution: Added explicit warning mechanism with ⚠️ markers in trajectory_summary flagging h-index/citation discrepancies; recommended pre-processing step clustering papers by semantic similarity, institutional affiliation, co-author network.

4. Coffee machine model appeared disassembled in simulation during env.reset(), but HDF5 replay showed correct assembly [Error Recovery Benchmark] (2026-03-16)#

Solution: Traced MimicGen commit 398af01b to discover missing locations_relative_to_corner coordinate transformation. Restored code blocks converting corner-relative to center-relative positions in _append_object().

5. Collecting error recovery demos for 6 tasks × 29 subtypes requires 1,740 samples which is prohibitively expensive [Error Recovery Benchmark] (2026-03-16)#

Solution: Group 29 subtypes into 5 Recovery Behavior Groups (RBGs) based on motion primitives; collect 5-10 demos per RBG per task; use MimicGen/IntervenGen augmentation to generate 100+ samples per subtype. Reduced to 329 demos (81% savings).

6. MimicGen augmentation failing: pos_delta computed but never applied to action warping, causing <5% success rate [Error Recovery Benchmark] (2026-03-19)#

Solution: Fixed _replay_with_warping to apply pos_delta to warped_actions[:, :3] before adding noise, enabling proper object-centric trajectory transformation. Expected success rate: 50%+ for D0 translation errors.

7. QCVLA DAgger Round 2 only improved +1pp (60%→61% SR), indicating saturation [QCVLA Latent Bridge] (2026-03-17)#

Solution: Proposed perturbation & bridging data augmentation: during DAgger rollout, randomly inject short coherent action sequences followed by reverse recovery actions. Captures large-magnitude deviations bridge would never naturally produce.

8. VLA inference task completed but infinite loop sending done messages because current_prompt was not reset [VLA/Robotics Infrastructure] (2026-03-17)#

Solution: Set current_prompt = "" after task completion and call _publish_state(“idle”); moved None check before task_completed.item() to avoid crashes on non-pi05 models.

9. MIHD NEEDS_COORDS_FUSIONS tuple duplicated across files, with adaln_attention missing from both [MIHD Spatial Transcriptomics] (2026-03-17)#

Solution: Centralized NEEDS_COORDS_FUSIONS as constant in models/Fusion.py, imported by both files. Added missing adaln_attention to tuple, fixing bug where this fusion would fail to receive spatial coordinates.

10. MIHD refine_labels_spatial_majority used O(N²) cdist causing OOM on Visium HD data with 17K+ spots [MIHD Spatial Transcriptomics] (2026-03-17)#

Solution: Replaced scipy.spatial.distance.cdist with scipy.spatial.cKDTree.query which has O(N*k) complexity for k-nearest neighbor search. Changed from full distance matrix to on-demand radius neighbor queries.

11. VLA training combined action_loss and task_loss into single value logged to wandb, unable to monitor each separately [VLA/Robotics Infrastructure] (2026-03-17)#

Solution: Modified compute_loss to return (combined_loss, {“action_loss”: …, “task_loss”: …}) tuple; training loop uses nnx.value_and_grad(…, has_aux=True) to unpack auxiliary metrics and add to info dict.

12. CalendarPro could only handle single intent per message, losing subsequent actions [Life Copilot] (2026-03-19)#

Solution: Replaced single-intent dual-verify flow with Claude CLI MultiIntentAnalyzer returning list of intents. Updated _process_message_queue to loop through intents and combine responses. Removed 7-layer keyword override system.

Learnings#

Architecture (architecture)#

  • Foundation Models’ key advantage for cross-sample tasks: provide shared embedding space without needing joint training; pretrained weights guarantee comparability across arbitrary inputs. Zero-shot cross-section retrieval fundamentally requires this property—no amount of per-sample optimization can substitute.
  • Batch correction methods like Harmony provide global distribution mixing but cannot preserve semantic alignment when independent training produces embeddings with misaligned dimension meanings. This requires shared training (Joint STAIG) or foundation models. Batch mixing and semantic alignment are orthogonal properties.
  • Single-intent classification is architectural constraint that cannot be fixed with keyword hacks. Multi-intent LLM analysis eliminates need for all heuristics. CalendarPro’s 7-layer keyword override system was treating symptoms of broken abstraction.
  • Git vs rclone responsibility boundary must be clearly defined: git manages reproducible public code and configuration, rclone manages personal data and runtime outputs. Mixing causes repo bloat (94MB personal data), privacy risks, and collaboration obstacles.
  • Multi-layered intent classification requires coordinated updates across 8+ files: enum, semantic routes, keyword scorer, LLM prompts, context assembly, dual verify defaults, handler + routing, plus derived data. Missing any layer causes classification failure.
  • Bootstrap migration strategy for source-of-truth cutover: when transitioning from scattered deploy targets to centralized staging, first-run detection with automatic content relocation prevents data loss and eliminates manual pre-migration burden.
  • Distributed collaboration state synchronization design principle: must sync not only input data (logs), but also intermediate outputs (reports), otherwise logic depending on output state (like finalized checks) will fail across different devices.
  • State machines require explicit reset of all related variables at each termination node; implicit assumptions that ’next loop will naturally change’ easily lead to unpredictable behaviors like infinite loops.
  • Spatial queries on large point clouds (10K+ spots) must use tree-based structures. O(N²) distance matrices cause OOM. cKDTree with O(N*k) neighbor queries scales to HD spatial omics data.
  • JAX/Flax’s value_and_grad(…, has_aux=True) elegantly returns additional monitoring metrics from loss function without affecting gradient computation—the standard pattern for multi-metric monitoring.
  • Hot path optimization golden rule: read path assumes resources exist, write path creates resources. DiskCache mkdir moved from read (_path_for called frequently) to write (put() called rarely) for 10x efficiency gain.

Tools (tools)#

  • Name disambiguation is #1 data quality issue in automated scholar profiling (affects ~30%). Requires multi-signal approach: h-index vs field coherence, citation temporal distribution, institutional metadata, co-author network analysis.
  • Ultra-large-scale text analysis engineering paradigm: ①automatic segmentation (150K chars ~15-20 papers optimal granularity) ②semantic merge (LLM-driven merge not concatenation) ③dynamic timeout (linearly scaled based on prompt length, 300s baseline+120s/50K chars) ④caching mechanism (avoid retry cost).
  • subprocess.run with capture_output=True has no stdin - any interactive prompts will hang indefinitely; always use –yes/–non-interactive flags for package managers and CLI tools in automation.
  • Cross-project technical transfer requires distinguishing ‘directly applicable’ from ‘conceptually adaptable’. FLARE→QCVLA transfer was mostly conceptual (perturbation thinking, expert mixing) rather than code-level.
  • Parallel agent exploration (3 agents for intent routing, task handling, general protection) is more efficient than sequential file reading for complex diagnostics. Each agent provides focused perspective that synthesizes into complete understanding.
  • Background Bash tasks in Claude Code don’t inherit conda init; must use either ‘conda run -n ENV’ or absolute Python executable paths. Direct conda activate fails with CondaError in non-interactive subshells.
  • Structured output validation should be architecturally decoupled from generation—two-stage ‘generate then repair’ pattern more robust than single-shot perfect output for complex nested schemas.
  • git filter-repo is the correct tool for purging large files/sensitive data from git history (22MB→516KB repo size reduction), but requires full workflow: backup .git, run filter-repo, restore structural files, force-push, update all clones.
  • Hugo theme updates require three-layer approach: ①git repository integrity (clean corrupted resource fork files), ②dependency alignment (theme version compatible with Hugo version), ③local customization preservation (manual conflict resolution).
  • Documentation-code drift is inevitable and accumulates silently over time; periodic exhaustive line-by-line verification (using parallel Explore agents for speed) is necessary to catch numeric values, path references, and internal contradictions.
  • Link-or-copy synchronization pattern: os.link provides zero-copy atomic updates when filesystem supports hard links; automatic fallback to shutil.copytree ensures cross-platform compatibility without manual intervention.

Debugging (debugging)#

  • MimicGen-style augmentation is fundamentally about coordinate transformation, not noise injection. pos_delta (source→target object offset) must be applied to action trajectories. Without this, 10→1000 augmentation collapses due to <5% success rate.
  • Coordinate transformation bugs in composite model systems can be masked by state replay mechanisms that overwrite XML-defined positions, surfacing only during fresh initialization (env.reset).
  • Large-scale code refactoring should use parallel Agent-based multi-dimensional review (reuse/quality/efficiency) to systematically discover issues. This uncovered 9 problems including latent bugs, performance bottlenecks, maintainability issues.
  • Proactive code review > reactive fixes. Even when user only asks ‘how it’s implemented’, expand to ‘are there problems’. Proactively discovering potential bugs is more valuable than fixing them after users encounter issues.
  • Class imbalance in binary classification requires explicit gradient balancing. Low overall loss can mask poor minority class learning when 96% of samples are one class. pos_weight effectively rebalances gradient contributions.
  • DateTime field parsing order matters critically—always prioritize ISO-format fields (start_iso) over display-format strings (start_time ‘08:00’) to avoid silent parse failures that break existence checks.
  • Error injection benchmark quality depends on task phase segmentation completeness; incomplete phase recognition causes some error types to be uncoverable. Need to verify all tasks cover expected phases during opportunity map stage.
  • Process GPU usage detection requires both environment variable inheritance check AND actual device file descriptor verification to avoid false positives from shell processes.
  • Download failure diagnosis should follow data-first approach: when systematic failures occur, verify data availability (grep for expected patterns in config files) before deep-diving into code logic.
  • Dataset format conversion must read schema files (like RLDS features.json) to confirm actual field names before assuming common conventions or guessing.
  • Shell script robustness principles: ①check git state before operations rather than relying on exit codes, ②add error handling for filesystem operations, ③explicitly exempt normal failure scenarios when using set -e.

Domain Knowledge (domain)#

  • Recovery demo data augmentation three-layer strategy: scene configuration (different error scenes as start states), cross-degree (D0→D1/D2 via rotation perturbation), cross-subtype (within-RBG cross-augmentation). Combined with MimicGen trajectory segmentation, reduces data requirements 81%.
  • When DAgger-style training saturates, expand data distribution via structured perturbations (coherent action sequences + recovery) rather than per-frame noise. For robotic manipulation, temporal coherence matters—physics and inertia make sequential perturbations more realistic.
  • ‘Why not before’ 反事实推理 framework reveals breakthrough timing dependencies: technology prerequisites (GPU/pretrained models), data availability (large-scale datasets), theoretical foundations (new concepts). Academic success highly dependent on ’technology stack maturity’ and ’time window’ matching.
  • LLM-based award identification without live API integration has fundamental recall limitations—training data cannot keep pace with conference proceedings. Hybrid approach needed: LLM screens candidates → DBLP/OpenReview API verifies → human resolves conflicts.
  • Robotics field paradigm shifts 2016-2023: deep imitation learning (2016-2019) → cross-embodiment large-scale datasets (2020-2022) → vision-language-action foundation models (2023+). Each window only 2-3 years; researchers who pivot during transitions capture disproportionate impact.
  • Academic trajectory analysis multi-angle validation method: Through iterative analysis of same researcher’s different paper subsets (from 1 paper to 100 papers), can build stereoscopic cognition from micro (single representative work deep analysis) to macro (complete publication list trajectory characterization).
  • Year-stratified analysis strategy superior to either full-corpus single-pass (prompt too long/analysis shallow) or random sampling (misses temporal evolution). Balances temporal dimension completeness with paper quality.
  • Research infrastructure building pathway: Yuke Zhu’s academic trajectory reveals field breakthroughs often accompanied by infrastructure-type work (AI2-THOR simulation, robosuite benchmark, RoboTurk crowdsourcing, MineDojo dataset). For emerging research directions, prioritizing reproducible evaluation benchmarks and data collection toolchains has more strategic value.
  • Recovery Behavior Groups (RBG) abstraction: classify 29 error subtypes by required motor primitives (re-grasp, retrieve, retract, redirect, realign) rather than error taxonomy. Enables data sharing within groups—more practical than per-error-type collection.
  • Fallback templates in AI systems should preserve useful LLM-generated content rather than blindly overwriting it. Check if LLM reply is substantive (>20 chars, not template phrase) before replacing with hardcoded fallback.
  • Keyword scoring can capture patterns that embedding similarity misses: 2+ distinct completion verbs is simple heuristic for batch updates that complements semantic routing. Explicit pattern detection still valuable in neural systems.
