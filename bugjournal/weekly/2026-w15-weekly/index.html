6. Float ball collapse animation on Windows showed visual flicker, but only when direction=‘left’ (ball on right side) [TokenMonitor] (2026-04-09)#

Solution: Root cause: SetWindowPos simultaneously moves and resizes, but WebView2 rendering is async. When ball position jumps (x=96→x=0) during resize, 1-2 frame gap shows old content clipped incorrectly. Fix: Wrap SetWindowPos with ShowWindow(SW_HIDE) before and SW_SHOWNOACTIVATE after, applied only to collapse (!expanded)

7. BasicContrastive.py contrastive loss performance bottleneck: per-node Python loop over 3000-4000 spots with tensor cloning each iteration [MIHD Spatial Omics] (2026-04-08)#

Solution: Vectorized loss computation using batch logsumexp operations, masked self-similarity with fill_diagonal_(-inf), boolean masks for positive neighbor filtering

Solution: Diagnosed fundamental architecture mismatch: DSW_embedding assumes continuous segments but LIPM zero-padding creates irregular gaps breaking temporal continuity. Recommended interpolation to uniform grid or time-aware models (GRU-D, BRITS)

Debugging (debugging)#

  • Code duplication hides bugs because fixes don’t propagate. 12 Error Skill files had inconsistent frame thresholds (e08: 30 vs standard 20) that code review revealed as unintentional. Extracting shared helpers forces consistency and makes bugs visible during refactoring.
  • Node.js 24+ has built-in environment variable proxy support (–use-env-proxy flag / NODE_USE_ENV_PROXY=1), but ONLY reads uppercase HTTP_PROXY/HTTPS_PROXY/NO_PROXY, not lowercase variants. Common pitfall when shell proxy setup scripts only export lowercase. Universal fix for all Node CLI tools
  • When refactoring LLM-driven pipelines from one language to another, validation must cover: (1) static prompt templates (2) runtime context injection (3) non-text outputs (charts/images) (4) code comments appearing in prompts (5) generated templates/scaffolds (6) error recovery logic. Parallel agent validation pattern (one per subsystem) more thorough than single-pass grep
  • DateTime timezone handling nuance: date_naive() on DateTime extracts date in source timezone, not local. For remote timestamp filtering, must explicitly convert to Local timezone first with with_timezone(&chrono::Local) before extracting date. Critical for any distributed system with cross-timezone data
  • StandardScaler NaN propagation mechanism: Single corrupt row with NaN in any feature column causes StandardScaler.fit() to compute NaN statistics (mean/std), propagating to all scaled data through (x - NaN) / NaN = NaN. Always validate data for NaN/Inf before normalization operations
  • LeRobot 0.4.0 shape=(1,) feature handling design contradiction: validate_frame requires ndarray but get_hf_features_from_features maps to scalar Value. Solution: insert type conversion in episode_buffer layer between validation and encoding
  • Always cross-validate aggregated metrics with ground truth. Log parsing bugs propagate incorrect statistics; verify against direct file counts (HDF5 demo_N groups) and structured reports (generation_report.json). Sanity checks from known input constraints catch reporting bugs early.
  • Adversarial code review catches issues missed in forward implementation. Critic subagent identified in-place mutation semantics, pattern variations across callsites, and defensive check locations that main agent overlooked during initial helper method extraction.
  • Subtask fragmentation cost: Each subtask boundary is independent ‘query object pose→re-warp→5-step interpolation’, error accumulates. 6 subtasks=5 error sources, far greater than original 1
  • Continuous closure detection importance: Robot grasping may have multiple attempts, first gripper_closed > threshold may fail. Need to detect continuous closure (≥10 frames) to confirm successful grasp
  • Multi-process debugging complexity: (1) each worker needs independent env+augmenter instance (2) path resolution more error-prone (relative paths easily miss hierarchy) (3) need explicit absolute paths like demos_dir (4) robosuite WARNING needs separate suppression in worker function
  • Asyncio state machine correctness invariant: state check + transition must be atomic within same lock; any lock-external check introduces race window
  • Interpolation vs extrapolation in temporal resampling: Linear interpolation should stay within observed data bounds. Forcing data into fixed global range causes extrapolation producing invalid values. Use per-sample adaptive ranges to avoid extrapolation artifacts
  • Format compatibility encompasses three layers: file structure (NPZ vs HDF5), metadata completeness (datagen_info fields), and semantic assumptions (starting state, task coverage). A file can pass structural checks but fail semantic requirements
  • MimicGen source demos require complete datagen_info metadata (eef_pose, object_poses, target_pose, gripper_action, subtask_term_signals) and semantic assumptions (clean initial state, complete task trajectory, valid subtask transitions). Recovery demos from error states need separate augmentation pipeline
  • Homebrew formula deprecated status is critical signal - when encountering low-level dependency issues, check upstream maintenance status first rather than debugging abandoned versions
  • WebGL rendering is asynchronous relative to native window geometry changes. SetWindowPos modifies window size/position synchronously, but Chromium needs 1-2 frames to render new content. For transparent windows with complex layouts, this async gap causes visible flicker. ShowWindow(SW_HIDE) before geometry change and SW_SHOWNOACTIVATE after prevents intermediate frames from being composited

  • Bash set -e scripts require defensive guards for optional operations. Find commands on potentially missing directories trigger non-zero exit codes; must add [ -d “$DIR” ] checks before find/compression operations to distinguish ‘directory empty’ from ‘command failed’.
  • Data format migrations are safer when execution is deterministic: saved (initial_state, actions) → replay in sim → reconstruct missing fields (target_poses, object_orientations). Enables backward-compatible augmentation without recollecting entire demo corpus.
  • ‘Uniform sampling’ in time-series contexts typically means equal temporal spacing (constant Δt), not balanced sample distribution across categories. Critical for sequence models like Transformers that assume regular timesteps. Scipy interpolation can transform irregular samples to uniform grids.
  • Cache systems in evolving data sources must include explicit version fields - TTL expiration alone insufficient to handle schema/source changes
  • Terminal resize synchronization needs full-chain coordination: ResizeObserver (frontend)→term.onResize→WebSocket message→SessionManager→PTY.resize(). Mismatch causes wrapping at wrong column. Debouncing prevents flooding during drag resize

  • React closure trap ref pattern: Third-party library callbacks (xterm.js onData) registered in useEffect capture stale props - common bug. Universal solution: props→ref, useEffect syncs ref.current, callback reads ref.current. React ecosystem best practice
  • WebSocket robustness three elements: reconnection (exponential backoff), backpressure (bufferedAmount check), heartbeat (not implemented but should consider). Distributed system network layer must assume ‘connection can break anytime’, not ‘connection should be stable’
  • Filesystem symlinks provide zero-overhead bidirectional sync for dual-boot systems. Replacing separate Ubuntu/Windows folders with symlink to single Windows partition eliminates sync lag and guarantees single source of truth without file duplication.
