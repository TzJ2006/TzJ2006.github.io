Problems & Solutions#

1. BasicContrastive.py contrastive loss bottleneck: per-node Python loop over 3000-4000 spots with tensor cloning each iteration [MIHD spatial omics] (2026-04-08)#

Solution: Vectorized loss computation using batch logsumexp operations eliminating loop; masked self-similarity with fill_diagonal_(-inf); boolean masks for positive neighbor filtering; yielded 10-50x speedup

2. Crossformer LIPM version performs 18x worse (MSE=9 vs 0.5) despite patch mechanism designed for irregular battery data [Crossformer time-series] (2026-04-08)#

Solution: Diagnosed fundamental architecture mismatch: DSW_embedding assumes continuous seg_len time steps but LIPM zero-padding creates irregular gaps breaking temporal continuity; recommended interpolation to uniform grid, time window aggregation, or irregular time-series models (GRU-D, BRITS, NAOMI)

3. 12 Error Skill files duplicating 150+ lines: frame checks (22 occurrences), direction normalization (8), list(dict.keys())[0] pattern (14); duplication hid bugs like e08’s 30-frame threshold vs standard 20 [Error Recovery Benchmark] (2026-04-08)#

Solution: Extracted 3 helper methods to BaseErrorSkill (check_remaining_frames, normalize_direction, random_direction); replaced list(keys())[0] with next(iter()); added 10 unit tests; eliminated 130+ lines duplication

4. MimicGen augmentation validation failures (0/10 scenes) - cubeA not lifted, remaining at 0.82 table height [MimicGen/Error Recovery] (2026-04-11)#

Solution: Multi-round diagnosis revealed root causes: subtask fragmentation (6 subtasks = 5 warp boundaries causing error accumulation), no post-grasp buffer, gripper detection bug (first closure vs sustained closure), retract subtask not warped, path parsing bug. Fixed by simplifying to 2 subtasks, adding 15-frame buffer, improving gripper detection, merging retract into re_acquire, fixing path parsing

Learnings#

Architecture (architecture)#

  • Vectorization of tensor operations eliminates Python loop overhead and enables GPU parallelism: BasicContrastive contrastive loss refactor from per-node loop to batch logsumexp operations yielded 10-50x speedup with identical mathematical behavior. Even mature deep learning code often contains un-vectorized loops from prototyping phase.
  • Zero-padding is NOT neutral for sequence models with positional encodings—it actively breaks semantic meaning: Crossformer’s DSW_embedding assumes continuous seg_len time steps; zero-padded patches with irregular gaps mislead model to learn ‘zero patterns’ instead of temporal dependencies. Architecture assumptions must match data preprocessing.
  • Faithful algorithm implementation requires understanding full execution model, not just data transformation: MimicGen’s success depends on waypoint-level closed-loop correction per subtask, which cannot be replicated by better open-loop action warping alone. Surface-level similarity (‘both use object poses’) can mask deep architectural differences.
  • Superpowers skill chain (brainstorming→ccplan→optimize) forms complete software lifecycle. ECL documents serve as cross-session persistent ‘design contracts’, enabling multi-round dialogue, multiple agents, even different models to maintain context consistency
  • Adversarial review value: Critic agent found 9 issues including 2 CRITICAL (unauthenticated PTY write, SSH parameter injection) and 4 HIGH (token brute-force, prototype pollution). Requirement-stage fixes cost « code-stage « production-stage
  • Original MimicGen design wisdom: (1) minimize subtask count (Stack uses only 2) (2) post-grasp buffer 10-20 frames ensures stable grasp (3) env.reset() to home position reduces pose gap (4) quasi-closed-loop execution (target_pose_to_action computes delta each step)
  • MimicGen’s core design is source demo + env.reset() randomization: treat error scene as fixed ‘initial state’, then use placement_initializer to generate variants, rather than re-injecting error for each target
  • Object-centric warping essence: EEF relative to object motion pattern is invariant, preserved through coordinate transform T_eef_new = T_obj_target × T_obj_src⁻¹ × T_eef_source. Two-step understanding: (1) eliminate source object absolute position (2) remap with new object position
  • Data provenance tracking must be explicit through every conversion pipeline step: HDF5→LeRobot conversion loses source_file+demo_idx mapping; solution is recording conversion_index.json during conversion to enable bridge-key linkage with generation-time metadata.
  • Error recovery datasets intentionally exclude error-causing actions to prevent models from learning failure modes; only corrective human demonstrations are used for imitation learning.
  • Benchmark papers evaluate paradigms, not methods - Nature Methods Analysis papers don’t need SOTA, but systematic answers to ‘how far can this methodology go’. Narrative shifts from ‘our method is best’ to ‘what are the boundaries and bottlenecks of this paradigm’
  • Porting validated patterns from mature open source projects more efficient and reliable than designing from scratch. Key: understand original design rationale (why 4 states not 3? why limit restart count? why exponential backoff?) then adapt to target context
  • Parallel agent strategy dramatically accelerates large-scale documentation - split 2246-line document into 3 independent parts, each generated by separate agent then assembled. Also accelerates research: 4 agents completed in 14s vs 45-60s sequential
  • Time-boundary partitioning (archive covers completed hours, live source covers current hour) elegantly solves deduplication without explicit record-level tracking - zero overlap means zero duplicate counting
  • Asyncio state machine correctness invariant: state check + transition must be atomic within same lock; any lock-external check introduces race window
  • JSONL (append-only log) + in-memory cache hybrid architecture highly effective for scenarios requiring both persistence and performance. Key design points: (1) atomic write (tmp → os.replace) prevents crash corruption (2) lazy load on read (3) bounded buffer (deque(maxlen)) (4) unified ISO 8601 UTC timestamp (5) periodic archival prevents infinite growth

Debugging (debugging)#

  • Code duplication hides bugs because fixes don’t propagate: 12 Error Skill files had inconsistent frame thresholds (e08: 30 vs. standard 20) that code review revealed as unintentional bugs. Extracting to shared helpers forces consistency.
  • Always cross-validate aggregated metrics with ground truth: log parsing bugs can propagate incorrect statistics. Verify against direct file counts (HDF5 demo_N groups) and structured reports (generation_report.json).
  • Adversarial code review catches issues missed in forward implementation: critic subagent identified in-place mutation semantics, pattern variations across callsites, and defensive check locations that main agent overlooked.
  • LeRobot 0.4.0 shape=(1,) feature handling contradiction: validate_frame requires ndarray but get_hf_features_from_features maps to scalar Value. Solution: insert type conversion in episode_buffer layer between validation and encoding.
  • StandardScaler NaN propagation mechanism: Single corrupt row with NaN in any feature column causes StandardScaler.fit() to compute NaN statistics (mean/std), propagating to all scaled data through (x - NaN) / NaN = NaN. Always validate data for NaN/Inf before normalization.
  • When refactoring LLM-driven pipelines from one language to another, validation must cover: (1) static prompt templates (2) runtime context injection (3) non-text outputs (charts/images) (4) code comments in prompts (5) generated templates/scaffolds (6) error recovery logic. Parallel agent validation more thorough than single-pass grep
  • DateTime timezone handling: date_naive() on DateTime extracts date in source timezone, not local. For remote timestamp filtering, must explicitly convert to Local timezone before extracting date. Critical for distributed systems with cross-timezone data
  • Subtask fragmentation cost: each subtask boundary is independent ‘query object pose → re-warp → 5-step interpolation’, errors accumulate. 6 subtasks = 5 error sources, far exceeding original’s 1
  • Code review layered remediation protocol: CRITICAL (security/races) → HIGH (deprecated APIs/private access) → MEDIUM (types/style) → LOW (linter), with test validation after each layer before proceeding to avoid cascading failures
  • WebView2 rendering is asynchronous relative to native window geometry changes. ShowWindow(SW_HIDE) before geometry change and SW_SHOWNOACTIVATE after prevents intermediate frames from being composited
  • Interpolation vs extrapolation in temporal resampling: Linear interpolation should stay within observed data bounds. Use per-sample adaptive ranges to avoid extrapolation artifacts

Domain Knowledge (domain)#

  • ‘Uniform sampling’ in time-series contexts typically means equal temporal spacing (constant delta t), not balanced sample distribution across categories. Critical for sequence models like Transformers that assume regular timesteps.
  • MimicGen-style data augmentation for robot manipulation uses object-centric coordinate transforms: source demo trajectories are warped relative to target object poses, enabling generalization across scene configurations while preserving action semantics.
  • Data format migrations can be non-destructive when execution is deterministic: saved (initial_state, actions) → replay in sim → reconstruct missing fields (target_poses, object_orientations). Enables backward-compatible augmentation without recollecting demos.
  • 1-to-N augmented dataset mapping requires aggregation across variant indices: for each clean episode, gather all bddl_idx (0-9 OSS variants) where orig_ep_idx matches, then convert aug_demo_idx to aug_lerobot_ep via conversion_index.
  • MimicGen source demos require complete datagen_info metadata (eef_pose, object_poses, target_pose, gripper_action, subtask_term_signals) and semantic assumptions (clean initial state, complete task trajectory, valid subtask transitions). Recovery demos from error states need separate augmentation pipeline
  • Format compatibility encompasses three layers: file structure (NPZ vs HDF5), metadata completeness (datagen_info fields), and semantic assumptions (starting state, task coverage). A file can pass structural checks but fail semantic requirements
  • Human demo data quality validation should be front-loaded to collection stage: teleop success alone insufficient, must simultaneously verify replay stability and augmentability
  • ccplan’s adversarial review (Phase 5) extremely valuable. In meeting-helper, code-reviewer as adversarial critic found 6 critical problems main planner missed, including CRITICAL Chinese-English mixed transcription limit (Whisper per-chunk detection causes poor intra-sentence code-switching).
  • Homebrew formula deprecated status is critical signal - when encountering low-level dependency issues, check upstream maintenance status first rather than debugging abandoned versions
  • Graph contrastive learning evolution in spatial transcriptomics: STAIG (node-level) → QueST (subgraph-level) → Loki (cross-modal), showing field progression from ‘one method per problem’ to ‘one foundation model for multiple tasks’
  • Terminology selection matters: label-free vs zero-shot - ‘zero-shot’ easily misunderstood as ’no training at all’, ’label-free / annotation-free’ accurately conveys core advantage of ’no manual annotation’
  • Self-supervised learning’s ‘free signal’ framing - spatial transcriptomics coordinates are structured information inherent to Visium chips. Viewing spatial graphs as ‘free supervision signal’ clarifies label-free value
  • NeurIPS Dataset Track review priorities (from 61 reviews): Most valued qualities are ‘comprehensive’ (17 mentions), ‘community’ (17), ’novel’ (15), ‘clear’ (28), ‘well-designed’ (13). Common criticisms: ’lack comparison’, ‘insufficient detail’, ’limited scope’
  • In multi-pipeline projects, technical terminology needs explicit project-specific definition. ‘Source demo’ can mean different things in recovery pipeline (RecoveryDemo) vs standard MimicGen pipeline (success_demos.hdf5). Documentation should establish clear namespace
  • LiPM paper demonstrates physics-informed pretraining for battery tasks: MMAE for cross-modal coupling, CIR for coulombic integration (charge conservation), dual-scale encoders for irregular sampling. Key insight: domain-specific foundation models outperform generic time series models when physical constraints properly encoded
