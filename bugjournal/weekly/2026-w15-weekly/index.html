Learnings#

Domain Knowledge (domain)#

  • The optimal fusion strategy for clustering and retrieval diverges: STAIG spatial graph contrastive learning is best suited for global domain clustering (Avg ARI 0.546), but its strong spatial homogenization leads to overly concentrated boundary retrieval (96% Epithelium); Attention fusion is more balanced for boundary microenvironment retrieval (60/40) — this is the core new argument of MIHD paper R6
  • Three new RM-IDEAL experiments: same-donor cross-section (151673↔151674) Layer 3 Spearman ρ as high as 0.86; cross-donor (151673↔151669) Layer 3 showed negative correlation (ρ=-0.73), suggesting systematic differences in Layer 3 morphology between donors
  • T cell markers (CD3D/CD3E, etc.) are extremely sparse in 8μm spots of HD colorectal cancer data (7.3% have expression); PCA gene expression detects T cell enrichment (3.8x) better than visual+gene fusion (STAIG 2.2x), indicating T cell signals are primarily driven by gene expression rather than tissue morphology
  • Paper Introduction opening strategy: directly presenting the most critical numerical finding (ARI 0.26→0.55) is more persuasive than broad background preamble; having each paragraph correspond to a Results section heading reinforces paper structural consistency

Tools (tools)#

  • Hugo site deployment has three independent dependencies that must all be in place simultaneously (hugo binary, theme repository, deployment git repository); any one missing causes failure at different stages with different error codes — these prerequisites should be explicitly listed in project documentation
  • Claude Code’s background bash commands do not inherit the shell’s conda activate state; background Python scripts must use conda run -n ENV python ... rather than conda activate ENV && python ...

Debugging (debugging)#

  • find operations on optional directories in bash set -e scripts must include a directory existence check ([ -d "$DIR" ]); otherwise the normal “directory not yet created” scenario triggers unexpected script termination
  • PID recording statements for parallel background tasks (& + pids tracking) must be precisely aligned with if/else control flow boundaries; placing shared code outside branches easily introduces duplicate PID tracking and silent file replacement omission bugs
  • Long-running batched experiments sharing an output file (e.g., summary.csv) need per-run output paths designed at the script level, or immediate backup after each batch completes; when background process output is empty, use ps aux to confirm process status before resubmitting
