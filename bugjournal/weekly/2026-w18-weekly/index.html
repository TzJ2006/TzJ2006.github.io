Problems & Solutions#

1. Window height oscillates rapidly (150-200ms cycles) when hovering chart bars, causing detail panel to flash on/off in continuous feedback loop [TokenMonitor] (2026-04-27)#

Solution: Implemented three-layer defense: (1) onLeave timer checks hoveredIdx >= 0 to skip fake leaves, (2) ResizeOrchestrator blocks observer-driven resize during chartHoverActive, (3) shrink-guard prevents window from shrinking during hover. Breaking the loop required intervention at multiple points, not just blocking resize.

2. SetFocus Win32 API requires calling thread to own foreground window, but tray click handler doesn’t have foreground ownership, causing first-click animation failure [TokenMonitor] (2026-04-27)#

Solution: Use SetForegroundWindow which has implicit foreground rights in tray icon context, combined with 200ms fallback timer in Svelte to handle edge cases. Win32 foreground window rules are context-dependent.

3. Window bottom edge drifts from taskbar during resize, creating visual gap; using current_rect.bottom causes bottom edge to jump unpredictably [TokenMonitor] (2026-04-27)#

Solution: Changed anchored_resize_origin from current_rect.bottom - height to work.bottom - height. Work area boundary is stable constant while current_rect reflects intermediate async SetWindowPos states. Anchor to fixed reference points instead of current state.

4. Hardcoded bottom-right anchor causes flashing when taskbar is elsewhere, or window gets repositioned by user or system [TokenMonitor] (2026-04-27)#

Solution: Built dynamic anchor detection: compare window center to work area center, store result in AtomicU8, use detected anchor in all positioning functions. Window anchor must be detected dynamically based on actual position.

5. Cursor usage warning appears even when rate limits API works correctly, confusing users about actual connection status [TokenMonitor] (2026-04-27)#

Solution: Downgraded remote usage events API failure from set_cursor_warning to logging; only warn when authentication genuinely unavailable (both pipelines fail). Multi-tier data pipelines serving different features should fail independently.

6. Mouse position detection using clientX/Y + getBoundingClientRect failed to prevent fake leave events during window movement [TokenMonitor] (2026-04-27)#

Solution: Abandoned coordinate-based detection; instead used hoveredIdx state check in timer callback. When window moves, pointermove doesn’t fire (mouse didn’t actually move), so tracked coordinates become stale; state-based detection is more reliable.

7. Users had to manually reopen Cursor and restart TokenMonitor to fix yellow ‘Connected’ warning state when auth token became stale [TokenMonitor] (2026-04-27)#

Solution: Implemented open_cursor_app command for cross-platform app launching (macOS open -a, Windows .exe paths, Linux cursor command) and retry_cursor_auth polling mechanism that automatically detects token refresh and clears warnings. Auto-retry pattern with timer cleanup prevents resource leaks; polling every 4 seconds for ~32 seconds balances responsiveness with resource usage.

8. Window bottom showed transparent gap due to 14px border-radius when bottom-anchored to taskbar [TokenMonitor] (2026-04-27)#

Solution: Query window anchor direction on startup via get_window_anchor_edge IPC, set data-anchor attribute on , conditionally remove bottom border-radius in CSS when data-anchor=‘bottom’. Transparent windows with border-radius require conditional styling based on runtime anchor position.

9. sqlite3 Command spawn missing CREATE_NO_WINDOW flag causes console windows to pop up on Windows [TokenMonitor] (2026-04-27)#

Solution: Added const CREATE_NO_WINDOW: u32 = 0x0800_0000 and .creation_flags() call with #[cfg(target_os = “windows”)] guard to cursor_parser.rs. All Windows process spawns need explicit CREATE_NO_WINDOW flag.

10. Window bottom edge drifts from taskbar when hovering over chart bars due to detail panel expansion triggering window resize beyond screen bounds [TokenMonitor] (2026-04-28)#

Solution: Introduced SCROLL_THRESHOLD_CAP = 500 constant; modified resolveScrollThresholdHeight to return Math.min of ratio-based calculation and cap; content now scrolls within fixed-height window instead of resizing. Hard-coded UI thresholds provide predictable behavior across all monitor sizes.

11. Initial v0.12.1 release failed CI checks for cargo fmt and clippy [TokenMonitor] (2026-04-28)#

Solution: Applied cargo fmt to window.rs to fix import ordering and line wrapping; removed needless return statements in config.rs; deleted old tag and re-released with fixes. Pre-release local validation should include full CI check suite to avoid tag churn.

Learnings#

Architecture (architecture)#

  • Feedback loops in UI systems can have non-obvious triggers: window resize → position change → browser event → state change → resize. Breaking the loop requires identifying correct intervention points at multiple layers, not just one.
  • Async window management (SetWindowPos) means ‘current state’ queries (GetWindowRect) during rapid changes return intermediate values. Stable anchors (work area bounds) are more reliable than querying current state for positioning calculations.
  • Multi-tier data pipeline design: when pipelines serve different features (rate limits vs usage events), failure handling should reflect feature independence. Don’t block one feature for another’s failure; fail gracefully per pipeline.

Debugging (debugging)#

  • Browser mouse events during window movement are ‘fake’ from user perspective but ‘real’ from browser perspective. Need state-based rather than coordinate-based detection to distinguish genuine user actions from window-movement artifacts. When window moves, pointermove doesn’t fire, making coordinates stale.
  • Iterative debugging with clear symptom description before root cause analysis prevents wasted effort. User wanted focus on visual behavior (‘window jumps up and down’) before technical diagnosis, ensuring alignment on what’s being fixed.

Tools (tools)#

  • Win32 foreground window rules are context-dependent: SetFocus requires calling thread to own foreground, but SetForegroundWindow has implicit rights from tray icon event context. Critical for system tray applications on Windows.
  • Pre-release CI validation must be run locally to maintain clean release history. cargo fmt –check and cargo clippy – -D warnings prevent tag churn and catch issues before public release tags are created.
  • Svelte 5 reactivity timing: $state updates queue DOM changes in micro-tasks, but CustomEvent dispatch is synchronous. Can cause race conditions when measurements happen before DOM updates. Understanding reactivity timing is crucial for correct behavior.
  • Auto-retry pattern design: use onDestroy/cleanup to prevent timer leaks; balance polling frequency (4s) and max attempts (8) for reasonable timeout window (32 seconds) that covers typical auth refresh time without excessive resource usage.
  • Git tag management for public releases requires deletion/recreation workflow (git tag -d, git push –delete, retag, push) rather than force-push or amend to maintain integrity and avoid confusion.

Domain Knowledge (domain)#

  • Transparent windows with border-radius create visual gaps when edges are meant to align with screen boundaries. Need conditional styling based on runtime anchor position (data-anchor attribute) to remove border-radius selectively.
  • Hard-coded UI thresholds (500px cap) can be more predictable than ratio-based calculations (0.75 * work area) for cross-device consistency. Ensures uniform behavior across 1080p, 1440p, and 4K monitors.
  • All Windows process spawns need explicit CREATE_NO_WINDOW flag (0x0800_0000); one missed spawn in any module breaks entire UX by causing console windows to pop up. Must be applied consistently across all subprocess creation.
