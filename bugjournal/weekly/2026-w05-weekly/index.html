关键任务#

  • ✅ STAIG vs MIHD GCN 实现系统性对比分析 (2026-02-01) — 对比激活函数应用位置、图构建算法效率、边权重方向、温度参数 τ、默认层数五个维度，并将 GCNGeneEncoder 默认行为对齐原始 STAIG

问题与解决方案#

1. ARI 从历史最高 0.37~0.53 回归至 0.13，多次参数调整（epoch、tau、num_layers）均无效 [MIHD / STAIG Fusion] (2026-02-01)#

解决方案: 通过 git stash + checkout 历史版本、逐步 diff 对比，精确定位到两处代码级 bug：① gene feature 输入应为 50 维 PCA embeddings 而非 3000 维原始基因表达；② edge_weight_method 应为 softmax_neglog（距离小→权重大）而非 softmax_log。移除 raw_gene_expr 逻辑并恢复正确边权重方法后 ARI 回升至 0.28~0.33

2. mclust 聚类在当前 base 环境下失败（无 rpy2） [MIHD / STAIG Fusion] (2026-02-01)#

解决方案: 改用 kmeans 聚类完成当前验证，后续切换至 General 环境（已有 rpy2 3.5.11）以恢复 mclust 并获得更高 ARI

学习收获#

调试 (debugging)#

  • 排查性能回归时，优先使用 git diff 历史成功版本比逐参数试探效率高一个数量级；当改动涉及多个文件时，git bisect / diff 是缩小搜索空间最有效的手段
  • “对齐原始实现"的代码修改本身可能引入 bug（伪修复），应在修改后立即运行完整基准测试验证性能，而非假设对齐即为正确

领域知识 (domain)#

  • STAIG Fusion 高 ARI（0.4+）依赖两个关键条件：① gene feature 必须是 50 维 PCA embeddings 而非原始基因表达；② 聚类必须使用 mclust（需 General 环境的 rpy2），kmeans 会导致显著下降
  • STAIG 原始边权重方向为 softmax(log(距离))（距离大→权重大），与直觉相反；MIHD 使用 softmax(-log(距离))（距离小→权重大）更符合直觉，两者产生不同聚类结果

架构 (architecture)#

  • STAIG 原始 GCN 对所有层（含最后层）应用激活函数（tau=0.5），适合端到端对比学习；MIHD GCNGeneEncoder 默认最后层不加激活，更适合作为特征提取器

AI 使用备注#

有效模式:

  • ✓ 并行探索两个大型代码库（STAIG net.py 与 MIHD GCNGeneEncoder）进行系统性多维度对比分析
  • ✓ 通过 git diff 历史版本自主缩小 bug 搜索范围，最终精确定位到代码级根本原因

局限性:

  • ✗ 在修复过程中缺乏系统性分析优先级，先尝试 epoch/激活函数/边权重等多个方向，走了多轮弯路才定位到真正原因
  • ✗ 无法感知 conda 环境上下文，多次使用错误的环境调用方式（conda activate 在 subprocess 中无效，应使用 conda run -n General）
  • ✗ 复杂 git 操作中产生状态管理错误（git stash pop 冲突后选择 drop，丢失部分中间状态）
