2. Configuration parameters (num_epochs, learning_rate) not being applied from YAML files [MIHD Spatial Transcriptomics Model] (2026-02-01)#

3. Edge weight method defaulting to incorrect directionality (softmax_log vs softmax_neglog) [MIHD Spatial Transcriptomics Model] (2026-02-01)#

Solution: Changed default from softmax_log to softmax_neglog to match proven high-performance configuration where small distance yields large weight

4. GCN architecture divergence from original STAIG implementation [MIHD Spatial Transcriptomics Model] (2026-02-01)#

Solution: Added apply_activation_to_last_layer parameter (default True) to align with STAIG’s design where all layers apply activation

Learnings#

Debugging (debugging)#

  • Environment configuration (conda environment, clustering libraries) can dominate code-level fixes in ML regression debugging. Always verify execution environment matches historical success conditions.
  • Git history comparison with last known good version is often faster than forward-fixing current code. Version archaeology should be early in debugging workflows for performance regressions.

Architecture (architecture)#

  • Input dimensionality mismatches (50D PCA vs 3000D raw features) cause catastrophic performance degradation in graph neural networks. Always validate feature dimensions match model expectations.
  • Clustering method choice (mclust vs kmeans) has first-order impact on spatial transcriptomics evaluation metrics. R integration via rpy2 is often necessary for reproducing bioinformatics paper results.
  • Seemingly minor architectural differences (last layer activation, edge weight direction) accumulate to cause significant divergence when replicating research implementations. Exhaustive parameter alignment is critical.

Tools (tools)#

  • Parallel agent deployment for cross-codebase comparison is highly effective for systematic architectural analysis. Multiple agents exploring different codebases simultaneously provides comprehensive comparison in one round.
