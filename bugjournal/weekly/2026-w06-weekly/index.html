Problems & Solutions#

1. GCN+STAIG fusion performance extremely poor (ARI 0.05-0.09) compared to PCA+STAIG (ARI 0.45) [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Two-phase diagnosis: (1) Compared with original STAIG discovering 4 algorithmic differences (edge dropout, feature clipping, edge weight direction, tau decay), (2) Analyzed original config revealing hyperparameter mismatches (learning_rate, num_layers, num_epochs, weight_decay)

2. apply_fusion() missing **kwargs parameter caused NameError preventing staig_fusion_e2e execution [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Added **kwargs to function signature to accept dynamically passed parameters like gene_expr_raw

3. STAIGTrainerE2E hardcoded assumption that input is raw gene expression (3000-dim), incompatible with PCA/MLP/GCN encoder outputs [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Added gene_preprocessing parameter (auto/standard/clip/none) to automatically select preprocessing strategy based on input dimensionality and source

4. Adaptive edge dropout implementation differs from PyG’s standard dropout_adj - requires custom logic based on edge probabilities [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Implemented custom adaptive_dropout_adj() where mask = rand() >= edge_prob, following original STAIG’s counter-intuitive strategy (drop similar edges, keep dissimilar)

5. experiments_log.md severely incomplete (14 entries vs 36 actual experiments), missing best results (ARI 0.528) [MIHD Spatial Transcriptomics] (2026-02-03)#

7. Bar charts rendered with inverse lengths (higher scores showed shorter bars) despite correct CSV values [Benchmark Infrastructure] (2026-02-06)#

Learnings#

Architecture (architecture)#

  • When implementing published methods, compare line-by-line with reference code BEFORE claiming alignment - papers omit critical implementation details. Both algorithmic alignment and hyperparameter configuration are critical for deep learning model performance reproduction. Dramatic underperformance (0.05 vs 0.45 ARI) signals implementation bugs, not tuning issues.
  • Deep learning project reproduction key is complete hyperparameter configuration alignment - even with correct code logic, subtle parameter differences (e.g., learning_rate from 0.0005 to 0.001) can cause cliff-like performance drops (ARI from 0.452 to 0.05).
  • Flexible software architecture requires strategy pattern at key nodes: gene_preprocessing’s auto/standard/clip/none design demonstrates how to support multiple input types without breaking interfaces. API flexibility matters more than narrow optimization - E2E training should support both raw and pre-encoded inputs.
  • End-to-end training vs two-stage training affects model performance and efficiency: original STAIG uses end-to-end approach (GCN embeddings trained within), while MIHD initially used two-stage (extract features then train) - core architectural difference.
  • Crowdsourced data pipelines require three-tier architecture (untrusted submit → trusted gateway → protected storage) with validation/audit trails to prevent abuse without granting write access.
  • Ethical data collection requires explicit opt-in consent with default-No prompts; automation scenarios need explicit flags rather than relying on interactivity detection.

Domain Knowledge (domain)#

  • STAIG fusion demonstrates significant improvement: PCA alone achieves ARI 0.202, but PCA+UNI2+STAIG reaches ARI 0.528 on section 151508 (123% improvement over PCA alone).
  • RM-Ideal (Region Matching with Ideal labels) uses WWL graph kernel: K-iteration neighbor label hashing → Hamming cost matrix → Wasserstein distance → similarity = 1-W. Captures spatial niche structure independent of learned embeddings.

Debugging (debugging)#

  • Evaluation metrics for niche query require careful mask design: Precision@K uses eval_mask (~niche to avoid self-match), SameLabel@K uses all_valid_mask (including niche for retrieval rate), Spearman uses eval_mask (rank correlation on non-self spots).

Tools (tools)#

  • Client-side visualization bugs often stem from serialization format mismatches rather than data correctness; explicit list conversion + version pinning prevents silent rendering failures.
  • When loading pretrained models (like UNI2), explicitly pass architecture parameters rather than relying on framework automatic inference, especially for shape-sensitive components like position encoding.
  • Privacy sanitization rules need precise boundary conditions; overly aggressive regex patterns can corrupt legitimate technical metadata sharing surface syntax with PII.
  • Documentation should be generated programmatically from source data (CSV) to prevent manual sync errors and incomplete records.
  • Environment dependency management best practice is incremental repair: directly run target program, install on-demand when encountering ModuleNotFoundError - more efficient than pre-scanning.
