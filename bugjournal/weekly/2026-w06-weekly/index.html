Problems and Solutions#

1. MIHD staig_fusion’s ARI (0.21/0.4849) is much lower than the original STAIG’s (0.562), root cause unknown [MIHD / STAIG Fusion] (2026-02-06)#

Solution: Compared two code paths, identified five root causes and ranked them by priority: ① Mixing coordinate scales for full-resolution images (most critical); ② mclust silently downgrading to kmeans when HPC is unavailable; ③ Reverse gene preprocessing order; ④ Pseudo-label cluster count difference of 300 vs 80; ⑤ Hyperparameters not fully aligned.

2. Plotly bar chart rendering error: Short bars for high scores, visual logic reversed [Benchmark Tool] (2026-02-06)#

Solution: The root cause was that Plotly 6.x used binary-encoded arrays serialization, which was incompatible with older CDN versions. Solution: Locked the CDN version to match the Python package version (3.3.1), forced a Python list input, and set rangemode=‘tozero’.

3. Strict STAIG mode forces visual encoder to UNI, users think it’s using UNI2, behavior unclear [MIHD / STAIG Fusion] (2026-02-06)#

Solution: Added clear hints about encoder override in logs; the UNI2 progress bar was not displayed because UNI was actually used.

4. create_trend_chart AttributeError: Method is commented out but still called in generate_html [Benchmark Tool] (2026-02-06)#

Solution: Added a stub method that returns an empty chart and completely removed the calling code.

Learning Outcomes#

Architecture#

  • The correct layered architecture for GitHub Pages submissions: Public users → relay (verification/throttling/masking) → repository_dispatch → queue files → daily batch processing (duplication/cleaning/appending CSV) → commit/push → automatic pages deployment. The relay layer is a key isolation layer to protect write permissions of the repository.
  • Interface names must strictly match actual behavior: The name ‘staig_fusion’ naturally implies the same semantics as STAIG. Allowing significant differences in default behavior can lead to many hidden errors. Global override behaviors (such as forcing UNI as the encoder) must be clearly indicated in logs.

Debugging#

  • Plotly 6.x uses binary-encoded arrays serialization by default, which causes bar length parsing errors when it’s incompatible with older CDN versions. Solution: Strictly lock the CDN version to match the Python package version, forcing a Python list input instead of pandas Series.
  • Low CPU and GPU utilization indicates that the program is waiting in single-threaded I/O, not just CPU. Visual feature extraction performance bottlenecks usually occur in data preprocessing (patch extraction) rather than model forward passes.
  • When R interfaces (like mclust) fail in Python, the error message isn’t intuitive. Sufficient pre-validation should be added at the Python level, and clear error messages with shape information should be provided. If an R package is unavailable, a clear warning must be given for silent degradation. The algorithm behavior shouldn’t be changed without any hints.

Domain Knowledge#

  • There are five quantifiable key differences between MIHD and original STAIG (ranked by priority): ① Mixing coordinate scales for full-resolution images (most critical); ② mclust silently downgrading; ③ Reverse gene preprocessing order; ④ Pseudo-label cluster count difference; ⑤ Hyperparameters and image transformation differences. The ARI gap (0.21→0.56) is mainly determined by points ① and ②.

Notes on AI Usage#

Effective Modes:

  • ✓ Systematic cross-repository code comparison: By comparing two code libraries line by line, we quantified and identified multiple implementation differences and ranked them by priority.
  • ✓ Multi-turn tool invocation combined with human observation: AI detected coordinate scale issues (technical details), while users noticed encoder override problems (runtime observations), creating a complementary approach.
  • ✓ Architecture design supplementation: Humans proposed core security intuitions (confirmation before upload, secure isolation), while AI translated them into complete executable technical architectures.

Limitations:

  • ✗ Insufficient fix-ups: When removing trend charts, only the calling function was removed initially, and additional fixes were needed.
  • ✗ Easy to overlook existing global override logic: Didn’t realize that strict mode would force visual encoder override when adding UNI2 tqdm.
  • ✗ Didn’t check HPC environment availability of modules (like scanpy) in advance, resulting in a ModuleNotFoundError during verification. This increased the number of debugging rounds.
  • ✗ Provided multiple options for low GPU utilization issues, but didn’t actively recommend using profile tools (py-spy, nvprof) to accurately identify the actual bottleneck.
