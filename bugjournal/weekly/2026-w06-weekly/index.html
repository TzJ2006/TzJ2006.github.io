Problems & Solutions#

1. apply_fusion() missing **kwargs parameter caused NameError, preventing staig_fusion_e2e branch execution [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Added **kwargs to apply_fusion() function signature to accept dynamically passed parameters (e.g., gene_expr_raw)

2. STAIGTrainerE2E hardcoded assumption that input is raw gene expression (3000-dim), incompatible with PCA/MLP/GCN encoder outputs [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Added gene_preprocessing parameter (auto/standard/clip/none) to automatically select preprocessing strategy based on input dimensionality and source; raw expression uses StandardScaler+Clip, pre-encoded embeddings use Clip only

3. GCN+STAIG fusion performance extremely poor (ARI 0.05-0.09) and unstable compared to PCA+STAIG (ARI 0.45) [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Two-phase diagnosis: (1) Compared with original STAIG discovering 4 algorithmic differences (edge dropout, feature clipping, edge weight direction, tau decay), (2) Analyzed original config revealing hyperparameter mismatches (learning_rate, num_layers, num_epochs, weight_decay)

4. MIHD STAIG implementation achieves ARI=0.05, far below original STAIG project’s 0.452 [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Compared original train_img_config.yaml revealing critical hyperparameter inconsistencies; need to change learning_rate to 0.0005, num_layers to 1, num_epochs to 300, weight_decay to 0.00001

5. Adaptive edge dropout implementation differs fundamentally from PyG’s standard dropout_adj [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Implemented custom adaptive_dropout_adj() function following original STAIG net.py:209-240, where mask = rand() >= edge_prob means higher-weight edges are MORE likely to be dropped (counter-intuitive strategy prevents in-domain overfitting)

6. UNI2 without parameters causes position encoding resampling error on GPU (shape mismatch) [MIHD Spatial Transcriptomics] (2026-02-05)#

Solution: Pass complete timm_kwargs (including img_size, patch_size, depth, num_heads, etc.) to let timm correctly initialize UNI2 model, avoiding reliance on automatic inference

7. experiments_log.md was severely incomplete (14 entries vs 36 actual experiments in CSV), missing best results [MIHD Spatial Transcriptomics] (2026-02-03)#

8. Public users cannot push to repository but need to contribute benchmark results [Benchmark Infrastructure] (2026-02-06)#

9. Bar charts rendered with inverse lengths (higher scores showed shorter bars) despite correct numeric values in CSV [Benchmark Infrastructure] (2026-02-06)#

Learnings#

Architecture (architecture)#

  • STAIG fusion demonstrates significant improvement: PCA alone achieves ARI 0.202, but PCA+UNI2+STAIG reaches ARI 0.528 on section 151508 (123% improvement). End-to-end vs two-stage training affects model performance - original STAIG trains GCN embeddings jointly within STAIG.train(), while MIHD initially used two-stage approach.
  • Flexible software architecture requires strategy pattern at key nodes: gene_preprocessing’s auto/standard/clip/none four-mode design demonstrates how to make a module support multiple input types without breaking interfaces. API flexibility matters more than narrow optimization - E2E training should support both raw and pre-encoded inputs.
  • Crowdsourced data pipelines require three-tier architecture (untrusted submit → trusted gateway → protected storage) with validation/audit trails to prevent abuse without granting write access. Ethical data collection requires explicit opt-in consent with default-No prompts.
  • Evaluation metrics for niche query require careful mask design: Precision@K uses eval_mask (~niche to avoid self-match), SameLabel@K uses all_valid_mask (including niche for retrieval rate), Spearman uses eval_mask (rank correlation on non-self spots).

Debugging (debugging)#

  • When implementing published methods, compare line-by-line with reference code BEFORE claiming alignment - papers omit critical implementation details like adaptive dropout mechanics. Both algorithmic alignment and hyperparameter configuration are critical for deep learning model performance reproduction.
  • Deep learning project reproduction key is complete hyperparameter configuration alignment - even with perfectly correct code logic, subtle parameter differences (e.g., learning_rate from 0.0005 to 0.001) can cause cliff-like performance drops (ARI from 0.452 to 0.05).
  • Dramatic underperformance (0.05-0.09 vs 0.45 ARI) signals implementation bugs, not hyperparameter tuning issues - reference comparison is faster than blind optimization. Ablation studies reveal which changes actually matter - expected improvements should be validated incrementally.
  • Always verify existing codebase state before implementing - documentation and actual code can be out of sync. Output semantic reasonableness validation is critical - 0.0 retrieval rate contradicting visual evidence indicates a bug, not true performance.

Tools (tools)#

  • Documentation should be generated programmatically from source data (CSV) to prevent manual sync errors and incomplete records. CSV files should be treated as source of truth; markdown logs should be generated to avoid inconsistencies.
  • When loading pretrained models (like UNI2), explicitly pass architecture parameters rather than relying on framework automatic inference, especially for shape-sensitive components like position encoding - explicit parameters avoid runtime errors.
  • Environment dependency management best practice is incremental repair rather than comprehensive upfront checking: directly run target program, install on-demand when encountering ModuleNotFoundError - more efficient than pre-scanning dependencies.
  • Client-side visualization bugs often stem from serialization format mismatches rather than data correctness; explicit list conversion + version pinning prevents silent rendering failures. Binary-encoded plot data (bdata) and CDN version drift can cause inverse bar chart bugs.
  • Privacy sanitization rules need precise boundary conditions; overly aggressive regex patterns can corrupt legitimate technical metadata sharing surface syntax with PII. Hostname regex should require TLD suffix to avoid false positives on version strings.

Domain Knowledge (domain)#

  • RM-Ideal (Region Matching with Ideal labels) uses WWL graph kernel: K-iteration neighbor label hashing → Hamming cost matrix (normalized by K+1) → Wasserstein distance → similarity = 1-W. Captures spatial niche structure independent of learned embeddings.
  • When porting graph algorithms between frameworks: squidpy.gr.spatial_neighbors() → sklearn NearestNeighbors + scipy sparse, torch_geometric.k_hop_subgraph() → networkx BFS. Pattern: translate to dependency-minimal equivalents. Always provide dependency-free fallbacks (e.g., scipy.optimize.linprog for Wasserstein when POT unavailable).
  • Caching strategy for expensive computations: RMIdealEvaluator precomputes all k-hop subgraphs at init (~8min for 4384 spots), saves to .npy cache, subsequent runs load instantly. Cache key should include spatial graph parameters (n_neighbors, k_hop).
