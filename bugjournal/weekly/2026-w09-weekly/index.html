Learnings#

Architecture (architecture)#

  • Missing instance attributes in PyTorch models can silently break checkpoint loading with strict=False, causing partial parameter initialization (17.7% random weights) that manifests as poor performance rather than errors. Always verify checkpoint key matching and log mismatches explicitly
  • Observation space design has nonlinear impact on task performance: missing object observations caused Coffee 0% SR but Stack achieved 98%. Different task complexities have vastly different information dependencies; templates cannot be one-size-fits-all
  • Batch processing critical for GPU efficiency: Python loops with repeated small GPU operations create severe underutilization (5% for sequential processing). Always aggregate into single GPU kernel launches through batching for 20-500x speedups
  • Dimensionality reduction’s value depends on downstream model capacity: helps simple methods (mean/attention) by reducing noise (+0.05-0.09 ARI), hurts sophisticated models (qformer/llava_mlp) by discarding information (-0.05 ARI). No universal answer
  • Task distribution mismatch between training and evaluation manifests as catastrophic failure (0% success). Pi0.5 trained on {Coffee, Stack, Threading, Assembly, Stack Three} failed completely when evaluated on PickPlace
  • BC-RNN significantly outperforms Pi0.5 on normal tasks (Stack: 98% vs 24%) but both fail at error recovery (~0%). Behavior cloning works in-distribution but doesn’t generalize to error scenarios without explicit recovery training
  • Large refactoring best practices: analyze dependencies → migrate in topological order (dependees first) → delete source files → global verification (grep + dry run), ensuring every intermediate state is runnable
  • Per-screen vs global state duality requires explicit routing: local methods must delegate to global evaluators when semantics are system-wide. Multi-file coordinated edits benefit from centralized orchestration
  • Pre-processed external datasets require explicit transformation history tracking to prevent pipeline corruption. Use metadata flags or value heuristics to detect already-applied transformations

Debugging (debugging)#

  • Vectorization delivers 100-1000x speedups for array operations: replacing Python nested loops with scipy/NumPy primitives (cdist, softmax, advanced indexing) almost always worth it. Key is recognizing O(n²) patterns and knowing the right library function
  • Robust error handling at iteration boundaries (scene-level try-except) prevents cascade failures in long-running batch processes while preserving partial results via resume mechanisms
  • Unbuffered output (PYTHONUNBUFFERED=1 + stdbuf -oL) non-negotiable for monitoring long-running ML training on remote clusters—buffering can hide failures for hours making debugging impossible
  • Proxy environment variables can intercept and break WebSocket connections to localhost even when raw TCP works—they operate at application layer. Always unset proxy vars when debugging local services
  • Training completion ≠ evaluation done. 100k-step trained models can sit unevaluated. Always verify completion via artifact inspection (checkpoints with timestamps), not log tails
  • Parallel exploration agents highly effective for understanding multi-component systems—exploring training setup, infrastructure, and pipeline independently then synthesizing enables faster comprehension than sequential investigation
  • Stub functions with TODO comments in production code can completely break features while appearing to work (no compiler errors). Critical to implement or remove stubs before shipping
  • Statistical tests (scipy.stats.kruskal) fail on degenerate distributions; always add edge case handling with informative messages rather than letting library exceptions propagate

Tools (tools)#

  • JAX requires explicit backend selection via JAX_PLATFORMS=cpu for CPU-only mode, not just CUDA_VISIBLE_DEVICES masking. Device masking alone causes backend initialization errors
  • HPC compute nodes typically lack internet access; always disable cloud logging (W&B, MLflow, Comet) via WANDB_MODE=disabled or configure offline mode for batch jobs
  • GPU memory leaks from idle model servers are major hazard in shared clusters. Phoenix server consumed ~407GB across 7 GPUs at 0% utilization. Need automated monitoring/cleanup or strict quotas
  • When loading 50+ GB models on shared GPUs, always check nvidia-smi and existing processes before new launches. Reusing zombie servers prevents CUDA OOM and saves startup time
  • VLA policy evaluation requires exact library version matching with training environment—even minor robosuite/mimicgen version differences cause API incompatibilities that silently invalidate benchmarking
  • Slurm pam_slurm_adopt enforces job-based SSH access—must allocate via salloc/sbatch before SSH to compute nodes. tmux + salloc provides stable interactive workflow resistant to connection drops
  • XLA_PYTHON_CLIENT_MEM_FRACTION controls JAX GPU memory pre-allocation: defaults to 0.75, can safely increase to 0.95 for dedicated training (from 61GB to 77.7GB per GPU)
  • Parallel execution with per-task timeouts essential for scanning large shared HPC filesystems where individual directories may exceed 2TB; sequential scanning infeasible for production-scale analysis
  • Multi-device development requires systematic documentation workflows; two-phase export-then-merge architecture enables asynchronous log collection with centralized consolidation
  • Batch historical report generation benefits from day-by-day subprocess isolation: prevents single date’s errors from blocking entire backlog, enables incremental progress tracking
  • macOS adaptive behaviors implementable via ProcessInfo query + NotificationCenter observation. isLowPowerModeEnabled queryable and observable via NSProcessInfoPowerStateDidChange

Domain Knowledge (domain)#

  • mclust probabilistic clustering doesn’t scale beyond ~5000 observations (40+ min for 17K bins); KMeans provides 360x speedup (2 min) with acceptable quality tradeoff (ARI ~0.17 vs 0.20-0.25) for exploratory spatial transcriptomics
  • RM-IDEAL benchmark reveals STAIG fusion (r=0.396) significantly outperforms simple strategies (concat/mean ≈0.25) by explicitly modeling spatial dependencies in gene expression embeddings
  • VisiumHD multi-resolution bin system (2um native with 4×4 aggregation to 8um) requires integer division coordinate transforms (r_big//4) for annotation alignment—empirically derived, not documented
  • Leiden community detection provides robust auto-clustering for unlabeled spatial transcriptomics via k-NN graph construction. Combined with silhouette score, enables unsupervised evaluation when ground truth unavailable
  • BC-RNN image vs low_dim modes have fundamentally different architectures: image needs ResNet18 encoder, larger RNN, smaller batches, 600 epochs; low_dim uses larger MLP, 2000 epochs. MimicGen paper’s primary results use image mode
  • Environment naming in robosuite/MimicGen has semantic implications: stripping ‘_D0’ suffix changes instantiated env class, altering object observables (65→37 dims). Base envs return fewer states than MimicGen variants
