Learnings#

Architecture (architecture)#

  • Batch processing critical for GPU efficiency: Python loops create severe underutilization (5% GPU for QFormer); batching with pre-computed padded tensors achieves 20-50x speedup processing all spatial locations in single GPU-parallel call
  • Dimensionality reduction’s value depends on downstream model capacity: helps simple aggregation methods (mean/attention +0.05-0.09 ARI) by reducing noise and curse of dimensionality, but hurts sophisticated learnable models (qformer/llava_mlp -0.05 ARI) that can exploit richer information—no universal answer, must match preprocessing to model complexity
  • Per-screen vs global state duality requires explicit routing: local methods must delegate to global evaluators when mode semantics require system-wide decisions. Multi-file coordinated edits benefit from constant extraction, centralized orchestration (AppState.didSet → global re-eval), comprehensive localization
  • Multi-task evaluation infrastructure needs task-agnostic design from start; retrofitting requires coordinated changes across configs, scripts, and orchestration layers. Task registry pattern maps task names to datasets/checkpoints/prompts enabling clean extensibility
  • Large refactoring best practices: analyze dependencies (grep all imports) → migrate in topological order (dependees first, dependents later) → delete source files → global verification (grep residual references + dry run tests), ensuring every intermediate state is runnable. Cannot safely delete ’legacy’ code without comprehensive dependency analysis
  • Stub functions with TODO comments in production code can completely break features while appearing to work (no compiler errors, type checks pass). Critical to implement or remove stubs before shipping, not leave as technical debt

Debugging (debugging)#

  • PyTorch checkpoint loading with strict=False silently skips mismatched keys; missing instance attributes preventing key remapping can cause 17%+ parameters to use random initialization with no error messages. Always verify state_dict key matching, log mismatches explicitly, and validate model behavior after loading
  • Task distribution mismatch is critical but often overlooked failure mode: models trained exclusively on successful task demonstrations fail catastrophically on OOD scenarios (<1% success). Pi0.5 trained on 5 MimicGen tasks evaluated on different task → 0% success; BC-RNN/VLAs trained on normal demos achieve ~0% on error recovery
  • Observation space deficiencies have vastly different impacts across task complexities: simple tasks (Stack 98% SR) can compensate via vision; complex tasks (Coffee 0% SR) completely fail when missing 57-dim object state. Configuration bugs can be silent killers—loss decreases normally but policy learns nothing useful
  • Environment naming conventions have semantic implications: stripping ‘_D0’ suffix in robosuite/MimicGen changes which env class is instantiated, altering object observables returned. Base envs return fewer states than MimicGen variants causing dimension mismatches (65→37 dims)
  • Reproducibility in RL/robotics evaluation requires explicit seeding of both policy RNG and environment RNG; comparing against reference implementations reveals missing initialization steps that documentation may omit
  • Robust error handling at iteration boundaries (scene-level try-except) prevents cascade failures in long-running batch processes while preserving partial results via resume mechanisms. Database contained two fingerprint groups (519 vs 130 scenes) requiring graceful degradation
  • Training completion ≠ evaluation done. Even 100k-step trained models can sit unevaluated in research workflows. On HPC systems, job logs reflect execution history not outcome completeness—always verify completion via artifact inspection (checkpoints, outputs) with timestamps, not log tails
  • Pre-processed external datasets (like Visium HD from collaborators) require explicit transformation history tracking to prevent pipeline corruption. Use adata.uns[’log1p’] flag or X.max() heuristics to detect already-applied transformations and skip redundant normalization

Tools (tools)#

  • Vectorization delivers 100-1000x speedups for mathematical operations on arrays: replacing Python nested loops with scipy/NumPy primitives (cdist for pairwise distances, softmax for per-row normalization, advanced indexing) is almost always worth it—the key is recognizing O(n²) patterns and knowing the right library function
  • GPU↔CPU transfer elimination matters at scale: small transfers (edge_attr tensor) seem negligible but accumulate across many iterations (1100 epochs × multiple dropout calls) to 1-2 seconds total. Always generate random numbers on device-local data with torch.rand(…, device=edge_attr.device)
  • mclust probabilistic clustering scales poorly beyond ~5000 observations (40+ min for 17,502 bins); KMeans provides 360x speedup (2 min vs 6+ hours for 11 sections) with acceptable quality tradeoff (ARI ~0.17 vs 0.20-0.25) for exploratory spatial transcriptomics
  • Unbuffered output (PYTHONUNBUFFERED=1 + stdbuf -oL) is non-negotiable for monitoring long-running ML training jobs on remote clusters—buffering can hide failures for hours and make debugging impossible. Add tqdm for visibility of long-running operations
  • Proxy environment variables (http_proxy, https_proxy, HTTP_PROXY, HTTPS_PROXY) can intercept and break WebSocket connections to localhost even when raw TCP layer works—they operate at application layer before socket creation. Always unset all proxy vars when debugging local services
  • When loading 50+ GB models on shared GPUs, always check nvidia-smi and existing processes first before attempting new launches. Reusing zombie servers prevents CUDA OOM, saves 3-5min startup time, and enforces resource awareness and multi-user cluster etiquette
  • JAX CPU-only execution requires both CUDA_VISIBLE_DEVICES="" (device masking) and JAX_PLATFORMS=cpu (explicit backend selection); former alone causes ‘No GPU/TPU found’ errors
  • HPC compute nodes typically lack direct internet access; always disable cloud logging services (W&B, MLflow, Comet) via WANDB_MODE=disabled or configure offline mode for batch training jobs to prevent hanging on connection attempts
  • XLA_PYTHON_CLIENT_MEM_FRACTION controls JAX GPU memory pre-allocation: defaults to 0.75, can safely increase to 0.95 for dedicated training workloads to maximize available memory (from 61GB to 77.7GB per GPU) enabling 6-GPU parallel training
  • Vision transformer input requirements are strict: ViTs require input dimensions divisible by patch size (UNI2 ViT-H/14 needs H%14=0, W%14=0). When adapting preprocessing designed for one architecture to another, verify dimensional constraints early; 224×224 is safe default for most ViTs
  • High-resolution spatial transcriptomics (Visium HD 8µm vs Visium 55µm bin spacing) requires dynamic visualization parameter scaling—spot size must scale with bin spacing to prevent invisible or overlapping dots. HD’s 7.3px spot diameter needs size=4.0 vs Visium’s size=1.0 default
  • Xcode ENABLE_USER_SCRIPT_SANDBOXING must be set to NO to allow build scripts to modify build products like Info.plist. PBXFileSystemSynchronizedRootGroup (objectVersion 77) automatically discovers new source files without manual project.pbxproj edits

Domain Knowledge (domain)#

  • BC-RNN significantly outperforms Pi0.5 Phoenix on normal MimicGen tasks (Stack D0: 98% vs 24%) but both fail catastrophically at error recovery (~0% success rate). Suggests behavior cloning from demonstrations works well for in-distribution tasks but doesn’t generalize to error scenarios without explicit training on recovery trajectories
  • RM-IDEAL Spearman correlation benchmark reveals STAIG fusion (r=0.396) significantly outperforms simple fusion strategies (concat/mean ≈0.25) by explicitly modeling spatial dependencies in gene expression embeddings. Learned fusion methods consistently rank higher than naive concatenation or averaging
  • Leiden community detection provides robust auto-clustering for unlabeled spatial transcriptomics data via k-NN graph construction with resolution parameter controlling granularity. Combined with silhouette score, enables unsupervised evaluation when ground truth labels unavailable (Visium HD high-resolution datasets)
  • VisiumHD’s multi-resolution bin system (2um native with 4×4 aggregation to 8um) requires integer division coordinate transforms (r_big//4) for pathologist annotation alignment; coordinate mapping empirically derived, not documented in Zenodo metadata
  • VLA policy evaluation requires exact library version matching with training environment—even minor robosuite/mimicgen version differences cause API incompatibilities (missing methods like get_bounding_box_half_size, changed signatures like env.seed()). Version mismatches can silently invalidate benchmarking results
