Problems and Solutions#

1. Pi0.5 full fine-tuning requires ~62GB/GPU. JAX FSDP=4 cannot effectively reduce memory. Repeated OOM in [Error Recovery Benchmark] (2026-02-23)#

Solution: Switch to the gemma_2b_lora+gemma_300m_lora architecture, which reduces training parameters by ~90% and reduces memory to ~22.5GB/GPU. Job 46553 ran smoothly at 2.0s/step.

2. All three-way evaluations in M14 crashed on scene 122/649: ~130 natural_* scenario xml_hash incompatibilities with the current mimicgen environment caused EnvironmentMismatchError, leading to a complete evaluation process crashing [Error Recovery Benchmark] (2026-02-23)#

Solution: Add a try-except for EnvironmentMismatchError in collect_on_scenes() of collector.py. Skip incompatible scenarios and record warnings. Resume the process with –resume. It was confirmed that there were approximately 519 actual effective evaluation scenarios.

3. Training logs had serious distortions due to stdout buffering: Pi0.5 training showed only ~580 steps, while actually reaching step 3000. This led to misjudgment of training termination and unnecessary restarts [Error Recovery Benchmark] (2026-02-23)#

Solution: Standardize the addition of PYTHONUNBUFFERED=1+stdbuf -oL in the training script and sbatch submission scripts. For long-term training, use sbatch to submit independent batch jobs and decouple them from the interactive session lifecycle.

4. The success rate in Pi0.5 evaluation was 0%, which was misjudged as a model quality issue [Error Recovery Benchmark] (2026-02-26)#

Solution: It was confirmed that the root cause was that the training task (MimicGen 5 tasks) did not overlap with the evaluation task (PickPlace). Instead, it was evaluated on the training task (Coffee/Stack/Threading/Assembly).

5. Observation dimension mismatch in BC-RNN evaluation (Expected 65, got 37) [Error Recovery Benchmark] (2026-02-26)#

Solution: Add import mimicgen to create env.registry during create_env(). Do not strip the _D0 suffix anymore. Keep all MimicGen observables (the _D0 environment provides additional object state observables).

6. In scGPT, TransformerModel.init omitted self.use_fast_transformer, causing key remapping in load_pretrained() to never execute. This led to random initialization of 17.7% attention weights. When strict=False, this was silently ignored [MIHD] (2026-02-28)#

Solution: Add a line of self.use_fast_transformer = use_fast_transformer to model.py. After fixing this, ARI improved by 44.4%, and all 11 DLPFC slices’ embeddings were re-extracted.

7. WebSocket client always encountered ConnectionRefusedError. Attempting different address formats (localhost/127.0.0.1/0.0.0.0) failed [Error Recovery Benchmark] (2026-02-28)Solution: It was found that the http_proxy environment variable was set to 127.0.0.1:10087. The websockets library failed to connect through this proxy. After unsetting all proxy variables at the beginning of run_eval.sh, the connection succeeded.#

8. The CoffeeMachineBodyObject lacks the get_bounding_box_half_size() method. An AttributeError was reported during the rollout initialization of the MimicGen Coffee task due to an interface gap between MimicGen and the current robosuite version. [Error Recovery Benchmark] (2026-02-28)#

Solution: The entire call chain (coffee_machine.py → CoffeeMachineBodyObject → CompositeBodyObject → MujocoXMLObject) was traced back. The get_bounding_box_half_size() method was implemented in each of the three different levels of base classes.

9. In HD data, the uns[‘spatial’] images appear to be full-sized but their coordinates are in the fullres pixel space after cropping. This mismatch leads to visual misalignment. Additionally, since X has been logged transformed, the pipeline will again perform normalize_total + log1p, resulting in a double transformation. [MIHD] (2026-02-28)#

Solution: In load_hd_data(), replace uns[‘spatial’] images with cropped_fullres.tif and recalculate the scalefactor. Add a skip_log parameter to preprocess_data(). For HD data with skip_log=True, only perform HVG filtering.

10. The Coffee BC-RNN loss converged normally (-7.66→-15.4), but the SR remained at 0% for a long time. Tasks configured with the same parameters achieved 64% after 20 epochs. This performance is highly similar to that of models with insufficient capacity. [Error Recovery Benchmark] (2026-03-01)#

Solution: By comparing the actual obs key structure in the HDF5 dataset, it was found that Coffee requires a 57-dimensional object state (required for multi-stage precise operations). The configuration template lacked this field. Add extra_low_dim=[‘object’] to all 9 tasks and regenerate the configuration.

11. There were four consecutive bugs when accessing image observations with robomimic BC-RNN: The logic for enabling the camera, the checkpoint config in JSON string format, issues with obtaining image resolution, and problems with converting from HWC to CHW formats. [Error Recovery Benchmark] (2026-03-01)#

Solution: ① Check the rgb modality from the checkpoint config JSON string (which requires json.loads). ② Read the actual 84×84 size from shape_metadata. ③ Manually perform HWC→CHW transposition and conversion from uint8 to float32/255 normalization (this is automatically handled by ObsUtils.process_obs in standard rollouts).

12. There were four consecutive bugs when starting Pi0.5 LoRA training: ① Setting JAX_PLATFORMS=cpu alone was ineffective. ② The norm_stats path was incorrect. ③ The boolean parameter –overwrite=False was ineffective. ④ Access to W&B network interfaces on HPC nodes was blocked. [Error Recovery Benchmark] (2026-03-01)#

Solution: ① Set CUDA_VISIBLE_DEVICES="" + JAX_PLATFORMS=cpu simultaneously. ② The actual path for norm_stats is assets//benchmark//norm_stats.json. ③ Use –no-overwrite instead of –flag=False. ④ Preset WANDB_MODE to disabled.

13. Adapting the pipeline path for VisiumHD HD data: Failure occurred when finding the image path for the vision encoder and loading spatial coordinates during the fusion stage, as it assumed the DLPFC directory structure. [MIHD] (2026-03-01)#

Solution: Add a cropped_fullres.tif pattern recognition to find_spatial_image(). Pass crop_dir.parent as data_root when calling the vision encoder. In the fusion stage, directly load coordinates from data.obsm[‘spatial’] without looking up the path.

14. When calculating norm_stats, the GPU util value was 0, but the memory was fully occupied. The task did not progress for a long time. The root cause was that PaligemmaTokenizer eagerly loaded all visible GPUs when calling get_config().model. [Error Recovery Benchmark] (2026-03-01)#

Solution: Set CUDA_VISIBLE_DEVICES="" + JAX_PLATFORMS=cpu simultaneously to force the CPU mode (norm_stats does not require model inference at all). Add CPU parallelism and sub-sampling (max-frames 10000) to reduce the calculation time from 10+ minutes to 2.5 minutes.

Learning Outcomes#

Architecture#

  • JAX FSDP only divides parameter storage, which does not affect the loading of the complete model during initialization or forward activation memory. The full training state for Pi0.5 requires ~62GB/GPU, and a single A800 cannot handle full fine-tuning with 80GB. LoRA (gemma_2b_lora) is the only feasible solution for the A800 cluster, reducing the memory requirement to ~22.5GB. Switch to LoRA when encountering the warning “Can’t reduce memory use below 62.46GiB”.
  • The observation keys configured for BC-RNN must match the task complexity: Tasks with multi-stage precise operations like Coffee require a 57-dimensional object state. A single 84×84 image cannot provide sufficient spatial precision. The performance of configuration bugs (normal loss but SR=0) is highly similar to insufficient model capacity. It is necessary to distinguish between them by comparing dataset structures.
  • Vectorization and batching significantly accelerate the process: O(n²) nested loops can be reduced by about 100-500x through cdist matrix operations. Python’s loop iteration can also be accelerated by about 20-50x through batch GPU forward operations. Norm_stats calculations do not require model inference at all. Forcing the CPU mode, sub-sampling, and parallelism can reduce the calculation time from 10+ minutes to 2.5 minutes.
  • Methodology for restructuring large repositories: Systematically grep to verify all imported dependencies, then execute in the order of “creating new modules → updating references → deleting old files”, and perform a dry-run verification. Before merging functions with the same name but different variants, carefully compare the boundary conditions (>vs>=vs< can lead to completely opposite results).

Debugging#

  • PyTorch model.load_state_dict(strict=False) will silently ignore non-matching keys. Keyed weights may run for a long time with random values without causing any errors. Production code should actively print missing_keys/unexpected_keys and verify the statistical values of parameters. There may also be similar bugs in the official open-source repository. It is necessary to audit the weight loading path actively rather than blindly trusting it.
  • Key details of robomimic BC-RNN inference: The checkpoint config is stored as a JSON string (which requires json.loads, not direct dict access). When bypassing the standard rollout process, manual HWC→CHW transposition and uint8→float32/255 normalization are required (this is automatically handled by ObsUtils.process obs in standard rollouts).

Domain Knowledge#

  • Key conclusions from M14: BC-RNN SR=0.28%, Pi0/Pi0.5 SR=0%. The Recovery Progress of Random (0.0199) was actually higher than all learning strategies. This directly proves the necessity of LoRA fine-tuning in M15.
  • The 649 scenario database consists of two types of environments: ~519 impulse/augmented compatible scenarios (xml_hash matching) and ~130 natural_* natural capture scenarios (xml_hash different). There were approximately 519 effective evaluation scenarios in M14. All document and target episode numbers need to be updated accordingly.
  • STAIG fusion is overall leading in the RM-IDEAL evaluation (average r=0.396 for 151673, peak r=0.644 for Layer_3). Multimodal fusion has a much better ability to capture spatial niche structures than single-modal baselines (gene/vision average r≈0.06-0.12). The effect of Vision Refine is bipolar: It is beneficial for strong fusion (attention +0.086 ARI), harmful for weak fusion (-0.054 for qformer). Strong fusion already has the ability to handle high-dimensional inputs, and reducing dimensions actually results in a loss of information.
  • Cross-task evaluation is a common pitfall in robot learning: If the training set task does not overlap with the evaluation task, it directly leads to 0% SR. It is necessary to ensure consistency between train/eval tasks during the experimental design phase. The MimicGen _D0 environment provides additional object observables. Removing the _D0 suffix from observations reduces the dimensionality from 65 to 37, disrupting the input consistency during reasoning.
  • VisiumHD coordinate mapping: 2um bin of r_big//4 results in an 8um grid row (similar for col). The visualization size parameters of scanpy spatial are multiples of spot_diameter_fullres. For HD with 8μm diameter (diameter≈7.3), the size should be approximately 4.0. For DLPFC with a diameter of approximately 144, use size=1.0. The Leiden community found that resolution=1.0 is the default clustering strategy for unlabeled HD data (crop10large k=17, Silhouette=0.302).

Tools#

  • Best practices for Slurm cluster training projects: For long-term training, sbatch should be used instead of srun to avoid session timeout due to SIGTERM. All training scripts should be standardized with PYTHONUNBUFFERED=1 + stdbuf -oL. Before starting the job, use fuser /dev/nvidiaX to clean up zombie GPU processes (monitoring tools like nvitop also hold CUDA contexts). MUJOCO_EGL_DEVICE_ID corresponds to the physical GPU number, not the logical CUDA_VISIBLE_DEVICES number.
  • Key configurations for JAX/Pi0.5 training: ① Set CUDA_VISIBLE_DEVICES="" + JAX_PLATFORMS=cpu simultaneously. ② XLA_PYTHON_CLIENT_MEM_FRACTION=0.95 is effective for A800 with 80GB (from 61 to 77.7GB). ③ HPC nodes must have WANDB_MODE set to disabled. ④ The argparse boolean flag –no-overwrite cannot be used instead of –flag=False. ⑤ The openpi norm_stats path is assets//benchmark//norm_stats.json.
  • The HTTP/HTTPS proxy environment variable (http_proxy) will be transparently proxied by the Python websockets library to establish WebSocket connections. When debugging local services on HPC clusters, it is necessary to unset the proxy variables. The MimicGen environment registers with robosuite through the side effects of importing mimicgen. It is necessary to explicitly perform this import before calling external tools. It is necessary to trace back the entire inheritance chain and implement methods that are missing in third-party libraries in all base classes. Only fixing the point where it is called most recently will cause other subclasses to still report errors.
  • The pam_slurm_adopt cluster’s GPU nodes do not support direct SSH access. srun –overlap can attach a new step to an existing interactive job to reuse node resources. Robomimic does not have a TTY environment, and will exit with EOFError if it encounters an existing checkpoint directory. It is necessary to completely clean the directory before starting each time. The checkpoint directory is the most reliable evidence of training completion. A SLURM job crashing does not mean that the training has failed.
