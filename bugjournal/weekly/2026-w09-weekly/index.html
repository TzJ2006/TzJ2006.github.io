Problems and Solutions#

1. Pi0.5 full fine-tuning requires ~62GB/GPU; JAX FSDP=4 cannot effectively reduce memory (only shards storage, not activation memory), causing repeated OOM [Error Recovery Benchmark] (2026-02-23)#

Solution: Switched to gemma_2b_lora+gemma_300m_lora architecture, reducing trainable parameters by ~90% and memory to ~22.5GB/GPU; Job 46553 running stably at 2.0s/step.

2. All three M14 evaluations crashed at scene 122/649: ~130 natural_* scenes have xml_hash incompatible with the current mimicgen environment, triggering uncaught EnvironmentMismatchError and crashing the entire evaluation process [Error Recovery Benchmark] (2026-02-23)#

Solution: Added try-except for EnvironmentMismatchError in collect_on_scenes() in collector.py; skips incompatible scenes and logs warnings; restarted with --resume, confirmed ~519 effective evaluation scenes.

3. Training logs severely distorted by stdout buffering: Pi0.5 training appeared to have run only ~580 steps but had actually reached step 3000, causing false diagnosis of training halt and unnecessary restarts [Error Recovery Benchmark] (2026-02-23)#

Solution: Standardized addition of PYTHONUNBUFFERED=1+stdbuf -oL in training scripts and sbatch submission scripts; long-running training switched to sbatch independent batch jobs, decoupling from interactive session lifetime.

4. Pi0.5 evaluation success rate was 0%, misdiagnosed as a model quality issue [Error Recovery Benchmark] (2026-02-26)#

Solution: Confirmed root cause as zero overlap between training tasks (MimicGen 5 tasks) and evaluation task (PickPlace); switched to evaluating on training tasks (Coffee/Stack/Threading/Assembly).

5. BC-RNN evaluation observation dimension mismatch (Expected 65, got 37) [Error Recovery Benchmark] (2026-02-26)#

Solution: Added import mimicgen registration in create_env(), no longer strips the _D0 suffix, preserving full MimicGen observables (the _D0 environment provides additional object state observables).

6. scGPT TransformerModel.__init__ missing self.use_fast_transformer, causing the Wqkv→in_proj_ key remapping in load_pretrained() to never execute, randomly initializing 17.7% of attention weights, silently ignored by strict=False [MIHD] (2026-02-28)#

Solution: Added self.use_fast_transformer = use_fast_transformer in model.py; after the fix, average ARI improved by 44.4%, re-extracted embeddings for all 11 DLPFC sections.

7. WebSocket client consistently getting ConnectionRefusedError; trying different address formats (localhost/127.0.0.1/0.0.0.0) all failed [Error Recovery Benchmark] (2026-02-28)#

Solution: Discovered http_proxy environment variable was set to 127.0.0.1:10087, causing the websockets library to transparently proxy WebSocket connections; unset all proxy variables at the start of run_eval.sh resolved the issue.

8. CoffeeMachineBodyObject missing get_bounding_box_half_size() method, causing AttributeError during MimicGen Coffee task rollout initialization (interface gap between MimicGen and current robosuite version) [Error Recovery Benchmark] (2026-02-28)#

Solution: Traced the full call chain (coffee_machine.py→CoffeeMachineBodyObject→CompositeBodyObject→MujocoXMLObject) and implemented get_bounding_box_half_size() in three different base classes.

9. HD data uns['spatial'] image is the full slide but coordinates are in post-crop fullres pixel space, causing misaligned visualization; also X was already log-transformed, and the pipeline would apply normalize_total+log1p again causing double transformation [MIHD] (2026-02-28)#

Solution: In load_hd_data(), replaced uns['spatial'] image with cropped_fullres.tif and recalculated scalefactor; added skip_log parameter to preprocess_data(), passing skip_log=True for HD data to only perform HVG filtering.

10. Coffee BC-RNN loss converges normally (-7.66→-15.4) but SR remains 0%; Stack with the same configuration reaches 64% SR by epoch 20 — closely resembling insufficient model capacity [Error Recovery Benchmark] (2026-03-01)#

Solution: Compared actual obs key structure in HDF5 datasets and discovered Coffee requires 57-dimensional object state (essential for multi-stage precise manipulation); configuration template was missing this field. Added extra_low_dim=['object'] for all 9 tasks and regenerated configurations.

11. robomimic BC-RNN image observation integration failing at 4 consecutive layers: camera enable detection logic / checkpoint config in JSON string format / image resolution retrieval / HWC→CHW format conversion all had issues [Error Recovery Benchmark] (2026-03-01)#

Solution: ① Detect rgb modality from checkpoint config JSON string (requires json.loads); ② read actual 84×84 size from shape_metadata; ③ manually HWC→CHW transpose + uint8→float32/255 normalization (handled automatically by ObsUtils.process_obs in standard rollout).

12. Pi0.5 LoRA training startup blocked by 4 consecutive bugs: ① JAX_PLATFORMS=cpu alone was ineffective; ② incorrect norm_stats path; ③ boolean argument --overwrite=False was ineffective; ④ HPC node W&B blocked from external network [Error Recovery Benchmark] (2026-03-01)#

Solution: ① Set both CUDA_VISIBLE_DEVICES="" and JAX_PLATFORMS=cpu; ② actual norm_stats path is assets/<config>/benchmark/<dataset>/norm_stats.json; ③ use --no-overwrite instead; ④ pre-set WANDB_MODE=disabled.

13. VisiumHD data pipeline path adaptation: vision encoder image path lookup and fusion stage spatial coords loading both assumed DLPFC directory structure and failed [MIHD] (2026-03-01)#

Solution: Added cropped_fullres.tif pattern recognition to find_spatial_image(); passed crop_dir.parent as data_root for vision encoder; loaded coordinates directly from adata.obsm['spatial'] in fusion stage to bypass path lookup.

14. GPU util=0 but VRAM fully loaded during norm_stats computation, task stalled indefinitely; root cause: PaligemmaTokenizer eagerly loaded onto all visible GPUs when get_config().model was called [Error Recovery Benchmark] (2026-03-01)#

Solution: Set both CUDA_VISIBLE_DEVICES="" and JAX_PLATFORMS=cpu to force CPU mode (norm_stats computation requires no model inference at all); added CPU parallelism + subsampling (max-frames 10000), reducing computation time from 10+ minutes to 2.5 minutes.

Lessons Learned#

Architecture#

  • JAX FSDP only shards parameter storage — it does not affect full model loading during initialization or forward activation memory. Pi0.5’s full training state requires ~62GB/GPU, exceeding a single A800 80GB capacity; LoRA (gemma_2b_lora) is the only viable option on the A800 cluster, bringing memory down to ~22.5GB. On seeing the ‘Can’t reduce memory use below 62.46GiB’ warning, switch to LoRA immediately.
  • BC-RNN observation keys must match task complexity: multi-stage precision tasks like Coffee require 57-dimensional object state that 84×84 images alone cannot provide sufficient spatial precision for. The signature of a configuration bug (normal loss but 0% SR) closely resembles insufficient model capacity — comparing dataset structure is required to distinguish them.
  • Vectorization and batching yield dramatic speedups: O(n²) nested loops → cdist matrix operations ~100–500x; Python per-spot loops → batched GPU forward ~20–50x. norm_stats computation requires no model inference at all — forcing CPU mode + subsampling + parallelism reduces it from 10+ minutes to 2.5 minutes.
  • Large repository refactor methodology: systematically grep-verify all import dependencies → execute in “create new module → update references → delete old file” order → dry_run validation. Functions with the same name but multiple variants require careful comparison of edge cases before merging (> vs >= vs < can completely invert behavior).

Debugging#

  • PyTorch model.load_state_dict(strict=False) silently ignores mismatched keys — critical weights can run at random values for a long time without any error. Production code should proactively print missing_keys/unexpected_keys and validate parameter statistics. Upstream open-source official repositories can also have the same class of bug; weight-loading paths require active auditing, not blind trust.
  • Critical robomimic BC-RNN inference details: checkpoint config is stored as a JSON string (requires json.loads, not direct dict access); when bypassing the standard rollout pipeline, HWC→CHW transpose and uint8→float32/255 normalization must be done manually (handled automatically by ObsUtils.process_obs in standard rollout).

Domain Knowledge#

  • M14 key conclusion: BC-RNN SR=0.28%, Pi0/Pi0.5 SR=0%, Random policy Recovery Progress (0.0199) exceeds all learned policies — learned policies suffer severe out-of-distribution generalization failure in error injection scenarios, directly proving the necessity of M15 LoRA fine-tuning.
  • The 649-scene database consists of two environment types: ~519 impulse/augmented compatible scenes (matching xml_hash) and ~130 natural_* naturally-captured scenes (different xml_hash). M14 actual effective evaluation scenes are ~519; all documentation and target episode counts need to be updated accordingly.
  • STAIG fusion ranked first in RM-IDEAL evaluation across the board (section 151673 average r=0.396, Layer_3 peak r=0.644), demonstrating that multimodal fusion captures spatial niche structure far better than single-modality baselines (gene/vision avg r≈0.06–0.12). Vision Refine shows polarized effects: beneficial for weak fusion (attention +0.086 ARI) but harmful for strong fusion (qformer -0.054) — strong fusion already has the capacity to handle high-dimensional inputs, and dimensionality reduction loses information.
  • Cross-task evaluation is a common pitfall in robot learning: zero overlap between training and evaluation tasks directly causes 0% SR; train/eval task consistency must be ensured at experimental design time. MimicGen _D0 environments provide additional object observables; stripping the _D0 suffix reduces observation dimensions from 65 to 37, breaking input consistency at inference time.
  • VisiumHD coordinate mapping: 2μm bin’s r_big//4 gives the 8μm grid row (same for col); scanpy spatial visualization size parameter is a multiple of spot_diameter_fullres — HD at 8μm (diameter≈7.3) needs size≈4.0, DLPFC (diameter≈144) uses size=1.0; Leiden community detection (resolution=1.0) is the default clustering strategy for unannotated HD data (crop10large k=17, Silhouette=0.302).

Tools#

  • Slurm cluster training best practices: long-running training must use sbatch (not srun, to avoid SIGTERM on session timeout); standardize PYTHONUNBUFFERED=1+stdbuf -oL in all training scripts; before launching, use fuser /dev/nvidiaX to clean up zombie GPU processes (monitoring tools like nvitop also hold CUDA contexts); MUJOCO_EGL_DEVICE_ID corresponds to physical GPU numbers, not CUDA_VISIBLE_DEVICES logical numbers.
  • JAX/Pi0.5 training key configurations: ① CPU mode requires both CUDA_VISIBLE_DEVICES="" and JAX_PLATFORMS=cpu; ② XLA_PYTHON_CLIENT_MEM_FRACTION=0.95 is effective for A800 80GB (61→77.7GB); ③ HPC nodes must pre-set WANDB_MODE=disabled; ④ argparse boolean flags use --no-overwrite, not --flag=False; ⑤ openpi norm_stats path is assets/<config>/benchmark/<dataset>/norm_stats.json.
  • HTTP/HTTPS proxy environment variables (http_proxy) are transparently applied to WebSocket connections by the Python websockets library; unset proxy variables when debugging local services on HPC clusters. MimicGen environments are registered into robosuite as a side effect of import mimicgen — this import must be explicitly called before any external tool invocations. Fixing missing methods in third-party libraries requires tracing the full inheritance chain and implementing in all base classes; patching only the nearest call site will leave other subclasses still failing.
  • pam_slurm_adopt cluster GPU nodes do not support direct SSH; srun --overlap can attach a new step to an existing interactive job to reuse node resources; robomimic exits with EOFError in non-TTY environments when a checkpoint directory already exists — the directory must be fully cleaned before each launch. The checkpoint directory is the most reliable evidence that training completed; a SLURM job crash does not mean training failed.
