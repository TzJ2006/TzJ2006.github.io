Problems and Solutions#

1. Drop Skill: The OSC controller actively maintains the EEF position (impedance control) in env.step(), and after the claw is opened, the object is held by a finger and cannot fall freely [ErrorRecoveryBenchmark] (2026-03-15)#

Solution: Bypass the controller and directly set MuJoCo qpos/qvel, then call mujoco.mj_step() to run 15 physical steps for initial separation, followed by standard control cycles

2. MIHD sample-crossing embedding comparison ineffective: per-section independent HVG selection + independent PCA fitting leads to non-comparable feature spaces, the conclusion that ‘PCA is better than STAIG’ indicates a methodological error [MIHD] (2026-03-15)#

Solution: Use the raw_shared intersection of shared HVGs (1137 genes) and unified processing as the correct baseline. Load directly from the original HDF5 instead of relying on per-section caches (there is an integerization bug in var_names)

3. Stack name parsing silent failure: stack.yaml uses cubeA/cubeB, MuJoCo’s actual name is cubeA_main, _sim_body_name2id returns -1, Python negative index causes all task phase detections to be misjudged as pre_reach [ErrorRecoveryBenchmark] (2026-03-15)#

Solution: Fix the body name segment, add a main/_body0 suffix backtracking logic in sim_body_name2id, and handle failure output by checking WARNING instead of silently returning -1

4. pi05 training is 33% slower than openpi (20h vs 15h), intuition suggests hardware differences, but the root cause is unknown [VLA-RoboTwin/pi05] (2026-03-11)#

Solution: By comparing parallel sub-proxies at the software level (pyproject.toml/uv.lock/wandb logs), identify the main reason as JAX version differences (0.5.0 vs 0.5.3). XLA compiler optimization has accumulated over time; use uv override-dependencies to resolve lerobot torch version upper limit conflicts

5. curobo pre-compiled.so and torch 2.7.1 ABI incompatibility (undefined symbol), JIT recompilation failed due to non-standard CUDA header path in conda [VLA-RoboTwin] (2026-03-12)Solution: Set CUDA_HOME to the root directory of the conda environment, set CPATH to targets/x86_64-linux/include/, and use pip install -e to recompile from source.#

6. Severe imbalance in online quota generation: premature_release naturally captures 7233 items, with 7 types completely at zero. The distribution of policy behavior is uneven and uncontrollable [ErrorRecoveryBenchmark] (2026-03-09)#

Solution: Establish an offline injection architecture: first collect trajectories during a full rollout, index injectable points offline, then selectively inject according to quotas; skip over已满 types.

7. CalendarPro intention misclassification: no confidence threshold (0.52 is also considered a valid classification), time expressions are misjudged by keyword routing, short confirmation words have no context understanding, Chinese tokens are estimated with a 3-fold deviation [CalendarPro] (2026-03-15)#

Solution: Add per-routing confidence thresholds (0.40-0.60), introduce a keyword scorer combined with an embedding routing method of 70/30 ratio. Split the system prompt into base and 11 fragments for injection as needed, and estimate Chinese tokens at ×1.5.

8. S2 co-author analysis is completely ineffective for top scholars (Levine/Abbeel/Finn, etc.) (all depth-2 fields are empty). There is a serious name ambiguity issue with Xiaolong Wang/Shuran Song [gadget] (2026-03-15)#

Solution: Reconstruct the strategy to prioritize homepage-first: preferentially capture student lists from professors’ personal pages, and co-authorship should only serve as a supplement. Use multiple strategies for URL discovery. Mark warnings for name ambiguities and recommend using S2 authorId for precise queries.

9. Incorrect assumption in VLA context replay architecture: a complete N-1 frame replay mechanism was designed, but most VLAs do not have a context window. This overhead is an ineffective design [ErrorRecoveryBenchmark] (2026-03-15)#

Solution: Reconstruct ContextReplayEngine into InjectionEngine, directly restoring the injected frame sim state. Limit data sources to MimicGen demo data, which is more controllable.

10. RTX 2080 Ti + PyTorch 2.9.0 triggers cuBLAS CUBLAS_STATUS_EXECUTION_FAILED for high-dimensional tensors with N>3500 [MIHD] (2026-03-09)#

Solution: First project to hidden_dim(128) using Linear projection, then index neighbors (to avoid high-dimensional large tensors entering cuBLAS). Change InfoNCE to a mini-batch comparison loss (batch_size=512).

11. MCP Server tool returns file paths, AI cannot consume the content directly [gadget] (2026-03-09)#

Solution: Reconstruct the tool to call underlying functions directly without using cmd_* wrappers, returning complete content (markdown/JSON). Control file writing through save parameters.

Learning Outcomes#

Architecture#

  • The necessary prerequisite for comparing embeddings across samples is shared feature space: independent HVG selection per section + independent PCA fitting equals dual irrelevance. An effective baseline must use a shared HVG intersection + joint processing, or a foundation model with fixed pre-trained weights.
  • There is a fundamental conflict between direct state operations in MuJoCo and the feedback controller (OSC): sim.forward() only updates kinematics, while mujoco.mj_step() advances dynamics and bypasses the controller. The simulation injection design must clearly choose one path.
  • The semantic split of error types should be based on ‘whether the recovery strategy is the same’ rather than ‘whether the injection mechanism is the same’: the detection conditions and recovery logic for drop_in_transit/drop_at_wrong_place/drop_with_interaction are completely different. Even if the injection actions are the same, they must be modeled separately.
  • The semantic router architecture has a flaw: embedding nearest neighbors always produce results that cannot express ‘uncertainty’. Combining confidence thresholds + alternative LLMs + keyword scorers is the most practical repair model, which can be applied to all vector retrieval-based classification systems (RAG routing, tool selection, etc.).
  • MCP tools should aim to consume AI as their primary goal: return complete content, with file writing being an optional side effect. General benchmarks should not assume that the model has a context window. InjectionEngine directly restoring the sim state is more versatile than context replay.
  • For top scholars (500+ papers), analysis of co-author frequencies cannot identify students - the first-author signal is diluted by many collaborators. Professors’ homepages explicitly list students, which is a factor of reliability. The citation graph (forward + backward) is a core feature of research toolchains. ‘Relevance’ and ’number of citations/popularity’ should be handled independently.
  • Offline injection architecture is better than online quota systems for building balanced error scenario datasets: decouple ’exploring injectability’ from ’executing injections’, allowing precise control over the number of each type of error; online natural capture is greatly affected by policy behavior distribution and cannot control type balance.

Debugging#

  • Upgrading to a new JAX version (0.5.0→0.5.3) can improve training speed by about 33%. The cumulative effect of XLA compiler optimization cannot be ignored.
  • In the conda environment, compile CUDA extensions: set CUDA_HOME to the root directory of the conda environment, and set CPATH to envs//targets/x86_64-linux/include/ (not /usr/local/cuda/include/); after upgrading torch to a new version, all .so files that depend on torch C++ ABI need to be recompiled.
  • Assumptions about the internal behavior of third-party frameworks must be verified through actual running: LeRobot automatically squeezes single-scalar feature into batch_size for DataLoader when shape=(1,) is inferred. This inference is unreliable. The actual effective values in training configuration need to be verified from wandb logs. Code default values may be overridden by CLI parameters.
  • Monitoring GPUs within K8s containers: scan/proc//fd/ for devices linked to/dev/nvidia*, and prioritize reading CUDA_VISIBLE_DEVICES. Processes that open all GPU devices but do not occupy memory are usually monitoring tools, which can be filtered based on this information.
  • Silent failures are the most dangerous bug mode: body_xpos[-1] negative index always returns the same position in two cubes; integerizing cached var_names results in zero intersection of gene names. Any parsing failure should immediately trigger a WARNING instead of returning a sentinel value. Perform integrity checks before using cached data.

Domain Knowledge#

  • scGPT independent benchmark (Genome Biology 2025) confirms that zero-shot is inferior to PCA/scVI; scGPT-spatial only compares with weak baselines (ARI≈0.30-0.40). SOTA (GraphST, ARI≈0.55-0.63) was not included, and there is no independent third-party verification. When evaluating new methods, it is necessary to verify whether their baseline is the current SOTA.
  • CALVIN evaluation is a purely online simulation that does not read episode data at all. Only validation/.hydra/merged_config.yaml is needed. The 1.3GB dataset can be compressed into a 600KB eval-only version.
  • A list of humanoid AI scholars and their mentors: Mingyu Ding←Jitendra Malik, Ruoshi Liu←Carl Vondrick, Xiaolong Wang←Abhinav Gupta, Shuran Song←Thomas Funkhouser, Yunzhu Li←Antonio Torralba, Yuke Zhu←Li Fei-Fei - presenting a systematic group of top-level perception/robotics mentors to humanoid AI.
  • Stream matching is becoming the mainstream action decoding architecture for VLA. Pi0 time convention: t=1 for pure noise → t=0 for target action. Using adaRMS to inject time conditions in Pi0.5 is better than simple splicing. In VLA-assisted tasks, isolating the main task gradient with stop_gradient is a safe starting point.

Tools#

  • Prompt engineering injection strategy: Split the system prompt into base (~50 lines) + intent-specific fragments (dynamically injected by category), which can reduce token consumption by 40-60%. The token density for Chinese characters is about 6 times that of English characters (1.5 tokens/character vs 0.25 tokens/character). Without correction, the context length will be systematically underestimated.
  • For projects with multiple tools coexisting, the output directory should be organized according to ‘file type priority’ (outputs/reports/summarize/, not summarize/reports/). The.gitignore can be simplified to a single line outputs/. The Python re-export shim mode (only containing from x import y; all=[…]) is an elegant migration method that remains backward compatible.
  • The PubMed esearch→efetch two-step E-utilities API can index metadata of subscription journals such as Nature/Cell/Science for free. The bioRxiv API is also open, and neither requires new dependencies (urllib.request). Verifying the feasibility of a pipeline with a small batch is better than using a full load.

Notes on AI Usage#

Effective Patterns:

  • ✓ Parallel sub-proxies accelerate multi-dimensional code analysis: rely on version differences for diagnosis. Start 3+ sub-proxies simultaneously during code library exploration to cover different file sets, significantly reducing analysis time.
  • ✓ Goal-driven delegation + iterative debugging loop: Users provide clear termination conditions (‘fix until no errors’). AI runs iteratively independently → reports an error → makes the smallest modification possible. There is an built-in error-correction mechanism.
  • ✓ Deep code library exploration identifies architectural challenges: CrossModalEnhancer actively detects degradation problems in single spots KV (each spot has only one vector), and proposes a solution to build KV sequences from spatial neighbors.
  • ✓ Hack sys.path → progressive refactoring of common/packages: The re-export shim mode remains backward compatible while eliminating duplicate code.
  • ✓ Verifying the feasibility of a small batch pipeline is better than using a full load (207 scenarios reveal 5 systematic defects). End-to-end integration testing reveals implicit dependencies at the pipeline level more effectively than unit testing.

Limitations:

  • ✗ Insufficient ability to reflect on experimental conclusions: Directly attribute numerical results without actively questioning the validity of the experimental design. The methodological flaws in MIHD embedding need to be corrected by users through external triggers.
  • ✗ Silent failure modes were not actively detected: Names of Stack bodies returned -1, negative indices in Python, integerizing cached var_names. All these require user discovery due to lack of integrity checks.
  • ✗ Over-engineering and incorrect architectural assumptions: VLA context replay is based on the wrong assumption that ‘all VLAs need a context window’. Incorrect inferences about LeRobot’s shape behavior were made and code was modified, which requires user correction or verification.
  • ✗ Insufficient ability to actively question the applicability of methodologies: S2 students continue to debug the code logic when they discover that it is ineffective. They do not actively question the boundaries of the methodology itself and need guidance from users before turning to the homepage solution.
  • ✗ Weak ability to handle name ambiguities in Semantic Scholar: There is a lack of active entity disambiguation for common Chinese-English translations. LLM analysis cannot automatically identify ambiguous researchers.
  • ✗ The signature was not verified before using APIs: The FastMCP version parameter and conda –no-banner were discovered to be incompatible only after running the program.
