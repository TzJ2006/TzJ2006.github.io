Problems & Solutions#

1. Error Recovery Benchmark v4’s physics-force injection cannot generate VLA-compatible temporal context - errors injected as initial states lack the behavioral history VLA models require [Error Recovery Benchmark v5.0] (2026-03-10)#

Solution: Designed v5.0 architecture with offline two-stage pipeline: Stage 0 collect clean trajectories (human + 10×VLA policy success runs) → Stage 1 offline scan opportunities per frame → Stage 2 quota scheduler (100 per task×category) → Stage 3 context replay (replay first N-1 frames to build VLA context window, inject error at frame N, let VLA freely infer from N onward). Shifted from force-based injection to action-based Error Skill modules.

2. PyTorch 2.9/CUDA 12.9 cuBLAS bug on RTX 2080 Ti causes crashes during backward pass of large matrix operations (3639×3639 logits for contrastive loss) [MIHD Spatial Transcriptomics] (2026-03-09)#

Solution: Redesigned contrastive loss to use mini-batch sampling (512 samples per batch) to avoid full N×N matrix computation. Added gradient clipping and LayerNorm for stability. For cross-sample visualization task, switched to CPU training as fallback when GPU incompatibility confirmed (PyTorch 2.9.0+cu129 vs CUDA 13.1 driver).

3. MCP server stdout corruption: gadget tools use print() extensively, but MCP uses stdout as JSON-RPC transport [gadget Tools Infrastructure] (2026-03-09)#

Solution: Created capture_stdout() context manager to redirect print() output, wrapped all tool invocations with _safe_call() that captures stdout/stderr and converts SystemExit to RuntimeError, returned captured logs in JSON response’s _logs field.

4. research_scout hardcoded arxiv_id throughout codebase, blocking multi-source expansion to bioRxiv and PubMed [gadget Tools Infrastructure] (2026-03-09)#

Solution: Introduced paper_id as canonical ID with source field, kept arxiv_id as backward-compatible alias, created _paper_id()/_paper_url() helpers, updated prompts to use paper_id, made merge logic check both fields when matching LLM responses.

5. MIHD patch query returned wrong physical spots: ‘queried patches don’t match current patch’ due to barcode ordering mismatch [MIHD Spatial Transcriptomics] (2026-03-11)#

Solution: Root cause: barcode ordering differs between cache (sc.read_10x_h5 order) and visualization (sc.read_visium order). Fixed by explicit barcode-to-cache-index mapping: {str(b): i for i, b in enumerate(cache_barcodes)}. Also fixed data path and coordinate conventions.

6. Robobrain Pi training loss oscillates significantly with 56% MSE loss dilution from zero-padded action dimensions [Robobrain Pi VLA Training] (2026-03-10)#

Solution: Deep analysis revealed 14-dim real actions (dual arm 6DOF+gripper each) padded to 32-dim for model architectural compatibility, but MSE loss computed over all 32 dimensions. This means 56% of loss (18/32) comes from zero-padded dimensions where model learns trivial noise→zero mapping. Recommended: action_loss = mean(square(v_t[…, :14] - u_t[…, :14])).

7. Robobrain Pi task completion binary classifier may learn trivial ‘always predict 0’ solution despite task_loss_weight=0.1 due to severe class imbalance [Robobrain Pi VLA Training] (2026-03-10)#

Solution: Root cause: severe class imbalance (only last 20 frames of ~200-frame episodes are positive = 9:1 ratio) combined with task_pos_weight=1.0 (no balancing). Solution requires independent tuning: keep task_loss_weight=0.1-0.15 (prevent auxiliary task from overpowering 50-step action prediction), but increase task_pos_weight to 5.0 (give positive class sufficient gradient signal via per-class weighting).

8. Training time for identical commands differed by 33% across two projects (openpi 15h vs pi05 20h), despite same hardware and configurations [Robobrain Pi VLA Training] (2026-03-11)#

Solution: Identified JAX version difference (0.5.0 vs 0.5.3) as primary cause through parallel exploration agents. Systematically aligned 6 dependency versions: JAX 0.5.0→0.5.3, torch>=2.5.1→2.7.1, orbax-checkpoint 0.11.1→0.11.13, numpy, beartype, transformers. Added override-dependencies to resolve lerobot’s torch<2.7 constraint conflict. Expected ~25% speedup from JAX optimization.

9. K8s container PID namespace isolation prevents nvidia-smi from showing process info - displays [Not Found] for all GPU processes [GPU Monitoring Tool] (2026-03-10)#

Solution: Root cause: PID namespace isolation. nvidia-smi queries kernel driver returning host PID, but container has different PID namespace numbering. Solution: developed gpumon.py tool combining psutil (reads container-internal /proc for process info: CPU/memory/cmdline/runtime) with nvidia-smi (GPU hardware state: VRAM/utilization/temperature), visualized via rich TUI with /proc/fd inspection for /dev/nvidia* device files and CUDA_VISIBLE_DEVICES env var.

10. Node.js ccusage tool cannot fetch latest model pricing from LiteLLM (1.4MB JSON), falls back to cached 46 models instead of 2696 [gadget Tools Infrastructure] (2026-03-10)#

Discovered Node.js v24’s built-in fetch (undici) doesn’t respect HTTPS_PROXY environment variable by default unlike curl/axios. Solution: NODE_OPTIONS="–use-env-proxy" npx ccusage@latest.

Learnings#

Architecture (architecture)#

  • VLA model temporal context dependency requires fundamental benchmark redesign: Cannot simply initialize episodes with error states - VLA needs behavioral history to make contextually appropriate recovery decisions. Solution: replay first N-1 frames to build context window, inject error at frame N, let VLA infer freely from N onward. This shifts error injection from ‘initial state perturbation’ to ’trajectory interruption’, matching real-world failure modes.
  • Large architectural refactoring requires detailed written plans with function signatures, line counts, and architectural decisions. Human architectural vision + AI tactical execution works better than AI improvisation. Phased execution with verification checkpoints prevents cascading failures.
  • Offline scan + quota scheduler architecture for balanced error datasets: Stage 0 collect clean trajectories (human + 10×VLA policy success runs) → Stage 1 offline scan opportunities per frame → Stage 2 quota allocation (100 per task×error category) → Stage 3 context replay. More efficient than online generation, guarantees dataset balance, enables reuse of expensive clean trajectory collection across multiple error injection experiments.
  • Parallelized phase-based refactoring: CalendarPro’s 7 phases implemented by 5 concurrent agents each owning a subset of changes, then integrated with test verification. Enables massive productivity gains for independent work.
  • Cross-attention on single tokens degenerates to linear projection. Spatial transcriptomics provides natural solution: k-NN neighbors form K/V sequences that encode neighborhood context. This pattern (spatial neighbors as attention context) is broadly applicable to spatial omics architectures.
  • MCP tool design principle: tools for Claude Code should return content by default (markdown reports, JSON data), with optional file persistence. This enables in-context use without filesystem round-trips. Summarize/research are ’enhancement skills’, not just file generators.
  • Multi-source data integration (homepage scraping + co-authorship + ArXiv + Semantic Scholar) with explicit source tracking provides robustness. Faculty homepages are more authoritative than co-authorship analysis for student discovery.
  • Re-export shims with lazy imports enable large-scale code refactoring while maintaining backward compatibility. Centralized infrastructure modules enable atomic refactoring - changing one constant propagates everywhere.
  • Modular architecture with hybrid integration: keeping monolithic CLI entry point while treating internal package as library code separates concerns beautifully. Reduces migration risk vs full rewrites.
  • When generalizing single-source systems to multi-source, introduce new abstraction (paper_id) while keeping old field (arxiv_id) as alias. Helper functions (_paper_id, _paper_url) centralize cross-source logic and make future expansions trivial.
  • Type-first organization (reports/, logs/, cache/) is more maintainable than tool-first when multiple tools share similar output patterns. Centralized path management enables atomic refactoring.
  • Documentation philosophy for research codebases: Users prefer ‘minimal usage-oriented’ structure (what/how/plan/results) over ‘comprehensive classification’ (detailed per-feature guides). Rationale: research code changes fast, detailed guides become stale quickly. Better to have thin integration docs + well-commented code.

Debugging (debugging)#

  • Zero-padding architectural compatibility trap: Padding for model compatibility (14→32 dims) can severely dilute gradient signals (56% waste) if included in loss computation. Always mask loss to cover only meaningful dimensions. In multi-dimensional outputs, this is subtle but critical - the model converges but learns inefficiently because majority of gradient signal optimizes trivial zero→zero mappings.
  • Independent hyperparameter tuning for imbalanced auxiliary tasks: loss_weight (balances task importance) and pos_weight (balances class contributions within task) serve different purposes and must be tuned independently. For auxiliary classification under severe imbalance: keep loss_weight low (0.1-0.15) to avoid overpowering main objective, increase pos_weight (e.g., 5.0) to address class imbalance.
  • Barcode-based alignment requirement in spatial transcriptomics: Explicit barcode mapping required at every pipeline junction (cache→load→render). Position-based matching breaks when adata reloaded from different sources or reordered by scanpy operations. Never assume index preservation across file I/O boundaries. Solution: maintain bc_to_cache_idx dictionaries throughout pipeline.
  • PyTorch 2.9 + CUDA 12.9 on RTX 2080 Ti has cuBLAS bug causing illegal memory access on large matrix operations (gemm failures at specific shapes like 128→1536, backward on N×N where N>3500). Workaround: mini-batch processing to keep matrices small. PyTorch CUDA build version (e.g., cu129) must match GPU driver CUDA version.
  • macOS TCC mechanism requires App Sandbox to create security-scoped bookmarks. Non-scoped bookmarks only resolve URLs without granting access. Sandbox + fallback re-authorization provides defense in depth.
  • Multi-source data merging: when sources have different biases (recency vs comprehensiveness), use most complete as foundation and enrich from specialized. ArXiv=recent, S2=historical → S2 base + ArXiv enrichment.
  • Error recovery should implement cost-graduated escalation (cheap model → expensive model → human intervention via logs) rather than binary fail/succeed. Maximizes automated recovery while preserving debugging information.
  • When plan mode repeatedly rejected, user may have unspoken blocker (environment unavailable, changed priority, technical constraint). After 2-3 rejections, switch from ’let me fix the plan’ to ‘what’s preventing us from proceeding?’.
  • Large-scale codebase refactoring requires phase separation: 1) Create new structure, 2) Migrate files, 3) Update code references, 4) Update docs (bulk sed acceptable for archived docs), 5) Verify final structure, 6) Cleanup old dirs. Doing code updates before file migration causes breakage.
  • PyTorch C++ extension ABI compatibility: torchvision custom operators compile-time link torch C++ API, versions must precisely match. torchvision 0.21→torch 2.6, 0.22→torch 2.7. Version mismatch breaks operator registration.
  • Base class data flow requires deep tracing in OOP hierarchies. Deleting overrides may not remove keys from output. Must trace base class execution paths (e.g., get_obs() → _take_picture() → pkl files) to ensure variables truly excluded.
  • Physical quantities need semantic bounds enforcement: Progress metrics derived from geometric formulas should be explicitly clamped to semantic range [0,1] even if formula theoretically produces valid values. Real-world edge cases break assumptions.

Tools (tools)#

  • JAX/XLA minor version upgrades (0.5.0→0.5.3) can deliver 20-30% training speedups through compiler optimizations; even patch-level framework updates merit performance audits in compute-intensive ML projects. Multi-project dependency drift is a stealth performance bug requiring periodic alignment.
  • PyTorch/torchvision strict version pairing: torch 2.7.x requires torchvision 0.22.x, 2.6.x needs 0.21.x, etc. C++ operator registration breaks if versions mismatch. Always explicitly pin both in dependencies when using torchvision, as indirect resolution can fail.
  • When wrapping existing CLI tools for protocol-based systems (MCP, JSON-RPC), stdout/stderr isolation is a first-class architectural concern. Use context managers for clean, reversible redirection without modifying original code.
  • K8s PID namespace isolation prevents nvidia-smi process detection. Solution: /proc//fd/ inspection for /dev/nvidia* device files + CUDA_VISIBLE_DEVICES env var. Monitoring tools opening all GPUs without CUDA_VISIBLE_DEVICES are false positives. Namespace isolation requires namespace-local data collection then aggregation.
  • Academic name disambiguation must be first-class concern in profiling systems. Common Chinese names create severe problems requiring mandatory ORCID/institution matching. Better to return ‘data unusable’ than propagate false profiles.
  • Node.js v24 fetch differs fundamentally from traditional HTTP clients (curl/axios/requests) - it ignores environment proxy variables by default. This is a compatibility trap requiring explicit –use-env-proxy flag or global-agent library. Critical for restricted network environments.
  • CUDA_VISIBLE_DEVICES GPU index remapping: Restricting to subset remaps indices to 0-N range within process CUDA context. Dependent libraries (MuJoCo EGL, rendering frameworks) must use remapped indices not original hardware IDs. Always set rendering device IDs to 0 when using single-GPU isolation.
  • Editable pip install environment scoping: Package availability scoped to Python environment executing install command. Artifacts (.egg-info) may exist but imports fail if installed elsewhere. Always verify ‘which python’ matches intended environment. Use –no-deps to install package structure without overwriting existing dependencies.
  • uv’s override-dependencies is a precise tool for resolving transitive dependency conflicts without forking or patching upstream packages; particularly useful when pinning git revisions with incompatible constraints.
  • Production logging requires two handlers: StreamHandler (INFO) for user-facing progress, RotatingFileHandler (DEBUG) for diagnostic details. Keep print() in interactive UI commands (list, show) but migrate all operational/diagnostic messages to logger.
  • Zero-dependency API clients (urllib.request + xml.etree) are viable for production tools when avoiding new dependencies is critical. bioRxiv and PubMed both implemented with stdlib only, maintaining gadget’s minimal dependency philosophy.
  • LLMs can serve as knowledge bases for unstructured metadata (conference awards, paper prestige) when APIs don’t provide it. ‘LLM as auxiliary database’ pattern is underutilized - requires prompt design but no infrastructure.
  • conda environment CUDA toolchain: pip-installed nvidia packages place headers in targets/x86_64-linux/include/ not standard include/. JIT C++ extension compilation needs CUDA_HOME pointing to conda env, CPATH including targets path.

Domain Knowledge (domain)#

  • RM-IDEAL spatial niche matching algorithm: (1) Build spatial adjacency graph (k-NN, n_neighbors=6), (2) WWL message passing (K=3 iterations aggregate neighbor labels), (3) Compute Hamming distance cost matrix, (4) Wasserstein optimal transport, (5) RM = 1 - W ∈ [0,1]. Cross-section evaluation: bidirectional query, compare RM-IDEAL scores vs embedding cosine similarity via Spearman ρ, P@K, SL@K metrics.
  • scGPT foundation model underperforms traditional methods (PCA, scVI, Harmony) in zero-shot clustering benchmarks (Genome Biology 2025).
