Problems & Solutions#

1. Error Recovery Benchmark v4’s physics-force injection cannot generate VLA-compatible temporal context - errors injected as initial states lack behavioral history VLA models require [VLA/Robot Learning Systems] (2026-03-10)#

Solution: Designed v5.0 architecture with offline two-stage pipeline: Stage 0 collect clean trajectories (human + 10×VLA policy success runs) → Stage 1 offline scan opportunities per frame → Stage 2 quota scheduler (100 per task×category) → Stage 3 context replay (replay first N-1 frames to build VLA context window, inject error at frame N). Shifted from force-based injection to action-based Error Skill modules.

2. MIHD spatial transcriptomics patch query visualization showing misaligned coordinates - query patches don’t match expected spatial locations [MIHD Spatial Transcriptomics] (2026-03-10)#

Solution: Identified 3 root causes: (1) Dataset/DLPFC/ path vs top-level path mismatch, (2) position-based assignment without barcode mapping (cache order ≠ adata order after scanpy operations), (3) marker overlay using cache coords instead of adata.obsm[‘spatial’]. Fixed by using consistent paths, bc_to_cache_idx dictionary for explicit barcode alignment, and barcode-based coordinate lookup.

3. Robobrain Pi training loss oscillates significantly - 14-dim real actions padded to 32-dim for model compatibility, but MSE loss computed over all 32 dimensions, causing 56% gradient dilution from trivial zero→zero mapping [VLA/Robot Learning Systems] (2026-03-10)#

Solution: Deep analysis revealed severe gradient dilution from zero-padded dimensions. Recommended: action_loss = mean(square(v_t[…, :14] - u_t[…, :14])) to mask loss to real dimensions only. Also corrected task_pos_weight from 1.0→5.0 to address 9:1 class imbalance in task completion classifier.

4. PyTorch 2.9/CUDA 12.9 cuBLAS bug on RTX 2080 Ti causes crashes during backward pass of large matrix operations (3639×3639 logits for contrastive loss) [MIHD Spatial Transcriptomics] (2026-03-09)#

Solution: Redesigned contrastive loss to use mini-batch sampling (512 samples per batch) to avoid full N×N matrix computation. Added gradient clipping and LayerNorm for stability. For cross-sample visualization, switched to CPU training as fallback when GPU incompatibility confirmed (PyTorch 2.9.0+cu129 vs CUDA 13.1 driver).

5. MCP server stdout corruption - gadget tools use print() extensively, but MCP uses stdout as JSON-RPC transport [gadget Tools Infrastructure] (2026-03-09)#

Solution: Created capture_stdout() context manager to redirect print() output, wrapped all tool invocations with _safe_call() that captures stdout/stderr and converts SystemExit to RuntimeError, returned captured logs in JSON response’s _logs field.

6. research_scout hardcoded arxiv_id throughout codebase, blocking multi-source expansion to bioRxiv and PubMed [gadget Tools Infrastructure] (2026-03-09)#

Solution: Introduced paper_id as canonical ID with source field, kept arxiv_id as backward-compatible alias, created _paper_id()/_paper_url() helpers, updated prompts to use paper_id, made merge logic check both fields when matching LLM responses.

7. Training time for identical commands differed by 33% across two projects (openpi 15h vs pi05 20h), despite same hardware and configurations [VLA/Robot Learning Systems] (2026-03-11)#

Solution: Identified JAX version difference (0.5.0 vs 0.5.3) as primary cause through parallel exploration agents. Systematically aligned 6 dependency versions: JAX 0.5.0→0.5.3, torch>=2.5.1→2.7.1, orbax-checkpoint 0.11.1→0.11.13. Added override-dependencies to resolve lerobot’s torch<2.7 constraint conflict.

8. Pi05 training failed at startup with ‘RuntimeError: operator torchvision::nms does not exist’ [VLA/Robot Learning Systems] (2026-03-11)#

Solution: Diagnosed torch 2.7.1 + torchvision 0.21.0 version mismatch (0.21.0 compiled for torch 2.6.x). Added explicit torchvision==0.22.1 to pyproject.toml dependencies, ran uv sync to upgrade. Verified: torch 2.7.1+cu126, torchvision 0.22.1+cu126.

9. K8s container PID namespace isolation prevents nvidia-smi from showing process info - shows [Not Found] for all GPU processes [Infrastructure & Monitoring] (2026-03-11)#

Solution: Root cause: PID namespace isolation. nvidia-smi queries kernel driver returning host PID, but container PID namespace has different numbering. Learnings#

Architecture (architecture)#

  • VLA model temporal context dependency requires fundamental benchmark redesign: Cannot simply initialize episodes with error states - VLA needs behavioral history to make contextually appropriate recovery decisions. Solution: replay first N-1 frames to build context window, inject error at frame N. This shifts error injection from ‘initial state perturbation’ to ’trajectory interruption’, matching real-world failure modes.
  • Offline scan + quota scheduler architecture for balanced error datasets: Stage 0 collect clean trajectories → Stage 1 offline scan opportunities → Stage 2 quota allocation (100 per task×error) → Stage 3 context replay. More efficient than online generation, guarantees dataset balance, enables reuse of expensive clean trajectory collection across multiple error injection experiments.
  • Large architectural refactoring requires detailed written plans with function signatures, line counts, and architectural decisions. Human architectural vision + AI tactical execution works better than AI improvisation. Phased execution with verification checkpoints prevents cascading failures.
  • Parallelized phase-based refactoring: CalendarPro’s 7 phases implemented by 5 concurrent agents each owning a subset of changes, then integrated with test verification. Enables massive productivity gains for independent work.
  • Cross-attention on single tokens degenerates to linear projection. Spatial transcriptomics provides natural solution: k-NN neighbors form K/V sequences that encode neighborhood context. This pattern (spatial neighbors as attention context) is broadly applicable to spatial omics architectures.
  • MCP tool design principle: tools for Claude Code should return content by default (markdown reports, JSON data), with optional file persistence. This enables in-context use without filesystem round-trips. Summarize/research are ’enhancement skills’, not just file generators.
  • Error taxonomy deformation dimension classification: D0 (position only), D1 (rotation only), D2 (position+rotation) systematically describes spatial deformations across all error types. More principled than phase-based classification, enables clearer validator logic by decomposing 6DOF changes into orthogonal components.
  • Multi-source data integration (homepage scraping + co-authorship + ArXiv + Semantic Scholar) with explicit source tracking provides robustness. Faculty homepages are more authoritative than co-authorship analysis for student discovery.
  • Re-export shims with lazy imports enable large-scale code refactoring while maintaining backward compatibility. Centralized infrastructure modules enable atomic refactoring - changing one constant propagates everywhere.
  • Modular architecture with hybrid integration: keeping monolithic CLI entry point while treating internal package as library code separates concerns beautifully. Reduces migration risk vs full rewrites.
  • Type-first organization (reports/, logs/, cache/) is more maintainable than tool-first when multiple tools share similar output patterns. Centralized path management enables atomic refactoring.

Debugging (debugging)#

  • Zero-padding architectural compatibility trap: Padding for model compatibility (14→32 dims) can severely dilute gradient signals (56% waste) if included in loss computation. Always mask loss to cover only meaningful dimensions. The model converges but learns inefficiently because majority of gradient signal optimizes trivial zero→zero mappings.
  • PyTorch 2.9 + CUDA 12.9 on RTX 2080 Ti has cuBLAS bug causing illegal memory access on large matrix operations (gemm failures at specific shapes like 128→1536, backward on N×N where N>3500). Workaround: mini-batch processing to keep matrices small. PyTorch CUDA build version (e.g., cu129) must match GPU driver CUDA version.
  • Independent hyperparameter tuning for imbalanced auxiliary tasks: loss_weight (balances task importance) and pos_weight (balances class contributions within task) serve different purposes and must be tuned independently. For auxiliary classification under severe imbalance: keep loss_weight low (0.1-0.15) to avoid overpowering main objective, increase pos_weight (e.g., 5.0) to address class imbalance.
  • Spatial transcriptomics barcode-based alignment requirement: Explicit barcode mapping required at every pipeline junction (cache→load→render). Position-based matching breaks when adata reloaded from different sources or reordered by scanpy operations. Never assume index preservation across file I/O boundaries. Solution: maintain bc_to_cache_idx dictionaries throughout pipeline.
  • JAX/XLA minor version upgrades (0.5.0→0.5.3) can deliver 20-30% training speedups through compiler optimizations; even patch-level framework updates merit performance audits in compute-intensive ML projects. Multi-project dependency drift is a stealth performance bug requiring periodic alignment.
  • PyTorch/torchvision strict version pairing: torch 2.7.x requires torchvision 0.22.x, 2.6.x needs 0.21.x. C++ operator registration breaks if versions mismatch. Always explicitly pin both in dependencies when using torchvision, as indirect resolution can fail.
  • Multi-source data merging: when sources have different biases (recency vs comprehensiveness), use most complete as foundation and enrich from specialized. ArXiv=recent, S2=historical → S2 base + ArXiv enrichment.
  • Error recovery should implement cost-graduated escalation (cheap model → expensive model → human intervention via logs) rather than binary fail/succeed. Maximizes automated recovery while preserving debugging information.
  • Physical quantities need semantic bounds enforcement: Progress metrics derived from geometric formulas should be explicitly clamped to semantic range [0,1] even if formula theoretically produces valid values. Real-world edge cases break assumptions.
  • conda environment CUDA toolchain: pip-installed nvidia packages place headers in targets/x86_64-linux/include/ not standard include/. JIT C++ extension compilation needs CUDA_HOME pointing to conda env, CPATH including targets path.
  • Base class data flow requires deep tracing in OOP hierarchies. Deleting overrides may not remove keys from output. Must trace base class execution paths (e.g., get_obs() → _take_picture() → pkl files) to ensure variables truly excluded.
  • macOS TCC mechanism requires App Sandbox to create security-scoped bookmarks. Non-scoped bookmarks only resolve URLs without granting access. Sandbox + fallback re-authorization provides defense in depth.

Tools (tools)#

  • When wrapping existing CLI tools for protocol-based systems (MCP, JSON-RPC), stdout/stderr isolation is a first-class architectural concern. Use context managers for clean, reversible redirection without modifying original code. Captured logs returned in JSON response’s _logs field.
  • When generalizing single-source systems to multi-source, introduce new abstraction (paper_id) while keeping old field (arxiv_id) as alias. Helper functions (_paper_id, _paper_url) centralize cross-source logic and make future expansions trivial.
  • K8s PID namespace isolation prevents nvidia-smi process detection. Solution: /proc//fd/ inspection for /dev/nvidia* device files + CUDA_VISIBLE_DEVICES env var. Monitoring tools opening all GPUs without CUDA_VISIBLE_DEVICES are false positives. Namespace isolation requires namespace-local data collection.
  • Academic name disambiguation must be first-class concern in profiling systems. Common Chinese names create severe problems requiring mandatory ORCID/institution matching. Better to return ‘data unusable’ than propagate false profiles.
  • Node.js v24 fetch proxy compatibility trap: Built-in fetch (undici) doesn’t respect HTTPS_PROXY/HTTP_PROXY environment variables unlike curl/axios/requests. Requires explicit NODE_OPTIONS="–use-env-proxy" or global-agent library. Critical for restricted network environments.
  • CUDA_VISIBLE_DEVICES GPU index remapping: Restricting to subset remaps indices to 0-N range within process CUDA context. Dependent libraries (MuJoCo EGL, rendering frameworks) must use remapped indices not original hardware IDs. Always set rendering device IDs to 0 when using single-GPU isolation.
  • Cross-environment tool invocation (conda run -n ) provides clean isolation for sibling repos with conflicting dependencies. Critical to set cwd to tool repo root, as many Python scripts use relative imports and sys.path manipulation.
  • Production logging requires two handlers: StreamHandler (INFO) for user-facing progress, RotatingFileHandler (DEBUG) for diagnostic details. Keep print() in interactive UI commands (list, show) but migrate all operational/diagnostic messages to logger.
  • uv’s override-dependencies is a precise tool for resolving transitive dependency conflicts without forking or patching upstream packages; particularly useful when pinning git revisions with incompatible constraints.
  • LLMs can serve as knowledge bases for unstructured metadata (conference awards, paper prestige) when APIs don’t provide it. ‘LLM as auxiliary database’ pattern is underutilized - requires prompt design but no infrastructure.

Domain Knowledge (domain)#

  • RM-IDEAL spatial niche matching algorithm: (1) Build spatial adjacency graph (k-NN, n_neighbors=6), (2) WWL message passing (K=3 iterations aggregate neighbor labels), (3) Compute Hamming distance cost matrix, (4) Wasserstein optimal transport, (5) RM = 1 - W ∈ [0,1]. Cross-section evaluation: bidirectional query, compare RM-IDEAL scores vs embedding cosine similarity via Spearman ρ.
  • scGPT foundation model underperforms traditional methods (PCA, scVI, Harmony) in zero-shot clustering benchmarks (Genome Biology 2025). AvgBIO scores on Pancreas/PBMC datasets show scGPT ~0.52-0.60 vs. HVG baseline ~0.60-0.77. Independent benchmarks more reliable than original paper claims.
  • Conditional Flow Matching for robot action generation: learns vector field u_t from noise to actions using linear interpolation path x_t = t·noise + (1-t)·actions, Beta(1.5,1) time sampling for stability near noise endpoint, Euler integration with 10 steps for inference. Time conditioning via sinusoidal embeddings + MLP.
  • DLPFC tissue layer characteristics directly impact embedding cross-sample matching: Layer_1 (superficial, thin) and Layer_4 (granular, cell-dense) show Spearman ρ>0.6; Layer_3 (largest, heterogeneous) shows ρ<-0.2. Embedding quality correlates with structural uniqueness and boundary clarity, not layer size.
  • Dataset conversion augmentation at source level: Random instruction sampling per episode (np.random.choice from available paraphrases) increases training diversity without requiring multiple dataset versions. One-line change with major impact on language-conditioned policy generalization.
  • Post-processing approach for ‘future-dependent’ variables: When per-frame variables require knowledge of move() completion state, recording frames normally then patching pickle files after move() finishes is cleaner than complex state machines.
  • For researcher trajectory analysis, temporal structure preservation (year-stratified sampling) is more valuable than global citation ranking. Shows domain-specific sampling strategies matter more than generic ‘sort by metric’ approaches.
  • JAX/Flax stop_gradient pattern for auxiliary tasks is common in multi-task learning: use features without backprop interference to main task. Enables richer training signals without gradient conflicts.
  • Special token mechanism requires 3-step implementation: tokenizer registers token ID, model extends embedding layer, data loader splices prompt with special tokens. All three must be coordinated.
