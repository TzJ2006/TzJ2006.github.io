问题与解决方案#

关键问题#

1. MIHD 融合管道中嵌入归一化不一致：UNI2/scGPT/PCA 已做 L2 归一化，但 HIPT/ResNet50/MLP/GCN 没有；融合后也没有归一化，影响下游聚类效果。#

解决方案: 设计三点归一化框架（post-encoder、pre-fusion、post-fusion），通过 utils/normalization.py 提供统一接口，所有方法均可通过 config 字段独立开关，L2 归一化对已归一化向量幂等，不影响 UNI2/scGPT。

关键洞察: 归一化一致性是多模态融合的基础，三点控制覆盖了编码器输出差异、融合前对齐和融合后分布统一三个不同问题，不能用单一策略替代。

2. Q-Former（BLIP-2）和 LLaVA MLP 常被混淆，用户提出"实现 Q-Former"时两种架构均有其价值。#

解决方案: AI 主动区分两者差异并提问，用户选择"Both"——Q-Former 用空间邻居 token 做交叉注意力，LLaVA MLP 是 2 层投影连接器，均纳入实施计划。

关键洞察: LLaVA 实际使用的是 MLP 连接器而非 Q-Former，名称混用是论文社区常见误解；正确区分有助于选择合适融合复杂度。

一般问题#

3. AI 未经规划直接开始实施，用户需要先有文档化的计划。#

解决方案: 用户主动打断并要求先将计划写入文件（docs/ENHANCEMENT_PLAN.md），之后再按阶段实施。

关键洞察: 大型多阶段实现任务应先文档化计划再编码，防止中途方向偏移且便于团队对齐。

4. CLAUDE.md 原版存在严重冗余（smart runner 命令出现两次、squidpy 警告出现 3 次）且缺少关键组件描述（GCN encoder、batch correction fusion、utils/ 模块）。#

解决方案: 通过并行 Task agent 全面探索代码库后，对照 AGENTS.md / GEMINI.md / README.md 三个参考文档重写，删除可由 ls 直接发现的文件列表和 STAIG changelog 历史，精简至 188 行。

关键洞察: CLAUDE.md 的价值在于跨文件的架构洞察，而非文件列表或可从代码直接读取的信息。

人类思路 vs AI 思路#

战略层面#

实施顺序管理：先规划还是先编码#

角色 思路
人类 用户在 AI 已开始创建 Task 并准备直接编码时，主动打断并要求先将完整计划写入文件，体现了工程化项目管理意识。
AI AI 倾向于在理解需求后立即开始创建任务并实施，缺少主动提出文档化计划的意识。

差异分析: 人类将计划文档视为沟通与对齐工具，而 AI 将任务完成视为首要目标，导致在缺乏文档的情况下过早进入编码阶段。

Q-Former 与 LLaVA MLP 的架构选择#

角色 思路
人类 用户最初提出"实现 Q-Former like LLaVA 的融合方法"，将两者混用。当 AI 提问后，用户决定"Both"，希望同时实现两种方案进行对比。
AI AI 正确识别了两者的技术差异（Q-Former=BLIP-2，LLaVA=MLP connector），主动提问澄清，并为两者各自设计了实现方案。

差异分析: AI 在技术细节辨别上优于用户，但用户的"Both"决策体现了实验驱动的研究思路，AI 未主动提出同时实现两种方案。

实现层面#

ExitPlanMode 与计划确认流程#

角色 思路
人类 用户在 AI 调用 ExitPlanMode 时拒绝了该操作，但未说明原因（可能是计划尚需调整，或不希望通过该工具确认）。
AI AI 在完成计划设计后按标准流程调用 ExitPlanMode 请求用户批准。

差异分析: 用户对 AI 工具调用流程有自己的预期，拒绝 ExitPlanMode 后未继续说明下一步，导致该会话以未完成状态结束。

AI 局限性#

重要局限#

  • 未主动提出先写计划文档再编码的工作流建议，在接到大型多阶段实施任务时直接开始创建 Task 和编码，需要用户手动打断才调整策略。

一般局限#

  • 在两个并行 Task agent 探索代码库后，主 agent 仍重复读取部分相同文件（如 run_benchmark.py、Fusion.py），造成上下文冗余。

今日收获#

核心收获#

  • QueST 使用 k-hop 子图池化做 niche representation，通过对比学习训练 niche encoder，并用 BatchDiscriminator（对抗训练）做 batch effect 校正，这套思路可直接移植到 MIHD 的 post-fusion 阶段。
  • MIHD 融合管道中三点归一化（post-encoder / pre-fusion / post-fusion）是解决异构 embedding 分布不一致的系统性方案，L2 归一化对已归一化向量幂等这一性质使其可安全应用于全部 encoder。
  • Q-Former（BLIP-2）使用可学习 query token + 交叉注意力从视觉特征中提取信息；LLaVA 实际使用的是 2 层 MLP 投影连接器，两者复杂度和训练成本差异显著，空间转录组场景中 Q-Former 可利用空间邻居作为上下文 token。

实践收获#

  • 大型多阶段实施计划应先文档化（写入 docs/ENHANCEMENT_PLAN.md 等）再按阶段编码，方便团队对齐和中途调整方向。
