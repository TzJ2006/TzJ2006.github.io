Problems & Solutions#

Critical Issues#

1. Tauri main window set_position() completely ignored by compositor on Linux/Wayland, window appears at screen center#

Solution: Detect WAYLAND_DISPLAY environment variable at run() entry; if in a Wayland session, force GDK_BACKEND=x11 to run the application via XWayland.

Key Insight: The Wayland protocol design prohibits clients from controlling absolute window positions; any retry strategy is futile — must fall back to the X11 protocol layer.

2. Float-ball expand layout anchors to screen boundary instead of ball position, causing visual jump when clicking to expand#

Solution: Modified expanded_rect_from_ball to ball-anchored mode: for Left anchor, expanded x = ball_rect.x; for Right anchor, expanded x = ball_rect.right - expanded_width; combined with clamped_y to keep y stable.

Key Insight: The expanded window’s reference point should anchor to the ball’s own position, not the screen safe-area boundary, to ensure visual position consistency before and after expansion.

3. host_cache_dir() in ssh_remote.rs uses assert!() which panics the entire program on path validation failure#

Solution: Changed function signature to Result<PathBuf, String>, propagated Result through all call chains, handled with unwrap_or_default() at call sites.

Key Insight: assert!() in production is equivalent to a time bomb; paths accepting external input must use Result to propagate errors.

4. On Linux GTK, setting resizable(false) causes programmatic set_size() to be rejected by WM, preventing float-ball from collapsing#

Solution: In the Linux branch, call set_resizable(true) before set_size to temporarily unlock, then call set_resizable(false) again after the operation completes.

Key Insight: The resizable(false) WM hint blocks both user drag and programmatic calls — it is not limited to user interaction.

5. FloatBallState does not store window geometry, reads outer_position/outer_size from WM each time, potentially reading stale values during expand/collapse sequencing causing layout errors#

Solution: Added FloatBallState.last_rect, created apply_and_store_rect helper function to synchronously write window and record state; added get_float_ball_position command for frontend use.

Key Insight: Maintaining authoritative geometry state at the application layer completely bypasses timing non-determinism of WM reads.

6. sync_ssh_host holds an RwLock read lock across SSH I/O await points that can last several seconds, blocking all readers#

Solution: Added #[derive(Clone)] to SshCacheManager, clone the manager before await to release the lock early.

Key Insight: Locks crossing async boundaries are a common hidden hazard in Rust async programming; the solution is to Clone the needed data before await to release the lock early.

7. provider parameter without validation can be used for path traversal attacks (e.g., passing ‘../../../etc’ to construct malicious paths)#

Solution: Added validate_provider() whitelist function allowing only ‘claude’ and ‘codex’; other values return Err.

Key Insight: Any user input concatenated into file paths must be whitelist-validated at system boundaries; allowlists are more reliable than denylists for preventing path traversal attacks.

8. load_cached_records silently returns an empty Vec for all I/O errors (including permission denied), masking real errors#

Solution: Changed to return Result, explicitly distinguishing ErrorKind::NotFound (legitimate, returns Ok(vec![])) from other I/O errors (returns Err).

Key Insight: File not found and unable to read are fundamentally different states; conflating them makes debugging extremely difficult.

9. ssh_command treats partial stdout with non-zero SSH exit code as success, accepting erroneous results#

Solution: Strictly check exit code first; only accept stdout on success(); log stderr on non-zero exit.

Key Insight: partial stdout + non-zero exit is a common SSH error pattern; non-empty stdout cannot be used as the success criterion.

10. Svelte 5 component uses both onMount and $effect to trigger data requests, causing double requests on mount#

Solution: Removed onMount call, kept $effect (in Svelte 5, $effect automatically executes once on mount).

Key Insight: Svelte 5’s $effect differs from Svelte 4 reactive statements — it executes immediately on mount and does not need to be paired with onMount.

11. FloatBall component registers Tauri event listeners asynchronously inside onMount with a race condition: component may be destroyed before listen() resolves, causing cleanup function not to be called and memory leaks#

Solution: Introduced a destroyed boolean flag; if already marked as destroyed when the Promise resolves, immediately call unlisten().

Key Insight: Any pattern of asynchronously acquiring cleanup handles in frontend components must account for the case where the component outlives the Promise.

Today’s Takeaways#

Core Takeaways#

  • Wayland compositors prohibit clients from setting absolute window positions at the protocol level; any set_position() retry strategy is futile. The correct fix is to detect WAYLAND_DISPLAY and force GDK_BACKEND=x11 to fall back to XWayland.
  • In Rust production code, assert!() and unwrap() on paths that accept external input (file paths, user parameters, SSH output) are potential crash points; the uniform pattern is to change function signatures to Result and propagate with ?, using unwrap_or_default() at call boundaries.
  • GTK’s resizable(false) WM hint blocks both user drag and programmatic set_size(); for transparent floating windows that need dynamic size adjustment, resizable must be temporarily unlocked before resizing.
  • Geometry state of transparent overlay windows should be authoritatively managed at the application layer (last_rect) rather than re-read from WM each time; WM’s outer_position/outer_size may have timing delays after resize, and reading stale values during expand/collapse sequencing causes layout errors.
  • Float window expand layout should anchor to the ball itself (ball-anchored) rather than the screen safe-area boundary, to ensure visual position consistency before and after expansion — no perceived jumping.
  • Locks crossing await points in Rust async code are hard-to-detect performance/deadlock hazards; the fix is to Clone or Arc::clone needed data before await to release the lock early.
  • Any user input concatenated into file paths must be whitelist-validated at system boundaries; ‘only allow known values’ is more reliable protection than ‘reject dangerous characters’, effectively preventing path traversal attacks.
  • In Svelte 5, $effect executes once automatically on component mount; using it together with onMount causes double requests. When asynchronously acquiring cleanup handles in frontend components, a destroyed flag is needed to handle the race condition where the component outlives the Promise.

Practical Takeaways#

  • git reset –soft is an efficient method for cleaning up commit history: no interactive rebase needed, can squash any number of commits into one semantic commit — suitable for CI environments and automation tools.
  • After adding new directory structures to rclone sync config, remote will not automatically create those directories if data has never been pushed; pull should distinguish between ‘remote does not exist’ (normal, treat as success) and ‘pull failed’ (error) to avoid false negatives.
