Problems and Solutions#

Key Issues#

  • Leiden Clustering: It needs to be compared with KMeans algorithm, but Leiden uses the resolution parameter by default to control the number of clusters. This makes it difficult to accurately match n clusters. Solution: Implement a binary search function in cluster_with_leiden(). Adjust the range of resolution to [0.01, 10) until the number of clusters matches. Up to 20 iterations may be required.

  • General Issues

  • HD process visual extraction failure: VisionExtractor tried to find images using data_root/section_id, but HD data was located in crop_dir. This resulted in failures in 11 out of 13 experiments. Solution: Add the crop_dir parameter to VisionExtractor.init. In extract(), use image_dir_override=crop_dir. Use image_dir_override as a priority during runtime. Create VisionExtractor using crop_dir during runtime."

  • SLURM Script Issues: Using source activate directly in the SLURM script caused problems with conda environment activation. All Leiden experiments were run on GPU-free login nodes. Solution: Replace source activate in the SLURM script with eval $(conda shell.bash hook) && conda activate. Resubmit the job after fixing the error."

  • run_pipeline.py Error: The default logic in the dataset configuration section caused the switch to HD data when hd_global was specified. This resulted in Leiden DLPFC experiments running in HD mode. Solution: Change the default value of datasets to only DLPFC. Use HD data only when –dataset HD is explicitly specified."

Today’s Summary#

Main Conclusions#

  • Leiden clustering performed similarly to KMeans on HD data (pca_leiden ARI≈0.333 vs pca ARI≈0.288 on DLPFC). It improved the performance of simple methods that used only genes, but its advantage was diminished for graph comparison methods like STAIG. The advantage of STAIG lies in spatial graph structures, which has nothing to do with clustering algorithms.
  • Simple fusion strategies (concat/mean/element_wise_sum) had lower ARI on HD data than single-modal PCA (0.19-0.20 vs 0.351). This indicates that simple fusion may lead to dilution of information in large-scale HD data. attention and staig_fusion were exceptions (attention ARI=0.342).
  • On HD data (true STHD data, with 3 clusters: epithelial tissue/connective tissue/smooth muscle), UNI2 with only visual evaluation had the highest NMI (0.387), but its ARI was lower than PCA (0.344 vs 0.351). It should be noted that NMI and ARI may not be consistent across different datasets.
  • In MISO, HD part used Visium HD colorectal cancer to show spatial expression of SPP1 (macrophages related to tumors) and CEACAM6. This can serve as a reference framework for verifying the markers in Fig 4 of MIHD.

Practical Summary#

  • The matching between STHD annotation file (8um_squares_annotation.csv) and data observation data required r_big/c_big coordinates. The match rate was 100%. After merging CE+Tumor data, three meaningful clusters were identified (Outside had only 4 negligible spots).
