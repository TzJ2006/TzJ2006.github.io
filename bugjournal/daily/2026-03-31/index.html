Problems & Solutions#

Critical Issues#

1. Leiden clustering needs to be compared with KMeans, but Leiden controls the number of clusters based on the resolution parameter, making it impossible to accurately match the number of clusters.#

Solution: Implement binary search in cluster_with_leiden(). Iterate through the [0.01, 10] range to adjust the resolution until the number of clusters matches, with a maximum of 20 iterations.

Key Insight: The relationship between Leiden’s resolution and the number of clusters is monotonic. Binary search can efficiently match the desired number of clusters, allowing fair comparison with KMeans.

General Issues#

2. HD pipeline Vision Extraction failed: VisionExtractor used the data_root/section_id path to find images, but HD data was in the crop_dir, causing 11/13 experiments to fail.#

Solution: Add the crop_dir parameter to VisionExtractor.init. Pass image_dir_override=crop_dir to extract(). Use image_dir_override preferentially in _load_image_and_coords(). Pass crop_dir to VisionExtractor when creating it in runner.py.

Key Insight: The image storage structures of DLPFC and HD data are completely different. The pipeline needs to explicitly distinguish the image paths for these two datasets.

3. SLURM scripts used source activate General, causing conda environment activation to fail. All Leiden experiments ran on GPU-free login nodes.#

Solution: Replace source activate in SLURM scripts with eval $(conda shell.bash hook) && conda activate. Resubmit the job.

Key Insight: SLURM nodes on the DCC cluster need to initialize the conda shell hook explicitly.不能直接使用source activate。

4. In run_pipeline.py, there is a default logic error in the dataset configuration. When the hd_global section exists, it automatically switches to the HD dataset, causing the Leiden DLPFC experiment to run in HD mode.#

Solution: Change the default value of the dataset to DLPFC. Only switch to the HD dataset when –dataset is explicitly specified.

Key Insight: The existence of the hd_global section in the configuration file should not affect the default dataset selection.

Learnings#

Key Learnings#

  • Leiden clustering performed similarly to KMeans on HD data (pca_leiden ARI≈0.333 vs pca ARI≈0.288 on DLPFC). It significantly improved for simple gene-only methods, but its benefits decreased for graph-based learning methods like STAIG. STAIG’s advantage comes from its spatial graph structure, which is independent of clustering algorithms.
  • Simple fusion strategies (concat/mean/element_wise_sum) on HD had lower ARI than single-modal PCA (0.19-0.20 vs 0.351), indicating that naive fusion may lead to information dilution in large-scale HD data. Attention and staig_fusion were exceptions (attention ARI=0.342).
  • On HD data (STHD Ground Truth, 3 clusters: Epithelium/Connective Tissue/Smooth Muscle), vision-only UNI2 had the highest NMI (0.387), but ARI was lower than PCA (0.344 vs 0.351). NMI and ARI may not be consistent for different data sets, so both metrics should be reported together.
  • The HD part of MISO (Nat Methods) used Visium HD colorectal cancer to show the spatial expression of SPP1 (tumor-associated macrophages) and CEACAM6. This can serve as a reference framework for verifying markers genes in MIHD Fig 4.

Practical Learnings#

  • The matching between STHD annotation files (8um_squares_annotation.csv) and adata obs can be achieved using r_big/c_big coordinates. The matching rate can reach 100%. After combining CE and Tumor data, 3 meaningful clusters were identified (Outside can be ignored for 4 spots).
