Problems & Solutions#

Critical Issues#

1. Leiden clustering needs a fair comparison with KMeans, but Leiden uses a resolution parameter to control cluster count and cannot exactly match n_clusters#

Solution: Implemented binary search in cluster_with_leiden(): iteratively adjusts resolution in the [0.01, 10] range until the cluster count matches, up to 20 iterations

Key Insight: The relationship between Leiden’s resolution and cluster count is monotonic, so binary search can efficiently match the target cluster count for a fair comparison with KMeans

General Issues#

2. HD pipeline Vision Extraction failure: VisionExtractor looked for images using the data_root/section_id path, but HD data is in crop_dir, causing 11/13 experiments to fail#

Solution: Added crop_dir parameter to VisionExtractor.__init__; passed image_dir_override=crop_dir in extract(); _load_image_and_coords() prioritizes image_dir_override; runner.py passes crop_dir when creating VisionExtractor

Key Insight: The image storage structures of DLPFC and HD data are completely different; the pipeline needs to explicitly distinguish image paths for the two datasets

3. SLURM script used source activate General, causing conda environment activation to fail; Leiden experiments all ran on the login node without GPU#

Solution: Replaced source activate in the SLURM script with eval $(conda shell.bash hook) && conda activate, then resubmitted the job

Key Insight: SLURM nodes on the DCC cluster require explicit conda shell hook initialization; source activate cannot be used directly

4. Default dataset logic error in run_pipeline.py: the presence of the hd_global config section automatically switched to HD dataset, causing Leiden DLPFC experiments to run in HD mode#

Solution: Changed the default dataset value to DLPFC, switching only when --dataset HD is explicitly specified

Key Insight: The presence of the hd_global section in the config file should not affect the default dataset selection

Today’s Takeaways#

Core Takeaways#

  • Leiden clustering performs comparably to KMeans on HD data (pca_leiden ARI≈0.333 vs pca ARI≈0.288 on DLPFC), with clear improvement on simple gene-only methods, but diminishing returns for graph contrastive learning methods like STAIG — STAIG’s advantage comes from spatial graph structure and is relatively independent of the clustering algorithm
  • Simple fusion strategies (concat/mean/element_wise_sum) on HD have lower ARI than unimodal PCA (0.19–0.20 vs 0.351), suggesting naive fusion may cause information dilution on large-scale HD data; attention and staig_fusion are exceptions (attention ARI=0.342)
  • On HD data (STHD Ground Truth, 3 clusters: Epithelium/Connective Tissue/Smooth Muscle), vision-only UNI2 has the highest NMI (0.387) but lower ARI than PCA (0.344 vs 0.351) — NMI and ARI can be inconsistent across datasets, and both metrics should be reported together
  • MISO (Nat Methods)’s HD section used Visium HD colorectal cancer to demonstrate spatial expression of SPP1 (tumor-associated macrophages) and CEACAM6, which can be directly used as a reference framework for marker gene validation in MIHD Fig 4

Practical Takeaways#

  • Matching the STHD annotation file (8um_squares_annotation.csv) to adata obs requires using r_big/c_big coordinates, achieving 100% match rate; after merging CE+Tumor, 3 meaningful clusters are obtained (Outside has only 4 spots and can be ignored)
