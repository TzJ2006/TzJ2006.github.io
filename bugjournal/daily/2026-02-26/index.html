Problems and Solutions#

Key Issues#

1. Pi0.5 evaluation success rate was 0%#

Solution: Change to evaluate on training tasks (Coffee/Stack/Threading/Assembly), no longer measure PickPlace.

Key insight: The most fundamental reason is that the training dataset does not overlap with the evaluation tasks, not model quality issues. This is a key discovery that AI initially overlooked and was triggered by human questions.

2. BC-RNN evaluation dimension mismatch (Expected 65, got 37)#

Solution: Add import mimicgen to _D0 environment in create_env(), no longer strip _D0 suffix, keeping all MimicGen observables.

Key insight: The _D0 suffix provides additional object state observables. Stripping it reduces the observation dimensionality from 65 to 37.

3. BC-RNN configuration was set to low_dim mode, which did not match the main results of the MimicGen paper#

Solution: Rewrite according to the official generate_training_configs_for_public_datasets.py and config_utils.py using set_learning_settings_for_bc_rnn() function to image mode.

Key insight: MimicGen does not provide pre-trained checkpoints, only datasets. The key difference between image mode and low_dim is RNN hidden_dim (400→1000), batch_size (64→16), epoch_every_n_steps (100→500), and ResNet18 + SpatialSoftmax encoder.

General Issues#

4. sbatch submission failed: User’s group not permitted to use this partition#

Solution: Found /usr/local/slurm.24051/bin/srun using source set-XY-I.sh, attached it to an existing interactive job with –jobid=45628 –overlap on node an49.

Key insight: There were multiple SLURM installations in the cluster. The script needed to be run in the correct environment to use the authorized version. Using the existing interactive allocation was the quickest way to bypass this issue.

Today’s Achievements#

Core Achievements#

  • MimicGen does not provide any pre-trained checkpoints, only provides HDF5 demo datasets. BC-RNN is its official benchmark algorithm. The image mode is consistent with the paper. The paper reports 100% success rate for Stack/Coffee, 98% for Threading, and 82% for Assembly.
  • MimicGen _D0 environment (e.g., Coffee_D0) provides additional object observables compared to the basic robosuite environment. Stripping the _D0 suffix reduces the observation dimensionality from 65 to 37, disrupting the consistency of input during reasoning.
  • Cross-task evaluation is a common pitfall in robot learning: If the training set and evaluation tasks do not overlap, it results in a 0% success rate. Consistency between train/eval tasks should be ensured during experimental design phase.
  • Core differences between BC-RNN image mode and low_dim: RNN hidden_dim from 400 to 1000, actor_layer_dims [1024,1024] to [], batch_size from 64 to 16, hdf5_cache_mode all to low_dim, epoch_every_n_steps from 100 to 500. Additionally, add ResNet18Conv+SpatialSoftmax(32kp)+CropRandomizer(76×76).

Practical Achievements#

  • When not having sbatch permissions, you can attach a new process to the existing interactive allocation using srun –jobid=EXISTING_JOB_ID –overlap. No need to apply for new GPU resources.
