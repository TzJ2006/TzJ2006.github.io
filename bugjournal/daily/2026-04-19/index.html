Problems & Solutions#

Critical Issues#

1. Tokenizer download failing with ‘brotli: decoder process called with data when can_accept_more_data() is False’ error#

Solution: Installed brotli C extension package to replace buggy brotlicffi, cleared HF cache, re-downloaded tokenizer successfully

Key Insight: brotlicffi has a known decoder bug; the brotli C extension is more reliable for HuggingFace model downloads

2. Translation engine failing with ‘requires accelerate’ error after tokenizer fix#

Solution: Installed vLLM (which includes accelerate as dependency) to provide GPU-accelerated VLLMEngine backend for Linux

Key Insight: On Linux, vLLM is the preferred translation backend (3x faster than transformers) and automatically pulls in all required dependencies

General Issues#

3. YAML parse errors in frontmatter: ‘mapping values are not allowed here’ for fan_chen.md summary field#

Solution: Changed YAML summary field from single quotes to double quotes to properly escape apostrophes in “This researcher’s” text

Key Insight: YAML single-quoted strings don’t escape internal apostrophes - need double quotes or literal block scalars for text with apostrophes

4. Hugo preflight check failing on empty markdown files with ‘missing frontmatter delimiters’ errors#

Solution: Deleted 0-byte files from outputs/site/ and website/content/ directories - these were remnants of incomplete monthly/weekly generation runs

Key Insight: Empty files are generated when monthly/weekly summary scripts run before enough data exists; incremental generation should skip these gracefully

5. Bash script failing with ‘syntax error near unexpected token $’\r’’ on line 29#

Solution: Used sed to convert Windows CRLF line endings to Unix LF line endings in update.sh

Key Insight: Bash interprets \r as part of syntax tokens rather than line endings; scripts edited on Windows must be converted before running on Linux

Learnings#

Key Learnings#

  • Local translation infrastructure requires careful dependency chain: brotli (not brotlicffi) for model downloads, vLLM (includes accelerate) for GPU inference on Linux, transformers fallback on Windows
  • YAML frontmatter validation must check for unescaped quotes/apostrophes in single-quoted strings and translation placeholder artifacts before Hugo build

Practical Learnings#

  • Cross-platform script compatibility requires explicit line ending normalization when files are edited on Windows and executed on Linux
  • uv package manager provides significantly faster dependency resolution than pip for large Python projects, especially when combined with conda environments
