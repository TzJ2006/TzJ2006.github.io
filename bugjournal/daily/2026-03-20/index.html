Key Learnings### Practical Learning Points#

  • When building a centralized synchronization infrastructure (sync.py), it’s important to support the --dry-run and --category flags from the beginning. This allows for safe incremental testing and selective synchronization without having to upload all data. Migrating rclone remote paths (gdrive:summarize/ → gdrive:gadget/) is less risky because atomic moves are used. When modifying remote paths, it’s necessary to check the integrity of the configuration migration. This includes updating the config.json in summarize/daily_summary.py, research/research_scout.py, test/sync logic, website/Hugo integration, sync.py global configuration, and existing remote data migration scripts.
