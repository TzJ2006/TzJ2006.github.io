Problems and Solutions#

Major Issues#

1. TEDDY’s gene-based ARI (0.186) is lower than PCA’s (0.255). The base model failed to outperform the simple baseline in clustering tasks.#

Solution: Analyzed root causes: TedDY was optimized for disease classification, not maximizing variance. The high-dimensional space of 1024d reduced KMeans performance. Training data was scRNA-seq data different from spatial transcriptomics distribution.

Key Insight: There is a fundamental contradiction between the “universal representation” of the base model and its “clustering adaptability.” PCA directly maximizes data variance, making it naturally suitable for KMeans. The base model learned representations specific to diseases, leading to systematic losses in downstream clustering tasks.

General Issues#

2. evaluation_planner.py miscached staig_gene_feat in slice 151676, causing the second phase to terminate with sys.exit(1).#

Solution: Modified the has_staig_gene_feat check logic to dynamically determine the actual number of top_genes in the slice configuration (default value changed from 3000 to 5000 for 151676) to ensure correct caching.

3. During STAIG fusion evaluation, model crash occurred in slice 151676, invalidating clustering results.#

Solution: Skipped this slice and reported results based on average ARI of 0.495 for the 10/11 slices.

4. Due to lack of Rust, webkit2gtk4.1-devel dependencies, and no sudo privileges, TokenMonitor couldn’t be built on HPC.#

Solution: Recommended three methods: build binary locally with permissions, request system dependencies from HPC administrators, or build Linux packages through GitHub Actions CI.

Today’s Summary#

Key Summary - The STAIG fusion framework demonstrates high robustness in handling gene encoders: Teddy 1024d (ARI=0.495) and PCA 50d (ARI=0.497) perform similarly under the STAIG framework, indicating that the STAIG’s spatial graph comparison learning mechanism has advantages in clustering performance. The impact of gene embedding quality is significantly reduced.#

  • The base model shows systematic transfer losses in cross-domain tasks: Teddy performs well in disease classification (ACC=0.72), but worse than PCA in spatial transcriptomics clustering (0.186 vs 0.255). The root cause lies in the mismatch between optimization goals (discrimination vs variance maximization).
  • HVG1500 (directly connected high-variance genes, no dimensionality reduction) has an ARI of 0.330 on slice 151673, outperforming all base models. This indicates that high-dimensional original features may be beneficial for certain slices by retaining more local variation information.

Practical Conclusions#

  • The hidden configuration split between slice-specific configurations and pipeline global defaults is a high-risk error. These should be dynamically resolved in cache checking logic, rather than being hardcoded.
