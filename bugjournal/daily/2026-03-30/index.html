Problems and Solutions#

Key Problems#

1. Teddy’s ARI based solely on genes (0.186) is worse than PCA’s ARI (0.255). The base model cannot surpass simple baselines in clustering tasks.#

Solution: Analyzed the root cause: Teddy is designed for classification of diseases, not for maximizing variance. The 1024d high-dimensional space worsened KMeans performance. The training data is separate scRNA-seq data, which differs from spatial transcriptomics distribution. Key Insight: There is a fundamental contradiction between the “general representation” of the base model and its “clustering adaptability.” PCA directly maximizes data variance, making it suitable for KMeans. However, the base model learns representations specific to certain diseases, leading to systematic transfer losses in downstream clustering tasks.

General Problems#

2. evaluation_planner.py incorrectly cached staig_gene_feat, causing sys.exit(1) during the second phase.#

Solution: Modified the has_staig_gene_feat check logic to dynamically parse the actual number of top_genes in slice configurations (151676 slices used 5000 instead of the default 3000), thus correctly identifying cached data. Key Insight: Implicit segmentation between slice-specific configurations (from staig_utils.py’s num_gene field) and the planner’s hard-coded defaults may lead to undetected false negatives.

3. In the STAIG fusion evaluation, 151676 slices crashed, resulting in invalid clustering results.#

Solution: Skipped this slice and reported the average ARI for 10/11 slices (ARI=0.495). Key Insight: 151676 is a known difficult slice. Its crash was unrelated to the gene encoder but due to the data characteristics of that slice, leading to a systematic issue.

4. Due to lack of Rust, webkit2gtk4.1-devel dependencies, and no sudo permission, TokenMonitor could not be built on HPC.#

Solution: Proposed three methods: build the binary on a local machine with permissions, ask HPC administrators to install dependencies, or use GitHub Actions CI to build Linux packages. Note that the Tauri application requires GTK/WebKit system libraries, which are usually not available in HPC environments, and the module system does not provide these libraries. HPC is not suitable as a target environment for GUI applications.

Learning Outcomes#

Main Learning Outcomes#

  • The STAIG fusion framework is highly robust to gene encoders: Teddy’s 1024d model (ARI=0.495) and PCA’s 50d model (ARI=0.497) perform almost the same under the STAIG framework. This indicates that the STAIG’s spatial graph contrast learning mechanism dominates clustering performance, while the quality of gene embeddings is greatly reduced.
  • The base model suffers from systematic transfer losses in cross-domain downstream tasks. Teddy is ahead in disease classification (ACC=0.72), but its performance in spatial transcriptomics clustering is worse than PCA (0.186 vs 0.255). This is because the optimization goals differ (discrimination vs variance maximization).
  • HVG1500 (directly connects original high-variance genes without dimensionality reduction) has an ARI of 0.330 on a single slice 151673, which is better than all base models. This suggests that high-dimensional original features may be more beneficial on some slices, as they retain more local variance information.

Practical Learning Outcomes#

  • The implicit separation between slice-specific configurations (e.g., staig num_gene) and global process defaults is a high-risk error. This should be dynamically parsed in the caching logic, rather than being hardcoded.
