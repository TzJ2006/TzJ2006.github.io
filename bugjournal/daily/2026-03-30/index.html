Problems & Solutions#

Critical Issues#

1. Teddy’s gene-only ARI (0.186) is worse than PCA’s (0.255). The Foundation Model fails to surpass simple baselines in clustering tasks.#

Solution: Analyzed the root cause: Teddy aims to classify diseases rather than maximize variance. The 1024d high-dimensional space worsens KMeans performance. The training data is dissociated scRNA-seq, which differs from spatial transcriptomics distribution. Key Insight: The fundamental tension between the “generalized representation” of the base model and its “clustering adaptability” exists. PCA directly maximizes data variance, making it naturally suitable for KMeans. However, the Foundation Model learns disease-specific representations, leading to systematic transfer losses in downstream clustering tasks.

General Issues#

2. evaluation_planner.py incorrectly cached staig_gene_feat for 151676, causing Phase 2 to terminate with sys.exit(1).#

Solution: Modified the has_staig_gene_feat check logic to dynamically parse the actual number of top_genes in slice configurations (151676 used 5000 instead of default 3000), correctly identifying cached data. Key Insight: The implicit configuration split between slice-specific configurations (top_genes from staig_utils.py’s num_gene field) and the planner’s hard-coded defaults can lead to unnoticed false negatives.

3. In the STAIG fusion evaluation, slice 151676 experienced model collapse, resulting in ineffective clustering results.#

Solution: Skipped this slice and reported the average ARI of 10/11 slices (ARI=0.495). Key Insight: 151676 is a known difficult slice for STAIG. Its collapse is unrelated to the gene encoder and is a systemic issue caused by the data characteristics of this slice.

4. TokenMonitor couldn’t be built on HPC due to lack of Rust, webkit2gtk4.1-devel dependencies, and no sudo privileges.#

Solution: Suggested three approaches: build the binary on a local machine with permissions, ask a HPC administrator to install dependencies, or use GitHub Actions CI to build a Linux package. Note that Tauri applications require GTK/WebKit system libraries, which are typically unavailable in HPC environments, and the module system doesn’t provide these libraries. HPC is not designed as a target environment for GUI applications.

Learnings#

Key Learnings#

  • The STAIG fusion framework is highly robust towards gene encoders: Teddy’s 1024d model (ARI=0.495) and PCA’s 50d model (ARI=0.497) perform almost the same under the STAIG framework. This indicates that the STAIG’s spatial graph contrast learning mechanism dominates clustering performance, and the impact of gene embedding quality is greatly diluted.
  • The Foundation Model suffers from systematic transfer losses in cross-domain downstream tasks. Teddy is SOTA in disease classification (ACC=0.72), but its performance in spatial transcriptomics clustering is inferior to PCA’s (0.186 vs 0.255). The reason is the mismatch in optimization goals (differentiation vs variance maximization).
  • HVG1500 (direct concatenation of original high-variance genes, without dimensionality reduction) has an ARI of 0.330 on single slice 151673, which is better than all Foundation Models. This suggests that high-dimensional original features may be beneficial on some slices by preserving more local variance information.

Practical Learnings#

  • The implicit split between slice-specific configurations (e.g., staig num_gene) and the pipeline’s global default values is a high-risk bug. It should be dynamically parsed in the cache checking logic instead of being hardcoded.
