Problems & Solutions#

Critical Issues#

1. Teddy gene-only ARI (0.186) underperforms PCA (0.255) — Foundation Model fails to beat a simple baseline on clustering#

Solution: Analyzed root causes: Teddy is optimized for disease classification, not variance maximization; the 1024d high-dimensional space degrades KMeans performance; training data is dissociated scRNA-seq, with a distribution mismatch from spatial transcriptomics

Key Insight: There is a fundamental tension between the “general representation” of foundation model embeddings and “clustering adaptability”: PCA directly maximizes data variance and is naturally suited for KMeans, whereas Foundation Models learn disease-discriminative representations, incurring systematic transfer loss on downstream clustering tasks

General Issues#

2. evaluation_planner.py false-positive on staig_gene_feat cache check for 151676, causing Phase 2 to terminate with sys.exit(1)#

Solution: Modified the has_staig_gene_feat check logic to dynamically resolve the actual top_genes count from the slice configuration (151676 uses 5000 vs. the default 3000), correctly recognizing the cache as present

Key Insight: Slice-specific configuration (top_genes from the num_gene field in staig_utils.py) diverging silently from the planner’s hard-coded defaults is a category of “implicit configuration fork” that easily produces hard-to-detect false-negative checks

3. Model collapse on slice 151676 during STAIG fusion evaluation, producing no valid clustering result#

Solution: Skipped the slice and reported results averaged over 10/11 slices (ARI=0.495)

Key Insight: 151676 is a known difficult slice for STAIG; its collapse is unrelated to the gene encoder and is a systematic issue driven by the data characteristics of that particular slice

4. Cannot build TokenMonitor on HPC (missing Rust, webkit2gtk4.1-devel, and other Tauri dependencies; no sudo access)#

Solution: Proposed three paths: build on a local machine with sufficient privileges and copy the binary, ask HPC administrators to install dependencies, or build a Linux release package via GitHub Actions CI

Key Insight: Tauri applications depend on GTK/WebKit system libraries that are typically unavailable in HPC environments, and the module system does not provide such libraries either; HPC systems are fundamentally not a target deployment environment for GUI applications

Today’s Takeaways#

Core Takeaways#

  • The STAIG fusion framework is highly robust to gene encoder choice: Teddy 1024d (ARI=0.495) and PCA 50d (ARI=0.497) perform nearly identically under the STAIG framework, indicating that STAIG’s spatial graph contrastive learning mechanism dominates clustering performance and substantially dilutes the impact of gene embedding quality
  • Systematic transfer loss of Foundation Models on cross-domain downstream tasks: Teddy achieves SOTA on disease classification (ACC=0.72), but underperforms PCA on spatial transcriptomics clustering (0.186 vs. 0.255); the root cause is a mismatch in optimization objectives (discriminative vs. variance maximization)
  • HVG1500 (raw highly variable genes concatenated directly, no dimensionality reduction) achieves ARI=0.330 on single slice 151673, outperforming all Foundation Models; this suggests that high-dimensional raw features may benefit certain slices by preserving more local variance information

Practical Takeaways#

  • Implicit forks between slice-specific configuration (e.g., STAIG num_gene) and pipeline global defaults are a high-risk bug category; cache check logic should resolve these dynamically rather than hard-coding values
