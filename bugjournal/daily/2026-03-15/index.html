问题与解决方案#

关键问题#

1. MuJoCo body name 缺少 _main 后缀（cubeA 应为 cubeA_main），导致 _sim_body_name2id 返回 -1，Python 负索引静默读取错误位置，Stack 任务所有阶段检测失效，首批 11 个 demo 视频全部在 pre_reach 注入#

解决方案: 将 stack.yaml 中 body_name 改为 _main 后缀，并在 env_wrapper._sim_body_name2id 中添加通用 fallback（依次尝试 {name}_main 和 {name}_body0），确保所有 task 的 yaml 均兼容

关键洞察: Python 负索引将 -1 静默转为有效访问而不报错；此类 bug 只能通过输出异常（如阶段分布极度偏斜）才能发现，应在 body name 查找处显式断言返回值有效性

2. v4 代码归档计划覆盖了主要路径，但遗漏了 policy_adapter.py → collector.py 的传递依赖，导致归档后 v5 框架 import 失败#

解决方案: 将 BasePolicy（抽象接口）和 PolicyResult（dataclass）两个轻量类直接内联到 policy_adapter.py，消除对已归档模块的依赖；归档后通过 init.py 导入验证快速发现问题

关键洞察: 静态 import 追踪可能遗漏传递依赖（A→B→C 链），归档操作后应立即执行 import 验证脚本作为快速检测器

3. LeRobot 标量特征定义 shape:(1,) 但写入时使用 np.float32(scalar)（0维数组），导致 pi0.py 中 [:, None] 产生 (b,1,1)、jnp.stack 产生 (b,1,2) 等错误形状，训练阻塞#

解决方案: 统一写入方式为 np.array([val])（shape:(1,)）；移除 pi0.py 中冗余的 [:, None]；将 jnp.stack 改为 jnp.concatenate(…, axis=-1) 将两个 (b,1) 拼为 (b,2)

关键洞察: LeRobot features 字典的 shape 定义与实际写入 array 的 shape 必须严格匹配；设计时应统一约定标量特征是 (b,) 还是 (b,1) 并全链路一致

4. 引用数量与论文评估分数的关系不明确，是否应纳入 composite_score 及何时触发分析#

解决方案: 明确决策：引用数仅用于对 top-5 论文排序（tiebreaker），不影响 composite_score（0.4相关性 + 0.3启发性 + 0.3*新颖性）；引用图分析始终触发无需额外 flag

关键洞察: 引用数衡量传播广度，与研究质量/创新性正交；混入评分会扭曲筛选结果，两个维度应独立维护

5. Research Scout 的 _call_llm 与 research 包的 call_llm 是两套独立 LLM 封装，合并时无法复用#

解决方案: 重写 research 包的 call_llm 增加 backend 参数，与 research_scout.py 的三后端逻辑对齐；保持 research_scout.py 的评估逻辑不变，仅在 profiler 侧补充多后端能力

关键洞察: monolithic 脚本与模块化包集成时，以 monolithic 脚本作为 CLI 入口、用 lazy import 引用模块化包，可避免大规模重构风险

AI 局限性#

重要局限#

  • pi0.py 中原始 aux_targets 形状处理错误（[:, None] 和 jnp.stack 使用不当）是 AI 在之前对话中写的代码缺陷，说明 AI 对数据流 shape 的端到端跟踪存在系统性盲区
  • 未进行充分的预验证（task yaml body_name 配置正确性、framework 完整 import 依赖图），直接执行导致产生错误的第一批 demo 视频和遗漏的归档依赖，均需额外 debug 轮次

一般局限#

  • 在有明确规格说明时，AI 仍倾向于先提供多个选项或主动发起探索性操作（如 /init 时的 bash 列目录），增加了不必要的交互轮次

今日收获#

核心收获#

  • MuJoCo robosuite 环境的对象 body name 通常带有 _main 后缀（如 cubeA_main），但 task config 逻辑名可能不带；_sim_body_name2id 返回 -1 时 Python 负索引静默读取最后一个 body 而不报错——应在 body name 查找处显式断言或添加 fallback，并以输出分布极度偏斜作为此类 bug 的早期信号
  • 引用数量与研究质量是正交维度：高引用反映传播广度/时间影响，不代表研究创新性；在论文评估中应分离两个维度（引用数仅排序，composite_score 评估研究质量）
  • LeRobot 数据集中 shape:(1,) 特征批量加载后为 (b,1) 而非 (b,)；数据写入（np.array([val])）与特征定义（shape:(1,)）必须严格匹配，且模型侧不得再做额外升维
  • 代码归档前需进行完整的传递依赖分析（A→B→C 链式依赖），仅静态依赖列表可能遗漏间接依赖；归档后立即执行 import 验证脚本可快速捕获遗漏；import 追踪是划定版本边界的可靠方法
  • monolithic 脚本与模块化包的集成最佳实践：以 monolithic 脚本为 CLI 入口，用 lazy import（函数内 import）引用模块化包，避免循环依赖且无需重构现有逻辑
  • Semantic Scholar API 支持 /paper/{id}/citations 和 /paper/{id}/references 端点，字段含 paperId、title、year、citationCount、authors、abstract，无需 API Key（有限速）；支持 arXiv ID 和 DOI 解析
  • 对于 BC-RNN 等带历史的策略，action-replay 方式重放轨迹时动作会累计漂移；用 set_sim_state 直接设置仿真状态才能精确重现训练轨迹的每一帧
