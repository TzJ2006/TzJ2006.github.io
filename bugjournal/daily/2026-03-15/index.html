バグレポート 2026-03-15

March 15, 2026 · 21 min

Problems and Solutions### Key Issues#

1. Fundamental flaws in MIHD’s cross-sample embedding methodology: AI concludes that “PCA is worse than STAIG” based on the wrong assumption that “input features themselves are weak,” without actively questioning the validity of the experimental design.#

Solution: After users questioned the effectiveness of cosine similarity, it was discovered through import tracing that both PCA and STAIG exhibit double-incomparability due to independent HVG selection per-section and independent PCA fitting. Comparing the two is ineffective; instead, using the raw_shared scheme with shared HVG intersection (1137 genes) as the correct baseline is recommended.

Key Insight: The effectiveness of cross-sample embedding comparisons depends on shared feature spaces. Embeddings fitted independently per-section have no mathematical meaning in terms of cross-sectional cosine similarity, regardless of the model used.

2. Drop skill objects do not actually fall: After opening the gripper and setting the object’s initial velocity, the OSC controller in env.step() re-applies clamping force to “pull” the object back.#

Solution: After opening the gripper and setting the object’s initial velocity, call mujoco.mj_step() to run 15 physical steps (completely bypassing the OSC controller) to complete the initial separation before entering the standard control loop.

Key Insight: sim.forward() only updates the kinematic state without advancing the dynamics. Only mujoco.mj_step() actually advances the MuJoCo physics engine, thereby bypassing all higher-level controllers. There is a fundamental conflict between direct state operations and feedback controllers. Simulation injection design must clearly choose one path.

3. Stack task body name parsing fails silently: stack.yaml uses cubeA/cubeB, while MuJoCo’s actual names are cubeA_main/cubeB_main. The _sim_body_name2id returns -1, and Python negative index body_xpos[-1] reads the last body. Task phase detection misjudges all as pre_reach.#

Solution: Fix the body_name field in stack.yaml. Add a fallback logic with the main/_body0 suffix in env_wrapper._sim_body_name2id. Output a WARNING instead of silently returning -1 when a failure occurs.

Key Insight: The negative index of body_xpos[-1] always returns the same position in both cubes. This silent error is extremely difficult to detect. Any parsing failure should trigger an alarm rather than returning a sentinel value.

4. VLA context replay architecture assumption error: AI designed a complete N-1 frame replay mechanism, believing it was necessary for providing correct observation history for VLA. At the same time, multiple data sources (demo+VLA rollout+BC rollout) were designed, ignoring the differences in controllability of each source.#

Solution: Users pointed out that most VLAs do not have a context window, and context replay is a useless overhead. Reconstruct ContextReplayEngine into InjectionEngine to directly restore the injected frame sim state. Limit data sources to MimicGen demo data (higher controllability).

Key Insight: The general benchmark should be efficient for models without context (BC-RNN, ACT, etc.). It cannot be over-engineered only for a few VLAs that support historical inputs. Users have a better understanding of the actual model range than AI’s theoretical inferences.

5. CalendarPro intent misclassification: Semantic routing has no confidence threshold (0.52 is also considered a valid classification). Sentences with time expressions are routed incorrectly due to keyword matching errors. The short confirmation word ‘ok’ lacks context understanding. The system prompt is too long (530 lines), and the estimation of Chinese tokens is three times off.#

Solution: Add a per-routing threshold (0.40-0.60). Cases below the threshold are handed over to the LLM. Introduce a keyword scorer (time regular boost schedule) combined with an embedding of 70/30 ratio. Split SYSTEM_PROMPT into BASE (~50 lines) and 11 intent-specific fragments for injection as needed. Change the estimation of Chinese tokens to chinese_chars × 1.5 + other_chars / 4.

Key Insight: Embedding’s nearest-neighbor routing lacks the ability to express “I’m not sure.” Keyword scoring precisely compensates for the embedding’s blind spot in dealing with time/digital expressions. The two complement each other. Chinese character tokens account for about six times that of English characters. Without correction, the context length will be systematically underestimated.

6. S2 co-author analysis is completely ineffective for top researchers like Levine/Abbeel/Finn (all depth-2 sections are empty), and there is a serious issue of name ambiguity (Xiaolong Wang matches veterinary/geological researchers, Shuran Song has only 2 papers in 2025).#

Solution: Reconstruct it as a homepage-first strategy: Prioritize extracting student lists from researchers’ homepage/laboratory pages. Co-authorship should be used as a supplement. Use multiple strategies for URL discovery (S2 homepage field + LLM suggestions + –homepage parameter). Warn of name ambiguity and suggest using S2 authorId for precise querying.

Key Insight: Academic homepages explicitly list students, which is a level one order of reliability compared to inferring from co-authorship relationships. Top researchers publish over 500 papers, and the first-author signal is diluted by many collaborators. There are fundamental applicability issues with co-author analysis methodology.

7. Research toolchain fragmentation: The functions of paper scout and researcher profiler overlap, and commands are scattered. There is complete duplicate code in New feature/directory. The dimension of citation relationships is missing in the toolchain.#

Solution: Use research_scout.py as a unified CLI entry point. Import the modular profiler as a library through lazy loading. Add a new profile/citations subcommand. Remove the New feature/directory and add the Semantic Scholar citation graph API. The three-stage report automatically runs a citation analysis on top 5 papers.

Key Insight: Retain the original CLI entry point and integrate new modules through lazy loading, rather than completely rewriting them. This ensures backward compatibility. Citation relationships (forward citations + backward references) are a core feature that is underestimated in the research toolchain.

8. Demo video scripts using action replay lead to cumulative open-loop errors in later frames. Phase detection in later frames is completely incorrect.#

Solution: Use set_sim_state() to directly restore the MuJoCo state vector for each frame, bypassing the accumulation of open-loop errors.

Key Insight: The stored clean trajectory contains a complete sim state vector. Restoring the state directly is much more accurate than replaying actions. Action replay is suitable for real-time control, while state restoration is suitable for offline analysis.

9. trajectory_regression cannot find any injection opportunities: can_inject() requires prev_phases to be at least 10 in length, but the pipeline never calls replay_and_label_phases(), and phase_labels is always None.#

Solution: Pass the –label_phases flag by default in Step 0 of run_v5_all_tasks.py.

Key Insight: Implicit dependencies (certain skills require a specific step of the pipeline to be explicitly activated) can only be exposed when that skill fails to run. End-to-end integration testing is more effective at detecting such pipeline-level defects than unit testing.

10. pi05 training error: The CheckpointWeightLoader structure does not match. The new progress layer (progress_mlp_in/out/cond_proj) is not in the checkpoint and does not match the hard-coded ‘..lora.’ regular expression.#

Solution: Add a configurable missing_regex field to CheckPointWeightLoader (defaulting to ‘..lora.’ for backward compatibility). Change the 4 progress experiment configurations to ‘..lora.|.progress.’.

Key Insight: Loading pre-trained weights with a new experimental module is a high-frequency scenario. The missing_regex should be designed as a configurable parameter. This is exactly the same requirement as LoRA fine-tuning, which is a common design pattern for transfer learning.

11. Aux_targets shape assumption error in pi0.py: AI inferred that LeRobot would maintain the shape of (b,1) after loading the shape of (1,) and modified the code accordingly. In reality, LeRobot will squeeze it to be a scalar (b,), resulting in a shape mismatch during training.#

Solution: Run the training and observe the log (‘aux_targets[…]: (32,)@float32’) to confirm the actual shape. Restore the original original[:, None] and jnp.stack operation.

Key Insight: LeRobot automatically squeezes the scalar feature of shape=(1,?) during DataLoader processing. This is a framework-level behavior. Hypotheses about the internal behavior of third-party frameworks must be verified through actual running and cannot be purely inferred.

General Issues#

12. adata_hvg cache bug: The HVG AnnData var_names in section 151673 were reset to integer indices (‘0’,‘1’,‘2’…), resulting in a zero intersection of gene names.#

Solution: Abandon reliance on the adata_hvg cache and load directly from the original HDF5 data (via load_dlpfc_data). Manually perform normalization and HVG selection.

Key Insight: Silent transformations may occur when writing key fields in cached data. A soundness check should be performed before use (e.g., whether var_names are gene symbols rather than integers).

13. When the LLM generates Chinese long-form JSON, Chinese quotation marks ("") mix into the JSON string values, leading to parsing failures. This issue is repeated in the profiles of many researchers such as Chelsea Finn, Yuke Zhu, and Mingyu Ding.#

Solution: Resubmit the malformed JSON to Claude, requesting only a corrected pure JSON. Automate processing through the repair_json_with_llm mechanism. Plan to explicitly require English quotation marks in the prompt as a long-term fix.

Key Insight: Generating JSON containing Chinese content should explicitly require English quotation marks in the prompt. Alternatively, perform format verification immediately after generation and fix the JSON as a fixed pipeline step, which is more reliable than relying on generation quality.

Human Thinking vs AI Thinking#

Strategic Level#

Methodological questioning of cross-sample cosine similarity effectiveness (MIHD)#

Role Thought
Human After AI concludes that “PCA is worse → input features themselves are weak,” humans instinctively ask, “Could it be that the cos similarity of cross-sample embedding is problematic?” This directly points to methodological flaws rather than numerical results.
AI AI tends to attribute problems directly from experimental numbers and does not actively question the validity of the experimental design itself (whether the feature space is comparable).

Difference Analysis: Humans have a prior ability to doubt methodologies and can ask, “Is the experiment designed correctly?” when they see abnormal results. AI is better at analyzing data under given assumptions, but requires external triggering for reflection on the assumptions themselves.

Difference Analysis: Users identify over-engineering based on the characteristics of the actual models and data controllability. AI starts from theoretical correctness and requires user practical experience to correct it.

Delegation mode for VLA training debugging#

Role Thought
Human Adopt a goal-driven delegation strategy: “Execute the training command yourself, fix all errors until no more errors occur, and then stop.” This gives clear termination conditions without interfering with specific steps.
AI AI follows a scientific method: run → observe errors → read source code to locate the problem → make minimal changes → re-run. However, it makes an error in the shape assumption of LeRobot and needs actual running logs to correct it.

Difference Analysis: Human delegation mode allows AI to debug independently. The wrong shape assumption is exposed through natural running. It is correct for humans to choose not to intervene in specific decisions—the error correction mechanism is built into the iterative loop.

AI Limitations#

Important Limitations#

  • Insufficient ability to reflect on experimental conclusions: After conducting the MIHD PCA diagnostic experiment, it directly draws wrong conclusions from surface numerical results without actively checking the validity of the experimental design itself (the comparability issue of independent per-section PCA). It requires external intervention by users to be corrected.
  • Silent failure modes lead to serious bugs being hidden for a long time: When parsing body names in stack.yaml fails, it silently returns -1. No warnings or assertions are designed, resulting in Python negative index bugs being completely invisible. Similar problems (integerization of var_names in adata_hvg cache) are also not discovered due to lack of soundness checks.
  • Over-engineering and wrong architectural assumptions: V5 context replay is over-designed based on the wrong assumption that “all VLAs require a context window.” It makes incorrect assumptions about the internal behavior of third-party frameworks (LeRobot), assuming that shape=(1,) features will be squeezed, and modifies the code accordingly. Both need to be corrected by users or verified through actual running.
  • Insufficient ability to actively question the applicability boundaries of methodologies: When S2 student discovery is completely ineffective for top researchers, AI continues to deeply debug the code logic (reading student_discovery.py, analyzing threshold settings), without actively questioning the applicability boundaries of the methodology itself. It requires user guidance to switch to the homepage solution.
  • Insufficient entity disambiguation ability in Semantic Scholar: For common Chinese-English translations such as Xiaolong Wang, Shuran Song, and Ming Yu, it almost always matches the wrong researchers. The LLM analysis cannot automatically identify “this is not the same person,” and can only issue warnings afterward. It lacks the ability to actively disambiguate entities.

General Limitations## Today’s Achievements#

Key Achievements#

  • Necessary prerequisite for cross-sample embedding comparison: The feature spaces must be shared. Independent HVG selection per section + independent PCA fitting results in double irreliability; effective cross-sample baselines must use the intersection of shared HVGs + joint PCA, or a foundation model with fixed pre-trained weights.
  • Standard method to bypass the upper-layer controller in MuJoCo: It is necessary to directly call mujoco.mj_step() (propagation dynamics) instead of sim.forward() (only updating kinematics), so as to complete the physical state change before the OSC controller intervenes. There is a fundamental conflict between direct state operations and feedback controllers. The simulation injection design must clearly choose one path.
  • Composite objects generated by MuJoCo’s CompositeBodyObject usually have a _main suffix in their body names (e.g., cubeA_main instead of cubeA). The body name parsing function of the env wrapper needs to include a backtracking logic for multiple candidate names ({name}→{name}_main→{name}_body0). When searching fails, it should output WARNING instead of silently returning -1.
  • The semantic split of error types should be based on ‘whether the recovery strategy is the same’ rather than ‘whether the injection mechanism is the same’: The detection conditions and recovery logic for drop_in_transit/drop_at_wrong_place/drop_with_interaction are completely different. Even if the injection actions are the same, they must be modeled separately. This is more meaningful for curriculum design during the training phase.
  • Architectural defect of the Semantic Router: The nearest neighbor always produces one result, unable to express ‘I’m not sure’. A combination of confidence thresholds + alternative LLMs + keyword scorers is the most practical repair pattern and can be applied to all vector retrieval-based classification systems (RAG routing, tool selection, etc.).
  • For top scholars who have published over 500 papers, the analysis of S2 co-author frequencies cannot reliably identify students—the first-author signal is diluted by a large number of collaborators. Listing students explicitly on the professor’s homepage is an order of magnitude more reliable than inferring reliability from collaborative relationships. This is the correct method for discovering students of top scholars.
  • Citation graphs (forward citations + backward references) are undervalued core features in the research toolchain: By analyzing ‘who cited this paper’, we can understand the evolution of research influence and popular follow directions; in the scoring system, ‘relevance’ and ’number of citations/popularity’ should be handled separately—citing count is used for ranking rather than scoring, preventing papers with high citations but low relevance from interfering with project direction filtering.
  • Mentor pedigree of embodied AI scholars: Mingyu Ding ← Jitendra Malik (Berkeley), Ruoshi Liu ← Carl Vondrick (Columbia), Xiaolong Wang ← Abhinav Gupta (CMU), Shuran Song ← Thomas Funkhouser (Princeton), Yunzhu Li ← Antonio Torralba (MIT), Yuke Zhu ← Li Fei-Fei (Stanford) — presenting a systematic student output pattern from top perception/robotics mentors to embodied AI.
  • In offline trajectory analysis, directly using set_sim_state() to restore the complete state vector for each frame is much more accurate and reliable than action replay, completely avoiding the accumulation of open-loop errors. Saving clean trajectories with states at the same time is a correct design decision.
  • The LeRobot dataset framework automatically squeezes single scalar features of shape=(1,) into (batch_size, ) when using DataLoader. In the model code, it is necessary to explicitly raise the dimension using[:, None]. When storing data, use np.float32(scalar) instead of np.array([scalar]). This is a framework-level behavior that must be verified through actual logs rather than inferred.
  • On-demand injection strategy in prompt engineering: Splitting the system prompt into a base (always included) + intent-specific fragment (dynamically injected based on classification results) can reduce token consumption by 40-60%. The token density of Chinese characters is about 6 times that of English characters (1.5 tokens/character vs 0.25 tokens/character). Without correction, the context length will be systematically underestimated.
  • The hack of sys.path.insert is a fragile cross-module dependency method: Any function renaming will cause a ImportError at runtime. The correct approach is to use common/packages + pip install -e..; the Python re-export shim pattern (the module only contains from x import y; all=[…]) is an elegant migration method that maintains backward compatibility.
  • In multi-tool coexistence projects, organizing output directories ‘by file type priority’ rather than ‘by tool name priority’ (outputs/reports/summarize/ instead of summarize/reports/) can reduce the.gitignore from multiple scattered rules to a single line outputs/, making it more friendly for CI/CD and disk quota management.
  • Verifying the feasibility of the simulation pipeline in small batches (100 total) is the correct iterative strategy: 207 scenarios exposed 5 systematic defects. If you directly rush to 2900 scenarios, you will waste a lot of GPU time on injections that are doomed to fail. End-to-end integration testing can discover implicit dependency defects at the pipeline level better than unit testing.
  • Flow matching is becoming the mainstream action decoding architecture for robot VLA: Models like π₀ converge to a combination of ‘pre-trained VLMs and flow-matching action heads’, which outperforms diffusion models in multi-modal modeling of continuous high-dimensional action spaces. Shuran Song’s Im2Flow2Act and UMI are two major breakthroughs in the field of robot data efficiency in 2024.
