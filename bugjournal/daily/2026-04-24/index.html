Problems & Solutions#

Critical Issues#

1. Claude Code JSONL format change caused 2.4x cost inflation: assistant messages with multiple content blocks written as separate lines with identical usage, but deduplication skipped when requestId missing#

Solution: Modified create_claude_unique_hash() to use message.id alone as fallback when requestId is None. Same message.id across multiple lines now correctly deduplicated as single API call

Key Insight: New Claude Code format lacks top-level requestId in many entries; message.id is sufficient unique identifier since content blocks share same message.id

2. SSH remote sync caused 10-25x data inflation: extraction script emitted duplicate content-block lines, and local append used -newer file filter causing full file re-extraction on any update, with no deduplication before append#

Solution: Two-layer fix: (1) Remote Python script now maintains seen={} by message.id during extraction. (2) Local sync_host() loads existing cache, builds key-set from (ts, model, in, out) tuples, filters new records before append

Key Insight: File-based incremental sync (-newer mtime) does not imply record-level deduplication. Every layer (extraction, ingestion, aggregation) needs explicit duplicate detection

3. ccusage reported ~1.75x higher token usage than TokenMonitor on same data, causing confusion about actual consumption#

Solution: Root cause identified in ccusage’s createUniqueHash() requiring both message.id AND requestId - when requestId is missing (13,960 of 20,116 entries), no deduplication occurs, causing duplicates to be counted multiple times. Additionally, ccusage uses ‘first-wins’ strategy which keeps early snapshots with low token counts

Key Insight: Streaming APIs write multiple JSONL snapshots per response with incrementing output_tokens; ‘first-wins’ deduplication keeps early snapshots with low token counts, while TokenMonitor’s ‘max output_tokens’ strategy correctly captures final values

4. Apple Speech subtitles stayed gray (partial state) and never turned white (finalized), with new text overwriting previous text instead of appending#

Solution: Modified _on_result callback condition from ‘if text and text != self._last_text’ to ‘if text and (is_final or text != self._last_text)’ to ensure final callbacks aren’t skipped by deduplication logic

Key Insight: Apple’s SFSpeechRecognizer returns identical text for final result and last partial result; deduplication must check is_final flag explicitly

5. Agent’s claude_parser.rs extraction reverted ssh.rs to old version, losing Task 3’s device_aggregation refactor#

Solution: Human identified issue by checking file line counts (ssh.rs back to 1536 lines). Re-applied all ssh.rs changes manually: removed aggregation functions, updated imports to device_aggregation::*, fixed all callers. Verified with cargo check + test

Key Insight: Agent-driven file modifications can revert uncommitted changes if system-reminder contains stale linter state. Always verify critical files after agent operations

6. Apple Speech audio capture stopped working with ‘TypeError: wrong type’ in ctypes.memmove when copying audio samples to AVAudioPCMBuffer#

Solution: Bypassed PyObjC bridge by using ctypes to directly call objc_msgSend with sel_registerName(‘floatChannelData’), obtaining native ctypes-compatible float** pointer instead of PyObjC-wrapped object

Key Insight: PyObjC bridge types are not directly compatible with ctypes functions; need to use objc_msgSend for low-level C API interop

7. SSH remote extraction only found ~/.claude/projects and ~/.config/claude/projects, missing custom installations like ~/.claude-code with different models#

Solution: Replaced hardcoded paths with glob pattern ‘for _cd in $HOME/.claude*; do [ -d $_cd/projects ] && …’ to auto-discover all .claude-prefixed directories; also source shell rc files to pick up CLAUDE_CONFIG_DIR environment variable

Key Insight: Users may have multiple Claude CLI installations with different configs in parallel directories; glob-based discovery is more robust than hardcoding

8. GitHub Actions cache approaching 10GB limit due to caching entire target/ directory including incremental build artifacts and docs#

Solution: Replaced actions/cache@v5 with Swatinem/rust-cache@v2 which only caches dependencies, added CARGO_INCREMENTAL=0 for CI job, achieved 80% cache reduction

Key Insight: Incremental compilation benefits local dev but wastes CI cache space since runners are stateless; specialized Rust cache actions provide smarter caching than generic file caching

9. Chart line Y-coordinates mismatched tooltip values: $51.45 point rendered near $0, while tooltip correctly showed $51.45#

Solution: Changed lineData segment lookup from find() to filter().reduce((sum, s) => sum + s.cost, 0) to sum all matching segments per bucket

Key Insight: Aggregated data (tooltip) and visual encoding (line position) must use same summation logic. Device-level disaggregation means one model can have multiple segments per time bucket

10. DevicesView and main chart showed different device breakdown totals. DevicesView used archive + frontier filtering, main page read raw remote-cache usage.jsonl without excluding archived hours#

Solution: Unified both code paths to call build_device_summary_merged() with archive.frontier() filter, skipping compact records already covered by archived entries

Key Insight: Archive system creates dual data sources (archived JSONL + live cache). All consumers must use frontier cutoff to avoid double-counting transition period

General Issues#

11. CI failed with ‘cannot find type FloatBallSizes’ — Linux #[cfg] blocks used FloatBallSizes but mod.rs didn’t import it for Linux#

Solution: Added #[cfg(target_os = “linux”)] use layout::FloatBallSizes; and #[allow(unused_imports)] on FloatBallAnchor (only used in non-Linux #[cfg] blocks). Verified with cargo clippy

Key Insight: Platform-conditional compilation requires platform-specific imports. Unused import warnings can be platform-dependent; use #[allow(unused_imports)] with #[cfg] documentation

12. Chart Y-axis scaling too coarse: $50→$100 jump made $51-75 data appear ‘flat’ (only using half chart height)#

Solution: Added intermediate thresholds to niceMax() sequence: 15/30 between 10-50, 75 between 50-100, 150/300 between 100-500, etc. Now $51 maps to $75 ceiling instead of $100

Key Insight: Nice number scales need denser steps in commonly-used ranges. 2x jump is too aggressive for cost data typically $10-100

Learnings#

Key Learnings#

  • Multi-layer data pipelines require defensive deduplication at every boundary: extraction (remote script), ingestion (local append), aggregation (parser), and archival (frontier filtering). File-based incremental sync does not guarantee record-level uniqueness
  • Correct JSONL dedup for streaming APIs: same message.id written multiple times with incrementing output_tokens. Must keep last entry or max(output_tokens). Relying solely on requestId (which may be absent) causes dedup failure. ‘First-wins’ strategy undercounts output tokens
  • Open-source tool correctness validation: ccusage has two unresolved bugs - #797 (requestId missing causes overcount) and #705 (first-wins undercount output_tokens). Cross-tool comparison + GitHub issue search effectively validates algorithm correctness
  • Apple’s SFSpeechRecognizer returns identical text for isFinal=True and last partial result; deduplication logic must explicitly check is_final flag to avoid skipping finalization callbacks
  • Factory closure pattern with shared mutable state (e.g., resizeOrchestrator with 19 state variables) should NOT be forcibly refactored. Cost-benefit analysis: only 6/573 lines are pure functions; extraction would require passing 19 parameters or introducing state object, increasing cognitive load
  • Reusing proven architectural patterns (LiteLLM fetch/cache) for similar features (exchange rates) ensures consistency and reduces bugs. Both use: fetch API → disk cache (TTL) → OnceLock<RwLock<» → startup load + async refresh + background loop check
  • GitHub Actions cache strategy: incremental compilation provides no benefit in CI (cold builds every run). Swatinem/rust-cache intelligently caches only dependencies vs entire target/ directory, reducing cache by 80% while maintaining speed
  • User domain intuition about system behavior (SSH sync repetition) is often faster than AI’s code-path analysis for identifying root causes. Hypothesis-driven debugging beats exhaustive tracing when symptoms have clear patterns
  • Streaming API deduplication requires ’last-wins’ or ‘max-value’ strategies rather than ‘first-wins’ because early chunks contain incomplete accumulated values (especially for output_tokens)
  • PyObjC bridge objects are not ctypes-compatible; use ctypes objc_msgSend with sel_registerName for direct C API interop to obtain native pointers
  • Glob-based path discovery (~/.claude*/) is more maintainable than hardcoding specific directory names when users may have parallel installations with different configs
  • Module extraction pattern for large Rust files: directory modules (foo/ with mod.rs + layout.rs) cleanly separate pure logic from stateful/platform-specific code. Applied successfully to float_ball.rs (geometry → layout.rs, IPC/window → mod.rs) and parser.rs (Claude parsing → claude_parser.rs)
  • Chart visual encoding and tooltip aggregation must use identical data pipeline. If tooltip sums all device segments but line Y uses first segment only, the mismatch is immediately visible even if both are ‘correct’ in their own context
  • jq as stateless JSON processor cannot implement deduplication - requires stateful tools like Python/awk. Multi-path extraction logic (jq/python/grep) must guarantee behavioral consistency
  • Archive system creates dual data sources (archived JSONL + live cache). All consumers must use frontier cutoff to avoid double-counting transition period
  • MimicGen augmentation requires deep semantic understanding: task phases encode manipulation progress, object anchoring prevents invalid scenes, warping preserves relative pose, and success validation is essential. Pure trajectory replay fails without context-aware adaptation
  • Deduplication key construction should use fallback strategies (e.g., message.id when requestId missing) rather than returning null, to avoid accidentally treating duplicates as unique entries

Practical Learnings#

  • Nice number scales need empirical tuning for the domain’s typical value range. Generic geometric progression (1/2/5/10/20/50/100) is too coarse for cost data concentrated in $10-100; intermediate steps (15/30/75) improve data-ink ratio
