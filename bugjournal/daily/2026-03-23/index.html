Problems and Solutions#

Key Issues#

1. MIHD slice retrieval almost completely failed (10 out of 14 queries have SL@50=0)#

Solution: Identified that PCA/STAIG processed each slice independently, resulting in non-comparable embedding spaces. The solution is to use the scGPT embedding algorithm, which provides a shared space with an average SL@50 of 0.416 and a hit rate of 100%.

Key Insight: Comparability of the embedding space (whether there is a shared coordinate system) is more important than the quality of the embedding itself. Different encoders should be used for slice-based tasks and single-slice clustering.

2. MIHD single-slice ARI differs from SOTA (currently 0.546, target above 0.60)#

Solution: Developed a four-stage ablation experiment plan (clustering ablation → batch effect analysis → probe retrieval → 151676 specific diagnosis). This plan has not been fully implemented yet.

Key Insight: Zero-variance of the 151676 embeddings caused mclust failure. Returning to KMeans could reduce the mean, but the root cause remains unclear.

General Issues#

3. Missing ~/.config/gadget/sync.json file on DCC, preventing sync.py from running#

Solution: AI read the existing summarize configuration to obtain rclone remote settings. Created a sync.json file and executed a pull operation after verifying the remote directory structure.

Key Insight: Configuration files do not migrate automatically when the device is initialized. They need to be inferred and recreated from the existing configuration.

4. When using cp -r to copy cchelper directory, files are copied to ~/.claude/skills/ directory instead of maintaining the sub-directory structure#

Solution: After identifying the problem, created a subdirectory with mkdir -p ~/.claude/skills/cchelper/. Then used the mv command to move the scattered files.

Key Insight: When the target directory already exists, cp -r will expand the contents of the source directory. Before executing this command, it is necessary to confirm the target path.

Today’s Summary#

Key Conclusions#

  • In spatial transcriptomics slice-based tasks, the comparability of the embedding space (shared coordinate system) is more important than the local quality of the embedding itself. The base model (scGPT) produces a fixed spatial representation, which is very suitable for slice-based retrieval. However, PCA/STAIG leads to non-comparable coordinate systems when dealing with independent slices.
  • The ideal method for single-slice clustering (STAIG) differs from slice-based retrieval methods (scGPT). For different tasks, different encoders should be used to achieve the best results, rather than pursuing a unified solution.

Practical Conclusions - The scavenger-h200 partitions on the DCC cluster provide H200×8 nodes, which are very important for training tasks that require a large amount of memory. The TRES limit is 100 CPUs and 500G of memory, which is a hard constraint when submitting large jobs.#
