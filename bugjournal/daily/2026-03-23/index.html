Problems & Solutions#

Critical Issues#

1. MIHD cross-slice retrieval almost completely failed (10 out of 14 queries had SL@50=0)#

Solution: It was determined that the root cause was PCA/STAIG processing each slice independently, resulting in an non-comparable embedding space. The solution is to use scGPT embedding (shared space, average SL@50=0.416, 100% hit rate). Key insight: The comparability of the embedding space (whether it has a shared coordinate system) is more fundamental than the quality of the embedding. Different encoders should be used for cross-slice tasks and single-slice clustering.

2. MIHD single-slice ARI is not as good as SOTA (current 0.546 vs target 0.60+)#

Solution: A four-stage ablation experiment plan was developed (clustering ablation → batch effect analysis → retrieval probe → 151676 special diagnosis). This plan has not been fully executed yet. Key insight: The zero variance of the 151676 embedding caused mclust to fail. Retrofitting KMeans reduced the mean value, but the root cause remains unknown.

General Issues#

3. No ~/.config/gadget/sync.json on DCC, so sync.py cannot be executed#

Solution: AI read the existing summarize config to get the rclone remote configuration. Manually created sync.json and verified the remote directory structure before executing pull. Key insight: Configuration files do not automatically migrate during initial setup of a new device. It’s necessary to infer from existing configurations and rebuild them.

4. When using cp -r to copy the cchelper directory, files are copied to ~/.claude/skills/ instead of maintaining a sub-directory structure#

Solution: After discovering the issue, I created a sub-directory using mkdir -p ~/.claude/skills/cchelper. Then, I moved the scattered files in mv. Key insight: cp -r will expand the source directory content directly if the destination directory already exists. It’s necessary to confirm the destination path before executing this command.

AI Limitations#

  • When analyzing MIHD projects, AI relied entirely on existing documents (plans.md/RESULTS.md/diagnostic reports). It couldn’t infer hidden progress or determine the priority of ablation experiments.

General Limitations#

  • When copying directories, it ignored the behavior difference of cp -r when the destination already existed. This resulted in files being copied instead of maintaining a sub-directory structure. This issue was discovered and corrected by the user himself/herself.

  • When searching for rclone configuration using glob, it timed out on large HPC file systems (20 seconds). It’s necessary to switch to a more precise path search strategy.

Learnings#

Key Learnings#

  • In spatial transcriptomics cross-slice tasks, the comparability of the embedding space (shared coordinate system) is more important than the local quality of the embedding. Foundation Model (scGPT) is suitable for cross-slice retrieval because it outputs a fixed representation space. PCA/STAIG, on the other hand, process slices independently, resulting in an non-comparable coordinate system.
  • The optimal method for single-slice clustering (STAIG) is different from the optimal method for cross-slice retrieval (scGPT). Different encoders need to be chosen for different tasks, rather than aiming for a unified optimal solution.

Practical Learnings#

  • The scavenger-h200 partition on the DCC cluster provides H200×8 nodes (although they are抢占式). These nodes are important resources for training tasks that require high memory. The TRES limit of 100 CPU cores and 500G of memory is a hard constraint when submitting large jobs.
