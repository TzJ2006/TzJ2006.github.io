Problems & Solutions#

Critical Issues#

1. MIHD cross-slice retrieval almost completely failed (10 out of 14 queries had SL@50=0)#

Solution: It was determined that the root cause was PCA/STAIG processing each slice independently, resulting in an uncomparable embedding space. The solution is to use scGPT embedding (shared space, average SL@50=0.416, 100% hit rate). Key insight: The comparability of the embedding space (whether it has a shared coordinate system) is more fundamental than the quality of the embedding. Different encoders should be used for cross-slice tasks and single-slice clustering.

2. The ARI of MIHD single slices differs from the SOTA (current 0.546 vs target 0.60+)#

Solution: A four-stage ablation experiment plan was developed (clustering ablation → batch effect analysis → retrieval probe → 151676 special diagnosis). This plan has not been fully executed yet. Key insight: The zero variance of the 151676 embedding caused mclust to fail. Retracing to KMeans reduced the mean value, but the root cause remains unknown.

General Issues#

3. There is no ~/.config/gadget/sync.json on DCC, preventing sync.py from running#

Solution: AI retrieved the existing summarize config to get the rclone remote configuration. Manually created sync.json and verified the remote directory structure before executing pull. Key insight: Configuration files do not automatically migrate during device initialization. It’s necessary to infer from existing configurations and rebuild them.

4. When using cp -r to copy the cchelper directory, files are copied to ~/.claude/skills/ instead of maintaining the sub-directory structure#

Solution: After identifying the issue, created sub-directories using mkdir -p ~/.claude/skills/cchelper. Then, moved the scattered files in mv. Key insight: cp -r will copy the contents of the source directory directly if the destination directory already exists. It’s necessary to confirm the destination path before executing this command.

AI Limitations#

Critical Limitations#

  • When analyzing MIHD projects, AI relied entirely on existing documents (plans.md/RESULTS.md/diagnostic reports). It couldn’t infer hidden progress or determine the priority of ablation experiments.

General Limitations#

  • Copying directories ignored the behavior difference of cp -r when the destination already existed. This resulted in files being copied instead of maintaining the sub-directory structure. This issue was discovered and corrected by AI itself.
  • Searching for rclone configuration using glob search timed out in large HPC file systems (20 seconds). It’s necessary to switch to a more precise path search strategy.

Learnings#

Key Learnings#

  • In spatial transcriptomics cross-slice tasks, the comparability of the embedding space (shared coordinate system) is more important than the local quality of the embedding. Foundation Model (scGPT) is suitable for cross-slice retrieval because it outputs a fixed representation space. However, PCA/STAIG processes slices independently, making the coordinate system non-comparable.
  • The optimal method for single-slice clustering (STAIG) is different from the optimal method for cross-slice retrieval (scGPT). Different encoders need to be chosen for different tasks, rather than trying to find a unified optimal solution.

Practical Learnings#

  • The scavenger-h200 partition on the DCC cluster provides H200×8 nodes (although they are抢占式). These nodes are important resources for training tasks that require high memory. The TRES limit of 100 CPU cores and 500G of memory is a hard constraint when submitting large jobs.
