Problems and Solutions#

Key Problems#

1. MIHD cross-slice retrieval almost completely failed (10 out of 14 queries had SL@50=0)#

Solution: Identified that the root cause was PCA/STAIG processing each slice independently, resulting in an embedded space that couldn’t be compared. The solution is to use scGPT embedding method (shared space, average SL@50=0.416, hit rate 100%). Key Insight: The comparability of the embedding space (whether it has a shared coordinate system) is more important than the quality of the embedding. Different tasks should use different encoders.

2. MIHD single slice ARI does not match the current optimal value (currently 0.546, target is above 0.60)#

Solution: Developed a four-stage ablation experiment plan (clustering ablation → batch effect analysis → retrieval probe → 151676 special diagnosis). This plan has not been fully executed yet. Key Insight: Zero variance of the 151676 embeddings caused mclust to fail. Backtracking to KMeans, the mean value decreased, but the root cause remains unclear.

General Problems#

3. No ~/.config/gadget/sync.json file on DCC, causing sync.py to fail#

Solution: AI obtained the existing summarize configuration to get rclone remote configuration. Manually created sync.json and verified the remote directory structure before executing pull. Key Insight: Configuration files do not migrate automatically when the device initializes. They need to be inferred and reconstructed from the existing configuration.

4. When using cp -r to copy the cchelper directory, files were copied to ~/.claude/skills/ instead of maintaining the sub-directory structure#

Solution: After identifying the problem, used mkdir -p ~/.claude/skills/cchelper to create the sub-directory. Then used mv to move the scattered files. Key Insight: If the target directory already exists, cp -r will directly copy the contents of the source directory. Before executing this command, you need to confirm the target path.

  • Using glob search to find rclone configuration timed out in the large HPC file system (20 seconds). A more precise path search strategy is needed.

Lessons Learned#

Key Lessons#

  • In spatial transcriptomics cross-slice tasks, the comparability of the embedding space (shared coordinate system) is more important than the local quality of the embedding. The base model (scGPT) is suitable for cross-slice retrieval because it outputs a fixed representation space. However, PCA/STAIG processes slices independently, making the coordinate system comparison impossible.
  • The best method for single-slice clustering is different from the best method for cross-slice retrieval. For different tasks, different encoders need to be used, rather than trying to find a unified optimal solution.

Practical Experience - The scavenger-h200 partition on the DCC cluster provides H200×8 nodes (although they are preemptive). These nodes are very important resources for training tasks that require a lot of memory. The TRES limit is 100 CPU cores and 500G of memory, which is a hard constraint when submitting large tasks.#
