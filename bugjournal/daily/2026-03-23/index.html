Issues & Solutions#

Critical Issues#

1. MIHD cross-slice retrieval almost completely failed (10 out of 14 query directions had SL@50=0)#

Solution: Diagnosis confirmed the root cause is that PCA/STAIG processes each slice independently, producing incomparable embedding spaces. The fix is to use scGPT embeddings directly (shared space, average SL@50=0.416, 100% hit rate).

Key insight: Embedding space comparability (whether a shared coordinate system exists) is more fundamental than embedding quality. Cross-slice tasks and single-slice clustering should use different encoders.

2. MIHD single-slice ARI gap vs. SOTA (current 0.546 vs. target 0.60+)#

Solution: A 4-phase ablation experiment plan was drafted (clustering ablation → batch effect analysis → retrieval probing → 151676 targeted diagnosis), but not yet fully executed.

Key insight: Zero-variance embeddings on slice 151676 cause mclust to fail and fall back to KMeans, which is the key factor dragging down the average. Root cause still unknown.

General Issues#

3. Missing ~/.config/gadget/sync.json on DCC, preventing sync.py from running#

Solution: AI read the existing summarize config to obtain the rclone_remote setting, manually created sync.json, verified the remote directory structure, then executed the pull.

Key insight: Config files are not automatically migrated to new machines and must be inferred from existing configs and rebuilt.

4. cp -r flattened the cchelper directory into ~/.claude/skills/ instead of preserving the subdirectory structure#

Solution: After discovering the issue, used mkdir -p ~/.claude/skills/cchelper to create the subdirectory, then moved the scattered files into it.

Key insight: When the target directory already exists, cp -r expands the source directory’s contents directly into it. The target path must be confirmed beforehand.

Today’s Takeaways#

Core Insights#

  • In cross-slice spatial transcriptomics tasks, the comparability of the embedding space (shared coordinate system) is more critical than local embedding quality. Foundation Models (e.g., scGPT) are naturally suited for cross-slice retrieval because they output a fixed representation space, whereas PCA/STAIG processes each slice independently, making coordinate systems incomparable.
  • The optimal method for single-slice clustering (STAIG) and the optimal method for cross-slice retrieval (scGPT) are different. Different encoders must be chosen for different task objectives rather than seeking a single unified optimal solution.

Practical Insights#

  • The DCC cluster’s scavenger-h200 partition provides H200×8 nodes (preemptive), which is an important resource for training tasks requiring large GPU memory. The account TRES limits of cpu=100/mem=500G are hard constraints when submitting large jobs.
