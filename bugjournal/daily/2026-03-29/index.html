Daily Report — 2026-03-29#

Today’s Highlights#

  • Completed Work: The TEDDY-G 400M base model has been successfully deployed in the DLPFC spatial transcriptomics dataset. The extraction process has been fixed and verified.
  • Implementation Method: Through systematic code review, four vulnerabilities were identified in the existing extract_teddy() function: incorrect word tokenizer, wrong model parameter names, incorrect output keys, and missing gene ID mapping. These vulnerabilities have been fixed and verified through batch runs in the teddy conda environment.
  • Purpose: To add the TEDDY encoder to the MIHD project, enabling it to be compared with existing models such as scGPT, Geneformer, and C2S. This will facilitate multi-model evaluations in disease state classification tasks.

The TEDDY single-cell base model has been analyzed on the HPC cluster. Four critical vulnerabilities in the extraction script have been fixed, and the embedding extraction work for the DLPFC dataset is now complete.

Today’s Tasks#

Architecture and Strategy#

  • ✅ Fixed vulnerabilities in the extract_teddy() script — Four critical vulnerabilities were identified and fixed: ① The incorrect word tokenizer was changed to GeneTokenizer; ② The model forward parameter input_ids was changed to gene_ids; ③ The output access key was changed to outputs['cell_emb'];; ④ A gene symbol-to-Ensembl ID mapping logic was added.
  • ✅ TEDDY paper analysis — Analyzed the arXiv paper number 2503.03485, learned about the TEDDY model architecture (two encodings: Teddy-G/X, six variants, with parameters up to 400M). Evaluated its performance in disease classification benchmarks and compared it with state-of-the-art models.

Implementation and Fixes#

  • 🔄 DLPFC embedding extraction — In the teddy conda environment, 1024-dimensional cell embeddings were extracted for 11 DLPFC regions. Region 151508 was completed (4384 points), and the remaining 10 regions are currently being processed in batches. The problem of insufficient GPU memory was solved by reducing the batch size from 32 to 8.

Problems and Solutions#

Key Issues#

1. The extract_teddy() function used the incorrect word tokenizer class and wrong model API (input ID, last hidden state), making the script unusable.#

Solution: By reviewing the TEDDY source code, it was confirmed that the correct APIs were used: GeneTokenizer, gene_ids parameters, and the output [‘cell_emb’] key. Additionally, a Ensembl ID mapping function was added.

Key Insight: TEDDY’s vocabulary uses Ensembl IDs instead of gene symbols. The model forward signature is different from the standard HuggingFace interface, so it needs to be reviewed in source code rather than inferred based on conventions.

General Issues#

2. When the batch size is set to 32 (32GB memory), there is a problem with insufficient GPU memory, as each sequence has up to 2048 tokens.#

Solution: Reduce the batch size from 32 to 8 to ensure manageable memory usage.

Key Insight: The ranking value encoding retains the first 2048 genes as token sequences, making the model longer than models like scGPT and requiring more memory.

3. Claude couldn’t recall previous TEDDY test conversations.#

Solution: Users provided a new arXiv link, and Claude re-analyzed the paper.

Key Insight: Cross-conversation memory wasn’t saved correctly, leading to context loss. It’s necessary to actively save memories at key research nodes.

Today’s Achievements#

Key Achievements#

  • TEDDY-G performed better than TEDDY-X (ranking value encoding outperformed binning). In multi-class tasks with retained donors, its performance was 8% higher than Nicheformer. However, in the retained disease binary classification task, its performance was comparable to Nicheformer. Linear tests showed that although the embedding quality was high, fine-tuning could lead to overfitting.
  • There are significant differences among single-cell base models in the HuggingFace ecosystem in terms of forward interfaces. It’s important not to assume the standard HuggingFace interface and instead verify the gene_ids parameter names and output dictionary structure through source code review.

Practical Achievements#

  • TEDDY’s vocabulary uses Ensembl IDs (43808 tokens). In the DLPFC dataset, 60% of genes (20044/33538) can be mapped to the vocabulary. The rest are non-protein-coding genes, such as lncRNA — which is a normal coverage rate.
