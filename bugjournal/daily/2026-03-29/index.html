Problems & Solutions#

Critical Issues#

1. extract_teddy() used a non-existent Tokenizer class and incorrect model API (input_ids, last_hidden_state), causing the script to fail entirely#

Solution: Read the Teddy source code to confirm the correct API: GeneTokenizer, gene_ids parameter, outputs['cell_emb'] output key, and added Ensembl ID mapping

Key insight: TEDDY vocab uses Ensembl IDs rather than gene symbols, and the model forward signature differs from the standard HuggingFace interface — reading the source directly is required rather than relying on convention

General Issues#

2. GPU OOM at batch_size=32 (32GB VRAM) because each sequence is up to 2048 tokens long#

Solution: Reduced batch_size from 32 to 8, bringing memory usage under control

Key insight: Rank-value encoding retains the top 2048 genes as the token sequence, which is longer than models like scGPT and requires more VRAM

3. Claude could not recall the conversation from the previous Teddy testing session#

Solution: User re-provided the arXiv link and Claude re-analyzed the paper

Key insight: Persistent memory across sessions was not properly saved, causing context loss — important research checkpoints require proactively saving memory

Today’s Takeaways#

Core Insights#

  • TEDDY-G outperforms TEDDY-X (rank-value encoding outperforms binned expression), and achieves 8% higher accuracy than Nicheformer on held-out donor multi-class tasks; however, results are mixed against Nicheformer on held-out disease binary classification; linear probe greatly outperforms fine-tuning (RA: 0.56→0.93), indicating high embedding quality but susceptibility to overfitting during fine-tuning
  • Single-cell foundation models in the HuggingFace ecosystem (TEDDY, scGPT, etc.) have significantly different forward interfaces — standard HuggingFace interfaces cannot be assumed; source code must be read to confirm the gene_ids parameter name and output dict structure

Practical Takeaways#

  • TEDDY vocab uses Ensembl IDs (43,808 tokens); 60% of genes in the DLPFC dataset (20,044/33,538) can be mapped to the vocab, with the remainder being lncRNAs and other non-protein-coding genes — this is a normal coverage rate
