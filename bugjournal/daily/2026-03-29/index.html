Problems & Solutions#

Critical Issues#

1. extract_teddy() used an incorrect tokenizer class and wrong model API (input_ids, last_hidden_state), causing the script to fail completely.#

Solution: Verified the correct API by reading the Teddy source code: GeneTokenizer, gene_ids parameter, outputs[‘cell_emb’] output key, and added Ensembl ID mapping.

Key Insight: TEDDY vocab uses Ensembl IDs instead of gene symbols, and the model forward signature differs from the HuggingFace standard interface. It’s necessary to read the source code directly rather than relying on conventions.

General Issues#

2. GPU OOM occurred when batch_size=32 (32GB memory), as each sequence contains 2048 tokens.#

Solution: Reduced the batch size from 32 to 8 to control memory usage.

Key Insight: Rank-value encoding retains the top 2048 genes as token sequences, which is longer than sequences in models like scGPT, resulting in higher memory requirements.

3. Claude couldn’t recall the previous Teddy test conversation.#

Solution: The user provided a new arXiv link, and Claude re-analyzed the paper.

Key Insight: Cross-session memory wasn’t properly saved, leading to context loss. It’s necessary to actively save memories at key research points.

Learnings#

Key Learnings#

  • TEDDY-G outperforms TEDDY-X (rank-value encoding is better than binned expressions). In the held-out donor multi-class task, it’s 8% better than Nicheformer. However, in the held-out disease binary task, it’s neck-and-neck with Nicheformer. Linear probing is much better than fine-tuning (RA: 0.56→0.93), indicating high-quality embeddings but potential overfitting from fine-tuning.
  • Forward interfaces of single-cell base models in the HuggingFace ecosystem differ significantly. Don’t assume standard HuggingFace interfaces; you must read the source code to confirm the gene_ids parameter names and output dict structure.

Practical Learnings#

  • TEDDY vocab uses Ensembl IDs (43808 tokens). In the DLPFC dataset, 60% of genes (20044/33538) can be mapped to the vocab. The rest are lncRNA and other non-protein-coding genes—this is a normal coverage rate.
