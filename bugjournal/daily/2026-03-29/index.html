Problems & Solutions#

Critical Issues#

1. extract_teddy() used an incorrect tokenizer class and wrong model API (input_ids, last_hidden_state), causing the script to fail entirely.#

Solution: Confirmed the correct API by reviewing the Teddy source code: GeneTokenizer, gene_ids parameter, outputs[‘cell_emb’] output key, and added Ensembl ID mapping.

Key Insight: TEDDY vocab uses Ensembl IDs instead of gene symbols, and the model forward signature differs from HuggingFace’s standard interface. It’s necessary to read the source code directly rather than relying on conventions.

General Issues#

2. GPU OOM occurred when batch_size=32 (32GB memory), as each sequence contains 2048 tokens.#

Solution: Reduced the batch size from 32 to 8 to control memory usage.

Key Insight: The rank-value encoding retains the top 2048 genes as token sequences, which is longer than sequences in models like scGPT, resulting in higher memory requirements.

3. Claude could not recall the previous Teddy test conversation.#

Solution: The user provided a new arXiv link, and Claude re-analyzed the paper.

Key Insight: Cross-session memory wasn’t properly saved, leading to context loss. Memory should be actively saved at key research points.

Learnings#

Key Learnings#

  • TEDDY-G outperforms TEDDY-X (rank-value encoding is better than binned expressions). It performs 8% better in multi-class tasks with held-out donors than Nicheformer, but competes with Nicheformer in binary classification tasks. Linear probing is much better than fine-tuning (RA: 0.56→0.93), indicating high embedding quality but potential overfitting in fine-tuning.
  • Forward interfaces of single-cell base models in the HuggingFace ecosystem vary significantly. Don’t assume standard HuggingFace interfaces; verify gene_id parameter names and output dictionary structures by reading the source code.

Practical Learnings#

  • TEDDY vocab uses Ensembl IDs (43808 tokens). In the DLPFC dataset, 60% of genes (20044/33538) can be mapped to the vocab. The rest are non-protein-coding genes like lncRNAs—this is a normal coverage rate.
