Issues and Solutions#

Key Issues#

1. ScGPT’s spatial clustering ARI score is very low (about 0.10), despite using pre-trained checkpoints; colleagues suspect flash memory attention incompatibility causing checkpoint loading issues#

Solution: The root cause is that TransformerModel.init never saves use_fast_transformer as an instance attribute, causing getattr(model, ‘use_fast_transformer’, True) to always return True, thereby preventing Wqkv→in_proj key remapping. This issue was fixed by adding self.use_fast_transformer = use_fast_transformer. Redeemed all 11 DLPFC embeddings, and after fixing the model, the average ARI increased from 0.118 to 0.170 (+44%), NMI increased from 0.208 to 0.277 (+33%).

Key Insight: When an instance attribute is missing in a PyTorch model class, even with strict=False, checkpoint loading may fail, leading to partial parameter initialization failures (17.7% of random weights), resulting in performance degradation rather than obvious errors. It is essential to verify whether the checkpoint key matches the model state dictionary and clearly record any mismatches.

2. Visium HD data lacks real labels, making traditional ARI/NMI metrics unusable for benchmark testing#

Solution: Used the Leiden community detection algorithm to automatically estimate the number of clusters from k-NN graphs, outputting only cluster visualizations with contour scores. Modified the compute_metrics() function to conditionally skip supervisory metrics calculations when labels are unavailable. The resolution parameter can control the granularity.

Key Insight: Unsupervised space transcriptomics data requires different evaluation strategies—focusing on visual consistency and internal consistency metrics (contour scores) rather than external validation. Tracking data sources is important for detecting pre-applied transformations.

3. In the multi-screen wallpaper application, the powerSave and powerSavePlus video pause modes do not work at all, despite having occlusion detection infrastructure#

Solution: Identified 4 critical function vulnerabilities: allOverlaysCompletelyCovered() and anyOverlayCompletelyCovered() hardcode returns false; reassignAllWindows() and windowScreenDidChange() have no effect. It was determined that the automatic mode is a feasible template. These four functions were implemented using the existing overlayWindows.values and.occlusionState.contains(.visible) modes.

Key Insight: Functions marked as pending in production code may completely destroy functionality even if they compile successfully—type checking can accept this, but runtime behavior does not. This is a dangerous technical debt that may remain undetected. The occlusion detection infrastructure works in the automatic mode, but needs to be connected.

4. During Pi0.5 server evaluation, WebSocket connection refused errors occurred, even though the TCP port is open and the server is listening on localhost:8000#

Solution: It was found that proxy environment variables (http_proxy=127.0.0.1:10087, https_proxy, HTTP_PROXY, HTTPS_PROXY, no_proxy, NO_PROXY) blocked WebSocket connections, even when connecting to localhost. Disabling all proxy variables immediately solved the problem. Key Insight: Even if the original TCP layer works fine, proxy environment variables can still interfere with localhost connections—they intercept them before socket creation. When debugging WebSocket issues on shared machines or shared clusters, it is essential to check and cancel the values of proxy variables.

5. HD adata_8um.h5ad has undergone log conversion (range [0, 6.05]), but preprocess_data() applies log1p by default, causing double conversion corruption#

Solution: Added the skip_log parameter to preprocess_data() and preprocess_for_encoding(). Checked the metadata to detect HD datasets and set skip_log=True to skip redundant normalization operations. Use the data.uns[’log1p’] flag or X.max() heuristic method for detection.

Key Insight: Preprocessed external datasets need to track conversion history to prevent pipeline corruption. HD spot diameter (7.3px) is 20 times smaller than Visium, so it is necessary to dynamically adjust the visualization parameter ratio (size=4.0 vs 1.0) to avoid invisible points.

6. Attempting to load 50.7 GiB of Pi0.5 model on GPU cluster resulted in insufficient CUDA memory—multiple zombie processes on GPU already consuming significant resources#

Solution: Ran nvidia-smi and ps to identify zombie server processes that are already loading and listening on port 8000. Reused existing server instances instead of restarting new ones, avoiding OOM issues and saving 3-5 minutes of startup time.

Key Insight: When loading models larger than 50GB on shared GPUs, it is essential to check nvidia-smi and existing processes before attempting a new startup. Reusing zombie servers is more efficient than killing and restarting them, and ensures reasonable resource allocation and proper handling of multi-user clusters.

7. Initial confusion regarding Pi0.5 training status—Slurm job 46553 crashed at step 5000, unclear whether 100k steps were completed#

Solution: Directly checked the checkpoint directory and found that checkpoints had timestamps up to February 24th, confirming successful training, although the job crashed early in the process (possibly due to failing to recover from a checkpoint). Then it was found that no evaluation was performed.

Key Insight: Slurm job logs do not reflect everything—they only show execution history, not the complete outcome. It is essential to verify the completion by checking actual checkpoint files and their timestamps, rather than just the log tail. Training completion does not equal evaluation completion.

8. Robosuite environment lacks the get_bounding_box_half_size method, and env.seed() API is not compatible with evaluating Pi0.5 Phoenix#

Solution: Configured PYTHONPATH to use the deps/ directory’s zhaoganlong robosuite/mimicgen, instead of the local mimicgen_workspace version. For effective evaluation, it is necessary to ensure that the library versions match those of the training environment.

Key Insight: VLA policy evaluation requires the library versions to be exactly the same as those in the training environment—even minor differences in robosuite/mimicgen versions can lead to API incompatibilities (missing methods, changed signatures). Version mismatches can result in invalid benchmark test results.

9. GPU acceleration exploration for visualization widgets—cuML/FAISS not installed, mclust not available (related to R_HOME issues)#

Solution: Analysis showed that the clustering process (100-300 milliseconds) is not the bottleneck; data loading and matplotlib rendering take up most of the time. It is recommended to use multi-process parallelization instead of optimizing only GPU performance.

Key Insight: Optimizing too early—analysis showed that I/O and rendering are the limiting factors, not computation. For workloads that rely on I/O, parallel CPU processing is more efficient than GPU optimization. Analyze carefully before making optimizations.

Lessons Learned#

Key Lessons#

  • When using PyTorch to load checkpoints, if strict=False is set, the system will silently skip keys that do not match. If an instance property is missing, this can lead to random initialization of more than 17% of parameters, without any error messages. In this case, due to the absence of the self.use_fast_transformer property, the mapping of the Wqkv→in_proj key failed, causing issues with the flash attention weight mapping. Therefore, it is essential to verify that keys in the state dictionary match, record any mismatches, and verify that the model behaves normally after loading.
  • The Pi0.5 Phoenix LoRA was fine-tuned on 9 MimicGen tasks (4500 samples per task), but its success rate was only 0-24% when evaluated in the same training environment. This significant gap between training and evaluation suggests issues such as overfitting of training data, insufficient training iterations, fundamental flaws in the architecture, or quality issues with the training data, which require further investigation.
  • Completion of training does not mean completion of evaluation. A model trained over 1 million steps may still be shelved during the evaluation process. In high-performance computing systems, job logs reflect the execution history, not the degree of completion. It is essential to verify the completion status by checking timestamped artifacts (checkpoints, output files), rather than just looking at the log tail. When discussing model performance, it is crucial to confirm the existence of the evaluation artifacts.
  • Proxy environment variables (http_proxy, https_proxy, HTTP_PROXY, HTTPS_PROXY) can intercept and disrupt WebSocket connections to the local host, even if the original TCP layer works properly. They act at the application layer before socket creation. When debugging local services on development machines or shared clusters, it is essential to disable these proxy variables.
  • The RM-IDEAL Spearman correlation benchmark shows that the STAIG fusion strategy (r=0.396) is significantly better than the simple fusion strategy (concatenation/average of approximately 0.25), as this strategy can explicitly model spatial dependencies in gene expression embeddings. The learned fusion methods always perform better than simple concatenation or averaging methods.
  • The Leiden community provides powerful automatic clustering capabilities for unlabeled spatial transcriptomics data, achieved through k-NN graph construction and controlled by resolution parameters. Combining contour scores allows unsupervised evaluation without real labels, which is common in Visium HD and other high-resolution spatial datasets.
  • Stub functions in production code with TODO annotations can completely destroy functionality without causing compilation or type checking errors. In the macOS wallpaper app, four stub functions prevented the powerSave/powerSavePlus functions from working for months. These stub functions must be implemented or removed before release, rather than being treated as technical debt.
  • When loading models larger than 50GB on shared GPUs, it is essential to check nvidia-smi and existing processes before attempting a new launch. Reusing zombie servers can prevent CUDA memory exhaustion, saving 3-5 minutes of startup time and ensuring resources are allocated reasonably, while respecting multi-user cluster etiquette. This is particularly important in shared high-performance computing infrastructure.
  • Preprocessed external datasets (such as those provided by collaborators for Visium HD) require clear tracking of conversion history to prevent pipeline damage. The data can be detected using the data.adata.uns[’log1p’] flag or X.max() heuristic methods, and duplicate standardization operations can be skipped. In multi-stage analysis processes, data sources are crucial.
  • For developing experimental research codebases, planning framework compatibility in advance is very important. Asking “What might this module be used for in the future?” can prevent architectural lock-ins. In the BC-RNN case, the standard robomimic checkpoint format (model.pth + config.json) can automatically enable error injection through the existing PolicyAdapter, without additional work.
  • VLA strategy evaluation requires library versions consistent with the training environment. Even minor differences in robosuite/mimicgen versions can lead to API incompatibilities (such as missing get_bounding_box_half_size method, or changes in signature, such as env.seed()). Version mismatches can result in invalid benchmark test results or runtime crashes.
  • The performance of spatial transcriptomics visualization processes is often affected by I/O (such as scanpy AnnData loading) and rendering (such as matplotlib), rather than clustering calculations. For this type of I/O-dependent workload, multi-process parallelization across independent samples is more effective than GPU acceleration. Performance analysis is necessary before optimization.
  • High-resolution spatial transcriptomics (Visium HD 8µm versus Visium 55µm bin spacing) requires dynamic adjustment of visualization parameter sizes. Spot size must match the bin spacing to avoid invisible or overlapping points. Visium’s 7.3px spot diameter requires size=4.0, while Visium’s default size is 1.0, so that the spatial plot area is correctly covered.
  • When discussing multivariate models in research, it is important to first specify the source of data (which data, how many samples, which tasks), rather than checking checkpoint paths. Training data is the conceptual foundation; checkpoint paths are technical details. This helps determine what to evaluate and compare.

Practical Experience#

  • The Xcode PBXFileSystemSynchronizedRootGroup (object version 77) can automatically discover new source files, eliminating merge conflicts in modern Xcode projects. It is important to set ENABLE_USER_SCRIPT_SANDBOXING to NO, allowing build scripts to modify build products like Info.plist.
  • In SwiftUI, double-click operations require using.onTapGesture(count: 2) closures, rather than just supporting single-click Button operations. This is useful for user interface modes where deliberate activation is necessary, as accidental single clicks can cause problems. This feature can be combined with a cooling timer to prevent rapid consecutive clicks.
  • Complex JSON files (such as Xcode .xcstrings localization files) should be edited using the language JSON library (Python’s json module), rather than manually replacing strings to maintain nested structures. Manual editing of strings can damage nested structures.
  • The tyro CLI framework uses positional subcommands (rather than –flag:subcommand syntax) and automatically adds dataclass prefixes to nested arguments (–args.field). Read the help output carefully to understand the argument structure.
  • On shared HPC file systems with millions of files, avoid recursive searches without strict depth limits. Such searches can time out.
