问题与解决方案#

关键问题#

1. scGPT-spatial的TransformerModel._init__未将use_fast_transformer保存为实例属性，导致load_pretrained()中getattr默认返回True，Wqkv→in_proj key重映射永不执行，12层attention的Q/K/V权重（17.7%参数）随机初始化，strict=False静默跳过无任何报错#

解决方案: 添加一行self.use_fast_transformer = use_fast_transformer，修复后所有186个模型参数均正确匹配，全部11个section的ARI和NMI均显著提升，平均ARI从0.1177提升至0.1700（+44.4%）

关键洞察: strict=False的checkpoint加载会静默丢弃不匹配的key；每个用于控制模型结构的构造参数都必须显式保存为实例属性，不能依赖getattr默认值——这是模型类设计的重要防御性编程原则

2. 代理环境变量（http_proxy=http://127.0.0.1:10087）拦截localhost WebSocket连接，导致evaluate_mimicgen.py客户端持续报ConnectionRefusedError，多次重试无效#

解决方案: 在客户端进程启动前执行unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY，通过对比加/不加代理环境变量的结果定位根因

关键洞察: 即使是localhost的WebSocket连接也会被http_proxy拦截；调试分布式ML推理连接问题时应第一时间检查并清除代理环境变量，再用原生TCP socket验证端口可达性

3. MimicGen与robosuite版本兼容性连环问题：①rollout时ModuleNotFoundError: mujoco_py；②环境注册失败Coffee_D1 not found in robosuite；③Coffee环境初始化AttributeError: get_bounding_box_half_size未实现（在mimicgen的coffee_machine.py中被调用7次，但robosuite三个相关类从未实现该方法）#

解决方案: ①将robomimic安装目录env_robosuite.py中顶层import mujoco_py改为try/except可选导入；②在env_robosuite.py顶部添加import mimicgen触发环境注册装饰器；③在MujocoXMLObject/CompositeBodyObject/BoxObject三个类分别实现get_bounding_box_half_size()方法

关键洞察: MimicGen环境通过Python import触发注册（非自动），且依赖robosuite中未实现的方法；评估必须使用与训练完全相同的代码分支（zhaoganlong fork），而非项目主线版本

4. HD数据集成存在两个预处理问题：①adata_8um.h5ad的X已log变换（range [0, 6.05]），但现有preprocess_data()会重复执行normalize_total+log1p；②spot_diameter_fullres=7.3px（远小于DLPFC的~144px），sc.pl.spatial默认size=1.0导致spot几乎不可见#

解决方案: ①为preprocess_data()和preprocess_for_encoding()新增skip_log参数，HD数据加载时设skip_log=True只执行HVG过滤；②将size改为4.0（bin间距29.2px / spot直径7.3px ≈ 4x）

关键洞察: 不同分辨率数据集的spot物理参数差异巨大，可视化参数需根据bin间距/spot直径比例动态调整；预处理函数需保证幂等性，避免log变换叠加导致数据失真

5. GPU资源管理连环问题：①遗留服务器进程（PID 2413737）占用约61GB显存导致后续启动OOM；②BC-RNN并行启动时因checkpoint目录已存在，robomimic在非TTY环境调用input()触发EOFError崩溃并留下僵尸进程#

解决方案: ①复用已运行的僵尸服务器（已监听8000端口完成模型加载），直接连接；②启动训练前清理所有已存在的checkpoint目录，通过nvidia-smi查询残留进程PID后kill清理

关键洞察: robomimic在checkpoint目录存在时调用input()等待交互，在脚本/非TTY环境中立即抛出EOFError；在GPU资源紧张环境中应先检查端口和进程状态再启动新服务

6. 两处信息误读：①AI基于job崩溃log（step 5000）误判Pi0.5训练失败；②tyro CLI语法错误——将union subcommand位置参数误写为–选项格式#

解决方案: ①直接检查checkpoint目录确认step 4000~99999完整存在（训练已于2月24日完成）；②通过–help确认tyro CLI接口格式，位置参数无需–前缀

关键洞察: 训练job崩溃不代表训练失败，应以checkpoint文件为准而非log文件结尾；tyro的union subcommand是位置参数，构建复杂CLI命令前必须先–help验证

一般问题#

7. SLURM作业提交与管理连环问题：①User’s group not permitted to use this partition；②–mem is not allowed违反QOS策略；③srun附加到已运行的交互式作业时输出为空命令未执行#

解决方案: ①添加–account=sysu_gbli2；②移除–mem=32G改用–gres=gpu:1（参考已成功运行的作业配置）；③添加–overlap标志：srun –jobid=xxx –overlap bash -c …

关键洞察: HPC集群需显式指定账号才能提交；不同集群对内存申请限制不同，参考已成功运行的作业是最可靠的配置来源；srun附加到有步骤运行的作业需要–overlap标志

8. run_benchmark.py存在独立的cluster_embeddings()函数副本，与utils/clustering.py不同步，HD运行时传入n_clusters=-1导致KMeans InvalidParameterError#

解决方案: 在run_benchmark.py本地的cluster_embeddings()中也添加n_clusters<=0的自动估计逻辑（调用estimate_n_clusters_leiden()），保持两处实现一致

关键洞察: 项目中存在功能重复的本地定义与utils模块，修改utils时必须同步检查scripts/中是否有独立副本

今日收获#

核心收获#

  • PyTorch模型checkpoint加载陷阱：strict=False会静默丢弃不匹配的key，每个用于控制模型结构的构造参数都必须显式保存为实例属性，否则getattr的默认值会导致隐蔽的随机初始化。scGPT官方GitHub上存在相同bug，发现后应考虑向上游提交修复。
  • MimicGen与robosuite版本兼容要点：①MimicGen环境变体通过import触发注册（非自动），需在env_robosuite.py顶部import mimicgen；②get_bounding_box_half_size方法在robosuite中从未实现但被mimicgen调用多次，需在MujocoXMLObject/CompositeBodyObject/BoxObject三个类手动实现；③评估必须使用与训练完全相同的环境代码分支（zhaoganlong fork）
  • MIHD RM-IDEAL基准结论（section 151673）：STAIG fusion最优（平均Spearman r=0.396，Layer_3峰值0.644）；QFormer/LLaVA-MLP/AdaLN-Attention等学习型融合（0.15-0.25）均优于concat/mean等简单融合；单模态baseline（pca/mlp/uni2）均在0.06-0.12；Layer_1全方法表现差（r<0.13），需特别关注
  • http_proxy代理环境变量会拦截localhost WebSocket连接；调试分布式ML推理连接问题时应第一时间unset所有代理变量，再用原生TCP socket验证端口可达性，而非直接重试连接
  • Visium HD数据处理要点：spot_diameter_fullres=7.3px（vs DLPFC ~144px），sc.pl.spatial的size参数应设为bin间距/spot直径≈4.0；adata_8um.h5ad的X已log变换，预处理需skip_log=True；Leiden在crop10large（17,502 spots）上估计出17-20 clusters；obsm[‘spatial’]坐标在裁剪后坐标空间，uns[‘spatial’] hires图像是全图需替换为cropped_fullres.tif
  • 项目中三类Pi0.5模型区别：①预训练base（zero-shot，M14，错误恢复评估成功率0%）；②9-task LoRA Phoenix（M16，训练完成至step 99999，4500 episodes，5个task prompt映射9个环境，4×A800，从未评估）；③zhaoganlong单任务微调版本（已有部分评估结果）
  • BC-RNN与错误注入pipeline的兼容性：现有RobomimicPolicyAdapter/BCPolicy/load_policy()原生支持robomimic checkpoint格式，BC-RNN训练产出标准checkpoint无需修改即可接入错误注入pipeline，只需保持标准输出结构

实践收获#

  • HPC环境调试经验汇总：①训练job崩溃不代表训练失败，应以checkpoint文件为准；②tyro CLI的union subcommand是位置参数而非–选项，构建命令前必须–help验证；③robomimic在非TTY环境checkpoint目录已存在时调用input()立即触发EOFError，启动前必须清理旧目录；④SLURM集群：账号sysu_gbli2，不允许–mem，srun附加需–overlap；⑤BC-RNN每任务仅用~2.2GB显存，9个任务可在单GPU并行运行
