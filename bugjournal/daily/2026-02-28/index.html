Problems and Solutions#

Key Problems#

1. The scGPT spatial clustering ARI score is very low (about 0.10), despite using pre-trained checkpoints; colleagues suspect flash memory attention incompatibility causing checkpoint loading issues.#

Solution: The root cause is that TransformerModel.init never saves use_fast_transformer as an instance attribute, causing getattr(model, ‘use_fast_transformer’, True) to always return True, thereby preventing Wqkv→in_proj key remapping. This issue was fixed by adding self.use_fast_transformer = use_fast_transformer. Redeemed all 11 DLPFC embeddings, and after fixing the model, average ARI increased from 0.118 to 0.170 (+44%), NMI increased from 0.208 to 0.277 (+33%).

Key Insight: When an instance attribute is missing in a PyTorch model class, even with strict=False, checkpoint loading may fail, leading to partial parameter initialization failures (17.7% of random weights), resulting in performance degradation rather than obvious errors. It is essential to verify that the checkpoint key matches the model state dictionary and clearly record any mismatches.

2. Visium HD data lacks real labels, making traditional ARI/NMI metrics unusable for benchmark testing.#

Solution: Used the Leiden community detection algorithm to automatically estimate the number of clusters from the k-NN graph, outputting only cluster visualizations with contour scores. Modified the compute_metrics() function to conditionally skip supervised metrics when labels are unavailable. The resolution parameter can control the granularity.

Key Insight: Unsupervised spatial transcriptome data requires different evaluation strategies—focusing on visual consistency and internal consistency metrics (contour scores) rather than external validation. Tracking data sources is important for detecting pre-applied transformations.

3. In the multi-screen wallpaper application, the powerSave and powerSavePlus video pause modes do not work properly, despite having occlusion detection infrastructure.#

Solution: Identified 4 critical function vulnerabilities: allOverlaysCompletelyCovered() and anyOverlayCompletelyCovered() are hardcoded to return false; reassignAllWindows() and windowScreenDidChange() have no effect. It was determined that the automatic mode is a feasible template. These four functions were implemented using the existing overlayWindows.values and.occlusionState.contains(.visible) modes.

Key Insight: Functions marked as pending in production code can completely ruin functionality even if they compile successfully—type checking can accept this, but runtime behavior does not. This is a dangerous technical debt that may go unnoticed. The occlusion detection infrastructure works in the automatic mode, but needs to be integrated.

4. During the Pi0.5 server evaluation, WebSocket connection rejection errors occurred, even though the TCP port was open and the server was listening on localhost:8000.#

Solution: It was found that proxy environment variables (http_proxy=127.0.0.1:10087, https_proxy, HTTP_PROXY, HTTPS_PROXY, no_proxy, NO_PROXY) blocked WebSocket connections, even for localhost connections. The problem was solved by canceling all proxy variables immediately. Key Insight: Even if the original TCP layer works fine, proxy environment variables can still interfere with localhost connections—they intercept them before socket creation. When debugging WebSocket issues on a development machine or shared cluster, it is essential to check and cancel the values of proxy variables.

5. HD adata_8um.h5ad has undergone log conversion (range [0, 6.05]), but preprocess_data() applies log1p by default, causing double conversion corruption.#

Solution: Added the skip_log parameter to preprocess_data() and preprocess_for_encoding(). Checked the metadata to detect HD datasets and set skip_log=True to skip redundant normalization operations. Used theadata.uns[’log1p’] flag or X.max() heuristic method for detection.

Key Insight: Preprocessed external datasets need to have their conversion history clearly tracked to prevent pipeline corruption. HD spot diameter (7.3px) is 20 times smaller than Visium, so it is necessary to dynamically adjust the visualization parameter ratio (size=4.0 vs 1.0) to avoid invisible points.

6. Attempting to load a 50.7 GiB Pi0.5 model on the GPU cluster resulted in insufficient CUDA memory—there were already processes consuming a lot of resources on the GPU.#

Solution: Ran nvidia-smi and ps to identify zombie server processes that were already loading and listening on port 8000 (PID 2413737). Reused existing server instances instead of restarting new ones, avoiding OOM issues and saving 3-5 minutes of startup time.

Key Insight: When loading models larger than 50GB on shared GPUs, it is essential to check nvidia-smi and existing processes before attempting a new startup. Reusing zombie servers is more efficient than killing and restarting them, and ensures proper resource allocation and respect for multi-user clusters.

7. Initial confusion regarding Pi0.5 training status—Slurm job 46553 crashed at step 5000, making it unclear whether 100k steps were completed.#

Solution: Directly checked the checkpoint directory and found that checkpoints had timestamps up to February 24th, confirming successful training, although the job crashed early in the process (possibly recovering from checkpoints). Then it was found that no evaluation was ever conducted.

Key Insight: Slurm job logs do not reflect the entire situation—they only show the execution history, not the complete results. It is essential to verify the completion by checking actual checkpoint files and their timestamps, rather than just the log tail. Training completion does not equal evaluation completion.

8. The Robosuite environment lacks the get_bounding_box_half_size method, and the env.seed() API is not compatible with evaluating Pi0.5 Phoenix.#

Solution: Configured PYTHONPATH to use the deps/ directory’s zhaoganlong robosuite/mimicgen, instead of the local mimicgen_workspace version. For effective evaluation, it is necessary to ensure that the library versions match those of the training environment.

Key Insight: VLA policy evaluation requires the library versions to be exactly the same as those of the training environment—even minor differences in robosuite/mimicgen versions can lead to API incompatibility (missing methods, changed signatures). Version mismatches can result in invalid benchmark test results.

9. GPU acceleration exploration for visual components—cuML/FAISS not installed, mclust not available (related to R_HOME issues).#

Solution: Analysis showed that the clustering process (100-300 milliseconds) was not the bottleneck; data loading and matplotlib rendering took up most of the time. It is recommended to use multi-process parallelization instead of optimizing only GPU performance.

Key Insight: Optimizing for GPU performance too early—analysis showed that I/O and rendering were the limiting factors, not computation. For workloads that rely on I/O, parallel CPU processing is more efficient than GPU optimization. Analyze thoroughly before making such optimizations.

Xcode’s User Script Sandbox Feature#

The user script sandbox feature introduced in the latest version prevents build scripts from accessing files by default. This is separate from the application’s runtime sandbox permissions. For build automation that modifies files, this feature must be disabled.

Lessons Learned#

Key Lessons#

  • Without human colleague guidance, the AI failed to independently identify errors in the scGPT upstream model class. Clear guidance was needed to investigate the checkpoint loading mechanism, not just to verify poor results.
  • It was unable to distinguish between Slurm job failures and successful training—assuming the training stopped at step 5000, even though checkpoints continued until step 99999 and timestamps were late. No active checks were performed to assess the absence of errors.
  • Before attempting to launch a new 50GB model server, it did not check the existing GPU processes using nvidia-smi, leading to CUDA OOM errors. This could have been avoided by reusing zombie processes.
  • Although the original TCP communication was normal, it was not anticipated that proxy environment variables would interfere with WebSocket local host connections. Systematic debugging was needed to identify this application-level interception issue.
  • Despite user interruptions, ExitPlanMode was repeatedly tried for HD implementation planning, indicating insufficient adaptability to allow discussion and optimization during the planning phase.

General Limitations#

  • The initial HD implementation plan aimed for the complete 545K point dataset, without considering phased rollouts. Users needed to modify this to start with the crop10large subset for faster validation.
  • The initially designed History function supported one-click activation. It was not anticipated that double-clicking with a cooldown period was necessary to prevent accidental wallpaper changes.
  • Initially, it was attempted to manually edit complex JSON localization files (Localizable.xcstrings) by string replacement, damaging the nested structure. Python should have been used for JSON operations from the start.
  • Inefficient recursive command searching was used, resulting in multiple timeouts on large cluster file systems. Targeted ls/grep operations should have been performed using workspace structure knowledge instead.
  • When asked for the location of visualization results, only image content was displayed instead of just file paths, causing unnecessary delays.
  • The ChangeLog version rules were misinterpreted, creating hotfix versions for day-to-day changes instead of grouping changes by date into major versions.
  • Due to network limitations, it was not possible to access the Phoenix CVPR 2025 paper from arxiv.org or openaccess.thecvf.com. Instead, network searches for abstracts were relied upon, rather than the original sources.

Practical Lessons#

Lesson Practical Application
Use of PyTorch for checkpoint loading When using PyTorch for checkpoint loading, ensure that strict=False is set to prevent unexpected key mismatches. If an instance property is missing, it may lead to random initialization of parameters, especially if self.use_fast_transformer is missing. Ensure that keys in the state dictionary match, and record any mismatches. After loading the model, verify that its behavior is normal.
Fine-tuning Pi0.5 Phoenix LoRA Pi0.5 Phoenix LoRA was fine-tuned on 9 MimicGen tasks with 4500 samples per task. However, the success rate was only 0-24% when evaluated in the same training environment. This indicates that there may be issues with overfitting training data, insufficient training iterations, fundamental architectural flaws, or quality issues with the training data. Further investigation is needed.
Importance of evaluating completion status Training completion does not equal evaluation completion. A model trained with 1 million steps may be shelved during the evaluation process. In high-performance computing systems, job logs reflect the execution history, not the completion status. It is essential to verify the completion status by checking timestamped artifacts such as checkpoints and output files, rather than just looking at the log tail. When discussing model performance, it is crucial to confirm the existence of the evaluation artifacts.
Proxy environment variables interfering with WebSocket connections Proxy environment variables can interfere with WebSocket local host connections. These variables act at the application layer before socket creation. When debugging local services on development machines or shared clusters, it is essential to disable these proxy variables.
RM-IDEAL Spearman Correlation Benchmarking The STAIG fusion strategy (r=0.396) is significantly better than the simple fusion strategy (average/series of about 0.25). This is because the strategy can explicitly model spatial dependencies in gene expression embeddings. The learned fusion method always performs better than simple series or average methods.
Automatic clustering capabilities of the Leiden community The Leiden community provides powerful automatic clustering capabilities for unlabeled spatial transcriptomics data, achieved through k-NN graph construction and control of granularity through resolution parameters. Combining contour scores allows for unsupervised evaluation without real labels, which is common in Visium HD and other high-resolution spatial datasets.
Stub functions in production code Stub functions in production code may completely destroy functionality without causing compilation errors or type checking failures. In the macOS wallpaper application, four stub functions caused the powerSave/powerSavePlus function to be unusable for months. These stub functions must be implemented or removed before release, rather than being retained as technical debt.
Checking GPU processes before launching models on shared GPUs When loading models larger than 50GB on shared GPUs, it is essential to check the nvidia-smi and existing processes before attempting a new launch. Reusing zombie servers can prevent CUDA memory exhaustion, saving 3-5 minutes of startup time and ensuring resources are allocated reasonably, while adhering to multi-user cluster etiquette. This is important in shared high-performance computing infrastructure.
Tracking conversion history for preprocessed external datasets Preprocessed external datasets, such as those provided by collaborators for Visium HD, need to have their conversion history tracked to prevent pipeline damage. The data can be detected using the datata.uns[’log1p’] flag or the X.max() heuristic method, and duplicate standardization operations can be skipped. Data source clarity is crucial in multi-stage analysis processes.
Planning framework compatibility for evolving experimental codebases Planning the framework compatibility for evolving experimental codebases is important. Asking “What might this module be used for in the future?” can prevent architectural lock-ins. In the BC-RNN case, the standard robomimic checkpoint format (model.pth + config.json) can automatically enable error injection through the existing PolicyAdapter, without additional work.
VLA strategy evaluation requires library versions consistent with the training environment Even minor differences in the robosuite/mimicgen version can lead to API incompatibilities (such as missing get_bounding_box_half_size method or changes in the signature of env.seed()). Version mismatches can result in invalid benchmark test results or runtime crashes.
Performance of space transcriptomics visualization processes The performance of space transcriptomics visualization processes is often affected by I/O (such as scanpy AnnData loading) and rendering (such as matplotlib), rather than clustering calculations. For such I/O-dependent workloads, multi-process parallelization across independent samples is more efficient than GPU acceleration. Performance analysis is necessary before optimization.
Adjusting visualization parameter sizes for high-resolution space transcriptomics For high-resolution space transcriptomics (Visium HD 8µm vs. Visium 55µm bin spacing), it is necessary to dynamically adjust the size of visualization parameters. The spot size must match the bin spacing to avoid invisible or overlapping points. The Visium 7.3px spot diameter requires size=4.0, while the default size is 1.0, so that the spatial drawing area is correctly covered.
Discussing multivariate models When discussing multivariate models in research, it is important to first specify the source of data (which data, how many samples, which tasks) rather than the checkpoint path. Training data is the conceptual foundation; checkpoint paths are technical details. This helps determine what to evaluate and compare.

Practical Tips#

  • The Xcode PBXFileSystemSynchronizedRootGroup (object version 77) can automatically discover new source files, eliminating merge conflicts in modern Xcode projects. It is important to set ENABLE_USER_SCRIPT_SANDBOXING to NO to allow build scripts to modify build products like Info.plist.
  • In SwiftUI, double-click operations require using.onTapGesture(count: 2) closures, rather than just supporting single-click Button operations. This is useful for user interface modes where deliberate activation is needed to avoid accidental clicks. This feature can be combined with a cooling timer to prevent rapid consecutive clicks.
  • Complex JSON files (such as Xcode .xcstrings localization files) should be edited using the language JSON library (Python’s json module), rather than string replacement to maintain nested structures. Manual string editing can damage nested structures.
  • The tyro CLI framework uses positional subcommands (rather than –flag:subcommand syntax) and automatically adds dataclass name prefixes to nested parameters (–args.field). Read the help output carefully to understand the parameter structure.
  • On shared HPC file systems with millions of files, avoid recursive searches without strict depth limits.
