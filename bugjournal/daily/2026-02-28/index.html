Problems & Solutions#

Critical Issues#

1. scGPT spatial clustering ARI scores very low (~0.10) despite using pretrained checkpoint; colleague suspected flash attention incompatibility causing checkpoint loading issues#

Solution: Root cause: TransformerModel.init never saved use_fast_transformer as instance attribute, causing getattr(model, ‘use_fast_transformer’, True) to always return True and prevent Wqkv→in_proj key remapping. Fixed by adding self.use_fast_transformer = use_fast_transformer. Re-extracted all 11 DLPFC embeddings with corrected model, improving average ARI from 0.118 to 0.170 (+44%), NMI from 0.208 to 0.277 (+33%)

Key Insight: Missing instance attributes in PyTorch model classes can silently break checkpoint loading when strict=False, causing partial parameter initialization (17.7% random weights) that manifests as poor performance rather than obvious errors. Always verify checkpoint keys match model state_dict and log mismatches explicitly

2. Visium HD data lacks ground truth labels, making traditional ARI/NMI metrics impossible to compute for benchmark evaluation#

Solution: Use Leiden community detection to auto-estimate n_clusters from k-NN graph, output only clustering visualization with silhouette score. Modified compute_metrics() to conditionally skip supervised metrics when labels unavailable. Resolution parameter controls granularity

Key Insight: Unsupervised spatial transcriptomics data requires fundamentally different evaluation strategy—focus on visual coherence and internal consistency metrics (silhouette score) rather than external validation. Data provenance tracking critical to detect pre-applied transformations

3. powerSave and powerSavePlus video pause modes completely non-functional in multi-screen wallpaper app despite occlusion detection infrastructure existing#

Solution: Traced to 4 critical stub functions: allOverlaysCompletelyCovered() and anyOverlayCompletelyCovered() hardcoded to return false; reassignAllWindows() and windowScreenDidChange() do nothing. Identified .automatic mode as working template. Implemented all four using existing overlayWindows.values and .occlusionState.contains(.visible) pattern

Key Insight: Stub functions with TODO comments in production code can completely break features while appearing to compile successfully—type checkers satisfied but runtime behavior absent. This is dangerous technical debt that can persist unnoticed. The occlusion detection infrastructure was already working for .automatic mode, just needed wiring

4. WebSocket connection refused error for Pi0.5 server evaluation despite TCP port being open and server listening on localhost:8000#

Solution: Discovered proxy environment variables (http_proxy=127.0.0.1:10087, https_proxy, HTTP_PROXY, HTTPS_PROXY, no_proxy, NO_PROXY) were intercepting WebSocket connections even to localhost. Unsetting all proxy vars fixed the issue immediately

Key Insight: Proxy environment variables can silently interfere with localhost connections even when raw TCP layer works—they intercept at application layer before socket creation. Always check and unset proxy vars when debugging WebSocket issues on development machines or shared clusters

5. HD adata_8um.h5ad already log-transformed (range [0, 6.05]), but preprocess_data() applies log1p by default, causing double-transformation corruption#

Solution: Add skip_log parameter to preprocess_data() and preprocess_for_encoding(). Detect HD dataset via metadata inspection and set skip_log=True to bypass redundant normalization. Use adata.uns[’log1p’] flag or X.max() heuristics for detection

Key Insight: Pre-processed external datasets require explicit transformation history tracking to prevent pipeline corruption. HD spot diameter (7.3px) is 20× smaller than Visium (~55µm), requiring dynamic visualization parameter scaling (size=4.0 vs 1.0) to prevent invisible dots

6. CUDA out of memory when attempting to load 50.7 GiB Pi0.5 model on GPU cluster—GPUs already full with existing processes#

Solution: Ran nvidia-smi and ps to identify zombie server process (PID 2413737) from previous launch already loaded and listening on port 8000. Reused existing server instance instead of launching new one, preventing OOM and saving 3-5min startup time

Key Insight: When loading 50+ GB models on shared GPUs, always check nvidia-smi and existing processes first before attempting new launches. Reusing zombie servers is more efficient than killing and restarting. Enforces resource awareness and multi-user cluster etiquette

7. Initial confusion about Pi0.5 training status—Slurm job 46553 crashed at step 5000, unclear if training ever completed to 100k steps#

Solution: Checked checkpoint directory directly, found checkpoints up to 99999 timestamped Feb 24, confirming training succeeded despite early job crash (likely resumed from checkpoint). Then discovered no evaluation was ever performed

Key Insight: Slurm job logs don’t tell the full story—they reflect execution history, not outcome completeness. Always verify completion by checking actual checkpoint artifacts with timestamps, not just log tails. Training completion ≠ evaluation done

8. Robosuite environment missing get_bounding_box_half_size method and env.seed() API incompatibility when evaluating Pi0.5 Phoenix#

Solution: Configured PYTHONPATH to use zhaoganlong’s robosuite/mimicgen from deps/ directory instead of local mimicgen_workspace versions. Exact environment matching with training setup required for valid evaluation

Key Insight: VLA policy evaluation requires exact library version matching with training environment—even minor robosuite/mimicgen version differences cause API incompatibilities (missing methods, changed signatures). Version mismatches can invalidate benchmarking results

9. GPU acceleration exploration for visualization pipeline—cuML/FAISS not installed, mclust unavailable (R_HOME issue)#

Solution: Profiling revealed clustering (100-300ms) is not the bottleneck; scanpy data loading and matplotlib rendering dominate. Recommended multi-process parallelization across 11 independent sections instead of GPU optimization

Key Insight: Premature GPU optimization attempt—profiling revealed I/O and rendering, not computation, as limiting factors. Parallel CPU processing more effective than GPU for this I/O-bound workload. Always profile before optimizing

General Issues#

10. Xcode build script to auto-update CFBundleVersion failed with ‘Operation not permitted’ when trying to modify built product’s Info.plist#

Solution: ENABLE_USER_SCRIPT_SANDBOXING = YES in project settings blocked build scripts from writing files. Set to NO to allow the PlistBuddy command to modify the built app’s Info.plist

Key Insight: Xcode’s user script sandboxing (introduced in recent versions) blocks build script file access by default, separate from app’s runtime sandbox entitlements. Must be disabled for build automation that modifies artifacts

Learnings#

Key Learnings#

  • PyTorch checkpoint loading with strict=False silently skips mismatched keys; missing instance attributes preventing key remapping can cause 17%+ parameters to use random initialization with no error messages. In this case, missing self.use_fast_transformer blocked Wqkv→in_proj remapping for flash attention weights. Always verify state_dict key matching, log mismatches explicitly, and validate model behavior after loading
  • Pi0.5 Phoenix LoRA fine-tuned on 9 MimicGen tasks (4500 episodes, 500/task) achieves only 0-24% success rate on those same training environments during evaluation. This severe training-evaluation gap suggests overfitting to training distribution, insufficient training iterations, fundamental architecture limitations, or training data quality issues requiring investigation
  • Training completion ≠ evaluation done. Even 100k-step trained models can sit unevaluated in research workflows. On HPC systems, job logs reflect execution history not outcome completeness—always verify completion via artifact inspection (checkpoints, outputs) with timestamps, not log tails. Explicitly verify evaluation artifacts exist when discussing model performance
  • Proxy environment variables (http_proxy, https_proxy, HTTP_PROXY, HTTPS_PROXY) can intercept and break WebSocket connections to localhost even when raw TCP layer works—they operate at application layer before socket creation. Always unset all proxy vars when debugging local services on development machines or shared clusters
  • RM-IDEAL Spearman correlation benchmark reveals STAIG fusion (r=0.396) significantly outperforms simple fusion strategies (concat/mean ≈0.25) by explicitly modeling spatial dependencies in gene expression embeddings. Learned fusion methods consistently rank higher than naive concatenation or averaging
  • Leiden community detection provides robust auto-clustering for unlabeled spatial transcriptomics data via k-NN graph construction with resolution parameter controlling granularity. Combined with silhouette score, enables unsupervised evaluation when ground truth labels unavailable (common in Visium HD and other high-resolution spatial datasets)
  • Stub functions with TODO comments in production code can completely break features while appearing to work (no compiler errors, type checks pass). In macOS wallpaper app, 4 stub functions rendered powerSave/powerSavePlus completely non-functional for months. Critical to implement or remove stubs before shipping, not leave as technical debt
  • When loading 50+ GB models on shared GPUs, always check nvidia-smi and existing processes first before attempting new launches. Reusing zombie servers prevents CUDA OOM, saves 3-5min startup time, and enforces resource awareness and multi-user cluster etiquette. This is critical on shared HPC infrastructure
  • Pre-processed external datasets (like Visium HD from collaborators) require explicit transformation history tracking to prevent pipeline corruption. Use adata.uns[’log1p’] flag or X.max() heuristics to detect already-applied transformations and skip redundant normalization. Data provenance is critical in multi-stage analysis pipelines
  • Framework compatibility planning matters upfront for research codebases with evolving experiments. Asking ‘what future uses might this need?’ prevents architectural lock-in. In BC-RNN case, standard robomimic checkpoint format (model.pth + config.json) automatically enables error injection via existing PolicyAdapter—no additional work needed
  • VLA policy evaluation requires exact library version matching with training environment—even minor robosuite/mimicgen version differences cause API incompatibilities (missing methods like get_bounding_box_half_size, changed signatures like env.seed()). Version mismatches can silently invalidate benchmarking results or cause runtime crashes
  • Visualization pipeline performance for spatial transcriptomics often dominated by I/O (scanpy AnnData loading) and rendering (matplotlib) rather than clustering computation. Multi-process parallelization across independent samples more effective than GPU acceleration for this I/O-bound workload pattern. Always profile before optimizing
  • High-resolution spatial transcriptomics (Visium HD 8µm vs Visium 55µm bin spacing) requires dynamic visualization parameter scaling—spot size must scale with bin spacing to prevent invisible or overlapping dots. HD’s 7.3px spot diameter needs size=4.0 vs Visium’s size=1.0 default for proper spatial plot coverage
  • For multi-variant model discussions in research, lead with data provenance (what data, how many samples, which tasks) rather than checkpoint paths. Training data is conceptual foundation; checkpoint paths are technical details. This enables better decision-making about what to evaluate and compare

Practical Learnings#

  • Xcode PBXFileSystemSynchronizedRootGroup (objectVersion 77) automatically discovers new source files without manual project.pbxproj edits, reducing merge conflicts in modern Xcode projects. ENABLE_USER_SCRIPT_SANDBOXING must be set to NO to allow build scripts to modify build products like Info.plist
  • SwiftUI double-click requires .onTapGesture(count: 2) closure, not Button actions which only support single taps. Useful for deliberate activation UX patterns where accidental single-click activation would be problematic. Combine with cooldown timer to prevent rapid-fire activation
  • Complex JSON files (like Xcode .xcstrings localization) should be edited via language JSON libraries (Python json module) rather than string replacement to maintain nested structure. Manual string editing risks breaking structure
  • tyro CLI framework uses positional subcommands (not –flag:subcommand syntax) and automatically adds dataclass name prefixes to nested arguments (–args.field). Read help output carefully to understand argument structure
  • On shared HPC filesystems with millions of files, avoid recursive find without tight depth limits—operations can timeout. Prefer direct path construction from known workspace conventions with targeted ls and grep commands
