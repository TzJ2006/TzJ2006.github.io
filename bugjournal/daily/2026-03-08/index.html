Problems and Solutions#

Key Problems#

1. VLA model exhibits systematic failures during critical task phases (incorrect pressing in click tasks,抓取/placement failures in operational tasks)#

Solution: Designed a distance-based heuristic algorithm combined with a gripper state filtering mechanism to annotate binary “critical regions” during data collection, thereby training the cls token classifier.

Key Insight: By training a binary classifier, it’s possible to predict areas that require extra attention for the agent, thus solving the systematic failure problems of VLA during critical task phases.

2. All critical_region labels are 0 after data collection, indicating that the threshold (0.10m) is too conservative.#

Solution: Added the get_ee_target_distances() method to record the original EE-to-target distances for all tasks, allowing empirical threshold adjustment.

Key Insight: When designing machine learning data annotation heuristics, it’s important to first record the original features before determining the label thresholds.

General Problems#

3. SAPIEN renderer encounters “RuntimeError: failed to find a rendering device” errors in headless environments.#

Solution: Added three Vulkan/EGL environment variables (VK_ICD_FILENAMES, __EGL_VENDOR_LIBRARY_FILENAMES, LD_LIBRARY_PATH) in collect_data.sh, pointing to the nvidia_libs directory.

Key Insight: Headless GPU servers require explicit Vulkan ICD and EGL vendor configurations to find NVIDIA rendering drivers, even if nvidia-smi works properly.

4. SSH command errors occur when running visualization scripts from a remote shell.#

Solution: Changed the relative paths of the project root directory and Python scripts to absolute paths, ensuring that the working directory is correctly set before execution.

Key Insight: SSH does not inherit the current working directory; it’s necessary to use “cd /absolute/path && command” or provide the absolute path for all files.

Learning Outcomes#

Key Learnings#

  • For training VLA in manipulation tasks, using a distance-based heuristic algorithm and grasping state filtering to二分 “critical regions” can systematically solve failure problems during critical task phases (pressing, grasping, placing).
  • The arXiv API supports author queries using the “au:author_name” syntax, which can be combined with keyword filters using AND/OR operators, and correlation sorting can be used for author searches.
  • When designing ML data collection heuristics, it’s necessary to achieve observability (logging original features) before adopting threshold-based labels.
  • research_scout.py now supports three different search modes with varying caching strategies: keywords/categories (sorted by date, with backoff time), conferences (sorted by relevance, with comment filtering), and authors (sorted by relevance, without post-processing).
  • Backward compatibility based on symbolic links results in complex dependency graphs that need to be fully mapped before refactoring (65 symbolic links × multiple import patterns = high risk of interruption if not carefully planned).

Practical Learnings#

  • MuJoCo visualization can generate from pre-saved.npz scene data without re-running the entire strategy process, enabling efficient post-production video generation.
  • SAPIEN’s headless GPU rendering requires three Vulkan/EGL environment variables: VK_ICD_FILENAMES, __EGL_VENDOR_LIBRARY_FILENAMES, and LD_LIBRARY_PATH, which point to the NVIDIA driver library directory.
