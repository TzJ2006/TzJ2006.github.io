Problems & Solutions#

Critical Issues#

1. VLA models fail consistently at specific task phases (imprecise pressing in click tasks, failed grasping/placing in manipulation tasks)#

Solution: Designed distance-based heuristics with gripper-state filtering to annotate binary ‘critical region’ labels during data collection, enabling cls token classifier training

Key Insight: Systematic VLA failures at task-critical phases can be addressed by training a binary classifier to predict when the agent is in high-stakes regions requiring extra caution

2. All critical_region labels were 0 after data collection, indicating threshold (0.10m) was too conservative#

Solution: Added get_ee_target_distances() method to record raw EE-to-target distances for all tasks, enabling empirical threshold tuning

Key Insight: When designing ML data labeling heuristics, always log raw features first before committing to threshold values for labels

General Issues#

3. SAPIEN renderer fails with ‘RuntimeError: failed to find a rendering device’ in headless environment#

Solution: Added three Vulkan/EGL environment variables to collect_data.sh (VK_ICD_FILENAMES, __EGL_VENDOR_LIBRARY_FILENAMES, LD_LIBRARY_PATH) pointing to nvidia_libs directory

Key Insight: Headless GPU servers require explicit Vulkan ICD and EGL vendor configuration to locate NVIDIA rendering drivers, even when nvidia-smi works correctly

4. SSH commands failed with ‘No such file or directory’ when running visualization script from remote shell#

Solution: Changed from relative paths to absolute paths for both project root and Python script, ensuring working directory was correctly set before execution

Key Insight: SSH does not inherit working directory; must use ‘cd /absolute/path && command’ or provide absolute paths to all file arguments

Learnings#

Key Learnings#

  • For VLA training on manipulation tasks, binary classification of ‘critical regions’ using distance-based heuristics + gripper-state filtering can systematically address failure patterns at task-critical phases (pressing, grasping, placing)
  • arXiv API supports author queries via ‘au:“Author Name”’ syntax, can be combined with keyword filters using AND/OR operators, and uses relevance-based sorting for author searches
  • When designing ML data collection heuristics, always implement observability (log raw features) before committing to threshold-based labels. Empirical tuning requires actual data distribution visibility
  • research_scout.py now supports three distinct search modes with different caching strategies: keyword/category (date-sorted, lookback-filtered), conference (relevance-sorted, comment-filtered), author (relevance-sorted, no post-filter)
  • Symlink-based backward compatibility creates complex dependency graphs requiring comprehensive mapping before refactoring (65 symlinks × multiple import patterns = high breakage risk without careful planning)

Practical Learnings#

  • MuJoCo visualization can be generated from pre-saved .npz scene data without re-running full policy rollouts, enabling efficient post-hoc video generation
  • Headless GPU rendering in SAPIEN requires three Vulkan/EGL environment variables: VK_ICD_FILENAMES, __EGL_VENDOR_LIBRARY_FILENAMES, and LD_LIBRARY_PATH pointing to NVIDIA driver library directory
