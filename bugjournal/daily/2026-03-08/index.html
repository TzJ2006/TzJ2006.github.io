Problems and Solutions#

Key Issues#

1. Natural scenario visualization has only 10 neutral frames, making it impossible to show the actual movement process of strategies (pre-error trajectory data was not persisted during generation)#

Solution: Record the complete action_history and initial_state in capture_natural_errors of rollout_generator.py and store them in NPZ. When visualizing, replay from the initial_state or 5 seconds before an error.

Key Insight: Pre-error trajectory data must be actively persisted during generation; separate storage structures need to be designed for demo replay (with HDF5) and policy replay (which requires additional storage).

2. All critical_region data collected was 0: The τ=0.10m threshold is purely based on parameter estimation, which does not match the actual robot workspace size.#

Solution: Instead of adjusting parameters directly, record original ee_target_distances (4-8 target distance variables per task). Analyze the actual data distribution before calibrating the threshold.

Key Insight: The robot task space scale deviates significantly from intuition. Experimental-driven methods (observing data first, then setting thresholds) are more reliable than parameter estimation. Recording original distances is also a valuable diagnostic tool.

3. Generating 0 effective scenarios in injection mode: Base was too stable, and the 3-35N thrust force was insufficient to pass verification.#

Solution: Visualize 158 natural error scenarios, covering 6 types of actual failures. Produce 4 MP4s on 3 GPUs in parallel.

Key Insight: Injection parameters need to be calibrated specifically for each task. Actively pivoting when encountering obstacles is a reasonable strategy, without needing user instructions.

4. AI initially suggested waiting until the injection task was completed before generating visualizations, but did not actively identify opportunities for parallel execution.#

Solution: Users identified that visualization could be generated immediately during injection tasks. AI then inserted lightweight visualization tasks during periods of low GPU load.

Key Insight: Long-term GPU tasks and lightweight visualization tasks can run in parallel on the same GPU. Such opportunities should be actively identified rather than conservatively waiting in serial mode.

General Issues#

5. GPU rendering environment configuration mismatch: MUJOCO_EGL_DEVICE_ID and CUDA_VISIBLE_DEVICES had physical numbers that did not match, or HPC headless nodes lacked Vulkan driver configurations leading to rendering initialization failures.#

Solution: Set MUJOCO_EGL_DEVICE_ID and CUDA_VISIBLE_DEVICES to the same physical number. Refer to eval.sh from the same repository and add VK_ICD_FILENAMES, __EGL_VENDOR_LIBRARY_FILENAMES, and LD_LIBRARY_PATH as environment variables in collect_data.sh.

Key Insight: EGL/Vulkan device configuration is independent of CUDA. Normalization of CUDA does not mean Vulkan/EGL are available. Solutions usually exist in other scripts from the same repository, so it’s better to reuse existing configurations.

6. Errors in .parent chain depth of sys.path.insert after symlink removal: Path.resolve() no longer folds symlinks, causing two pipeline scripts to fail to find the project root directory.#

Solution: Change .parent.parent to .parent.parent.parent and recalculate according to the actual directory hierarchy (pipeline/ → scripts/ → error_benchmark/ → project_root).

Key Insight: After removing symlinks, all paths that depend on folded symlinks must be rechecked thoroughly. This is a potential hidden issue that can be easily overlooked in large-scale migrations.

7. SSH remote execution script path errors: SSH defaults to the home directory, making it impossible to find project scripts. Multiple fine-tuning attempts failed.#

Solution: Use absolute paths for script paths in SSH commands and place the cd operation within the same bash -c string as the main command.

Key Insight: SSH connections do not inherit the local cwd. Absolute paths must be explicitly specified. Diagnosis should focus on the root cause rather than iterative fine-tuning.

AI Limitations#

Important Limitations### General Limitations#

  • Repetitive error diagnosis is inefficient: The SSH path issue was only resolved after multiple fine-tuning steps to locate the root cause (absolute paths were required). Diagnosis should target the root cause rather than making incremental adjustments.
  • Insufficient context management: The pre-error replay feature was interrupted due to context exhaustion before the visualization script could complete. Large-scale feature development should plan session segmentation points in advance.

Today’s Achievements#

Key Achievements#

  • 4-way parallel sub-agents + direct processing of main threads: This allowed for the large-scale migration of 65 symlinks within a single session. The key was dividing agent tasks based on file collections without overlap, with the main thread handling non-parallelizable Makefile/Documentation work.
  • Experimental-driven approach is better than parameter estimation: The robot task space scale differs significantly from intuition. Key parameters (such as distance thresholds) should be analyzed using small samples before setting them. The original records of ee_target_distances are valuable diagnostic tools.
  • Pre-error trajectory data (action_history + initial_state) must be stored in NPZ during scenario generation; otherwise, visualization cannot reproduce the actual behavior of the strategy. Demo replay (with HDF5) and policy replay (requiring additional storage) should be distinguished in design.
  • The physical characteristics of three_piece_assembly make the injection mode ineffective (base is too stable). Error scenarios must be collected using natural_capture. Natural error distributions (32% grasp_wrong_pose, 25% premature_release) reveal that BC-RNN’s main bottleneck lies in grasping rather than insertion phase.
  • In inheritance systems, using the template method pattern (base class returns default values, subclass overrides as needed) is more consistent with the open-close principle compared to using conditional branches in get_obs(). New tasks only need to implement subclass methods without modifying the core process.
  • arXiv supports precise author queries by specifying au:“Author Name” and can combine it with keywords (au:“Name” AND (kw1 OR kw2)). This is an effective way for the system to track researchers’ latest work.

Practical Achievements#

  • GPU rendering environment configuration is independent: EGL device IDs must match CUDA physical numbers. Vulkan drivers (VK_ICD_FILENAMES) are independent of CUDA. If CUDA works, Vulkan can also work. Solutions often exist in other scripts within the same repository; prioritize reuse.
  • After removing symlinks, the depth of the sys.path parent chain needs to be recalculated based on the actual directory hierarchy. Path.resolve() no longer folds symlinks. The original parent chains that worked before removing symlinks have insufficient depth after removal, which is a potential issue in large-scale migrations.
