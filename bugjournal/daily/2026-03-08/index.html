Problems and Solutions#

Key Problems#

1. The VLA model always fails at certain task stages (incorrect pressing in click tasks, failure to grasp/place in manipulation tasks)#

Solution: Designed a distance-based heuristic algorithm, considering the influence of gripper states on critical regions, to annotate binary “critical regions” during data collection, thereby supporting the training of the cls token classifier. Key Insight: By training a binary classifier, it is possible to predict areas where agents need special attention, thus addressing systematic VLA failures at critical task stages.

2. All critical_region labels become 0 after data collection, indicating that the threshold (0.10 meters) is too conservative#

Solution: Added the get_ee_target_distances() method, recording the original EE-to-target distances for all tasks, allowing for empirical threshold adjustment. Key Insight: When designing machine learning data annotation heuristics, it is important to record the original features before determining the label thresholds.

General Problems#

3. The SAPIEN renderer encounters “RuntimeError: failed to find a rendering device” errors in headless environments#

Solution: Added three Vulkan/EGL environment variables (VK_ICD_FILENAMES, __EGL_VENDOR_LIBRARY_FILENAMES, LD_LIBRARY_PATH) to collect_data.sh, pointing to the nvidia_libs directory. Key Insight: Headless GPU servers require explicit Vulkan ICD and EGL vendor configurations to find NVIDIA rendering drivers, even if nvidia-smi works properly.

4. When running visualization scripts from a remote shell, the SSH command returns “No such file or directory” errors#

Solution: Changed the relative paths of the project root directory and Python scripts to absolute paths, ensuring that the working directory is correctly set before execution. Key Insight: SSH does not inherit the working directory; you must use “cd /absolute_path&&command” or provide the absolute paths of all files.

Lessons Learned#

Key Lessons#

  • For VLA training in manipulation tasks, distance-based heuristics and filtering of grasping states can be used to systematically handle failure modes at critical task stages (pressing, grasping, placing).
  • The arXiv API supports author queries using the “au:” syntax, which can be combined with keyword filters using AND/OR operators, and relevance sorting can be used for author searches.
  • When designing machine learning data collection heuristics, observability should be achieved before using threshold-based labels (record original features). Empirical adjustments require visibility into the actual data distribution.
  • research_scout.py now supports three different search modes with different caching strategies: keywords/categories (sorted by date, with backoff time), conferences (sorted by relevance, with comments filtered), authors (sorted by relevance, without post-processing filtering).
  • Backward-compatibility with symbolic links creates complex dependency graphs, requiring a comprehensive mapping before refactoring (65 symbolic links × multiple import patterns = high risk of disruption if not carefully planned).

Practical Lessons#

  • MuJoCo visualizations can be generated from pre-saved.npz scene data without rerunning the entire strategy, enabling efficient post-production video generation.
  • SAPIEN’s headless GPU rendering requires three Vulkan/EGL environment variables: VK_ICD_FILENAMES, __EGL_VENDOR_LIBRARY_FILENAMES, and LD_LIBRARY_PATH, pointing to the NVIDIA driver library directory.
