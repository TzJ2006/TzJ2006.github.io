Problems & Solutions#

Critical Issues#

1. SameLabel@K metric returned 0.0 even though embedding clearly retrieved same cell types#

Solution: Root cause: eval_mask excluded all niche spots via ~niche_mask, leaving zero Layer_3 spots. Fixed by computing SameLabel over all_valid_mask (including niche) while keeping Precision@K on eval_mask (excluding niche to avoid self-match bias)

Key Insight: Different evaluation metrics require different exclusion masks - retrieval rate needs the full valid set, while rank agreement needs to exclude the query region itself

2. Initial plan called for implementing RM-Ideal from scratch, but code review showed it was already implemented#

Solution: Human redirected to verify existing implementation rather than duplicate work; ran comprehensive tests confirming identical/different subgraphs return 1.0/0.0 as expected

Key Insight: Always verify existing codebase state before implementing - documentation and actual code can be out of sync

General Issues#

3. Node ID collision risk when combining query and reference subgraphs in shared label dict#

Solution: Applied node ID offset of adata_q.n_obs to all reference node IDs before creating combined label dict, ensuring no key collisions

Key Insight: When merging graphs from separate AnnData objects, node indices overlap - use deterministic offsets rather than relying on disjoint ID spaces

4. Wasserstein distance solver dependency - POT library not installed in quest conda environment#

Solution: Implemented two-tier solver: try import ot.emd2() first (fast C backend), fallback to scipy.optimize.linprog with HiGHS solver (no extra dependency, sufficient for small subgraphs ~30-100 nodes)

Key Insight: Always provide dependency-free fallbacks for optional performance optimizations - scipy is ubiquitous, POT is not

Learnings#

Key Learnings#

  • RM-Ideal (Region Matching with Ideal labels) uses WWL graph kernel: K-iteration neighbor label hashing → Hamming cost matrix (normalized by K+1) → Wasserstein distance → similarity = 1-W. Captures spatial niche structure independent of learned embeddings.
  • Evaluation metrics for niche query require careful mask design: Precision@K uses eval_mask (~niche to avoid self-match), SameLabel@K uses all_valid_mask (including niche for retrieval rate), Spearman uses eval_mask (rank correlation on non-self spots)

Practical Learnings#

  • When porting graph algorithms between frameworks: squidpy.gr.spatial_neighbors() → sklearn NearestNeighbors + scipy sparse, torch_geometric.k_hop_subgraph() → networkx BFS. Pattern: translate to dependency-minimal equivalents.
  • Caching strategy for expensive computations: RMIdealEvaluator precomputes all k-hop subgraphs at init (~8min for 4384 spots), saves to .npy cache, subsequent runs load instantly. Cache key should include spatial graph parameters (n_neighbors, k_hop).
