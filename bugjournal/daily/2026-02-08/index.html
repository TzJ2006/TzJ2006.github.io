Today’s Summary#

Key Conclusions#

  • The mechanism of sharing hash tables between two subgraphs in the WWL graph kernel is key to the correctness of RM-Ideal: Only nodes with the same neighborhood structure can be hashed into the same integer, ensuring comparability across graphs. Node IDs must be globally unique (separated by offsets) to avoid incorrect hash table lookups.
  • Evaluation metrics need to distinguish between “rank quality” (Precision@K, excluding self-matching errors) and “type retrieval rate” (SameLabel@K, not excluding similar nodes). Using the same eval_mask for both issues is a design flaw.
  • RM-Ideal caching (precomputing k-hop subgraphs and storing them as.npy files) reduces batch evaluation time from 8 minutes to almost zero seconds, which is crucial for iterative experiments.
  • The results of uni_staig_fusion on Layer_3 151508 show: Spearman r=0.42 (statistically significant, p~4e-131), SameLabel@50=1.0, Precision@50=0.26 —— Embeddings can accurately retrieve the same cell type, but their ability to capture fine structures within specific domains is limited.
