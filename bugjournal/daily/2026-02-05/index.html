Problems and Solutions#

Key Problems#

1. GCN+STAIG’s performance is very poor and unstable (average ARI of about 0.16, CV of about 106%). The root cause lies in four key differences between the implementation and the original STAIG paper.#

Solution: By comparing the number of lines in the net.py and staig.py of the original STAIG code, we identified four differences: lack of adaptive edge dropout, gene trimming, incorrect edge weight direction, and missing tau decay. All verification tests passed.

Key Insight: The adaptive edge dropout mechanism of the original STAIG (rand() >= edge_prob, where edges with higher similarity were more likely to be discarded) was contrary to intuition. However, this mechanism prevented overfitting within the domain. Multiple deviations could lead to performance crashes, and the impact of each fix cannot be evaluated independently.

2. There are two key flaws in the end-to-end fusion module: ① Misunderstanding of the gene encoder architecture (AI mistakenly understood E2E as “no need for a gene encoder” and designed a gene_encoder=none interface); ② The apply_fusion() function signature lacked **kwargs parameters, resulting in a NameError crash at runtime.#

Solution: After the user corrected the misunderstanding about the architecture, AI redesigned the gene preprocessing parameters to support any input dimension. At the same time, **kwargs was added to the end of the apply_fusion() signature to ensure that the fusion kwargs dictionary could be passed correctly. These two flaws were completely fixed in the second round of tests, and the end-to-end process passed verification.

Key Insight: The core of end-to-end training lies in the joint optimization of GCN and STAIG, rather than skipping gene encoding. Mismatches between function signatures and parameters will be exposed at runtime. Unified checks should be carried out during the design phase to ensure consistency of interfaces.

3. MIHD’s STAIG fusion ARI (0.452) is lower than that of the original STAIG, but the reason is unclear.#

Solution: By comparing the train_img_config.yaml and run_benchmark.py default parameters, we found four parameter differences: the learning rate was doubled, the number of GCN layers increased by one, the training cycles were increased by 250 times, and the weight decay was increased by 10 times. Performance differences were caused by hyperparameters rather than algorithm logic. Key Insight: Comparing learning models is very sensitive to hyperparameters. Smaller learning rates and fewer layers actually improved performance in STAIG, indicating a high risk of overfitting. When evaluating performance differences, we should first check hyperparameters instead of suspecting algorithm errors.

4. When the timmkwargs is not provided, the UNI2 model throws a shape invalid error (a shape invalid problem occurred with a size of 391680).#

Solution: All timmkwargs must be provided (such as img_size, patch_size, depth, etc.) so that timm can correctly calculate the position encoding grid size.

Key Insight: The size of the pre-trained weight position encoding table on HuggingFace Hub does not match the default inference results of timm. When loading these data, accurate architecture parameters must be provided.

General Problems#

5. Lack of conda environment dependencies: torch_geometric is missing in the AI environment, while pandas and other dependencies are missing in the general environment.#

Solution: The problem with torch_geometric was solved by switching to the general environment. The general environment fixed the missing dependencies by installing them step by step using pip/conda. Priority was given to fixing base dependencies with inconsistent metadata.

Key Insight: This project uses two conda environments (scgpt_3 and General), which need to be switched according to tasks. Using pip and conda together in the same environment may lead to inconsistent metadata. Using conda is more reliable for fixing base dependencies.

Today’s Achievements#

Core Achievements#

  • The adaptive edge dropout mechanism of the original STAIG (rand() >= edge_prob, where edges with higher similarity were more likely to be discarded) was contrary to intuition. However, this mechanism prevented overfitting within the domain and enhanced cross-domain learning capabilities. This difference had a fundamental impact on the quality of GCN messaging compared to uniform random dropout.
  • The performance ceiling of end-to-end training (joint optimization using GCN embeddings for multi-modal comparison) was much higher than that of two-stage separation methods (first extracting fixed GCN embeddings, then training STAIG). The original STAIG could achieve an ARI of 0.68, while the two-stage separation was only about 0.16. This difference stemmed from whether the GCN embeddings contained cross-modal supervision signals.
  • The configuration of STAIG hyperparameters had counterintuitive characteristics: smaller learning rates (0.0005), fewer GCN layers (1 layer), shorter training time (300 cycles), and smaller weight decay coefficients (1e-5) actually brought better performance. This indicated that comparing learning models was at high risk of overfitting. Therefore, priority should be given to checking performance differences rather than suspecting algorithm logic.
  • The ‘auto’ strategy for gene preprocessing (using StandardScaler+Clip when dim>100, otherwise using Clip only) is a simple and universal solution for processing genes from different sources. This avoided the need for repeated normalization of standardized embeddings.

Practical Achievements#

  • Two key points in project environment management: ① Use two conda environments (scgpt_3 for scGPT/gene analysis, general environment for image embedding/PCA). Switch between these environments according to tasks. ② CUBLAS_WORKSPACE_CONFIG must be set at the top of the program (before all imports), otherwise CUDA operators would choose an uncertain path during initialization, and setting it later would be invalid.
