Problems and Solutions#

Key Issues#

1. GCN+STAIG performance is extremely poor and unstable (ARI average ≈0.16, CV ≈106%). The root cause is four critical differences between the implementation and the original STAIG paper.#

Solution: By comparing lines of the original STAIG’s net.py and staig.py, we identified four differences: missing adaptive edge dropout, gene clipping, incorrect edge weight direction, and missing tau decay. All verification tests passed.

Key insight: The original STAIG’s adaptive edge dropout (rand() >= edge_prob, where higher similarity edges were more likely to be dropped) was contrary to intuition. However, this mechanism prevented overfitting within the domain. Multiple deviations could lead to performance crashes, and the impact of each fix cannot be evaluated separately.

2. There are two key defects in the end-to-end fusion module: ① Misunderstanding of the gene encoder architecture (AI misinterpreted E2E as “no need for a gene encoder” and designed a gene_encoder=none interface); ② The apply_fusion() function signature lacked **kwargs, causing a runtime NameError crash.#

Solution: After the user corrected the misunderstanding about the architecture, AI redesigned the gene preprocessing parameters to support any input dimension. At the same time, **kwargs was added to the end of the apply_fusion() signature, ensuring that the fusion_kwargs dictionary could be passed correctly. Both defects were fully fixed in the second round of testing, and the end-to-end pipeline passed verification.

Key insight: The core of end-to-end training is the joint optimization of GCN and STAIG, not skipping gene encoding. Mismatches between function signatures and parameters are exposed during runtime. Uniform checks should be performed at the design stage to ensure consistency in interfaces.

3. MIHD’s STAIG fusion ARI (0.452) is lower than that of the original STAIG, but the reason is unknown.#

Solution: By comparing train_img_config.yaml and default parameters in run_benchmark.py, we found four parameter differences: learning rate was twice as high, one more layer in GCN, 250 more epochs, and weight decay was 10 times higher. Performance differences were caused by hyperparameters rather than algorithmic logic.

Key insight: Comparative learning models are highly sensitive to hyperparameters. Smaller learning rates and fewer layers may actually improve performance in STAIG, indicating a high risk of overfitting. When evaluating performance differences, hyperparameters should be checked first, rather than suspecting algorithmic errors.

4. The UNI2 model throws a shape invalid error when timm_kwargs is not provided (shape invalid for size 391680).#

Solution: It is necessary to provide all of timmkwargs (img_size, patch_size, depth, etc.) so that timm can correctly calculate the position encoding grid size.

Key insight: The size of the pre-trained weight position encoding table on HuggingFace Hub does not match timm’s default inference. Accurate architecture parameters must be provided when loading this data.

General Issues#

5. Missing conda environment dependencies: AI environment lacked torch_geometric, and the General environment lacked pandas and other dependencies.#

Solution: The problem with torch_geometric was solved by switching to the General environment. The General environment was fixed by iteratively installing missing dependencies using pip/conda. Condos were prioritized for fixing underlying dependencies with inconsistent metadata.

Key insight: The project uses two conda environments (scgpt_3 and General), which must be switched based on tasks. Using both pip and conda in the same environment can lead to inconsistent metadata. It’s better to use condos for fixing underlying dependencies more reliably.

Today’s Achievements#

Core Achievements#

  • The original STAIG’s adaptive edge dropout (rand() >= edge_prob, where higher similarity edges were more likely to be dropped) was contrary to intuition. However, this mechanism prevented overfitting within the domain and enhanced cross-domain learning. Compared to uniform random dropout, this difference had a fundamental impact on GCN message passing quality.
  • The performance ceiling of end-to-end training (GCN embedded within STAIG for joint optimization with multi-modal comparison) was much higher than that of two-stage separation (first extracting fixed GCN embeddings and then training STAIG). The original STAIG could achieve an ARI of 0.68, while the two-stage separation was only approximately 0.16. The difference stemmed from whether the GCN embedding contained cross-modal supervision signals.
  • The configuration of STAIG hyperparameters was counterintuitive: smaller learning rates (0.0005), fewer GCN layers (1 layer), shorter training duration (300 epochs), and smaller weight decay (1e-5) actually led to better performance. This indicates that comparative learning models have a high risk of overfitting, and performance differences should be prioritized by checking hyperparameters rather than suspecting algorithmic logic.
  • The ‘auto’ strategy for gene preprocessing (StandardScaler+Clip when dim>100, otherwise just Clip) is a simple and universal solution for handling genes from different sources. This avoids repeated normalization of standardized embeddings.

Practical Achievements#

  • Two key points in project environment management: ① Use two conda environments (scgpt_3 for scGPT/gene analysis, General for image embedding/PCA). Switch between them based on tasks. ② CUBLAS_WORKSPACE_CONFIG must be set at the top of the program (before all imports), otherwise CUDA operators will choose an uncertain path during initialization, and setting it later will be ineffective.
