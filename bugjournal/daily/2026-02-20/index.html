Problems and Solutions#

Key Problems#

1. run_benchmark.py re-instantiates encoders and embeddings every time it runs, wasting UNI2/SCAN features that have already been calculated.#

Solution: Integrated CacheManager into has/load/save_*_embedding() methods; added cache checks before encoder instantiation, with variation support (standard/frequency/strict state); propagated coverage flags to both calling points.

Key Insight: Users’ expectations are correct—run_benchmark.py should have similar caching functionality. AI initially proposed workaround solutions (separate evaluation scripts, manual caching) rather than solving the root cause.

2. BC-RNN strategy expects input size to be 65, but the environment provides 37 dimensions, causing RuntimeError.#

Solution: The root cause is mismatched object counts (8 objects = 56 dimensions for training, 4 objects = 28 dimensions in the evaluation environment). Fixed _to_robosuite_obs() function to map ‘object-state’ to ‘object’, using raw robosuite obs from environment._env._get_observations(), instead of using StateExtractor.

Key Insight: The initial fix only applied to the injection path (_generate_from_single_rollout); the same problem exists in the natural capture path. When fixing the observation process, all code paths need to be checked.

3. staig_fusion ARI in eval_scan_fusion.py is 0.39, which is lower than expected.#

Solution: It was found that there was a double normalization error—staig_alignment_config was not passed, so STAIGTrainer applied StandardScaler after upstream preprocessing, while sc.pp.scale() already existed. Additionally, no previous 151673 staig_fusion baseline data was available for comparison.

Key Insight: “Lower than expected” without baseline data is speculative. Before assuming regression, the existence of baseline data must be verified. Also, independent evaluation scripts easily overlook configuration details that the standard process can handle correctly.

General Problems#

4. MimicGen dataset augmentation failed, with the error “env_class unexpected keyword argument”.#

Solution: Fixed robomimic_utils.py to use inspect.signature() to filter kwargs based on robomimic 0.3.0 API signature, but this was still hindered by mujoco_py dependency issues in pip installations. Decided to skip MimicGen and expand scale by using more BC-RNN/VLA iterations.

Key Insight: Environmental isolation issues require in-depth dependency analysis. When encountering obstacles, alternative methods should be pursued instead of forcing incompatible dependencies.
