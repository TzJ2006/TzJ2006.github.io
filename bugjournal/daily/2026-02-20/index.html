Problems & Solutions#

Critical Issues#

1. run_benchmark.py re-instantiated encoders and extracted embeddings every run, wasting hours on already-computed UNI2/SCAN features#

Solution: Integrated CacheManager with has/load/save_*_embedding() methods; added cache checks before encoder instantiation with variant support (standard/freq/staig_strict); propagated override flag to both call sites

Key Insight: User’s expectation was correct—run_benchmark.py should have had caching like pipeline does. AI initially proposed workarounds (separate eval script, manual caching) instead of fixing the root cause.

2. BC-RNN policy expected input_size=65 but environment provided 37 dimensions, causing RuntimeError#

Solution: Root cause was object count mismatch (trained with 8 objects=56 dims, eval env has 4 objects=28 dims). Fixed _to_robosuite_obs() to map ‘object-state’→‘object’ key and use raw robosuite obs from env._env._get_observations() instead of StateExtractor

Key Insight: Initial fix only covered injection path (_generate_from_single_rollout); natural capture path (_capture_single_rollout) had same issue. Need to check all code paths when fixing obs flow.

3. eval_scan_fusion.py had staig_fusion ARI~0.39 result that seemed lower than expected#

Solution: Investigation revealed double normalization bug—didn’t pass staig_alignment_config, so STAIGTrainer applied StandardScaler after upstream preprocessing already did sc.pp.scale(). Also found no prior 151673 staig_fusion baseline existed to compare against

Key Insight: “Lower than expected” without baseline data is speculation. Always verify baseline exists before assuming regression. Also: independent eval scripts easily miss config details that canonical pipelines handle correctly.

General Issues#

4. MimicGen dataset augmentation failed with ’env_class unexpected keyword argument’ error#

Solution: Fixed robomimic_utils.py to filter kwargs based on robomimic 0.3.0 API signature using inspect.signature(), but still blocked by mujoco_py dependency in pip-installed robomimic. Decided to skip MimicGen and scale via more BC-RNN/VLA rollouts instead

Key Insight: Environment isolation issues require deep dependency analysis. When blocked, pivot to alternative approaches rather than forcing incompatible dependencies.

  • Fixed BC-RNN obs mismatch in injection path but missed identical issue in natural capture path, requiring second fix iteration—incomplete code analysis when debugging

Learnings#

Key Learnings#

  • When user questions redundant work (“why re-instantiate if embeddings exist?”), they’re often identifying missing infrastructure that should be there—investigate the expectation before proposing workarounds
  • Observation pipeline bugs in multi-path codebases (injection vs natural capture) require exhaustive analysis—fix in one path doesn’t guarantee fix in all paths; grep for all uses of the pattern
  • Cache variants (standard/freq/staig_strict) prevent collisions when same encoder produces different embeddings based on preprocessing flags—variant system is more robust than separate encoder names

Practical Learnings#

  • Large-scale GPU jobs benefit from parallel launch across devices with distinct logs (m12_bc_rnn_injection.log, m12_bc_rnn_natural.log) for independent monitoring
  • Environment dependency issues (mujoco_py in robomimic 0.3.0) may be showstoppers—when blocked, pivot to alternative data sources rather than forcing incompatible deps
