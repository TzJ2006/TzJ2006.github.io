Issues and Solutions#

Key Issues#

1. All background services of CalendarPro (GapAnalyzer/AutonomousExecutor/ReminderEvaluator) have never been run in a production environment: setup_hook and main.py did not call BackgroundCoordinator.start_all().#

Solution: Added an await coordinator.start_all() call in the discord_bot.py’s on_ready() method.

Key Insight: Registration patterns + lifecycle management can lead to silent failures where “registered but never started”; 196 unit tests passed without detecting this integration flaw, indicating that test coverage does not equal system availability.

2. On line 114 of zhaoganlong create_5hz_dataset_new_motion.py, the save_path contains a .testc后缀, but downstream create_speed_dataset.py reads it without this suffix, resulting in silent data loss; when rerunning with h5py append writing mode, create_group reports a ValueError.#

Solution: Changed _adjust_llava_motion.testc.hdf5 to _adjust_llava_motion.hdf5; changed h5py append writing (‘a’) to overwrite writing (‘w’) to ensure idempotency.

Key Insight: Inconsistent file names in upstream and downstream scripts can silently skip data without an error, making it one of the most hidden bugs in a pipeline; HDF5 training data generation scripts should use write mode rather than append mode.

3. In Openpi-moe, norm_stats.json lacks the prev_actions key but training does not report errors; simultaneously, in VAE, unnormalized histActions and normalized actions are spliced directly, causing scale inconsistency.#

Solution: Modified compute_norm_stats.py to dynamically detect the presence of prev Actions, adding RunningStats if present and writing it into norm_stats.json; regenerate statistical files with prev Actions.

Key Insight: apply_tree(strict=False) in transforms.py遍历数据键而不是 norm_stats 键, silently skipping missing keys—new training features must be updated simultaneously with normalization scripts, otherwise it will affect training quality due to silent scale mismatch.

4. All 452 utterances in CalendarPro are hard-coded; the accumulated LLM correction records in data/intent_mismatches.jsonl have never been used as training signals.#

Solution: Externalize them into JSON files, implement UtteranceAugmenter that automatically learns from mismatch logs, register daily augmentation tasks, and clear used mismatch records after processing.

Key Insight: AI system’s mismatch records are free annotation data that should be automatically reverted to the improvement loop; externalization of utterances + automatic augmentation is a low-cost continuous learning mechanism.

5. The 16 except Exception: pass silent anomalies in CalendarPro are distributed across new services, masking all runtime errors; the execute_step() loop body in executor.py only has pass, and get_progress() always returns None, effectively disconnecting the entire Agent execution chain.#

Solution: Replace all silent anomalies with except Exception as e: logger.error(...); remove dead code, implement real step distribution logic and progress tracking functions.

Key Insight: Silent failures at the state tracking layer and event bus layer are much more harmful than ordinary business layers—services seem to be running but all errors are swallowed up, making monitoring and debugging impossible; new features pass unit tests but their empty implementations at integration points disconnect the entire chain.

6. Pi0.5 experienced OOM on a single A800 80GB card (model + activation values require ~64GB+), and relative_to ValueError occurred due to incorrect symlink resolve path during startup.#

Solution: Add –fsdp-devices 2 parameter to split the model across GPU 2+3; set OPENPI_DATA_HOME to point to the actual cache directory to bypass pathlib.resolve() for symlink resolution.

Key Insight: Even using LoRA, Pi0.5 requires at least 2 80GB GPUs, and FSDP is a necessary condition rather than an optimization measure; openpi’s get_cache_dir() uses pathlib.resolve() to resolve symlinks, which must be explicitly set through environment variables to specify the cache root path.

7. In MIHD cross-sample evaluation, negative Spearman correlations were observed in Layer_3 and Layer_6 (ρ≈-0.21～-0.36), with P@K zero for all layers, coexisting with significantly positive Spearman values.#

Solution: Negative correlations are identified as data characteristics that blur boundaries between adjacent layers in the embedding space (not code errors); coexistence of P@K=0 and Spearman>0.4 is a reasonable phenomenon—both indicators measure exact set overlap (extremely strict) and overall ranking monotonicity.

Key Insight: Zero-shot fusion embeddings can be effective globally but lack precise positioning capabilities; Layer_1/5 perform excellently due to their unique structure, while intermediate transition layers are inherent weaknesses; do not deny one with just one indicator.

8. In CalendarPro, the 9 IntentTypes (SET_GOAL/LOG_MEAL/REVIEW_DECISIONS) have no corresponding processors in the Discord Bot, and user-related instructions default to the original response.#

Solution: Added an elif branch in discord_bot.py’s handle_intent to implement corresponding handle_xxx method calls for GoalTracker/DietService/ThoughtIncubator services.

Key Insight: The intent routing layer and model layer have been updated, but the view layer (Bot Handler) has not been synchronized—this is a typical multi-layer update inconsistency issue that cannot be automatically detected in end-to-end integration testing.

Today’s Gains#

Core Gains#

  • apply_tree(strict=False) is an implicit risk in ML training pipelines: When adding new training features that need normalization, if the norm_stats calculation script is not updated synchronously, it will lead to a silent scale mismatch affecting training quality; when concatenating historical actions and predicted actions with VAE, it must be ensured that both use the same normalization scale.
  • Standard three-item checks after large-scale implementation: (1) Whether new services are connected to the startup link; (2) Whether cross-layer (route → handler) updates are consistent; (3) Whether silent exceptions have masked runtime errors. Passing tests is a necessary condition, not a sufficient one.
  • Before designing an Agentic system, actively research mature similar projects—OpenClaw’s EventBus/CronScheduler/Plugin registration patterns and GSD’s STATE.md persistent memory/ContextAssembler/multi-Agent context isolation are highly reusable architectural patterns. Researching first before designing can avoid architectural rework.
  • The common silent failure mode in registration patterns + lifecycle management is “registered but never started”; silent exceptions (except: pass) are particularly dangerous in asynchronous service architectures: the service seems to run, but all errors are swallowed up, making monitoring and debugging impossible. The correct approach is to always at least record logger.exception().
  • The mismatch records of AI systems are free annotation data—each time the LLM corrects a classification error is a training sample; unsupervised continuous self-improvement can be achieved through automatic backflow with UtteranceAugmenter, making it a cost-effective online learning mechanism.
  • Parallel multi-Agents (4 processing different files simultaneously) are extremely effective in large-scale code implementation tasks, compressing serial time to about 1/4 and ensuring file isolation to avoid conflicts; high-quality early-stage architectural documents (clear dependencies, EventBus events, registration methods, test requirements) are the key prerequisites for AI’s one-time efficient implementation.
  • The layer characteristics of cross-sample zero-shot fusion embeddings: The highly unique Layer_1/5 perform exceptionally well due to their obvious structural differences (Spearman 0.42-0.66), while the intermediate transition layers (Layer_3/6) show a negative correlation with adjacent layers in the embedding space; P@K=0 but Spearman>0.4 is reasonable—both measure exact position matching and global ranking monotonicity.
  • Pi0.5 (PaliGemma 2B + action expert 300M) also requires at least 2 80GB GPUs under LoRA fine-tuning mode; the data dependencies for each training stage of the zhaoganlong framework: Pi0.5 can be immediately started with independent LeRoBot data, Diffusion Policy requires Step 2 (speed dataset), and LLaVA MPM requires all data preparation to be completed.
  • Philosophy of testing dialogic AI systems: Unit tests verify component correctness, but system value needs to be verified through actual dialogue testing (Discord message-driven), as the two cannot replace each other; GSD’s hierarchical context assembly (PROJECT→ROADMAP→STATE→EXECUTION) is an effective engineering solution for solving multi-agent context rot.

Practical Gains#

  • Practical tips for offline HPC clusters: Dependencies can be installed by copying site-packages directly between Python version conda environments; CUDA header files may be in targets/x86_64-linux/include/ instead of the standard path, and when compilation fails, first search for cuda_runtime_api.h before setting CPLUS_INCLUDE_PATH.
