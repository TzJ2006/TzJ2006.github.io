Problems and Solutions#

Key Issues#

1. cchypothesis lacks runtime debugging capabilities — All hypotheses are verified only through static code analysis, lacking dynamic state/time/data issues#

Solution: Implemented third-stage intelligent classification: static hypotheses remain in parallel read-only mode; runtime hypotheses use sequential plugin-based detection with [DEBUG Hx] labeled logs, Git safety checkpoints, and automatic cleanup via git

Key Insight: Dual-track investigation can maintain speed when not needed (parallel static), and increase depth when needed (plugin-based detection) — avoiding slowing down all debugging sessions

2. Dashboard consistently experiences 1-2 Hz vertical jitter, caused by three-layer positive feedback loops: data refresh → UI redrawing → window resizing → data refresh#

Solution: Multi-layer interrupt handling: (1) Window: 16ms to 100ms settling delay, 3throttle per second; (2) Data: shallow PayloadEqual bypasses unnecessary.set() operations; (3) Parser: 2-minute active timeout to prevent five_hour_cost oscillations

Key Insight: Self-oscillation errors require simultaneous intervention at all feedback layers; fixing one layer alone will only weaken the loop, not eliminate it

3. Git storage stack conflict: ccplan proposal added third-stage storage above existing fourth-stage storage, causing pop-up order confusion and potentially overwriting uncommitted work#

Solution: Canceled fourth-stage Git storage and used Git restore for non-destructive cleanup; now only third-stage storage is used (single storage model)

Key Insight: Multiple storage levels in automated workflows can lead to fragile state management — it’s better to use non-storable cleanup (Git restore) rather than stateful undoing (storage)

4. TokenMonitor window appears in the center of the screen instead of near the system tray; when resized, the window extends downward to the taskbar instead of maintaining its bottom edge#

Solution: Implemented direct Win32 API targeting of tray windows (FindWindowW → TrayNotifyWnd); changed front-end resizing from webviewWindow.setSize() to backend set_window_size_and-align IPC for atomic size and position updates

Key Insight: Tauri’s cross-platform locator plugin abstracted platform differences but lost accuracy; Windows system tray positioning requires specific platform Win32 API for pixel-level precision alignment

5. 42 Rust compilation errors: command file included corrupted imports of deleted modules (change_stats, integrations, parser) as well as duplicate code from new commands/submodems#

Solution: Rewrote command file into a thin 80-line module root structure, declaring 6 common modules, retaining only AppState/UsageDebugReport and 4 shared helper functions, and fixing visibility and type issues

Key Insight: Incomplete refactoring (mixed old and new code) is worse than monolithic code; migration or rollback must be completed

6. Window anchor detection is unreliable: visible:false initially, there’s a race condition between position_near_tray and GetWindowRect, continuous resizing causes anchor flipping#

Solution: Completely eliminated dynamic detection; tray pop-up windows are always positioned at the bottom of the working area based on position_near_tray, so the bottom anchor is hardcoded

Key Insight: For known types of windows, static configuration is better than dynamic detection; runtime detection increases complexity and introduces failure modes without real benefits

General Issues#

7. Criticism that Git restore doesn’t clean up untracked files (e.g., .debug/hypothesis.log created during plugin process)#

Solution: Explicitly execute rm -rf .debug/ after each plugin cycle; move log output directory from.claude/ (configuration directory) to.debug/ (temporary project directory) and include a gitignore note

Key Insight: Git restore only affects tracked files — automated cleanup workflows must combine Git commands with explicit file system operations to achieve thorough cleanup

Lessons Learned#

Key Lessons#

  • Multi-stage planning process (ccplan) performed well in dealing with complex integrations with conflicting issues — adversarial review phase identified key git stash conflicts that could lead to data loss
  • Cascading feedback loops require multi-layer intervention: fixing window size issues (16ms→100ms) alone wasn’t enough; also dealing with data layer (load equality) and computation layer (timeout)
  • Metric-driven debugging requires careful lifecycle management: each hypothesis needs independent instrument→run→analyze→cleanup cycles to avoid cross-contamination and legacy debugging code
  • Incomplete module refactoring leads to hidden dependency hells: commands.rs still imports crate::parser, while new commands/calendar.rs imports crate::usage::parser — these two paths cannot coexist
  • Dynamic runtime detection adds failure modes when static configuration is sufficient: tray windows always use position_near_tray, so the bottom anchor should be hardcoded
  • Cross-platform window positioning in Tauri: plugin abstraction sacrificed accuracy for portability; perfect pixel positioning (especially system tray alignment) requires native APIs for specific platforms

Practical Experience#

  • Comprehensive test suites can catch refactoring issues: 218 Rust tests and 191 front-end tests verified that fixes wouldn’t disrupt existing behavior
  • Automated cleanup workflows must distinguish between tracked and untracked files: git restore handles tracked files; untracked files (logs, temporary directories) require explicit rm/rmdir operations
