Problems & Solutions#

Critical Issues#

1. cchypothesis lacked runtime debugging capability — all hypotheses verified via static code analysis only, missing dynamic state/timing/data flow issues#

Solution: Implemented Phase 3 intelligent triage: static hypotheses remain parallel read-only; runtime hypotheses use sequential instrumented probing with [DEBUG Hx] tagged logs, git safety checkpoints, and automatic cleanup via git restore

Key Insight: Dual-track investigation preserves speed (parallel static) while adding depth (instrumented probing) only when needed — avoids slowing down all debugging sessions

2. Dashboard persistent 1-2 Hz vertical jitter caused by three-layer positive feedback loop: data refresh → UI re-render → window resize → data refresh#

Solution: Multi-layer breaking: (1) Window: 16ms→100ms settle delay + 500ms/3ops throttle (2) Data: shallowPayloadEqual to skip redundant .set() (3) Parser: 2-minute grace period for is_active boundary to prevent five_hour_cost oscillation

Key Insight: Self-oscillation bugs require simultaneous intervention at all feedback layers; fixing one layer alone only weakens but doesn’t eliminate the cycle

3. Git stash stack collision: ccplan proposal added Phase 3 stash on top of existing Phase 4 stash, causing pop-order confusion and potentially burying user’s uncommitted work#

Solution: Eliminated Phase 4’s git stash in favor of git restore (non-destructive cleanup); only Phase 3 uses stash now (single-stash model)

Key Insight: Multiple stash levels in automated workflows create brittle state management — prefer stateless cleanup (git restore) over stateful undo (stash) when possible

4. TokenMonitor window appeared in screen center instead of near system tray; resizing caused window to grow downward into taskbar instead of maintaining bottom edge#

Solution: Implemented Win32 API direct tray positioning (FindWindowW → TrayNotifyWnd); switched frontend resize from webviewWindow.setSize() to backend set_window_size_and_align IPC for atomic size+position updates

Key Insight: Tauri’s cross-platform positioner plugin abstracts away platform differences but loses accuracy; Windows system tray positioning requires platform-specific Win32 API for pixel-perfect alignment

5. 42 Rust compile errors: commands.rs contained broken imports to deleted modules (change_stats, integrations, parser) and duplicated code from new commands/ submodules#

Solution: Rewrote commands.rs as thin 80-line module root declaring 6 pub mods, retained only AppState/UsageDebugReport and 4 shared helpers, fixed visibility and type issues

Key Insight: Incomplete refactoring (hybrid old/new code) is worse than monolithic code; must complete the transition or revert

6. Window anchor detection unreliable: visible:false initial mount, race condition between position_near_tray and GetWindowRect, continuous resize causing anchor flip#

Solution: Removed dynamic detection entirely; tray popup windows are always positioned at work.bottom by position_near_tray, so hardcode bottom-anchor behavior

Key Insight: Static configuration beats dynamic detection for known window types; runtime detection adds complexity and failure modes without benefit

General Issues#

7. Critic agent identified git restore . doesn’t clean untracked files (e.g., .debug/hypothesis.log created during instrumentation)#

Solution: Added explicit rm -rf .debug/ after each instrumentation cycle; changed log output directory from .claude/ (config dir) to .debug/ (temporary project dir) with gitignore guidance

Key Insight: git restore only affects tracked files — automated cleanup workflows must combine git commands with explicit filesystem operations for complete hygiene

Learnings#

Key Learnings#

  • Multi-phase planning workflows (ccplan) excel at complex integrations with conflicting concerns — adversarial review phase caught critical git stash collision that would have caused data loss
  • Cascading feedback loops require simultaneous multi-layer intervention: fixing window resize alone (16ms→100ms) wasn’t enough until data layer (payload equality) and computation layer (grace period) were also addressed
  • Instrumented debugging requires careful lifecycle management: each hypothesis needs isolated instrument→run→analyze→cleanup cycle to avoid cross-contamination and leftover debug code
  • Incomplete module refactoring creates hidden dependency hell: commands.rs still importing crate::parser while new commands/calendar.rs imports crate::usage::parser — both paths can’t coexist
  • Dynamic runtime detection adds failure modes without benefit when static configuration suffices: tray windows always use position_near_tray, so bottom-anchor should be hardcoded
  • Cross-platform window positioning in Tauri: plugin abstractions sacrifice accuracy for portability; pixel-perfect positioning (especially system tray alignment) requires platform-specific native APIs

Practical Learnings#

  • Comprehensive test suites catch refactoring issues: 218 Rust + 191 frontend tests verified fixes didn’t break existing behavior
  • Automated cleanup workflows must distinguish tracked vs untracked files: git restore handles tracked files; untracked files (logs, temp dirs) need explicit rm/rmdir
