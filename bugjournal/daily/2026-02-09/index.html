Problems and Solutions#

Major Issues#

1. Local copy of create_clustering_visualization in run_benchmark.py completely blocks the同名 function in utils/visualization.py.#

Solution: Moved the local copy to line 1644 of run_benchmark.py, applying the same metric parameters and fig/axes layout adjustments. Key Insight: AI only modified theutils module without noticing the local copy, resulting in invalid verification runs. The problem was identified using Grep search.

2. scanpy’s sc.pl.spatial(save=filename) cannot display chart objects, so text cannot be added to the charts.#

Solution: Reorganized the code to use a clear fig, axes = plt.subplots(1, 2) layout, specifying ax parameters when calling sc.pl.spatial(), and then using fig.text() and fig.savefig() for rendering. Key Insight: scanpy’s save= parameter is a convenient API, but it sacrifices control over chart objects. It’s necessary to comment out figure.show() and use bbox_inches=‘tight’ to ensure text boxes aren’t cropped.

General Issues#

3. Parallelizing 3 GPU training processes (PCA/GCN/E2E + STAIG_FUSION) led to GPUOOM errors, causing process termination.#

Solution: Stopped all parallel tasks and switched to sequential execution for each configuration.

4. AI misjudged the best configuration and tried to run uni_staig_fusion again (without a gene encoder).#

Solution: The user explicitly instructed to use the pca_uni_staig_fusion folder. After checking, AI confirmed using PCA+UNI+STAIG_FUSION+-skip_cached. Key Insight: It’s necessary to review the existing directory structure before viewing historical experiment results.

5. Slices like 151510/151672 reported ValueError: staig_fusion requires spatial coordinates.#

Solution: This was a data issue (missing spatial coordinate files in slices). Skipped these slices and continued processing others. Key Insight: Some DLPFC slices had incomplete data, while staig_fusion relied heavily on spatial coordinates. These slices need to be checked for completeness.

Today’s Summary#

Key Takeaways - When modifying functions in large scripts, use Grep to find local copies of the same function. Ensure no local copies block module-level implementations.#

  • Scanpy’s spatial visualization can be integrated with matplotlib using a clear fig, axes layout. It’s recommended to use fig.savefig(bbox_inches=‘tight’) instead of scanpy’s save parameter.

Practical Suggestions#

  • In single-GPU high-performance computing environments, STAIG_FUSION requires a lot of memory for training. Parallel processing is not feasible; batch experiments should use && connections or separate slurm jobs.
  • Among the 11 DLPFC dataset slices, some slices like 151510/151672 lack spatial coordinates or have full-resolution images. Staig_fusion and UNI-based models rely heavily on these data.
