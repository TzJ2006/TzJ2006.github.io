Problems and Solutions#

Key Issues#

1. A local copy of create_clustering_visualization existed in run_benchmark.py, completely obscuring the同名 function in utils/visualization.py.#

Solution: Located the local copy at line 1644 of run_benchmark.py and applied the same metrics parameters and fig/axes reorganization. Key insight: AI only modified theutils module without noticing the local copy, resulting in ineffective verification runs. It was discovered through Grep search.

2. scanpy’s sc.pl.spatial(save=filename) does not expose a figure object, making it impossible to add text to charts.#

Solution: Reorganized to use an explicit fig, axes = plt.subplots(1, 2) layout, calling sc.pl.spatial() with ax parameter, and then using fig.text() and fig.savefig() for rendering. Key insight: scanpy’s save= parameter is a convenient API that sacrifices control over the chart object. It’s necessary to comment out figure.show() and use bbox_inches=‘tight’ to ensure text boxes aren’t cropped.

General Issues#

3. Parallelly launching 3 GPU training processes (PCA/GCN/E2E + STAIG_FUSION) led to GPU OOM errors, causing processes to be killed.#

Solution: Stopped all parallel tasks and switched to sequential execution for each configuration.

4. AI misjudged the optimal configuration and tried to re-run uni_staig_fusion (without a gene encoder).#

Solution: The user explicitly instructed to use the pca_uni_staig_fusion folder. After checking, AI confirmed using PCA+UNI+STAIG_FUSION+-skip_cached. Key insight: It’s necessary to review the existing directory structure before examining historical experiment results.

5. Slices like 151510/151672 reported ValueError: staig_fusion requires spatial coordinates.#

Solution: This was a data issue (missing spatial coordinate files in slices). Skipped these slices and continued with others. Key insight: Some DLPFC slices have incomplete data, and staig_fusion relies heavily on spatial coordinates. It’s necessary to check the completeness of these slices later.

Today’s Takeaways#

Key Takeaways#

  • Before modifying functions in large scripts, use Grep to search for local copies of the same function. Ensure no local copy obscures module-level implementations.
  • Scanpy’s spatial visualization can be integrated with matplotlib using an explicit fig, axes layout. It’s key to use fig.savefig(bbox_inches=‘tight’) instead of scanpy’s save parameter.

Practical Takeaways#

  • In a HPC single-GPU environment, STAIG_FUSION requires high memory for training. Parallel processing is not feasible; batch experiments should use && concatenation or separate slurm jobs.
  • Among the 11 DLPFC dataset slices, some slices like 151510/151672 lack spatial coordinates or have full-resolution images. Staig_fusion and UNI-based models rely heavily on these data.
