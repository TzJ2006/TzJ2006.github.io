Issues and Solutions#

Key Issues#

Despite modifying utils/visualization.py, metric overlay still did not appear on the charts.#

Solution: Discovered a duplicate create_clustering_visualization function in line 1644 of run_benchmark.py, covering the utils version. Both versions were modified accordingly.

Key Insight: In large codebases, it’s necessary to grep all files for function definitions to determine whether individual modifications are effective.

There is a risk of overwriting existing good results in the pca_uni_staig_fusion directory.#

Solution: Use --skip_cached instead of --override to preserve cached embeddings from completed regions.

Key Insight: Always check existing results before running experiments again. Use the cache flag to distinguish between incremental completion and full reruns.

General Issues#

When running 3 STAIG fusion experiments simultaneously, OOM termination occurred due to parallel GPU tasks.#

Solution: Switch to sequential execution by using && commands within a single Bash call.

Key Insight: In deep learning benchmarks involving large amounts of spatial transcriptomics data, exclusive GPU access is required. Parallelization should only be performed on CPU-intensive tasks.

Regions 151510, 151672, and 151674 failed due to missing spatial coordinates or OpenCV errors.#

Solution: The script continued processing the remaining regions. These were identified as data integrity issues that required separate investigation.

Key Insight: A robust benchmark process should validate data prerequisites (spatial coordinates, image files) and gracefully fail when duplicates are found, providing clear error messages.

Lessons Learned#

Key Lessons#

  • In large codebases with modular utility tools, always use grep to find all occurrences of function names. Local copies can overwrite imports, causing silent modifications to fail.
  • In deep learning benchmarks involving spatial transcriptomics data (more than 4000 points, 1024-dimensional embeddings, STAIG training), exclusive GPU access is required. Parallelization should only be performed on CPU-intensive tasks like RM-Ideal WWL graph kernels.
  • Experiment management requires clear caching strategies: use --override for full reruns, --skip_cached for incremental completion, and avoid using any flags to fail on duplicate items.

Practical Experience#

  • Visualization refactoring pattern: Explicitly creating fig/axes makes it possible to add custom annotations (metric text overlay). However, Scanspy’s save= parameter doesn’t support this.
