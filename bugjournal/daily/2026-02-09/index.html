Problems & Solutions#

Critical Issues#

1. Metrics overlay not appearing on plots despite modifying utils/visualization.py#

Solution: Discovered duplicate create_clustering_visualization() function at line 1644 in run_benchmark.py that shadowed the utils version - modified both

Key Insight: In large codebases, always grep for function definitions across all files before assuming a single modification will take effect

2. Risk of overwriting existing good results from pca_uni_staig_fusion directory#

Solution: Used –skip_cached flag instead of –override to preserve cached embeddings for completed sections

Key Insight: Always check existing results before re-running experiments - use caching flags to distinguish between incremental completion and full re-run

General Issues#

3. Parallel GPU tasks caused OOM kill (exit code 137) when running 3 STAIG fusion experiments simultaneously#

Solution: Switched to sequential execution by chaining commands with && in a single Bash call

Key Insight: Deep learning benchmarks with large spatial transcriptomics data require exclusive GPU access - parallelize across CPU-bound tasks only

4. Sections 151510, 151672, 151674 fail with missing spatial coordinates or OpenCV errors#

Solution: Script continues with remaining sections - identified these as data integrity issues requiring separate investigation

Key Insight: Robust benchmark pipelines should validate data prerequisites (spatial coords, image files) and fail gracefully with clear error messages

AI Limitations#

Critical Limitations#

  • Failed to detect duplicate function definitions - modified utils/visualization.py without checking if run_benchmark.py had its own local copy that would shadow it
  • Used –override flag by default without checking if valuable existing results would be lost

General Limitations#

  • Attempted to run 3 GPU-intensive deep learning tasks in parallel without considering GPU memory constraints, leading to OOM kill

Learnings#

Key Learnings#

  • In large codebases with modular utilities, always grep for all occurrences of a function name before modifying - local copies can shadow imports and cause silent modification failures
  • Deep learning benchmarks on spatial transcriptomics data (4000+ spots, 1024-dim embeddings, STAIG training) require exclusive GPU access - parallelize only CPU-bound tasks like RM-Ideal WWL graph kernels
  • Experiment management requires explicit caching strategy: –override for full re-run, –skip_cached for incremental completion, no flags to fail on duplicates

Practical Learnings#

  • Visualization refactoring pattern: explicit fig/axes creation enables custom annotations (metrics text overlay) that scanpy’s save= parameter doesn’t support
