Problems & Solutions#

Critical Issues#

1. LeRobot 0.4.0 save_episode error: ‘only 0-dimensional arrays can be converted to Python scalars’ on observation.dones field (shape=(1,) np.array)#

Solution: Root cause: LeRobot’s get_hf_features_from_features() maps shape=(1,) to scalar Value, but validate_frame() requires ndarray. Inserted loop before save_episode() to convert all dones arrays to Python int via .item()

Key Insight: When framework validation and encoding logic conflict, optimal insertion point is the buffer between them (episode_buffer), not modifying feature definitions or bypassing validation

2. Screenpipe 0.2.13 (Homebrew) startup failed with ‘Protobuf parsing failed’ error when loading wespeaker_en_voxceleb_CAM++.onnx model#

Solution: Identified Homebrew version as deprecated (does not build). Uninstalled via brew, deleted corrupted model cache (~/Library/Caches/screenpipe/models/), switched to npx screenpipe@latest or official desktop app, updated all documentation

Key Insight: Homebrew formula deprecated status is critical signal. When encountering low-level dependency issues, check upstream maintenance status first rather than debugging abandoned versions

3. Battery training loss immediately became NaN from first iteration despite data appearing normal#

Solution: Systematically traced pipeline: found cycle column had 4 NaN values from corrupt row (battery 31, cycle 833). StandardScaler computed NaN mean/std, causing (data - NaN) / NaN = NaN for all scaled values. Removed single corrupt row

Key Insight: NaN propagation through normalization: Single NaN in training data corrupts StandardScaler statistics (mean/std), causing all subsequent scaled values to become NaN. Always validate data cleanliness before normalization operations

4. TokenMonitor costs don’t match Claude Code /cost output#

Solution: Discovered TokenMonitor used 2.0x multiplier for 1h cache tokens while Claude Code uses unified 1.25x rate. Changed cache_write_1h pricing in pricing.rs to match Claude Code exactly

Key Insight: Claude Code’s /cost deliberately simplifies pricing (unified rate) despite API providing granular breakdown. UI simplicity can justify data aggregation — not every field needs exposure

5. CircuitBreaker HALF_OPEN race condition: state check outside lock allowed multiple probes to enter simultaneously#

Solution: Moved state check inside ‘async with self._lock’ block to ensure atomic state check + transition

Key Insight: Concurrent state machines must perform state check and transition atomically within same lock. Any lock-external check introduces race window

6. MIHD paper positioning unclear — initially framed as ‘fusion strategy benchmark’ with vague core contribution#

Solution: Repositioned as ‘frozen FM + self-supervised fusion paradigm evaluation’ answering ‘can this replace fine-tuned methods’. Changed narrative from method proposal to systematic boundary exploration

Key Insight: Benchmark papers evaluate paradigms, not methods. Core value is systematic boundary exploration, not SOTA claims. Precise problem definition drives correct narrative framing

7. LiteLLM pricing database missing major Chinese AI models (GLM, Qwen, DeepSeek) — systematic gap in Chinese AI ecosystem#

Solution: Integrated OpenRouter API (92 Chinese model entries) with concurrent fetch using LiteLLM-priority merge: OpenRouter provides base coverage, LiteLLM overwrites where available. Added 7-day TTL auto-refresh

Key Insight: Global AI pricing requires multi-source strategies with geographic diversity. Western databases systematically lack Chinese models; Chinese sources lack Western models. Redundancy is essential not optional

8. Initial uniform sampling used fixed 0-20 second range, producing invalid IR values (16, 15, 14…) instead of expected 0-0.024 range#

Solution: Changed strategy: interpolate within each cycle’s actual chargetime range (typically 10-13 seconds) rather than forcing into global fixed range. Avoided extrapolation beyond observed data points

Key Insight: Interpolation vs extrapolation: Forcing data into global range beyond observed bounds causes extrapolation with invalid values. Use per-sample adaptive ranges to stay within interpolation regime

9. ‘Zero-shot’ terminology easily misunderstood — STAIG fusion trains 550 epochs, reviewers would challenge if truly ‘zero-shot’#

Solution: Switched to ’label-free / annotation-free’ emphasizing no manual annotation needed. Clarified ‘zero-shot’ refers to encoder level (frozen FM), while fusion is self-supervised using spatial coordinates as free signal

Key Insight: Precise terminology prevents reviewer confusion. Layered definition (frozen encoder + self-supervised fusion) avoids ambiguity and highlights practical value (no annotation cost)

10. tianhe cluster GitHub access failed: ‘Failed to connect to 172.16.31.200 port 3138: No route to host’#

Solution: Diagnosed outdated proxy configuration (172.16.31.200). User provided ‘source /app/bin/proxy.sh’ command. Updated git config http/https.proxy to new proxy (99.72.0.200:3138), npx openskills install succeeded

Key Insight: HPC cluster network proxies change frequently. When network errors occur, verify proxy effectiveness (env | grep proxy, git config –list) rather than assuming configuration correctness

Learnings#

Key Learnings#

  • LeRobot 0.4.0 shape=(1,) feature handling design contradiction: validate_frame requires ndarray but get_hf_features_from_features maps to scalar Value. Solution: insert type conversion in episode_buffer layer between validation and encoding
  • StandardScaler NaN propagation mechanism: Single corrupt row with NaN in any feature column causes StandardScaler.fit() to compute NaN statistics (mean/std), propagating to all scaled data through (x - NaN) / NaN = NaN. Always validate data for NaN/Inf before normalization operations
  • Benchmark papers evaluate paradigms, not methods — Nature Methods Analysis papers don’t need SOTA, but systematic answers to ‘how far can this methodology go’. Narrative shifts from ‘our method is best’ to ‘what are the boundaries and bottlenecks of this paradigm’
  • Homebrew formula deprecated status is critical signal — when encountering low-level dependency issues, check upstream maintenance status first rather than debugging abandoned versions
  • Graph contrastive learning evolution in spatial transcriptomics: STAIG (node-level) → QueST (subgraph-level) → Loki (cross-modal), showing field progression from ‘one method per problem’ to ‘one foundation model for multiple tasks’
  • Code review layered remediation protocol: CRITICAL (security/races) → HIGH (deprecated APIs/private access) → MEDIUM (types/style) → LOW (linter), with test validation after each layer before proceeding to avoid cascading failures
  • Asyncio state machine correctness invariant: state check + transition must be atomic within same lock; any lock-external check introduces race window
  • Global AI pricing requires multi-source strategies with geographic diversity — Western databases (LiteLLM) systematically lack Chinese models; Chinese sources lack Western models. Redundancy essential not optional
  • Rust performance patterns: avoid unnecessary clone(), use borrowing structs for serialization, prefer line counting over full deserialization when only metadata needed
  • Terminology selection matters: label-free vs zero-shot — ‘zero-shot’ easily misunderstood as ’no training at all’, ’label-free / annotation-free’ accurately conveys core advantage of ’no manual annotation’. Layered definition (encoder zero-shot + fusion self-supervised) avoids ambiguity
  • Parallel agent strategy dramatically accelerates large-scale documentation — split 2246-line document into 3 independent parts (Paper Plan, Repo Structure, Module Specs), each generated by separate agent then assembled. Also accelerates research: 4 agents completed in 14s vs 45-60s sequential
  • Self-supervised learning’s ‘free signal’ framing — spatial transcriptomics coordinates are structured information inherent to Visium chips. Viewing spatial graphs as ‘free supervision signal’ clarifies label-free value and strengthens practical utility (no additional annotation cost)
  • Interpolation vs extrapolation in temporal resampling: Linear interpolation should stay within observed data bounds. Forcing data into fixed global range causes extrapolation producing invalid values. Use per-sample adaptive ranges to avoid extrapolation artifacts
  • Multi-stage data processing pipeline correctness requires end-to-end verification — special values (like -1) handling must be consistent across all stages (preprocessing → Dataset → Loss → evaluation)
  • Claude Code’s /cost deliberately simplifies pricing presentation (unified 1.25x cache rate) despite API providing granular 5m/1h breakdown. UI simplicity can justify data aggregation — not every field needs exposure
  • HPC cluster network proxies change frequently — establish standard check workflow: env | grep proxy + git config –list + test connection, rather than assuming configuration correctness
  • Pydantic v2 migration pattern: dict → model_dump(), but test mocks must synchronously provide same interface via helper functions like _make_settings_mock()
  • JSONL session logs don’t contain all billable attributes — Fast mode tier invisible in logs despite affecting actual costs, limiting what cost-tracking tools can reconstruct from historical data

Practical Learnings#

  • Weekly pricing refresh (7-day TTL) with background checking strikes optimal balance for LLM cost tracking: frequent enough to catch new models and rate changes, infrequent enough to avoid API quota concerns
