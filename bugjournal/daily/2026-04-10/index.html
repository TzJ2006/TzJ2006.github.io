Learning Outcomes#

Key Learnings#

  • The contradiction between shape=(1,_) in LeRobot 0.4.0 and feature processing design: validate_frame requires ndarray, while get_hf_features_from_features maps to scalar values. The solution is to insert a type conversion layer between validation and encoding.
  • NaN propagation mechanism of StandardScaler: Any row containing NaN in any feature column will cause StandardScaler.fit() to calculate NaN statistical information (mean/stddev), and affect all scaled data by multiplying it by (x - NaN) / NaN = NaN. It’s necessary to verify the NaN/Inf situation before normalization operations.
  • The benchmark paper evaluated the paradigm rather than the method - Papers in Nature Methods Analysis do not require SOTA, but need systematic answers to the question “How far can this method go.” The narrative shifted from “Our method is the best” to “What are the boundaries and bottlenecks of this paradigm?”
  • The outdated status of the Homebrew formula is an important signal - When encountering low-level dependency issues, it’s better to check the upstream maintenance status rather than debugging deprecated versions.
  • Development in graph contrast learning in spatial transcriptomics: STAIG (node-level) → QueST (subgraph-level) → Loki (cross-modal), indicating a shift from “one method for each problem” to “one base model for multiple tasks.”
  • A layered repair scheme for code review: CRITICAL (security/competition) → HIGH (outdated API/private access) → MEDIUM (type/styling) → LOW (lint tools). Each level must be tested and verified to avoid cascading failures.
  • The correctness invariant of the Asyncio state machine: State checks + transitions must be atomic within the same lock; any check involving an external lock introduces a competitive window.
  • Global AI pricing requires multi-source strategies with geographical diversity - Western databases (LiteLLM) systematically lack Chinese models; Chinese sources lack Western models. Redundancy is necessary, not optional.
  • Rust’s performance patterns: Avoid unnecessary clone(); use borrowing structures for serialization; when only metadata is needed, choose line counting over full deserialization.
  • Term selection is important: No-label vs zero-sample - “Zero-sample” can be misunderstood as “no training at all,” while no-label/no-annotation accurately conveys the core advantage of “no need for manual annotation.” Layered definitions (zero-sample encoder + fusion self-supervised) avoid ambiguity.
  • Parallel proxy strategies significantly accelerated the generation of large-scale documents - Divided the 2246-line document into 3 independent parts (paper plan, repository structure, module specifications), each generated by a separate proxy, and then assembled. This also accelerated the research: 4 proxies completed in 14 seconds, while sequential processing took 45-60 seconds.
  • The “free signal” framework for self-supervised learning - Spatial transcriptomics coordinates are structured information belonging to Visium chips. Treating spatial graphs as “free supervision signals” clarifies the value of no-label and enhances practicality (without additional annotation costs).
  • Interpolation and extrapolation in time resampling: Linear interpolation should remain within the observed data range. Forcing data into a fixed global range leads to extrapolations that produce invalid values. Using sample-adaptive ranges can avoid extrapolation artifacts.
  • The correctness of multi-stage data processing procedures requires end-to-end verification - Handling special values like -1 must be consistent across all stages (preprocessing → dataset → loss → evaluation).
  • Claude Code’s /cost intentionally simplifies pricing display (uniform 1.25x cache rate), although the API provides detailed 5m/1h breakdowns. The simplicity of the UI can justify data aggregation - not every field needs to be made public.
  • HPC cluster network proxies often change - Establish a standard check workflow: env | grep proxy + git config –list + test connection, rather than assuming the correctness of configuration.
  • Pydantic v2 migration pattern: dict → model_dump(), but test simulations must provide the same interface through auxiliary functions like make_settingsMock().
  • JSONL session logs do not contain all billed attributes - The quick mode level is invisible in the log, although it affects actual costs, which limits which cost tracking tools can reconstruct data from historical data.

Practical Learning Outcomes#

  • Weekly updates to pricing (7-day time limit), combined with background checks, are the best balance for tracking LLM costs: frequent enough to capture new models and rate changes, but not so frequent as to cause API quota issues.
