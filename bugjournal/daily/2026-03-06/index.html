Problems & Solutions#

Critical Issues#

1. CalendarPro bot fails to understand follow-up messages like ‘你安排在几点了呀’ (What time did you schedule?)#

Solution: Identified three-layer issue: (1) general_keywords pre-check executes before AI sees conversation history, (2) pre-check uses overly broad keywords like ‘几点了’, (3) AI provider calls don’t receive history parameter. Designed fix to remove pre-check bypass and pass full conversation history to AI

Key Insight: Context loss is typically a multi-layer problem: data not collected → data not passed → data not used. Must trace the complete call chain from storage through provider interface to prompt construction

2. SAPIEN ‘Render Error’ in headless Docker container - RuntimeError: failed to find a rendering device#

Solution: Discovered missing NVIDIA Vulkan/OpenGL rendering libraries (libGLX_nvidia.so, etc). Container had CUDA compute libraries but not rendering stack. Downloaded matching NVIDIA driver version 535.104.12 to match kernel driver, extracted libraries to user directory, configured VK_ICD_FILENAMES and LD_LIBRARY_PATH environment variables in eval.sh.

Key Insight: Docker containers with NVIDIA GPUs need both CUDA runtime AND Vulkan/OpenGL libraries for graphics-based simulation. Driver version must exactly match between kernel module and user-space libraries or Vulkan ICD fails to load.

3. Phoenix training checkpoints exist in zhaoganlong’s directory, unclear whether to reuse or retrain#

Solution: User pointed out should check existing checkpoints and symlink when available, only train missing ones on an53 free GPUs

Key Insight: Before designing training plan, inventory existing resources (checkpoints, datasets, configs) to avoid wasting compute. Establish checkpoint reuse strategy first

4. User assumed normalized data came from STHD pipeline, but data characteristics didn’t match pipeline code#

Solution: Traced code to discover STHD doesn’t perform normalization - it uses raw counts directly. The normalized data user saw was from external scanpy preprocessing (normalize_total + log1p)

Key Insight: Need to distinguish between pipeline-internal processing and external preprocessing. Code inspection is more reliable than assumptions based on data appearance

General Issues#

5. OpenPI config missing warmup training configurations for 3 out of 9 MimicGen tasks#

Solution: Added finetune and inference configs for coffee_task_D1, stack_three_task_D1, and three_piece_assembly_task_D1 using add_finetune_config and add_inference_config helper functions

Key Insight: Use config generation functions to maintain consistency across task configurations rather than manual duplication

6. FrozenInstanceError: cannot assign to field ‘asset_id’ in policy_config.py#

Solution: DataConfig was defined with @dataclasses.dataclass(frozen=True), making it immutable. Replaced direct assignment (data_config.asset_id = value) with dataclasses.replace(data_config, asset_id=value) which creates a new instance.

Key Insight: Frozen dataclasses enforce immutability at the language level - modifications require explicit use of dataclasses.replace() to create new instances rather than mutating in-place.

7. FileNotFoundError with duplicated path: assets/norm_stats.json/norm_stats.json#

Solution: When robotwin_repo_id is provided, asset_id was incorrectly set to ’norm_stats.json’ (filename instead of directory). Modified code to load directly from assets/ directory using normalize.load() without appending asset_id subdirectory.

Key Insight: Path construction bugs often arise from mixing directory identifiers with filenames. Explicit handling of special cases (local files vs remote repositories) prevents path duplication.

8. MulticoreTSNE build failure due to CMake version incompatibility (system 3.26.4 vs required <=3.18)#

Solution: Identified that MulticoreTSNE is only used for optional TSNE visualization, not core training/eval. Suggested commenting out dependency in requirements.txt to proceed with installation (user rejected this approach).

Key Insight: Distinguish between critical dependencies and optional visualization tools. In research code, it’s often acceptable to skip non-essential dependencies that cause build issues.

Learnings#

Key Learnings#

  • Context loss requires tracing the complete call chain: data collection (conversation_history.json storage) → data passing (provider call signatures) → data usage (prompt construction). Missing any layer breaks context
  • Before designing training plans, inventory existing resources (checkpoints, datasets, configs) and establish reuse strategy to avoid wasting compute resources. Ask ‘what exists?’ before ‘what to build?’
  • Docker containers running NVIDIA GPU workloads need two distinct library sets: CUDA runtime (compute) AND Vulkan/OpenGL (rendering). Many GPU-enabled containers ship with only CUDA. For simulation frameworks like SAPIEN that use graphics APIs, both stacks must be present with exact version matching to kernel driver.
  • In spatial transcriptomics data processing, adata.X can be either normalized values (scanpy pipeline) or raw counts (STHD pipeline) - must check code rather than assume from data type
  • Parallel multi-agent exploration accelerates understanding of complex codebases, but requires careful planning to avoid redundant work. Use agents for independent dimensions (history management, message flow, provider interface) not overlapping concerns
  • Two-phase data collection (seed discovery + deterministic replay) is a robust pattern for robotics simulation: first phase filters for successful task completions and saves trajectory plans, second phase replays with saved plans to collect clean training data. This separates stochasticity from data quality.
  • Symlink-based dependency sharing can dramatically reduce disk usage for large research projects (78GB -> 37MB) while maintaining independent project structure. Critical to symlink data directories rather than copy, and use relative paths for portability.
  • Documentation optimization core principle: remove content discoverable through other means (like ‘make help’ for commands), preserve ‘big picture’ insights requiring cross-file understanding
  • Vulkan ICD (Installable Client Driver) discovery requires both the nvidia_icd.json manifest AND the actual driver libraries to be present and version-matched. VK_ICD_FILENAMES environment variable overrides system search paths for containerized environments.

Practical Learnings#

  • Python frozen dataclasses provide compile-time immutability guarantees but require explicit use of dataclasses.replace() for modifications. Direct assignment raises FrozenInstanceError. This pattern enforces functional programming style for config objects.
