Problems and Solutions#

Key Issues#

Issue Solution
1. SAPIEN reported a Render Error in the headless Docker container. The bare except option hid the actual error, and the mismatched driver version (535.104.12) with the apt repository version (535.288.01) caused an invalid installation. Four steps to fix: ① Modify bare except to expose the real traceback; ② Confirm it was a missing NVIDIA Vulkan ICD (libGLX_nvidia.so); ③ Extract all libnvidia-*.so user-mode GL libraries from the official NVIDIA package 535.104.12 .run and place them in the user directory; ④ Fix the rendering issue by injecting environment variables like VK_ICD_FILENAMES, __EGL_VENDOR_LIBRARY_FILENAMES, and LD_LIBRARY_PATH.
2. Users pointed to incorrect h5ad files (raw counts with values/integer features not matching descriptions). Direct toarray() on large sparse matrices caused OOM (exit code 137). Identify file path mismatches by comparing X value characteristics (integers and ranges). Use sc.read_h5ad(backed=‘r’) in low-memory mode for local sampling. Once the user confirmed, locate the correct file all_region/adata_8um.h5ad.
3. CalendarPro Discord bot lost context when following messages (“What time are you arranged?” and “OK”). It was misclassified as GENERAL or triggered a new conversation. Three fixes: ① Trim general_keywords to remove time-related terms and confirmation words; ② Let llm_classify receive the last three conversations; ③ Reset GENERAL semantic routing and remove confirmation word classes from utterances.
4. OpenPI config.py lacked three FLARE warmup training configurations (coffee_task_D1, stack_three_task_D1, three_piece_assembly_task_D1). Added corresponding add_finetune_config() and add_inference_config() calls to the CONFIGS list, adding a total of six new configurations.

Issue Solution
5. Role Thinking

AI AI uses systematic methods like exposing real errors through bare except, checking system status, and comparing value characteristics to narrow down the problem scope. In the Render Error case, it failed because it didn’t check the driver version first.

AI Limitations#

Important Limitations#

  • Checked critical version information only after performing operations in the system environment (e.g., installing NVIDIA rendering libraries without checking the version first). This led to additional diagnostic steps after installing incorrect versions. Similarly, for HPC big data files, memory safety mode wasn’t used by default, and direct toarray() caused OOM.

General Limitations#

  • Lack of session state awareness: Unable to recognize that the same task had been completed in previous sessions (e.g., 17:37, 20:00, 20:02). Task output returned “No task found” when querying for background tasks, indicating reliability issues with tools.
  • Complex dependency scenarios require multiple rounds of iteration to uncover all missing items. It took several attempts to find all missing.so files for Vulkan dependencies, ultimately resolving them by copying all libnvidia-*.so files. This reflects AI’s limitations in predicting dependencies.

Today’s Takeaways#

Key Takeaways- The STHD pipeline uses raw UMI counts throughout the process. The model directly treats X as Poisson observations. During VisiumHD binning from 2μm to 8μm, counts are aggregated using sum (total UMI is conserved). For HVG selection, flavor=‘seurat’ is used on log1p data. It’s important to distinguish between the internal processing links of STHD and those of the standard scanpy preprocessing links.#

  • When implementing Phoenix+FLARE training for 9 MimicGen tasks, it’s necessary to differentiate between two paradigms: Phoenix with a single model for multiple tasks vs FLARE with independent Pi0.5 LoRA per task (additional warmup data generation is required to generate R45T03 parameters and a 5-stage pipeline). Large HPC datasets should be shared through symbolic links rather than copied.
  • Fixing the complete solution for SAPIEN/Vulkan rendering in the headless Docker environment: Extract the user-mode GL library from the NVIDIA official.run driver package (no need to install kernel modules) to the user directory. Inject it through the VK_ICD_FILENAMES and LD_LIBRARY_PATH environment variables. No root privileges are required. The driver version must be strictly consistent with the kernel module version in /proc/driver/nvidia/version.
  • Key points for designing Discord bot context awareness: Keyword pre-checking should not截断 all messages containing specific words statically. Verbs like “ok” and time-related phrases like “what time is it” should have clear contextual meanings in follow-up messages. LLM classification must receive the conversation history to correctly process follow-up messages. The training set for semantic routing utterances should not include general verbs.
  • RoboTwin’s two-stage data collection mode: In stage one, only lightweight simulations are used to find successful seeds (motion planning trajectories are saved). In stage two, deterministic seeds are used to replay and collect complete multimodal HDF5 training data. This separation ensures data quality (only successful trajectories are collected), supports breakpoint continuation of data collection, and both stages can be run independently.

Practical Achievements#

  • Python bare except is a high-risk debugging trap: In the rendering framework, critical information like RuntimeError may be hidden. During debugging, it’s better to change bare except to except Exception as e: traceback.print_exc(). This is often the first step in finding the root cause.
  • The cluster setproxy.sh script routes traffic to the internal proxy server (172.16.31.200:3138) for accessing external networks by setting http_proxy/https_proxy/git proxy. It must be executed with source to take effect in the current shell. Shared proxy bandwidth decreases as the number of concurrent users increases. It’s recommended to use it during off-peak hours for large file downloads.
