Problems and Solutions#

Key Issues#

1. MIHD staig_fusion’s ARI (0.21/0.4849) far below the original STAIG notebook’s (0.562), root cause unknown#

Solution: The system compared two code paths, identifying five root causes: ① Mixed coordinate scaling for full-resolution images (13332x13332) still multiplied by a hires scaling factor (0.15), leading to severe deviation of patch points from tissue areas (most critical); ② mclust silently downgrading to kmeans when HPC was unavailable; ③ Gene preprocessing order reversed (MIHD first HVG then normalize/log/scale vs original STAIG); ④ Pseudo-label cluster count 300 vs 80; ⑤ Hyperparameters not fully aligned and image transformations differences Key insight: The name staig_fusion inherently promises ’equivalent to STAIG’ semantics. Allowing significant differences in default behavior can lead to many hidden errors. Mixed coordinate scaling is the top priority issue to fix

2. Plotly bar chart rendering error: High scores correspond to short bars, visual logic inverted#

Solution: Locked the Plotly JS CDN version to match the Python installation version (3.3.1), changed Series to list, and set y-axis rangemode=‘tozero’ Key insight: Plotly 6.x uses binary-encoded array serialization, which causes data parsing errors when incompatible with plotly-latest CDN version. Version consistency must be strictly maintained

3. mclust clustering reports error ‘svd(data, nu=0): a dimension is zero’, and silently downgrading to kmeans when HPC environment is unavailable#

Solution: Added shape verification at the Python level (2D, no rows/columns, sample count >= cluster count). The root cause will be confirmed after the next run. Need to install rpy2 and R mclust packages to eliminate silent degradation issues Key insight: Error messages from the R side are not intuitive. Sufficient pre-verification should be added at the Python level. Silent degradation must include clear warnings rather than changing clustering methods without any prompts

General Issues#

4. Strict STAIG mode forces visual encoder to UNI, users unaware (thinking they’re using UNI2, tqdm not appearing)#

Solution: Added clear hints about encoder override in logs; UNI2 progress bars not displayed because UNI was actually run Key insight: Global override behavior is opaque to users. Clear hints must be added in logs; otherwise, users will spend time debugging non-existent problems

5. Low GPU utilization (both CPU and GPU usage were very low)#

Solution: Not fully resolved. Analysis indicated a single-threaded I/O wait bottleneck: patch extraction was the main bottleneck. Proposed solutions include adjusting batch size, DataLoader multi-processing, and patch cache pre-extraction Key insight: Low CPU and GPU usage together indicates program is waiting in single-threaded IO, not CPU being the bottleneck. Performance bottlenecks for visual feature extraction are usually found in data preprocessing rather than model forward passes

6. create_trend_chart AttributeError: Function was commented out but still called in generate_html#

Solution: Added a stub method that returns an empty chart and completely removed the calling code Key insight: Commenting out function definitions does not equal removing them. All calling parties must be removed simultaneously

Strategic Level#

Role Thought
Human Humans directly pointed out the problem (staig_fusion scores far below original STAIG), provided example.ipynb as a ground truth, and decided ‘staig_fusion’s semantics are equivalent to STAIG, so it should be overridden without retaining old behavior’. Continuously asked ‘why is the effect poor’ to drive deeper analysis. The additional observation that ‘CPU usage was also low’ directly ruled out the assumption that CPU was the bottleneck
AI AI systematically traced code (measured image size 13332x13332, calculated coordinate compression ratio, line-by-line comparison of two code bases), identified 5 quantitative differences after multiple tool calls. However, it required users to clearly decide on the direction to proceed

Difference analysis: Human decisions are more strategic (names inherently imply semantics, direct override allows significant differences in default behavior without causing many hidden errors). Mixed coordinate scaling is the top priority issue to fix

Difference analysis: Human intuition includes ‘confirm consent before uploading’ and ‘security isolation’. AI is responsible for translating intuition into executable technical architectures

Identification of global override logic and gene preprocessing order#

Role Thought

AI AI compared MIHD prepare_gene_features with original STAIG adata_processing.py, discovered that the order of normalize/log/scale and HVG selection was swapped. However, when implementing tqdm, it overlooked existing global override logic

Difference analysis: Humans are more aware of their own running environment and expected behavior, quickly identifying log anomalies. AI can systematically compare code details but easily overlook existing global logic, requiring users to observe to compensate

AI Limitations#

Important Limitations#

  • Inadequate thoroughness of multiple operations: When fixing mclust dimension errors, only shape protection checks were added without tracing the root cause upstream. When removing trend charts, only the calling was removed at first without handling function definitions simultaneously, requiring a second fix. Systematic comparison across code bases is inefficient, and gap analysis is gradually identified through multiple tool calls, failing to output a complete structured list at once

General Limitations#

  • Easy to overlook existing global override logic: When adding UNI2 tqdm, it was not realized that strict STAIG mode would force visual encoder override. Before implementing STAIG alignment modifications, no active checks were performed on numerous unsubmitted dirty files. Users must be explicitly informed to proceed
  • No pre-check of available modules in the HPC environment (e.g., scanpy), resulting in ModuleNotFoundError errors when verifying Python scripts. Only indirect verification methods could be used, increasing debugging iterations. Optimistic estimates of local tool reliability were made (Windows environment conda run occasionally failed)
  • Multiple options for GPU utilization low issues were provided, but no profile tools (e.g., py-spy, nvprof) were actively suggested to accurately locate the actual bottleneck. Instead, guesses were made based on code reading

Today’s Takeaways#

Core Takeaways#

  • There are five quantifiable key implementation differences between MIHD and the original STAIG (priority ranked): ① Mixed coordinate scaling for full-resolution images (most critical, coordinates incorrectly compressed from x:2579-11821 to x:386-1773, patch points severely deviating from tissue areas); ② mclust silently downgrading to kmeans when HPC was unavailable; ③ Gene preprocessing order reversed (HVG first vs normalize/log/scale first); ④ Pseudo-label cluster count 300 vs 80; ⑤ Hyperparameters and image transformations differences. The ARI gap (0.21→0.56) is mainly determined by ①②③
  • Correct hierarchical architecture for GitHub Pages static publishing and public data collection: Public users → relay (verification/rate-limiting/masking) → repository_dispatch → queue file → daily batch processing workflow (deduplication/cleaning/append CSV) → commit/push → automatic Pages deployment
  • Interface names must strictly match actual behavior: The name ‘staig_fusion’ inherently promises equivalent semantics to STAIG. Allowing significant differences in default behavior can lead to many hidden errors. Global override behaviors (e.g., forcing encoder to UNI) must be clearly indicated in logs
  • Plotly 6.x uses binary-encoded array serialization by default; when incompatible with older CDN versions (plotly-latest), it causes bar length errors. Solution: Lock the CDN version to match the Python package version and force the input to be a Python list

Practical Takeaways#

  • Error messages from R interfaces (e.g., mclust) are not intuitive in Python. Sufficient pre-verification should be added at the Python level, with clear error messages including shape information. Silent degradation of mclust and other R packages must include clear warnings; clustering methods cannot be changed without any prompts
