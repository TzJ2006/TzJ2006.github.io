Key Learnings### Practical Learning Outcomes#

  • ArXiv HTML Rendering is now more suitable for full-text extraction than PDF: faster, with no OCR errors, and maintaining structure. For papers not processed in HTML format, PDF can be used as an alternative. This indicates that the evolution of infrastructure requires regular review of “best practices”.
  • Ambiguous numeric logs without semantic tags: “1736651” could represent count/ID/size. It’s always better to use clear labels: “Author 1736651” or “1,736,651 papers”. The extra cost is just two words, but it eliminates errors in interpreting the entire category.
  • For academic analysis tools, narrative parts prefer Chinese output, while technical terms prefer English. A dual-language approach: describe structures in English (breakthroughs, why not before, impacts), and explain them in Chinese.
  • When there’s a selection logic upstream (year-based stratification → ≤10/year), downstream stages should not use different criteria for re-filtering. This can lead to inconsistent patterns (papers seen by fast modes differ from those by detailed modes).
  • Multi-agent exploration pattern: Use the Explore agent to understand the code library, the Plan agent to design, and implement them in a sequential manner through user verification. This clearly separates concerns and reviews plans before execution.
