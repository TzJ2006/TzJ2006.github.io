Key Learnings### Practical Learning Outcomes#

  • ArXiv HTML rendering is now more suitable for full-text extraction than PDF: Faster, with no OCR errors, and retains the structure. For papers with pre-Html, PDF can be used as an alternative. This indicates that the evolution of infrastructure requires regular review of “best practices”.
  • Numerical records without semantic tags can lead to ambiguity: “1736651” could refer to a count/id/size. Always use clear labels: “Author 1736651” or “1,736,651 papers”. The extra two words are less costly but can avoid misinterpretation of the entire category.
  • For academic analysis tools, narrative sections prefer Chinese output, while technical terms prefer English. A dual-language approach: write the structure in English (breakthroughs, why not before, impact), and provide explanations in Chinese.
  • When there is selection logic upstream (annual stratification → ≤10 years), downstream stages should not use different criteria for re-filtering. This can lead to inconsistent patterns (fast mode seeing different papers than detailed mode).
  • Multi-agent exploration pattern: Use the Explore agent to understand the code library, the Plan agent for design, and implement them in sequence through user verification. This clearly separates concerns and allows for planning before execution.
