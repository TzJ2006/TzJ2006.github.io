Problems & Solutions#

Critical Issues#

1. JSON parsing fails catastrophically when Haiku repair fails, losing original LLM response and making debugging impossible#

Solution: Implemented escalating repair chain (haiku→sonnet→opus) with _try_parse_json helper to avoid recursion, saving failed responses to output/logs/json_repair_failed/.txt for debugging

Key Insight: When LLM-based repair is involved, must avoid infinite recursion by separating pure parsing logic (_try_parse_json) from repair-invoking logic (parse_json_response), and always preserve original output on total failure

2. ArXiv only returns recent 100 papers, Semantic Scholar has 500 papers with citation data, but merge logic discarded S2 papers without ArXiv matches, losing most historical high-impact work#

Solution: Reversed data source priority: S2 becomes primary source (has year/citation/venue/abstract), ArXiv enriches with arxiv_id/pdf_url/categories, then add orphan ArXiv papers. Result: comprehensive timeline coverage

Key Insight: When merging datasets with different biases (ArXiv=recent, S2=comprehensive), use the more complete source as foundation and enrich from the specialized source, not vice versa

3. Sorting papers only by citation count lost temporal research trajectory, making it impossible to see how researcher’s focus evolved over career stages#

Solution: Implemented year-stratified selection: group by year, sort within year by award weight then citations, take ≤10/year, display chronologically. Preserves trajectory narrative while highlighting top work per period

Key Insight: For longitudinal analysis, temporal structure is often more valuable than global ranking; stratified sampling maintains both dimension

4. manip_progress_distance_left/right contained negative values and values >1 in collected data#

Solution: Added np.clip(progress_dist, 0.0, 1.0) to constrain both left and right progress distance to valid [0,1] range

Key Insight: Curved robot trajectories can cause current EE position to temporarily exceed start-to-target distance during interpolation, breaking the 1 - |current-final|/|start-final| formula assumptions. Explicit bounds enforcement is necessary for progress metrics.

5. ArXiv and Semantic Scholar APIs don’t provide award data (best paper, spotlight, etc.), but this is critical for identifying breakthrough work#

Solution: User suggested leveraging LLM training data: send paper titles+venues to LLM, ask it to recall known awards. Implemented as new analysis stage with separate caching

Key Insight: LLMs can serve as auxiliary knowledge bases for data not available via APIs, especially for prestigious but unstructured metadata like conference awards

6. critical_region still appeared in HDF5 files after deleting the override method#

Solution: Analyzed base class _base_task.py:510 and found get_obs() unconditionally writes pkl_dic[‘critical_region’]. Added post-processing step to pop this key from patched pickle files.

Key Insight: Deleting override methods only changes return value; base class code paths must be traced to ensure keys are truly removed from output. Post-processing pkl files before HDF5 conversion is effective for removing unwanted keys.

General Issues#

7. Log message ‘[S2] 获取论文: 1736651’ ambiguous—looks like 1.7M papers fetched, actually author ID. User spotted this immediately#

Key Insight: Log messages must be unambiguous even when read out of context. Numbers should always have explicit labels indicating their semantic type

Solution: Removed ‘sorted(…, reverse=True)[:10]’ filter, changed to ‘papers_for_download = [p for p in papers if p.arxiv_id]’. Now downloads full text for all filtered papers

Key Insight: When selection logic already enforces quality threshold upstream, downstream shouldn’t re-filter with different criteria—causes inconsistency between modes

AI Limitations#

Critical Limitations#

  • Used smaller dataset (ArXiv 100 recent) as merge base rather than comprehensive dataset (S2 500 historical), prioritizing algorithmic simplicity over semantic completeness for trajectory analysis
  • Static code analysis insufficient for tracing data flow: Claude deleted get_critical_region_label() override but failed to trace that base class get_obs() unconditionally writes pkl_dic[‘critical_region’], requiring user to discover this through runtime testing
  • Initially defaulted to traditional data pipeline thinking (API scraping, structured datasets) for award information, missing the insight that LLMs already ‘know’ famous conference awards from training data
  • Implemented single-stage LLM repair (haiku only) rather than multi-stage escalation, treating error recovery as binary rather than graduated cost-benefit problem
  • Limited physical reasoning about edge cases: Did not anticipate that curved robot trajectories could cause progress_distance to exceed [0,1] bounds, implementing formula without considering trajectory geometry constraints
  • Over-reliance on method inheritance logic: Assumed deleting override would remove variable from output, without verifying all code paths that write to data structures. Needed empirical testing to catch base class unconditional writes.

General Limitations#

  • Failed to anticipate log message ambiguity: wrote ‘[S2] 获取论文: 1736651’ without considering how bare number appears ambiguous to readers. Required user correction to make subject-verb-object structure explicit
  • Did not proactively verify that detailed mode’s paper selection logic aligned with year-stratified selection—allowed inconsistency where selection caps at ≤10/year but download recaps at global top 10

Learnings#

Key Learnings#

  • For researcher trajectory analysis, temporal structure preservation (year-stratified sampling) is more valuable than global citation ranking. Shows domain-specific sampling strategies matter more than generic ‘sort by metric’ approaches
  • Post-processing approach for ‘future-dependent’ variables: When per-frame variables require knowledge of move() completion state (like target poses), recording frames normally then patching pickle files after move() finishes is cleaner than complex state machines during frame capture
  • LLMs can serve as knowledge bases for unstructured metadata (conference awards, paper prestige) when APIs don’t provide it. This ‘LLM as auxiliary database’ pattern is underutilized—requires prompt design skill but no infrastructure
  • When implementing LLM-based repair in recursive parsing logic, must extract non-LLM stages into separate helper to avoid infinite loops. Pattern: _try_parse (pure) vs parse_response (with repair)
  • Multi-source data merging: when sources have different biases (recency vs comprehensiveness), use most complete as foundation and enrich from specialized. ArXiv=recent, S2=historical → S2 base + ArXiv enrichment
  • Error recovery should implement cost-graduated escalation (cheap model → expensive model → human intervention via logs) rather than binary fail/succeed. Maximizes automated recovery while preserving debugging information
  • Base class data flow requires deep tracing: In complex OOP hierarchies with per-frame data collection, deleting overrides may not remove keys from output. Must trace base class execution paths (e.g., get_obs() → _take_picture() → pkl files) to ensure variables are truly excluded.
  • Physical quantities need semantic bounds enforcement: Progress metrics derived from geometric formulas should be explicitly clamped to their semantic range [0,1] even if the formula theoretically produces valid values, as real-world trajectories introduce edge cases

Practical Learnings#

  • ArXiv HTML rendering now preferred over PDF for full-text extraction: faster, no OCR errors, preserves structure. PDF as fallback for pre-HTML papers. Shows infrastructure evolution requires periodic revisiting of ‘best practices’
  • Logging numbers without semantic labels creates ambiguity: ‘1736651’ could be count/ID/size. Always use explicit labels: ‘author 1736651’ or ‘1,736,651 papers’. Cost is two extra words, benefit is eliminating entire class of interpretation errors
  • For academic analysis tools, Chinese output is preferred for narrative sections but English for technical terms. Bilingual approach: structure in English (breakthrough, why_not_before, impact), explanation in Chinese
  • When selection logic exists upstream (year stratification → ≤10/year), downstream stages should not re-filter with different criteria. Causes mode inconsistency (fast mode sees different papers than detailed mode)
  • Multi-agent exploration pattern: Using Explore agent for codebase understanding → Plan agent for design → sequential implementation with user validation creates clear separation of concerns and allows plan review before execution
