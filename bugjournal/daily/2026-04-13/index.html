Problems & Solutions#

Critical Issues#

1. Teddy+UNI2 multimodal fusion degrades white matter discrimination: Teddy-only ARI=0.186, concat ARI=0.151 (-19%), mean ARI=0.153 (-18%). Root cause: Teddy arrives pre-L2-normalized in 1024d with per-dim variance 4.3e-5 (460x smaller than PCA 50d at 2.0e-2). After pipeline L2-normalization, gene signal contributes only 13.3% of total variance in concat (2560d) while vision dominates at 86.7%.#

Solution: Background agent analyzed section 151673 embeddings and discovered variance distribution mismatch. Fix options: PCA-reduce Teddy before fusion, variance-weighted concatenation, or standard scaling instead of L2 normalization.

Key Insight: Pre-normalized high-dimensional embeddings lose signal strength after dimension-agnostic L2 normalization. Fusion strategies must account for input variance distributions. High dimensionality alone doesn’t guarantee discriminative power—variance per dimension is critical.

2. STAIG fusion shows no difference between Teddy and PCA encoders (Teddy avg=0.495, PCA avg=0.467, diff=+0.028 within noise std=0.119). All gene encoder comparisons using STAIG fusion appear invalid.#

Solution: Code inspection revealed models/Fusion.py line 1253: staig_fusion passes gene_feat=staig_gene_feat_3000.npz (raw HVG), completely ignoring the gene_emb argument. Both Teddy+STAIG and PCA+STAIG use identical inputs. Observed ARI differences are stochastic GCN training noise, not systematic gene encoder effects.

Key Insight: STAIG fusion architectural bug: gene encoder parameter is a no-op. All gene encoders collapse to raw HVG input regardless of specified encoder. This invalidates comparative evaluation of gene encoders when using STAIG as fusion method.

3. Single cursor (_current_line_start) in MeetingHelper cannot track two independent partial lines (mic and system audio), causing text corruption and overwrites during concurrent transcription output.#

Solution: Refactored to region-based architecture: split subtitle window into two zones (MIC/SYS), each with independent cursor, file handle, partial line buffer, and finalization logic. Adversarial audit by Opus subagent caught this fatal concurrency bug before implementation.

Key Insight: Concurrent transcription output requires isolated state management per source. Shared mutable state (cursor, buffer) is the root cause of concurrency bugs. ccplan adversarial audit phase prevents implementing fundamentally broken architectures.

4. Portable collection scripts hardcoded 6 tasks across pack/run scripts and tutorial, requiring manual updates in multiple files for each new task addition.#

Solution: Centralized task definition in task_registry.yaml and made all scripts dynamically derive task lists. Implemented automatic portable registry generation during packaging with fail-fast asset validation (task_config, HDF5, scenes).

Key Insight: Centralized configuration with dynamic derivation eliminates multi-file consistency problems and scales better than manual synchronization across scattered hardcoded lists.

5. Five interrelated bugs in error-recovery pipeline caused 100% MimicGen augmentation failure (0/10 success rate): controller cache desync, quaternion format mismatch, leading zero-action frames, missing settling mechanism, and demo validation order.#

Solution: Systematically fixed all 5 bugs in causal chain: (1) controller sync after state modification, (2) quaternion wxyz→xyzw conversion, (3) trim leading zero-action frames, (4) add settling mechanism, (5) validate demo before augmentation. Success rate improved to 40% (4/10). Fixes applied to recovery mainline but only partially propagated to training source data pipeline.

Key Insight: Complex system failures are rarely single-point issues but cascading effects of multiple subtle bugs. Requires systematic identification and repair of entire causal chain. Code reuse across dependency paths requires explicit tracking to ensure fixes propagate everywhere.

6. libero2lerobot_parallel.py slows down drastically after processing many episodes (O(n²) performance degradation as dataset grows).#

Solution: Removed dataset.hf_dataset = dataset.create_hf_dataset() from the per-episode loop (line 224). This call rescans all existing parquet files, causing O(n²) cumulative cost as dataset grows. save_episode() already persists data; hf_dataset rebuild only needed at end if dataset will be read.

Key Insight: Incremental data operations should avoid full dataset reconstruction in loops. Always profile long-running pipelines for hidden O(n²) patterns—particularly operations that ‘rescan from scratch’ on each iteration.

7. HD basic_contrastive and QFormer fusion failed with CUDA illegal memory access and OOM errors on RTX 2080 Ti (11GB VRAM). RTX 2080 Ti insufficient for HD 17,502 spots × 550 epochs.#

Solution: Resubmitted jobs to gpu-common partition with explicit GPU constraint gres=gpu:5000_ada:1 (24GB VRAM), increased memory to 64GB, and extended time limit to 2 days. Separated QFormer into dedicated experiment config to isolate memory-intensive fusions.

Key Insight: Large-scale spatial data (17K+ spots) requires ≥24GB VRAM for contrastive learning. SLURM GPU type specification syntax: –gres=gpu:: (e.g., gres=gpu:5000_ada:1) to request specific GPU models within heterogeneous partitions.

8. LeRobot 0.4.0 updated add_frame() signature to require task as separate positional argument instead of embedding in frame dict, breaking existing conversion code.#

Solution: Modified worker function to return task_instruction separately; changed add_frame call from add_frame(frame_data) to add_frame(frame_data, task=task_instruction); removed task injection into frame dict.

Key Insight: API breaking changes in dependencies require signature inspection and adaptation. Old pattern of embedding metadata in data dict replaced by explicit parameters in LeRobot 0.4.0.

Learnings#

Key Learnings#

  • NeurIPS Datasets & Benchmarks track paper structure: 9-page main body with Methods in appendix, ML-oriented language, emphasis on systematic evaluation over algorithmic novelty. Core contribution: fusion strategy dominance (ARI range 0.16-0.55 across fusions vs <0.05 across encoders). Strong ‘difficulty proof’ essential: Pi0.5 FT 3.7% RSR vs 60-70% normal SR catastrophic gap.
  • Pre-normalized high-dimensional embeddings lose discriminative power in dimension-agnostic fusion: Teddy 1024d pre-L2-normalized has 460x less per-dim variance than PCA 50d; after pipeline L2-normalization, gene signal is invisible in concat/mean with UNI2 1536d. Solution: variance-weighted fusion, PCA-reduce high-dim embeddings before fusion, or standard scaling instead of L2 norm.
  • ccplan workflow prevents ‘implementing the wrong thing’ and ‘missing critical edge cases’. Multi-stage pipeline (intent calibration → hypothesis validation → adversarial audit → review gate) caught 9 issues in MeetingHelper including fatal single-cursor concurrency bug before implementation. Structured divergence + convergence superior to straight-line coding. Adversarial audit by independent Opus subagent is essential—cannot self-critique effectively.
  • CCPlan (Evolving Constraint Planning) workflow scales well for multi-task projects: 11 P0/P1 tasks organized into 3 waves (immediate → analysis → refinement) with parallel agent execution and structured ECL document tracking. Key: continuous execution mandate (phases flow without user prompts) and AskUserQuestion tool for checkpoints.
  • Multi-agent exploration pattern for complex codebases: launch 3 parallel agents (results, methodology, plans) with targeted prompts, then synthesize findings into structured deliverable. Effective for large projects with distributed documentation. Used successfully for MIHD paper writing.
  • Cross-chemistry battery degradation modeling can leverage CV domain generalization: disentangle domain-invariant (z ~ degradation state) from domain-specific (chemistry) representations. Physical anchoring (internal resistance, coulombic loss, dQ/dV peaks) provides interpretable initialization for learned z. Three-phase verification: physics candidates → normalization validation → contrastive learning encoder.
  • LeRobot 0.4.0 dtype semantics: ‘image’=bytes inline in parquet (PNG per frame), ‘video’=MP4 in videos/ directory. meta.video_keys only returns dtype:‘video’ features. For large datasets, always use dtype:‘video’ for efficient MP4 encoding rather than storing PNG bytes per frame.
  • Incremental data pipelines must avoid O(n²) operations: rebuilding entire dataset after each element adds cumulative cost. libero2lerobot_parallel’s dataset.create_hf_dataset() in loop caused dramatic slowdown after ~1000 episodes. Always profile long-running pipelines for hidden O(n²) patterns—particularly operations that ‘rescan from scratch’.
  • Centralized configuration with dynamic derivation scales better than scattered hardcoded lists. Fail-fast validation at build time prevents incomplete artifacts better than runtime warnings. Applied in error-recovery portable scripts: task_registry.yaml as single source of truth eliminates multi-file consistency problems.
  • STAIG fusion architectural insight: gene_emb parameter is ignored in models/Fusion.py line 1253; fusion always uses raw HVG 3000d from staig_gene_feat_3000.npz. This explains why Teddy+STAIG and PCA+STAIG have identical performance (diff within noise). All gene encoder comparisons using STAIG are invalid.
  • DE analysis validation workflow: load raw adata (not HVG subset) → run rank_genes_groups with wilcoxon → use Hungarian matching to align predicted clusters to ground truth layers → check marker gene ranks in expected layers. Key markers for DLPFC: PCP4/NEFL (Layer 5), MBP/MOBP (WM), CARTPT (Layer 3). Poor marker recovery can reflect genuine biological clustering difficulty, not just algorithmic failure—always validate GT baseline.
  • Bootstrap confidence intervals essential for small-n comparisons: STAIG 10-section avg ARI=0.495 [0.387, 0.597] vs PCA=0.265 [0.230, 0.296] shows non-overlapping CIs confirming significance. Implemented via utils/statistics.py with n=1000 resampling; 95% CI = (percentile_2.5, percentile_97.5).
  • SLURM GPU type specification syntax: use –gres=gpu:: format (e.g., gres=gpu:5000_ada:1, gres=gpu:a6000:1) to request specific GPU models within heterogeneous partitions. gpu-common and scavenger-gpu both have mixed GPU types; explicit constraint prevents allocation to underpowered GPUs. RTX 2080 Ti (11GB) insufficient for HD spatial data; need ≥24GB VRAM.
  • Complex system failures are rarely single-point issues but cascading effects of multiple subtle bugs. error-recovery pipeline: 5 interrelated bugs (controller cache, quaternion, zero-action frames, settling, validation order) caused 100% failure. Requires systematic identification and repair of entire causal chain. Code reuse across dependency paths requires explicit tracking to ensure fixes propagate.
  • Region-based UI architecture ideal for concurrent output: when multiple async data sources need to update the same UI simultaneously, allocating independent regions (cursor, buffer, draw logic) per source is cleaner and less error-prone than shared state + locks. Applied in MeetingHelper dual audio transcription.

Practical Learnings#

  • Project-level Claude Code settings (.claude/settings.json) can dramatically improve iteration speed for exploratory work by using global ‘Bash’ allow with targeted ‘deny’ list for destructive ops (rm, rmdir, git push –force, git reset –hard, git clean -f). Settings propagate to all subdirectories and sessions. Applied in MIHD project.
  • Shell SIGPIPE handling requires care when downstream commands (head, grep -m) exit early. Use dedicated extraction functions or process substitution to avoid broken pipe errors polluting output. Fixed in error-recovery run_collection.sh by replacing pipeline with direct first-match function.
