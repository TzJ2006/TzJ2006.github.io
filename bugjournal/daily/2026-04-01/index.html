问题与解决方案#

关键问题#

1. Linux悬浮球点击后位置跳变~36px#

解决方案: 将Rust侧expanded_rect_from_ball()的Left/Right anchor分支从"吸附到inner_bounds边缘"改为"以ball_rect.x为锚点，仅溢出时最小化夹紧"，保证球体坐标在展开/收起前后不变。

关键洞察: 展开窗x坐标应等于ball_rect.x（左扩）或ball_rect.x-(expanded_width-ball_size)（右扩），只在监视器真的放不下时才向内推；Debug跨平台UI位置问题时优先问"哪层代码在改坐标"而非"坐标如何换算"，Rust/native层比前端更可能是根因。

2. Linux收起悬浮球时出现完整展开窗残影（WebKitGTK透明窗口）#

解决方案: Linux上将collapse改为native-first顺序：先invoke set_float_ball_expanded(false)缩小原生窗口，再翻转Svelte expanded状态，避免大透明窗口持续暴露在屏幕上。

关键洞察: WebKitGTK/compositor不会主动清除透明窗口的旧像素缓冲区；只要native窗口还是展开尺寸，CSS动画期间屏幕上就始终能看到旧内容，无论前端如何隐藏元素。

3. STAIG HD实验结果ARI=1.0异常（旧缓存污染）#

解决方案: 排查npz文件时间戳发现STAIG npz来自Feb 28旧跑，根因是staig_gene_feat缓存缺失导致实验静默失败但保留旧文件。手动准备HD STAIG基因特征缓存后删除旧npz重跑。

关键洞察: pipeline的skip_completed机制在文件存在但内容无效（gt_labels全为-1）时无法检测，需要主动校验gt_labels合法性。

4. 首轮Bug修复未解决根因（AI错误定位根因）#

解决方案: 用户反馈后AI重新审查，将关注点从"前端坐标换算和延后shrink时机"转移到"Rust锚点布局本身和Linux native collapse顺序"，锁定真正根因后二次实现。

关键洞察: 第一轮过于关注前端坐标换算，忽略了Rust布局函数主动改变位置这一根本原因；人类坚持描述症状而非接受"需要更复杂实现"结论，是触发正确修复路径的关键。

5. HD学习式融合GPU job超时：HD数据集(17502 spots)实际运行时间远超预估#

解决方案: Kill本地进程后提交GPU job，发现6h仍不够，改为4个独立SLURM job，每种融合策略分开提交并设置充足时间限制（24h+）。

关键洞察: learning-based fusion训练时间随数据集规模超线性增长，HD(17502 spots)的时间不能从DLPFC(~4000 spots)线性推算。

一般问题#

6. 工程配置类问题：Rust/Cargo未在PATH、SLURM脚本误删GT缓存、conda ToS强制接受#

解决方案: Rust PATH：构建前source ~/.cargo/env；SLURM脚本：删除’rm -f ground_truth.npz’行并手动重建GT缓存；conda ToS：运行conda tos accept –override-channels指定channel。

关键洞察: Rust安装只修改~/.bashrc，子进程不继承交互shell env；conda 26.x新增ToS强制接受机制，无人值守脚本需预先处理。

今日收获#

核心收获#

  • 核心实验结论：高质量embedding本身比聚类方法选择更重要。STAIG+mclust(0.546) > STAIG+Leiden(0.513)，Leiden对弱embedding帮助大（+29%）但对强embedding几乎无额外收益（+6%），融合策略的质量决定上限。
  • Linux WebKitGTK不会主动清除透明窗口的旧像素缓冲区：native窗口保持大尺寸时CSS隐藏元素无法消除残影；正确做法是native-first collapse（先缩小native窗口，再驱动前端状态变化）。
  • Tauri跨平台悬浮窗的展开/收起布局应以球体实际位置为锚点（而非safe-area边界），保证球体视觉上固定不动；Debug跨平台UI位置问题时优先问"哪层代码在改坐标"，Rust/native层比前端更可能是根因。
  • HD维度不平衡问题：concat融合(1586d = 50d PCA + 1536d UNI2)反而比PCA单模态差，高维模态完全主导；多模态融合需要维度对齐或加权，简单拼接在维度悬殊时不可行。
  • HD niche query发现：多模态attention融合的border query能捕获组织交界处的混合微环境，而基因only的PCA query更倾向同一组织类型，说明多模态embedding编码了更丰富的空间上下文。

实践收获#

  • SLURM GPU job设计原则：learning-based fusion训练时间随数据集规模超线性增长，HD(17502 spots)不能从DLPFC(~4000 spots)线性推算，应为每种策略创建独立job并设置充足时间（24h+）。
  • Tauri Linux悬浮球右键点击应在pointerdown层就过滤（button!==0），而非依赖preventDefault，否则右键会触发和左键相同的展开/收起逻辑。
