Problems & Solutions#

Critical Issues#

1. Hugo not installed — script terminated at Step 5 with ‘hugo: command not found’ (exit 127)#

Solution: User installed Hugo extended v0.154.5 via the system package manager

Key insight: set -e converts any command failure directly into script termination; a missing Hugo binary prevents the entire 7-step pipeline from ever reaching the critical build stage

2. themes/PaperMod/ and public/ directories both absent — Hugo build and deployment cannot run#

Solution: User cloned the PaperMod theme and the tzj2006.github.io deployment repo

Key insight: Hugo site deployment depends on three independent components (hugo binary, theme repo, deployment git repo); any one missing causes either a silent or a hard failure — prerequisites should be explicitly documented

General Issues#

3. static/videos directory does not exist — find returns an error, set -e causes Step 4 to terminate unexpectedly#

Solution: Added a directory existence check before find ([ -d “$DIR” ]); skip the step if the directory does not exist

Key insight: find operations on optional/empty directories in set -e scripts must be guarded; an ordinary “directory not yet created” scenario will otherwise trigger unexpected script termination

4. HandBrakeCLI inline fallback branch PID added to pids array twice, and compressed file not used to replace original (output is .hb.mp4)#

Solution: Moved pids+=($!) and echo statements into their respective if/else branches; added && mv after the HandBrakeCLI command to replace the original file

Key insight: PID tracking logic for parallel background tasks must align precisely with if/else control flow boundaries; placing shared code outside branches easily introduces duplicate execution and silent data-loss bugs

5. git push failed due to unconfigured GitHub authentication (‘could not read Username for https://github.com’)#

Solution: User configured an SSH key or gh auth, then re-ran the script successfully end-to-end

Key insight: HTTPS remotes require pre-configured credentials in non-interactive shells; the deployment repo should use an SSH remote to avoid environment-specific failures

Today’s Takeaways#

Core Takeaways#

  • Hugo site deployment has three independent prerequisites that must all be in place simultaneously (hugo binary, theme repo, deployment git repo); any one missing causes failures at different stages — these prerequisites should be explicitly listed in project documentation

Practical Takeaways#

  • find operations on optional directories in bash set -e scripts must include a directory existence check; otherwise an ordinary “directory not yet created” scenario triggers unexpected script termination
  • Code for parallel background tasks (& + pids tracking) must align precisely with if/else control flow boundaries; placing shared code outside branches easily introduces duplicate PID tracking and silent file-replacement omission bugs
