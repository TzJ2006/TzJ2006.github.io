Problems & Solutions#

Critical Issues#

1. Hugo was not installed, causing the script to terminate at Step 5 with “hugo: command not found” (exit code 127)#

Solution: The user installed Hugo extended v0.154.5 using the system package manager.

Key Insight: set -e will convert any command failure into script termination. The absence of Hugo prevented the 7-step process from reaching the critical build stage.

2. The themes/PaperMod/ and public/ directories were missing, preventing Hugo build and deployment#

Solution: The user cloned the PaperMod theme and tzj2006.github.io deployment repository.

Key Insight: Hugo site deployment depends on three independent components: the Hugo binary, the theme repository, and the deployment Git repository. Any missing component will cause a silent or terminated failure. These conditions should be clearly stated in the documentation.

General Issues#

3. The static/videos directory was missing, causing an error with the find command and set -e to terminate Step 4 unexpectedly#

Solution: A directory existence check ([ -d “$DIR”]) was added before the find command. If the directory did not exist, the step was skipped.

Key Insight: Operations on optional/empty directories must be protected in the set -e script. Otherwise, normal “directory not created” scenarios could cause unexpected script termination.

4. The PIDs of the HandBrakeCLI rollback branch were duplicated in the pids array, and the original files were not replaced after compression (output was .hb.mp4)#

Solution: The pids+=($!) and echo statements were moved to their respective if/else branches. The && mv command was added after the HandBrakeCLI command to replace the original files.

Key Insight: The PID tracking logic for parallel background tasks needs to be carefully aligned with the if/else control flow. Putting common code outside the branches can lead to duplicate execution and silent file replacement errors.

5. git push failed due to insufficient GitHub authentication (could not read Username for ‘[[[HYMTPH_0]]’)#

Solution: The user configured SSH keys or used gh auth before running the script. The script passed successfully.

Key Insight: HTTPS remotes require pre-configured credentials in non-interactive shells. The deployment repository should use SSH remotes to avoid environmental differences.

Learnings#

Key Learnings#

  • Hugo site deployment requires three independent dependencies to be met simultaneously: the Hugo binary, the theme repository, and the deployment Git repository. Any missing dependency can cause failures at different stages. These conditions should be clearly listed in the project documentation.

Practical Learnings#

  • In the bash set -e script, a directory existence check must be added to the find operation on optional directories. Otherwise, normal “directory not created” scenarios could cause unexpected script termination.
  • The code for parallel background tasks (& + pids tracking) must be carefully aligned with the if/else control flow. Putting common code outside the branches can lead to duplicate PID tracking and silent file replacement errors.
