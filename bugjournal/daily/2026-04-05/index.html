Problems and Solutions#

Key Issues#

1. Linux WM fails to handle transparent borderless window sizing atomically, leading to systematic position offset +305px/+113px, flickering detection/compensation mechanisms, and inconsistent sizes between WebView and native windows.#

Solution: Set the window size to expanded state (152×56), without any adjustments. Use GDK input_shape_combineRegion to control the click area: only the ball area is displayed when folded (56×56), while the entire window is shown when unfolded or during dragging. Key Insight: This is a platform architecture issue, not a computational error. Linux WM interprets the same rect request differently. Solutions need to be found at the sizing level; local fixes can only temporarily solve the problem.

2. Life-Copilot Discord bot continues to report “OAuth authentication required” for calendar queries. There are two overlapping issues at play: (1) Claude -p does not load project-level MCP configurations; (2) permission denied errors occur when using MCP tools in non-interactive mode.#

Solution: Add –mcp-config and –permission-mode bypassPermissions parameters to the claude_adapter.py command. Running Claude -p directly revealed the root cause of permission denied errors. Since there is a conflict with MCP server initialization, the –allowedTools wildcard is unreliable in non-interactive mode. Using bypassPermissions is the correct method for backend service calls.

3. After implementing the fixed window layout, three platform-specific issues arose: ball display deformation (window builder still uses 56×56 size), inability to expand, and dragging failures (input shape gets blocked after leaving the ball area).#

Solution: (1) The window builder should use the expanded size and lock min=max based on GTK geometry_hints. (2) CSS justify-content should always take effect based on data-direction. (3) A new set_float_ball_dragging command was added to temporarily expand the input shape during dragging, with it being restored after the process is complete.

Key Insight: WebKitGTK has default minimum size limits. The X11 layer intercepts input shape events before setPointerCapture, so synchronizing the area expansion in Rust is necessary.

General Issues#

4. When using structlog to wrap standard library logging, extra {} fields disappear silently from the JSON output.#

Solution: Add the ExtraAdder() processor to the shared_processors entry, ensuring that additional fields are merged into the structured output.

Key Insight: When using structlog to wrap standard library logging, additional parameters do not get passed automatically. It is necessary to explicitly add these parameters so that the field disappears silently without reporting any errors.

5. When users request a backout of “changes made during this session,” the AI mistakenly uses git stash, which may lead to loss of historical work.#

Solution: After using git stash pop, manually identify and correct the specific changes made during this session.

Key Insight: Git stash affects all uncommitted changes, so it is not suitable for situations where only “changes made during this session” need to be corrected. It is better to use git diff to clearly define the scope of changes before making individual corrections.

Achievements Today#

Core Achievements#

  • Linux X11 WM does not guarantee atomicity for transparent borderless windows when sizing. Different WMs interpret the same coordinates differently, leading to systematic offset issues. The only reliable solution is to eliminate sizing functionality (fixed window + GDK input shape). Local patches can only temporarily solve the problem.
  • Claude CLI -p mode requires two steps to complete MCP configuration: (1) Explicitly pass –mcp-config to load project-level tools; (2) Add –permission-mode bypassPermissions to avoid permission confirmation in non-interactive mode. In this case, the –allowedTools wildcard is unreliable because it conflicts with MCP server initialization. BypassPermissions is the correct method for backend service calls.
  • GDK Input Shape intercepts events on the X11 layer. setPointerCapture cannot override it. In dragging scenarios, a set_float_ball_dragging command in Rust should be used to temporarily expand the input shape to the entire window. With cairo-rs 0.18 and gdk 0.18, input_shape_combineRegion can be used directly in Tauri with_webview callbacks, eliminating the need for additional complex dependencies.
  • When debugging nested process CLI tools (main process → Claude CLI → MCP server), it’s more efficient to first record the actual command parameters and then isolate and verify the effects of each parameter.
  • The ccplan method effectively transforms complex platform architecture problems into executable plans by exploring alternative solutions, going through a review process, and testing feasibility. The ECL documentation provides cross-session context persistence, reducing the cost of repeated analysis.

Practical Achievements#

  • When using structlog to wrap standard library logging, it’s necessary to explicitly add extra={} parameters as ExtraAdder() processors so that they appear in the output fields. Otherwise, the field will disappear silently without reporting any errors.
