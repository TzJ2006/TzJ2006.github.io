Problems & Solutions#

Critical Issues#

1. Linux WM cannot atomically handle dynamic resize of transparent borderless windows, causing systematic +305px/+113px position offset, probe/corrective mechanism flickering, and size mismatch clipping between WebView and native window#

Solution: Fixed window size to expanded (152×56) permanently with no resizing; used GDK input_shape_combine_region to control click-through: collapsed shows only ball area (56×56), expanded and dragging accept events across the full window

Key Insight: This is a platform architecture problem, not a calculation bug: Linux WM interprets the same rect request with different semantics; eliminating resize is the cure, not local patches that merely shift symptoms around

2. Life-Copilot Discord Bot continuously replied “OAuth authentication required” to calendar queries, root cause being two compounding issues: (1) claude -p does not load project-level MCP config; (2) MCP tool call permissions are silently denied in non-interactive mode#

Solution: Sequentially added –mcp-config .claude/mcp_servers.json and –permission-mode bypassPermissions to the command in claude_adapter.py; reproduced permission_denials field by running claude -p directly to locate root cause

Key Insight: Claude CLI session-less mode and interactive mode have different MCP loading behavior; –allowedTools wildcards are unreliable in non-interactive mode due to a race condition with MCP server initialization; bypassPermissions is the correct mode for backend service calls

3. Three platform-specific issues appeared after the first fixed-window implementation: ball displayed squashed (window builder still using 56×56), unable to expand (CSS not applying per data-direction), drag broken (input shape blocking X11 events after mouse leaves ball area)#

Solution: (1) Changed window builder to use expanded dimensions and locked min=max with GTK geometry_hints; (2) CSS justify-content always applied based on data-direction; (3) Added set_float_ball_dragging command to expand input shape to full window during dragging and restore afterward

Key Insight: WebKitGTK has default minimum natural size constraints; X11-layer input shape event interception occurs before setPointerCapture, requiring synchronous region expansion on the Rust side

General Issues#

4. structlog wrapping stdlib logging causes extra={} fields to silently disappear from JSON output#

Solution: Added structlog.stdlib.ExtraAdder() to shared_processors to merge extra fields into structured output

Key Insight: When structlog wraps stdlib, extra parameters are not automatically passed through — ExtraAdder must be explicitly added, as fields are silently dropped without any error

5. When the user requested rollback of “current session changes,” the AI incorrectly used git stash, nearly causing loss of historical work#

Solution: After git stash pop restored changes, manually identified the three specific modifications from that session and precisely reverted them

Key Insight: git stash affects all uncommitted changes and is not appropriate for “only revert current session changes”; git diff should be used first to understand the scope of changes before reverting them individually

Today’s Learnings#

Core Learnings#

  • Linux X11 WM provides no atomicity guarantees for resize requests on transparent borderless windows; different WMs interpret the same coordinates with different semantics, causing systematic offsets; completely eliminating resize (fixed window + GDK input shape) is the only reliable solution — local patches merely shift symptoms around
  • Complete MCP configuration for Claude CLI -p mode requires two steps: (1) explicitly pass –mcp-config to load project-level tools; (2) add –permission-mode bypassPermissions to bypass permission confirmations that cannot be completed in non-interactive mode; –allowedTools wildcards are unreliable in this mode due to race conditions with MCP server initialization, and bypassPermissions is the correct mode for backend service calls
  • GDK input shape intercepts events at the X11 layer; setPointerCapture cannot override it; drag scenarios require using the set_float_ball_dragging command on the Rust side to temporarily expand the input shape to the full window; under cairo-rs 0.18 + gdk 0.18, input_shape_combine_region can be used directly in Tauri with_webview callbacks without adding complex dependencies
  • When debugging nested process CLI tools (main process → Claude CLI → MCP server), logging the actual command-line arguments being passed and then isolating and verifying parameter effects one by one is more efficient than guessing parameter behavior
  • The ccplan skill’s divergent exploration (6 approaches) + adversarial review + feasibility probing workflow effectively transforms complex platform architecture problems into executable plans; ECL documents provide cross-session context persistence, reducing repeated analysis overhead

Practical Learnings#

  • When structlog wraps stdlib logging, stdlib extra={} parameters require explicitly adding ExtraAdder() processor to appear in output fields; otherwise fields are silently dropped with no error
