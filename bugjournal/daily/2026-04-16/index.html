Issues and Solutions#

Key Issues#

1. There is an error on line 96 in revalidate_demos.py: The obj_orientations quaternion data is incorrectly written into the obj_positions dictionary, causing data loss.#

Solution: Changed the code on line 96 from ‘obj_positions[key[4:-5]]’ to ‘obj_orientations[key[4:-5]]’ to correctly write the quaternions into the dictionary. Key Insight: Mistakes in data loading code may silently destroy multi-field data structures.

2. The determine_counts_toward_target function returns the cached counts_toward_target value when the field exists, preventing re-evaluation under the new quota rules.#

Solution: Clears the counts_toward_target and quota_reason fields before calling determine_counts_toward_target during revalidation. Key Insight: Functions that need recalculation should explicitly clear the cache when checking pre-calculated values.

General Issues#

3. The NumpyEncoder class exists repeatedly in 4 different files (resegment_recovery_demos.py, 3_mimicgen_recovery_augment.py, 2_collect_recovery_demos.py, revalidate_demos.py).#

Solution: Moved the NumpyEncoder to the shared scriptUtils.py file and updated all consumers to import from this file. Key Insight: Shared JSON encoding logic should be placed in common utilities to prevent maintenance burdens.

4. There is initial uncertainty about the correct CLI syntax—the repository README mentions /plugin commands, but the CLI uses the ‘claude plugin’ structure.#

Solution: Use ‘claude –help’ and ‘claude plugin –help’ to verify the actual CLI functionality before executing commands, ensuring the correct syntax is ‘claude plugin marketplace add’ and ‘claude plugin install’. Key Insight: External documentation may not match the actual local CLI syntax; always check the local help document before executing installation commands.

5. The tag_npz_validation and save_manifest functions exist repeatedly between revalidate_demos.py and 2_collect_recovery_demos.py.#

Solution: Moved both functions to the shared recovery_collection_validation.py module and updated consumers to import from this module. Key Insight: Validation and persistence logic should be centralized in the validation framework module.

  • agent-skills provides 20 production-level engineering skills, distributed across six development stages (Specification/Refinement; Planning: Architecture/Test-Driven Development; Building: Incremental Coding; Verification: Testing; Reviewing: Code Review; Deployment): These skills represent mature engineering workflows.
