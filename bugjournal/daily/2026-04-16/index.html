Problems & Solutions#

Critical Issues#

1. Bug in revalidate_demos.py:96 where obj_orientations quaternion data was incorrectly written to obj_positions dictionary, causing data loss#

Solution: Changed line 96 from ‘obj_positions[key[4:-5]]’ to ‘obj_orientations[key[4:-5]]’ to write quaternions to correct dictionary

Key Insight: Copy-paste errors in data loading code can silently corrupt multi-field data structures

2. determine_counts_toward_target returns cached counts_toward_target value when field exists, preventing re-evaluation under new quota rules#

Solution: Clear counts_toward_target and quota_reason fields before calling determine_counts_toward_target during revalidation

Key Insight: Functions that check for pre-computed values need explicit cache invalidation when re-computing is required

General Issues#

3. NumpyEncoder class duplicated in 4 separate files (resegment_recovery_demos.py, 3_mimicgen_recovery_augment.py, 2_collect_recovery_demos.py, revalidate_demos.py)#

Solution: Moved NumpyEncoder to shared script_utils.py and updated all consumers to import from there

Key Insight: Shared JSON encoding logic should live in common utilities to prevent drift and maintenance burden

4. Initial uncertainty about correct CLI syntax - repository README mentioned /plugin commands but actual CLI uses ‘claude plugin’ structure#

Solution: Verified actual CLI capabilities using ‘claude –help’ and ‘claude plugin –help’ before executing commands, confirming the correct syntax is ‘claude plugin marketplace add’ and ‘claude plugin install’

Key Insight: External documentation may not match exact local CLI syntax; always verify with local help documentation before executing installation commands

5. tag_npz_validation and save_manifest functions duplicated between revalidate_demos.py and 2_collect_recovery_demos.py#

Solution: Moved both functions to shared recovery_collection_validation.py module and updated consumers to import from there

Key Insight: Validation and persistence logic should be centralized in the validation framework module

Learnings#

Key Learnings#

  • Error recovery demo validation supports four quota rules (success_only, replay_only, augmentable_only, replay_and_augmentable) with different strictness levels; changing rules requires batch revalidation of existing data
  • agent-skills provides 20 production-grade engineering skills organized across six development phases (Define: spec/refine; Plan: architecture/TDD; Build: incremental coding; Verify: testing; Review: code review; Ship: deployment), representing codified senior engineering workflows
  • Systematic code deduplication across multiple files can yield significant line reduction (80 lines in this case) while improving maintainability
  • Functions that cache computed values must explicitly invalidate cache when re-computation is needed; determine_counts_toward_target checks for pre-existing counts_toward_target field
  • Claude Code plugin installation follows marketplace-first pattern: use ‘claude plugin marketplace add ’ to add the plugin source, then ‘claude plugin install @’ to install globally

Practical Learnings#

  • Injection replay warnings about missing clean trajectories are UX issues, not data quality issues; fallback to direct state load still produces correct post-injection state
