Problems & Solutions#

Critical Issues#

1. Bot fails to classify multi-task status updates (e.g., ‘A完成了，B也解决了，C需要改时间’), falling back to ‘I don’t understand’ despite substantive user input#

Solution: Added BATCH_UPDATE intent with semantic route (21 utterances), keyword scorer boost for 2+ completion verbs, LLM prompt teaching updates array schema, and handler that searches calendar, deletes completed tasks, reschedules/notes others, logs completions, and emits EventBus events. Also fixed GENERAL fallback to use substantive LLM replies instead of discarding them

Key Insight: Intent classification failures often stem from three-layer gaps: (1) no enum member, (2) no semantic route examples, (3) no LLM prompt fragment. Fixing all three layers plus keyword scoring creates robust classification. GENERAL fallback should preserve useful LLM responses rather than blindly templating.

2. Semantic router scores batch updates at ~0.20 vs 0.50 threshold, causing LOW_CONFIDENCE classification#

Solution: Created BATCH_UPDATE_ROUTE with diverse Chinese/English utterances (‘A完成了，B也搞定了’, ’tasks A and B completed, C needs rescheduling’) and added keyword scorer rule: 2+ distinct completion keywords → +0.30 boost, capturing the defining characteristic of batch updates

Key Insight: Embedding-based semantic routing misses multi-entity status patterns. Explicit keyword counting (2+ completion verbs) is a simple but effective heuristic for batch detection that complements embedding similarity.

3. LLM classifier has no batch_update in allowed intent list, defaults to GENERAL with clarification type, which triggers hardcoded ‘I don’t understand’ template that discards useful AI-generated replies#

Solution: Added batch_update to INTENT_PROMPT_FRAGMENTS with updates array schema (task_name, action, note, new_time) and to allowed intent list in build_prompt(). Improved GENERAL handler to check if clarification-type replies have substantive LLM content (>20 chars, not template phrase) and use it instead of hardcoded template

Key Insight: Fallback templates should be last resort, not default. When LLM generates substantive replies, preserve them even if confidence is low. Template overrides waste the LLM’s reasoning and create worse UX.

Learnings#

Key Learnings#

  • Multi-layered intent classification requires coordinated updates across 8+ files: enum (models.py), semantic routes (intent_routes.py), keyword scorer (keyword_scorer.py), LLM prompts (prompts.py), context assembly (context_assembler.py), dual verify defaults (dual_verify.py), handler + routing (discord_bot.py), plus derived data (route JSON). Missing any layer causes classification failure
  • Parallel agent exploration (3 agents for intent routing, task handling, general protection) is more efficient than sequential file reading for complex diagnostics. Each agent provides focused perspective that synthesizes into complete understanding
  • Fallback templates in AI systems should preserve useful LLM-generated content rather than blindly overwriting it. Check if LLM reply is substantive (>20 chars, not template phrase) before replacing with hardcoded fallback
  • Keyword scoring can capture patterns that embedding similarity misses: 2+ distinct completion verbs is a simple heuristic for batch updates that complements semantic routing. Explicit pattern detection is still valuable in neural systems

Practical Learnings#

  • When implementing features from user-provided plans, reading all affected files in parallel first (rather than one-by-one) enables better understanding of dependencies and allows Claude to batch edits more efficiently
