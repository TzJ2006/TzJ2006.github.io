Problems and Solutions#

Key Problems#

1. Bots cannot classify multi-task status updates (e.g., “A is completed, B is also resolved, C needs time adjustment”), and still revert to “I don’t understand” even with substantial user input.#

Solution: Added BATCH_UPDATE intent, including semantic routing (21 expressions), enhanced keyword scorer for 2 or more completion verbs, LLM prompt fragments for learning update array patterns, and handlers for searching calendars, deleting completed tasks, rearranging/annotating other tasks, recording completion status, and publishing EventBus events. Also fixed the GENERAL default response by using meaningful LLM responses instead of discarding them.

Key Insight: Intent classification failures usually stem from three levels of defects: (1) no enumeration members, (2) no semantic routing examples, (3) no LLM prompt fragments. Solving these three levels along with keyword scoring can create powerful classification capabilities. GENERAL default responses should retain useful LLM responses rather than blindly using templates.

2. The semantic router scores for batch updates are about 0.20, below the threshold of 0.50, resulting in LOW_CONFIDENCE classification.#

Solution: Created BATCH_UPDATE_ROUTE with multiple Chinese/English expressions (“A is completed, B is also done”, “Tasks A and B are completed, C needs rearrangement”) and added keyword scoring rules: 2 or more different completion keywords → increase score by 0.30 to capture batch update features.

Key Insight: Embedding-based semantic routing cannot recognize multi-entity status patterns. Clear keyword counting (2 or more completion verbs) is a simple but effective batch detection heuristic that can complement embedding similarity.

3. The LLM classifier doesn’t have batch updates in the allowed intent list, defaults to GENERAL with clarifying types, which triggers the hardcoded “I don’t understand” template, thereby discarding useful AI-generated responses.#

Solution: Added batch updates to INTENT_PROMPT_FRAGMENTS, including update array patterns (task name, operation, notes, new time), and added it to the allowed intent list in build_prompt(). Improved the GENERAL handler by checking if clarifying responses have meaningful LLM content (>20 characters, not template phrases), and used it instead of the hardcoded template.

Key Insight: Reverting to the template should be a last resort, not the default option. When the LLM generates meaningful responses, even if the confidence is low, they should be retained. Template overrides waste the LLM’s reasoning ability, leading to worse user experiences.

Learning Points#

Key Learnings#

  • Multi-level intent classification requires coordinated updates of 8 or more files: enumeration files (models.py), semantic routing files (intent_routes.py), keyword scorers (keyword_scorer.py), LLM prompt files (prompts.py), context assembly files (context_assembler.py), double verification default values (dual_verify.py), handler + routing files (discord_bot.py), and derived data (route JSON). Missing any layer will lead to classification failure.
  • Parallel agent exploration (3 agents responsible for intent routing, task handling, and overall protection) is more efficient than sequential file reading, suitable for complex diagnostic tasks. Each agent provides a focused perspective, ultimately leading to a complete understanding.
  • Backtracks in AI systems should retain useful LLM-generated content, rather than blindly overwriting it. Before replacing the hardcoded backtrack content, it’s important to check if the LLM response has meaningful content (>20 characters, not template phrases).
  • Keyword scoring can capture patterns that embedded similarity cannot capture: 2 or more different completion verbs is a simple batch update heuristic that can complement semantic routing functions. In neural systems, explicitly detecting patterns remains valuable.

Practical Learnings#

  • When implementing the functions outlined in the user’s plan, it’s better to read all affected files in parallel, rather than individually. This helps understand dependencies and enables Claude to edit files more efficiently in batches.
