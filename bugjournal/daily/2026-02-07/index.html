Problems and Solutions#

Key Issues#

1. ARI performance in the STAIG strict mode (approximately 0.09) is much lower than that of the original STAIG (approximately 0.45+), despite claims of full alignment#

Solution: Through three-way parallel exploration (MIHD implementation, original STAIG code, image preprocessing process), we identified and fixed four key differences. In section 151673, ARI’s performance improved to 0.5435.

Key Insight: The original STAIG’s image preprocessing process only performed Resize→ToTensor operations, without performing ImageNet normalization. This had a fundamental impact on the embedding distribution of the visual encoder, thereby affecting subsequent steps such as PCA, pseudo-labeling, and edge weights.

2. mclust clustering failure: ‘Error in svd(data, nu = 0) : a dimension is zero’ (appeared in section 151676)#

Solution: Encapsulated numpy2ri.activate()/deactivate() within try/finally blocks. Added float64 conversions, NaN/Inf clearing, and removed zero-variance columns.

Key Insight: The original STAIG called mclust only once per training session, while the benchmark loop called 12 times. rpy2’s activate() was not automatically deactivated, causing damage to the conversion stack and ultimately resulting in R receiving zero-dimensional matrices.

General Issues#

3. Patch extraction is very slow, taking a long time to process individual parts#

Solution: Converted np.array(pil_img) inside the loop to an external loop (approximately 3500 times). Used numpy slicing to extract patches and pre-allocated output tensors.

Key Insight: The real bottleneck was the repeated full-image conversion of large images with dimensions of 20000×20000, not the processing logic for individual patches. Identifying bottlenecks is more important than uniformly optimizing them.

4. ‘RM-ideal score’ definition cannot be found in the code library or arXiv paper#

Solution: Found ‘RM score’ = Region Matching Score in QueST’s source code (QueST-paper/QueST/QueST.md). This is based on WWL Wasserstein Weisfeiler-Lehman graph kernel. The user has a separate note file RM_Ideal_implementation_notes.md, and this implementation will be integrated later.

Key Insight: RM-ideal may be a user-defined extension to the metric, rather than the metric defined in the paper. Minor differences in terminology can lead to complete search failures.

Today’s Summary#

Key Points#

  • When replicating deep learning papers, image preprocessing processes (especially whether normalization was performed) are details that can easily be overlooked but have a significant impact. Do not assume that “standard practices” are consistent with the original implementation; compare each step separately.
  • Multiple small implementation differences can lead to significant performance improvements (ARI improved from 0.09 to 0.5435). Systematic three-way comparison (original implementation, new implementation, configuration) is more reliable than checking each difference individually.

Practical Points#

  • rpy2’s numpy2ri.activate() must be used together with deactivate(). When calling R functions inside loops, if the conversion lifecycle is not handled properly, it can lead to unpredictable failures.
  • Identifying bottlenecks during performance optimization is crucial: Removing the complete image numpy conversion from the loop (from 3500 times to 1 time) is more effective than any other optimization. This demonstrates the practical value of Amdahl’s Law.
