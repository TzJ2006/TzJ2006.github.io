Problems & Solutions#

Critical Issues#

1. ARI dropped from 0.4+ to 0.13, unclear why previous configuration succeeded#

Solution: Version comparison revealed code had been modified to pass 3000D raw gene expression instead of 50D PCA embeddings to STAIG training; reverted to using PCA embeddings

Key Insight: run_benchmark_core.py contained an erroneous ‘fix’ that passed raw_gene_expr (3000D) when gene_encoder=‘pca’, breaking the original workflow that used 50D PCA features

2. Test results show ARI=0.13 instead of expected 0.4 even after fixes#

Solution: Identified that tests ran in base/AI environment with kmeans clustering; historical success used General environment with rpy2+mclust clustering

Key Insight: Clustering method (mclust vs kmeans) has massive impact on downstream metrics; environment configuration is critical for reproducibility

3. Configuration file parameters (num_epochs, learning_rate, num_hidden) not being read correctly#

Solution: Updated STAIGFusionTrainer.init to read all training parameters from staig_alignment_config, not just tau and num_layers

Key Insight: Incomplete config parsing caused training with wrong hyperparameters despite correct YAML configuration

4. edge_weight_method defaulted to wrong value (‘softmax_log’ vs ‘softmax_neglog’)#

Solution: Changed default from ‘softmax_log’ (distance large → weight large) to ‘softmax_neglog’ (distance small → weight large) to match proven high-performance configuration

Key Insight: Edge weight directionality significantly impacts graph-based learning; STAIG uses log(distance) while optimal MIHD config uses -log(distance)

5. MIHD GCNGeneEncoder architecture diverged from original STAIG implementation#

Solution: Added apply_activation_to_last_layer parameter (default True) to align with STAIG’s design where all layers including the last apply activation

Key Insight: Subtle architectural differences (last layer activation) can accumulate and cause divergence from reference implementations

Learnings#

Key Learnings#

  • In machine learning regression debugging, environment configuration (conda environment, clustering libraries) can dominate code-level fixes; always verify execution environment matches historical success
  • When debugging performance regressions, git history comparison with last known good version is often faster than forward-fixing current code; version archaeology should be early in the workflow
  • Input dimensionality mismatches (50D PCA vs 3000D raw features) can cause catastrophic performance degradation in graph neural networks; always validate feature dimensions match model expectations
  • Seemingly minor architectural differences (last layer activation, edge weight direction) accumulate to cause significant divergence when replicating research implementations; exhaustive parameter alignment is critical
  • Clustering method choice (mclust vs kmeans) has first-order impact on spatial transcriptomics evaluation metrics; R integration via rpy2 is often necessary for reproducing bioinformatics paper results
  • Parallel agent deployment for cross-codebase comparison is highly effective for systematic architectural analysis; two agents exploring STAIG and MIHD simultaneously provided comprehensive comparison in one round
