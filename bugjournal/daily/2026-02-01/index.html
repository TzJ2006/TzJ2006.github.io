Issues and Solutions#

Key Issues#

1. ARI dropped from 0.4+ to 0.13, unclear why previous configurations worked#

Solution: Version comparison showed that the code was modified, passing 3000D raw gene expression instead of 50D PCA embeddings to STAIG training; reverted to using PCA embeddings.

Key Insight: run_benchmark_core.py contained incorrect “fix” code, passing 3000D raw gene expression when geneencoder=‘pca’, disrupting the workflow that originally used 50D PCA features.

2. Even after fixing, test results showed ARI at 0.13 instead of expected 0.4#

Solution: Found that tests were conducted using kmeans clustering in the base/AI environment; historically, successful cases were achieved using rpy2+mclust clustering in the General environment.

Key Insight: Clustering methods (mclust vs kmeans) have a significant impact on downstream metrics; environment configuration is crucial for reproducibility.

3. Configuration parameters (num epochs, learning rate, num_hidden) were not read correctly#

Solution: Updated STAIGFusionTrainer.init to read all training parameters from staig_alignment_config, not just tau and num_layers.

Key Insight: Incomplete configuration parsing led to incorrect hyperparameters being used despite correct YAML configuration.

4. Edge_weight_method default value was incorrect (‘softmax_log’ vs ‘softmax_neglog’)#

Solution: Changed the default value from ‘softmax_log’ (distance large → weight large) to ‘softmax_neglog’ (distance small → weight large) to match proven high-performance configurations.

Key Insight: The direction of edge weights significantly affects graph-based learning; STAIG uses logarithmic distances, while optimal MIHD configurations use negative logarithmic distances.

5. MIHD GCNGeneEncoder architecture did not align with the original STAIG implementation#

Solution: Added apply_activation_to_last_layer parameter (default True) to align with STAIG design, where activation functions are applied to all layers including the last layer.

Key Insight: Minor architectural differences (last layer activation) can accumulate and lead to differences from the reference implementation.

Lessons Learned#

Key Lesson Learned: In machine learning regression debugging, environment configuration (such as conda environments, clustering libraries) may be more important than code-level fixes; ensure the execution environment matches historical successful cases.#

When debugging performance regression issues, comparing git history with the last known good version is usually faster than directly modifying the current code; version tracing should be an early step in the workflow. Mismatch in input dimensions (50D PCA vs 3000D raw features) can lead to serious performance degradation in graph neural networks; ensure feature dimensions match model expectations. Seemingly minor architectural differences (last layer activation, edge weight direction, etc.) can accumulate and lead to significant differences when replicating research implementations; comprehensive parameter alignment is crucial. The choice of clustering methods (e.g., mclust vs kmeans) has a significant impact on spatial transcriptomics evaluation metrics; to replicate the results of biological informatics papers, it is usually necessary to integrate with R using rpy2. The deployment of parallel proxies for comparing across codebases is very effective for systematic architecture analysis; two proxies simultaneously exploring STAIG and MIHD provided a comprehensive comparison result during one round of testing.
