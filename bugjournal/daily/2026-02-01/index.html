Issues and Solutions#

Key Issues#

1. STAIG fusion benchmark test: ARI dropped from 0.4 to 0.13#

Solution: Identified that the root cause was an error in the code that passed 3000-dimensional original gene expression to STAIG training, rather than using 50-dimensional PCA embedding. Removed the raw_gene_expr logic from run_benchmark_core.py.

Key Insight: STAIG performance is highly sensitive to gene feature dimensions. Using features reduced by PCA (50 dimensions) is crucial, not original expression data (3000 dimensions).

2. Missing rpy2 dependency caused inability to use mclust clustering method, forcing the choice of kmeans#

Solution: Found that rpy2 3.5.11 was installed in the General environment. Tests should be conducted in the General environment, not in the base/AI environment.

Key Insight: Environment selection is important: The General environment has rpy2 required for mclust, which may explain the gap between the current 0.13 ARI (kmeans) and the target of 0.4+ ARI (mclust).

3. MIHD GCNGeneEncoder only applies activation function to non-final layers, deviating from the original STAIG design#

Solution: Modified the forward propagation to apply activation functions to all layers by default (added the apply_activation_to_last_layer=True parameter for backward compatibility).

Key Insight: The original STAIG applied activation functions to all GCN layers, including the last layer (net.py:29-32). This architectural detail has an impact on representation learning.

General Issues#

4. Configuration parameters (num_epochs, learning_rate, num_hidden) were not read from the staig_alignment YAML configuration#

Solution: Added code to read all training parameters from the config['training'] section, rather than just tau and num_layers.

Key Insight: Incomplete configuration parsing led to using default values instead of verified hyperparameters from the alignment configuration.

Lessons Learned#

Key Lesson - Key Takeaway: When debugging ML performance regression issues, first compare complete configurations (environment, dependencies, hyperparameters, data preprocessing) with known good states, rather than just code changes.#

  • Gene feature dimensions have a significant impact on STAIG performance: 50-dimensional PCA embedding works well, but 3000-dimensional original expression leads to severe performance degradation (ARI dropped from 0.33 to 0.09).
  • Insights from STAIG architecture: The original STAIG applied activation functions to all GCN layers, including the last layer (different from GNN implementations that usually ignore final activation functions). This detail is crucial for reproducing the problem.
  • Clustering method selection is critical: mclust (which requires rpy2/R) performs significantly better than kmeans in STAIG embedding. The expected ARI was above 0.4, but the actual observation was 0.13.
  • Key differences between STAIG and MIHD GCN implementations: Activation method for the last layer, way of calculating edge weights (softmax(log(d) vs softmax(-log(d))), graph construction efficiency (O(n² vs O(n log n) with KD-tree).
