Problems and Solutions#

Key Issues#

1. Independent PCA/STAIG per-section generate non-comparable embedding spaces, resulting in 10 groups having SL@50=0 across 14 combinations in cross-slice retrieval; normalization cannot fix this issue.#

Solution: Use a pre-trained Foundation Model (scGPT): All slices share the same model weights, ensuring that SL@50 increases from 0.013 to 0.416, and accuracy improves from 14% to 100%.

Key Insight: The root cause of failure is not insufficient gene feature information (Raw HVG’s 86% accuracy proves there is information available), but rather different principal component axes in per-section PCA make cosine similarity meaningless. Normalization cannot solve the coordinate system inconsistency problem (mathematically impossible). Foundation Model is the only correct zero-shot solution.

2. CompositeBodyObject gets scattered during env.reset() (lid floating, base offset 0.1-0.2m), but behaves normally when replayed via HDF5 (set_sim_state_flat covers body position to hide the problem).#

Solution: Add a self.locations_relative_to_corner instance property storage in init of generated_objects.py (with assertions) and restore the coordinate transformation logic from corner to center during append_object(), consistent with MimicGen commit 398af01b.

Key Insight: set_sim_state_flat() overwrites all bodies’ pos/quat when restoring saved state from HDF5. The root cause was accurately identified through git diff.

3. High cost of VLA error recovery data collection: 6 tasks × 29 subtypes × 10 entries = 1740 human demonstrations.#

Solution: Divide 29 error subtypes into 5 RBG groups based on recovery motion primitives, allowing demos across subtypes to share augmentation; MimicGen generates over 1000 demos from 10 source demos; the total demand is reduced to 329 entries (81% savings).

Key Insight: Error type classification (by triggering reason) and recovery behavior classification (by motion primitive) are two different dimensions. The latter’s structural similarity is key to implementing cross-error-type data reuse.

4. Semantic Scholar academic database incorrectly combines papers by multiple researchers, leading to profiles across completely unrelated fields with severely distorted h-index statistics (140 papers but only h-index of 4; renowned professor Pieter Abbeel mis-matched as a person with h=4).#

Solution: Two-tier approach: (1) Analysis layer: LLM actively identifies confusion and warns, verifying through three-dimensional signal cross-validation of h-index/number of citations/field span; Tool layer: Reconstructing disambiguation scoring weights (quantitative indicators overriding string similarity) + adding a three-level parsing chain (exact ID→paper reverse search→name search) + –author-id CLI parameters for manual specification.

Key Insight: Three signals for detecting name confusion: abnormal ratio of h-index to paper count, impossible coexistence of field span at the methodological level, contradiction between citation statistics and publication years. Disambiguation core: The gap in academic output among similar scholars is often a matter of orders of magnitude; quantitative indicators should dominate disambiguation.

5. UNI2 visual Foundation Model performs unexpectedly poorly (71% accuracy rate, some directions have negative Spearman r).#

Solution: Accept UNI2’s limitations in cross-sample scenarios and use scGPT as the main model. Root cause: H&E histology images have staining/ preparation batch effects, making visual features unreliable across samples.

Key Insight: Performance difference between Gene FM and Vision FM (100% vs 71%) reveals that gene expression has stronger consistency across samples compared to morphological images, providing general guidance for multi-modal FM selection.

6. S2 API has a limit of 429 when限速，_s2_request() recursion retries without a stop condition, causing the program to hang indefinitely; Yiran Chen failed the paper reverse search due to speed limit, leading to incorrect matching with a medical scholar with h=10 instead of h=65.#

Solution: Change recursion to a for loop + exponential backoff (5→10→20→40→60 seconds) + throwing S2RateLimitError after up to 5 retries; during speed limits, use WebSearch to find the correct authorId (5442167), using –author-id parameter to bypass disambiguation.

Key Insight: Recursion is a potential resource leak hazard; there needs to be a downgrade option for external API limits (manual ID specification). This confirms the necessity of adding –author-id parameters.
