Lessons Learned#

Key Lessons - The Critical Advantage of Basemodels in Cross-Sample Tasks Lies in Their Ability to Provide Shared Embedding Spaces Without Joint Training; Pre-trained Weights Ensure Comparability for Any Input. Zero-shot Cross-Sampling Retrieval Essentially Depends on This Feature—Any Optimization for Each Sample Cannot Replace It.#

  • Name ambiguity is the primary data quality issue in automated character profiling—affecting approximately 30% of analysis subjects. Multiple signal approaches are needed: h-index and domain consistency, citation time distribution, institutional metadata, collaborator network analysis. A single data source is not enough.
  • Three-Layered Strategy for Restoring Data Augmentation: Scene configuration (using different error scenarios as initial states), cross-degree (rotating perturbations from D0 to D1/D2), cross-type (cross-enhancement within RBG). Combined with MimicGen trajectory segmentation technology, data requirements can be reduced by 81% (from 1,740 samples to 329 samples).
  • Coordinate transformation errors in composite model systems can be masked through a state replay mechanism—this mechanism covers the positions defined in XML, only becoming apparent at new initializations. This explains long-standing errors that are difficult to detect through visual inspection during demonstrations.
  • Why Not Before: The “Before” counterfactual reasoning framework reveals the time dependence between technical prerequisites (GPU/pre-trained models), data availability (large-scale datasets), and theoretical foundations (new concepts). Academic achievements highly depend on matching “technical stack maturity” and “time window”—not just individual talent.
  • Users prefer using the Claude Cli backend locally (claude –print) rather than the cloud API with keys. This preference must be remembered when making future gadget/research invocations. Repeated authentication failures should trigger questions about backend assumptions.
  • Award recognition based on LLMs has basic recall limits without real-time API integration—training data cannot keep up with conference papers. A hybrid approach is needed: (1) LLM screening candidates; (2) verification using DBLP/OpenReview APIs; (3) manual resolution of conflicts.
  • Paradigm Shifts in the Robotics Field Between 2016-2023: Deep Reinforcement Learning (2016-2019) → Large-Scale Datasets for Cross-Perspective Performance (2020-2022) → Vision Language Models-Based Models (since 2023). Each period lasted only 2-3 years; researchers working during these transitions had disproportionately significant impacts.
  • Golden Rule of Hot Path Optimization: Read paths assume resources exist, write paths create them. DiskCache mkdir has been moved from read operations (frequently called path_for) to write operations (rarely called put()), resulting in a 10-fold efficiency improvement. This rule applies to all persistent layers such as file systems, databases, and caches.
  • Background Bash tasks in Claude Code do not inherit from conda init; ‘conda run -n ENV’ or absolute paths (${CONDA_PREFIX}/bin/python) must be used. In non-interactive sub shells, directly calling conda activate causes a CondaError.
  • Structured output verification should be separated from the generation process—the two-stage “generate then fix” pattern is more suitable for complex nested patterns than single perfect outputs; this makes it possible to have an unknowable verification layer.
  • Restoring Behavioral Group Abstraction: Classify 29 types of errors based on required motion primitives (re-grasping, retrieval, retraction, redirection, rearrangement), rather than classifying them by type. This allows sharing data within the same group—this is more practical than collecting data separately by error type.
