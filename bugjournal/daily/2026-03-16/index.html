Problems & Solutions#

Critical Issues#

1. Per-section independent PCA/STAIG training produces incomparable embedding spaces, causing cross-section retrieval to fail (14-29% hit rate)#

Solution: Use Foundation Models (scGPT gene FM, UNI2 vision FM) that apply same pretrained weights to all sections, producing naturally comparable embeddings. scGPT achieves 100% hit rate with avg SL@50=0.416

Key Insight: Zero-shot cross-section retrieval requires shared embedding space; joint PCA/Procrustes/joint-training all violate zero-shot constraint by requiring both sections’ data. Only pretrained FMs provide true zero-shot capability. User’s immediate recognition of this constraint prevented wasted effort on non-viable solutions.

2. Scholar database name disambiguation failure - multiple researchers with identical names being merged into single profiles with incoherent paper lists spanning 5-6 unrelated fields (affects ~30% of analyzed researchers)#

Solution: Added explicit warning mechanism in trajectory_summary field flagging h-index/citation discrepancies and field inconsistencies; recommended pre-processing step to cluster papers by semantic similarity, institutional affiliation, co-author network before analysis

Key Insight: Name collision is systematic issue requiring multi-signal approach: h-index vs field coherence, citation temporal distribution, institutional metadata, co-author network analysis. Single data source (Semantic Scholar) insufficient for reliable disambiguation.

3. Coffee machine model appeared disassembled in simulation (lid floating, base offset) during env.reset(), but HDF5 demo replay showed correct assembly#

Solution: Traced MimicGen commit 398af01b to discover missing locations_relative_to_corner coordinate transformation. Restored two code blocks: instance attribute storage with assertion, and coordinate transformation in _append_object() converting corner-relative to center-relative positions

Key Insight: Bug was masked by set_sim_state_flat() overwriting XML body positions with saved states. Only env.reset() (which generates new initial states from XML) exposed the incorrect placement. This explains 3-month latency despite daily use.

4. Collecting error recovery demos for 6 tasks × 29 subtypes requires 1,740 samples (174×10) which is prohibitively expensive#

Solution: Group 29 subtypes into 5 Recovery Behavior Groups (RBGs) based on motion primitives (re-grasp, re-navigate, back-approach, etc.); collect 5-10 demos per RBG per task; use MimicGen/IntervenGen augmentation to generate 100+ samples per subtype

Key Insight: Recovery motions share structure across error types; augmentation via trajectory segmentation + object/pose randomization can scale 10 source demos to 100+ training samples while preserving recovery skill.

5. Research profiler failed with ‘Please set ANTHROPIC_API_KEY’ when using –api anthropic backend after 4 rounds of environment fixes#

Solution: User explicitly instructed to switch to –api claude_cli backend, which calls local ‘claude –print’ command instead of cloud API

Key Insight: AI spent 4 rounds fixing environment variables, conda paths, and module installations while assuming –api anthropic was correct; never questioned the backend choice until user intervened. Pattern of authentication failures should have triggered root cause questioning earlier.

6. LLM award identification has near-zero recall - 12 out of 13 queries returned empty results due to conservative ‘only mark if certain’ policy#

Solution: Current approach prioritizes precision over recall; LLM correctly refuses to hallucinate awards but lacks access to up-to-date conference proceedings; human accepted this tradeoff

Key Insight: Award metadata is sparse in training data and highly dynamic (new conferences yearly); reliable award detection requires integration with live APIs (OpenReview, DBLP) rather than pure LLM generation.

7. discover_homepage_urls() function called LLM even when S2 homepage already existed, wasting API cost#

Solution: Added early return logic: when S2 homepage available, directly skip LLM call phase

Key Insight: LLM calls are expensive operations; should only serve as fallback after all deterministic data sources (S2 API, homepage fields) fail, not run unconditionally.

8. JSON format instability when generating Chinese analysis with nested structures - 6 outputs (20%) required manual repair via dedicated fixing prompts#

Solution: Implemented two-stage approach: (1) generate analysis, (2) pass through JSON repair prompt if parsing fails; human maintained strict schema enforcement requiring exact key names

Key Insight: Chinese-language nested JSON is at edge of Claude’s format stability; adding explicit schema validation prompt with example structure significantly improves compliance. English JSON is far more reliable.

9. DiskCache._path_for() executed mkdir on every read operation, producing unnecessary filesystem calls in hot path#

Solution: Moved mkdir logic to put() method (write path only); read path assumes directory exists or file doesn’t exist

Key Insight: Hot path optimization golden rule: read path assumes resources exist, write path creates resources. Applies to filesystem, database, cache - all persistence layers.

Learnings#

Key Learnings#

  • Foundation Models’ key advantage for cross-sample tasks: provide shared embedding space without needing joint training; pretrained weights guarantee comparability across arbitrary inputs. Zero-shot cross-section retrieval fundamentally requires this property - no amount of per-sample optimization can substitute.
  • Name disambiguation is #1 data quality issue in automated scholar profiling - affects ~30% of analyzed researchers. Requires multi-signal approach: h-index vs field coherence, citation temporal distribution, institutional metadata, co-author network analysis. Single data source insufficient.
  • Recovery demo data augmentation three-layer strategy: scene configuration (different error scenes as start states), cross-degree (D0→D1/D2 via rotation perturbation), cross-subtype (within-RBG cross-augmentation). Combined with MimicGen trajectory segmentation, reduces data requirements 81% (1,740→329 demos).
  • Coordinate transformation bugs in composite model systems (robosuite’s CompositeBodyObject) can be masked by state replay mechanisms (set_sim_state_flat) that overwrite XML-defined positions, surfacing only during fresh initialization (env.reset). Explains long-latent bugs passing visual inspection during demo playback.
  • ‘Why not before’ 反事实推理 framework reveals breakthrough timing dependencies: technology prerequisites (GPU/pretrained models), data availability (large-scale datasets), theoretical foundations (new concepts). Academic success highly dependent on ’technology stack maturity’ and ’time window’ matching - not just individual genius.
  • User’s preferred workflow: local claude_cli backend (claude –print), not cloud API with keys. Must remember this preference for future gadget/research invocations. Pattern of repeated auth failures should trigger backend assumption questioning.
  • LLM-based award identification without live API integration has fundamental recall limitations - training data cannot keep pace with conference proceedings. Hybrid approach needed: (1) LLM screens candidates, (2) DBLP/OpenReview API verifies, (3) human resolves conflicts.
  • Robotics field paradigm shifts 2016-2023: deep imitation learning (2016-2019) → cross-embodiment large-scale datasets (2020-2022) → vision-language-action foundation models (2023+). Each window only 2-3 years; researchers who pivot during transitions capture disproportionate impact.
  • Hot path optimization golden rule: read path assumes resources exist, write path creates resources. DiskCache mkdir moved from read (_path_for called frequently) to write (put() called rarely) for 10x efficiency gain. Applies to filesystem, database, cache - all persistence layers.
  • Background Bash tasks in Claude Code don’t inherit conda init; must use either ‘conda run -n ENV’ or absolute Python executable paths (${CONDA_PREFIX}/bin/python). Direct conda activate fails with CondaError in non-interactive subshells.
  • Structured output validation should be architecturally decoupled from generation - two-stage ‘generate then repair’ pattern more robust than single-shot perfect output for complex nested schemas; enables model-agnostic validation layer.
  • Recovery Behavior Groups (RBG) abstraction: classify 29 error subtypes by required motor primitives (re-grasp, retrieve, retract, redirect, realign) rather than error taxonomy. Enables data sharing within groups - more practical than per-error-type collection.
