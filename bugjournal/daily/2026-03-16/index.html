Problems and Solutions#

Key Problems#

1. Independent PCA/STAIG training for each part results in an uncomparable embedding space, leading to failure in cross-part retrieval (recall rate of 14-29%)#

Solution: Use base models (scGPT gene FM, UNI2 visual FM). These models apply the same pre-trained weights to all parts, resulting in a naturally comparable embedding space. ScGPT has a recall rate of 100%, with an average SL@50 of 0.416.

Key Insight: Zero-shot cross-part retrieval requires a shared embedding space. Joint PCA/Procrustes/joint training require data from both parts, thus violating the zero-shot constraint. Only pre-trained base models can provide true zero-shot capabilities. Users immediately realized this constraint, avoiding wasting time on infeasible solutions.

2. Failure in handling name ambiguity in scholar databases – Multiple researchers with the same name were merged into one profile, and their paper lists covered 5-6 unrelated fields (affecting about 30% of the analysis subjects)#

Solution: A clear warning mechanism was added to the trajectory_summary field, indicating inconsistencies in h-index/citation count and field consistency. It was recommended to cluster papers based on semantic similarity, institutional affiliation, and author networks before analysis.

Key Insight: Name conflicts are a systematic problem that requires multi-signaling approaches: h-index and field consistency, citation time distribution, institutional metadata, and author network analysis. A single source of data (Semantic Scholar) is insufficient for reliable ambiguity handling.

3. During env.reset(), the coffee machine model appears to be disassembled during simulation (lid floating, base offset), but the HDF5 playback shows correct assembly status#

Solution: The commit 398af01b of MimicGen was tracked, and the missing position_relative_to_corner coordinate transformation was found. Two code blocks were restored: instance attribute storage with assertions, and the coordinate transformation in append_object(). The relative corner positions were converted to relative center positions.

Key Insight: Since set_sim_state_flat() overwrote the XML body position, the error was masked. Only envReset() (generating a new initial state from XML) exposed the incorrect placement. This explains the 3-month delay despite daily use.

4. Collecting error recovery demonstrations for 6 tasks × 29 sub-types requires 1,740 samples (174 × 10), which is very expensive#

Solution: The 29 sub-types were divided into 5 recovery behavior groups based on motion primitives. Each task collected 5-10 demonstrations per group. MimicGen/IntervenGen enhancement techniques were used to generate over 100 samples per sub-type.

Key Insight: Recovery actions have the same structure across different error types. By using trajectory segmentation + object/pose randomization, 10 source demonstrations could be extended to over 100 training samples while maintaining recovery skills.

5. After 4 rounds of environment fixing using –api anthropic backend, the research概况器 encountered the error “Please set ANTHROPIC_API_KEY”#

Solution: Users explicitly instructed to switch to the –api Claude CLI backend, which calls the local ‘claude –print’ command instead of the cloud API.

Key Insight: The AI spent 4 rounds fixing environment variables, conda paths, and module installations, assuming –api anthropic was correct. The authentication failure pattern should have triggered a investigation into the root cause earlier.

6. Recall rate of LLM award recognition is almost zero – Due to a conservative “only marking in specific cases” policy, 12 out of 13 queries returned empty results#

Solution: The current approach prioritizes accuracy over recall. The LLM correctly rejected awards, but due to lack of access to the latest conference records, it couldn’t accurately identify awards.

Key Insight: The metadata for winning awards in the training data is very sparse and dynamic (new conferences are added every year). To reliably identify awards, integration with real-time APIs (OpenReview, DBLP) is needed, rather than relying solely on LLM generation.

7. The discover_homepage_urls() function still calls the LLM when the S2 homepage already exists, wasting API resources#

Solution: A pre-rejection logic was added: when the S2 homepage already exists, the LLM call phase is skipped.

Key Insight: LLM calls are very expensive operations. They should only be used as a backup when all deterministic sources (S2 API, homepage fields) fail.

8. When generating Chinese analysis, the nested JSON format is unstable – 6 out of 20 outputs need to be manually repaired using special prompts#

Solution: A two-step approach was implemented: (1) generating the analysis; (2) if parsing fails, using JSON repair prompts. Humans need to strictly follow the patterns to ensure correct key names.

Key Insight: Chinese nested JSON is at a marginal level of stability in Claude’s format. Adding explicit pattern validation prompts with example structures can significantly improve compliance. English JSON is more reliable.

9. DiskCache._path_for() executes mkdir operations every time it reads, causing unnecessary file system calls in hot paths#

Solution: The mkdir logic was moved to the put() method (only for writing paths). The read path assumes the directory exists or the file does not exist.

Key Insight: The golden rule for hot path optimization is: read path assumes resources exist, write path creates resources. This applies to all persistent layers like file systems, databases, and caches.
