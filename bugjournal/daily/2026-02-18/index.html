Key Issues#

1. Pi0/LIBERO checkpoints completely failed in PickPlace tasks (zero success rate), VLA cannot be generalized across tasks.#

Solution: Plan to use MimicGen to expand the source demonstration data from 10 to 500, and then fine-tune the dedicated model using Pi0.5 LoRA. Key Insight: In the robosuite environment, VLA needs to be benchmarked against weights for specific tasks; generic pre-trained weights cannot be generalized across tasks.

2. The can/square/transport task directory in robomimic_checkpoints is empty, missing key benchmark checkpoints.#

Solution: Need to retrain or download the pre-trained weights for the corresponding task. Currently, only the Lift task can be used. Key Insight: Although the checkpoint directory structure was planned, actual model files were missing, affecting the comprehensive evaluation of multi-task benchmarks.

3. Basic contrastive training is very slow, each part takes about 1 hour, resulting in a total time of over 10 hours for 88 experiments.#

Solution: Wait for the background process to complete and use this time to handle other tasks on Tiansheng. Key Insight: The computational cost of conditional output buffering is much higher than that of simple fusion strategies (concat/mean), which becomes a bottleneck in benchmark test runtime.

4. Conditional output buffering mechanism prevents real-time monitoring of the process progress.#

Solution: Use file system polling (find -mmin) to monitor the modification time of intermediate files, indirectly determining the process progress. Key Insight: By observing the addition of new content in the output file, it is possible to bypass the buffer of the conda run. This actually monitors the completion status at the part level, rather than real-time progress.

Today’s Summary#

Main Conclusions#

  • Coordinate Correction Effect: Concatenated fusion achieved an ARI of 0.2751, exceeding the PCA baseline of 0.2613. This indicates that the corrected visual features effectively supplemented gene expression information. The improvement in uni2 alone increased from 0.065 to 0.121 (+86%), confirming that before correction, patch errors were the main performance bottleneck.
  • UNI/UNI2 Patch Extraction Standards: Input size is 224×224 pixels, patch size = 14, ImageNet normalization; HEST-1k uses a region of 112µm×112µm with a resolution of 0.5µm/px = 224×224 pixels. These are standard practices in the field of space transcriptomics.
  • Robosuite Environment: VLA failed to achieve generalization across tasks (pi0_libero had a success rate of 0% in PickPlace), requiring fine-tuning for specific tasks. The MimicGen pipeline was tested on zhaoganlong (10 source demonstrations → 500 generated demonstrations → LeRobot format → LoRA fine-tuning), which is a proven feasible path.

Actual Summary#

  • STAIG-style contrastive learning is the computational bottleneck in multimodal fusion benchmarks, with each part taking about 1 hour. This should be considered when designing benchmarks; possibly requiring separate grouping or increasing computing resources.
  • Best practices for managing checkpoints in error recovery benchmarks: Use symbolic link directories to manage model weights across multiple paths. The configuration file only references symbolic link paths, facilitating migration and maintenance.
