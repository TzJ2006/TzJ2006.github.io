Problems and Solutions#

Key Problems#

1. Due to the X/Y coordinate swapping in tissue_positions_list.csv parsing, visual embeddings are extracted from transposed image positions#

Solution: Changed column names to pxl_row/pxl_col (semantic names), swapped coordinate order (X, Y) = (pxl_col, pxl_row), cleared visual cache, and ran experiments again

Key Insight: Spatial data requires explicit Y as row and X as column semantics; a 10-fold scale factor verification can detect coordinate system errors early

2. Pi0_libero checkpoints have a 0% success rate on PickPlace tasks, but 96-100% on Coffee/Stack tasks#

Solution: Identified the root cause: Pi0_libero is fine-tuned for the LIBERO domain, lacking PickPlace-specific training data (only 10 source demonstration samples, while other tasks have 500); proposed a process of MimicGen → LeRobot → LoRA

Key Insight: VLA’s cross-task generalization fails when the task domains differ (LIBERO vs PickPlace has different object/scene layouts); fine-tuning requires more than 500 demonstration samples, which can be expanded from 10 source samples using MimicGen

General Problems#

3. Unable to visually inspect VLA policy behavior on PickPlace tasks, making it impossible to diagnose the 0% success rate#

Solution: Created visualize_policy_rollout.py script, supporting EGL off-screen rendering, frame-by-frame capture, HUD overlay (policy/steps/reward/status), and MP4 export functionality for each release

Key Insight: Video releases can reveal behavior patterns (e.g., random failures vs purposeful failures), and success rate alone cannot distinguish

4. Uneven distribution of checkpoint paths in multiple HDD_POOL directories, making configuration management fragile#

Solution: Created a centralized checkpoints/ directory, using task-organized symbolic links (bc_rnn/pick_place, vla/pi0_libero, robomimic/lift), and updated all configurations to use relative paths

Key Insight: Symbolic link-based checkpoint organization allows atomic path updates, while clearly distinguishing original storage from project references

AI Limitations#

General Limitations#

  • Before recommending Pi0_libero, no active search for task-specific VLA checkpoints was conducted, requiring manual prompting to investigate zhaoganlong’s complete checkpoint directory
  • Multiple searches were needed to find that zhaoganlong’s dataset was organized by task (Coffee/Stack), with 500 generated demonstration samples per task, with initial focus on directly finding PickPlace

Lessons Learned#

Key Lessons - In medical imaging, spatial coordinate systems require clear semantic naming (pxl_row=Y, pxl_col=X), verified with physical units (pixel × scaling factor = micron); generic pixel_x/pixel_y naming is error-prone.#

  • Integration of visual base models is crucial in multimodal learning, relying on spatial alignment; even if the architecture is correct, incorrect segment extraction (from transposed positions) can lead to half the performance.
  • VLA models trained in one task domain (e.g., LIBERO) have little generalization ability in different operating environments, despite similar action spaces; task-specific fine-tuning is essential.
  • MimicGen can expand 10 human demonstrations to 500 training examples through automatic mutation generation, matching the data size used for successful VLA fine-tuning (Coffee: 96%, Stack: 100%).

Practical Lessons#

  • Checkpoint management based on symbolic links enables version control of path references while preserving the original data locations; this is important for collaboration on shared file systems.
