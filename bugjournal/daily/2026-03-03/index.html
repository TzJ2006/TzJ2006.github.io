Problems & Solutions#

Critical Issues#

1. SwiftUI .ultraThinMaterial showed fixed color instead of transparent glass over video#

Solution: SwiftUI material uses .behindWindow blend mode (samples desktop behind window) not .withinWindow (samples same window content). Replaced with NSViewRepresentable wrapping NSVisualEffectView with explicit .withinWindow mode and .clear backgroundColor on NSHostingView

Key Insight: SwiftUI abstractions hide critical rendering pipeline details; NSView-level control required for complex visual effects like within-window transparency

2. 151676’s STAIG fusion embeddings all zeros (embedding similarity all 0, Spearman r is NaN), model completely collapsed#

Solution: Added np.allclose(embeddings, 0) detection in load_fused_embeddings; if true, print warning and automatically trigger –recompute_fusion path to reload gene/vision embeddings from cache and retrain STAIG (300 epochs, ~50s on RTX 2080 Ti); embeddings normal after fix (64-dim, norm=1.0)

Key Insight: Phase 2 saved STAIG fusion embeddings may collapse to zeros due to training instability; need proactive detection and auto-recompute in benchmark script to avoid manual intervention

3. Video switching after NSOpenPanel selection still played old video#

Solution: ensureFileAccess resolved per-screen bookmark to old video URL instead of requested new URL. Added guard to compare resolved vs requested URL paths; if different, return original URL (which has temporary sandbox permission from NSOpenPanel)

Key Insight: Per-screen bookmark storage conflicts with multi-video history; guard needed to detect when bookmark points to stale file

4. Coffee task BC-RNN showed 0% success rate, suspected evaluation bug#

Solution: Checked training logs and confirmed coffee BC-RNN training itself achieved 0% SR at all epochs (600 epochs trained). Not an evaluation bug; training genuinely failed for this task

Key Insight: Always verify training metrics before debugging evaluation pipeline; downstream zeros may reflect upstream training failure

5. an53 node crashed with NODE_FAIL at 08:30, causing loss of tmux sessions and running job 46798#

Solution: Investigated via sacct logs, determined crash caused by resource overload (8 GPUs 100% util, high CPU). AI operations ruled out as cause (last completed at 05:07, 3h before crash). SSH host key mismatch resolved with ssh-keygen -R an53.

Key Insight: srun ‘Requested nodes are busy’ errors indicate node overload, not command failure. Slurm reports minimal crash details without root access to system logs.

6. P100 GPU (sm_60) CUDA kernel incompatible with PyTorch (requires sm_70+); torch.cuda.is_available() returns True but operations fail asynchronously#

Solution: Initially tried try/except to catch torch.zeros(1).cuda() exception, but P100 async failure cannot be caught. User switched to RTX 2080 Ti (sm_75) and ran successfully. Eventually added CUDA_VISIBLE_DEVICES="" environment variable for forced CPU fallback logic in script

Key Insight: torch.cuda.is_available() only checks if driver and library exist, not if GPU compute capability is compatible with current PyTorch. CUDA operations may fail asynchronously and cannot be caught by Python try/except. Most reliable approach is to switch GPU or use environment variables to force CPU

7. LLM-generated JSON truncated (…truncated marker), causing daily_summary.py parsing failure for 2026-02-14 and other reports#

Solution: Sent truncated raw text as independent prompt to AI, requesting only fix/extract valid JSON portion; AI successfully restored complete JSON structure

Key Insight: LLM generating ultra-long JSON has truncation risk (especially multi-segment merge scenarios); should add truncation detection and automatic JSON repair retry logic in merge phase, rather than relying entirely on manual discovery and intervention

8. Screensaver clock text update left 0.2s ghost of previous digits#

Solution: CALayer has implicit ~0.25s animation for contents property changes. Wrapped blurView.layer?.mask updates in CATransaction with setDisableActions(true) to force immediate replacement

Key Insight: CALayer animates nearly all property changes by default; explicit transaction control required for instant visual updates

9. BC-RNN policy evaluation crashed with ‘Cannot handle data type <class ’numpy.float64’>’#

Solution: robomimic’s TensorUtils.to_tensor() cannot handle numpy scalar types. Modified _to_robosuite_obs to filter _raw_obs dict and only pass keys with array/image values to robomimic policy

Key Insight: Robomimic expects structured obs dicts with array values; numpy scalars from robosuite (like gripper width float64) must be filtered before passing to TensorUtils

Visualization script failed multiple times: Bash heredoc != escaped as != (SyntaxError), filename prefix mismatch (151673_filtered_feature_bc_matrix.h5 vs filtered_feature_bc_matrix.h5), spatial/ data path wrong (in Dataset/DLPFC/ not directly in section directory), load_dlpfc_data() return value unpacking error (returns 1 not 2), read_10x_h5 missing uns[‘spatial’] causing sc.pl.spatial() failure#

Solution: Created independent scripts/_viz_rm_ideal.py file instead of heredoc; used glob("*filtered_feature_bc_matrix.h5") to find correct filename; directly used sc.read_visium("/path/to/Dataset/DLPFC/151673") for loading (auto-handles spatial/ subdirectory); fixed load_dlpfc_data() unpacking to single return value; abandoned load_dlpfc_data, directly used sc.read_visium to ensure uns[‘spatial’] exists

Key Insight: Bash heredoc is sensitive to special character escaping, complex scripts should be written as files; DLPFC data filenames have section prefix; spatial data is in Dataset/DLPFC/{section}/spatial/ not {section}/spatial/; load_dlpfc_data uses read_10x_h5 not read_visium, missing spatial metadata; sc.pl.spatial strongly depends on uns[‘spatial’], must use read_visium for loading

Learnings#

Key Learnings#

  • RM-IDEAL cross-section evaluation complete workflow: ① load string labels (from annotation files) aligned by cache barcode; ② load fusion embeddings (from Phase 2 outputs or recompute); ③ use RMIdealEvaluator.compute_niche_scores_cross_section() to compute RM-IDEAL scores (WWL graph kernel + Wasserstein distance); ④ compute embedding cosine similarity (niche centroid vs all spots); ⑤ use Spearman r / Precision@K / Same-label rate to evaluate correlation; ⑥ generate spatial heatmap visualization (sc.pl.spatial)
  • LoRA base model selection is critical for transfer learning: pi0.5_base (general), pi0.5_libero (Libero benchmark), pi0.5_mimicgen_phoenix (MimicGen tasks). Using wrong base may explain poor performance.
  • WWL (Wasserstein Weisfeiler-Lehman) graph kernel algorithm details: ① K rounds of WL message passing, each round collects neighbor label multiset → sort and concatenate → global hash table lookup/assign new integer; ② query subgraph and reference subgraph share same hash tables, ensuring label encoding consistency; ③ output per-node (K+1)-dim feature vector [a^0, a^1, …, a^K]; ④ cost matrix uses normalized Hamming distance (different bits / total bits); ⑤ POT library ot.emd2() solves Wasserstein distance (10x speedup), scipy linprog fallback; ⑥ RM score = 1 - W, W ∈ [0, 1]
  • Cross-sample niche query evaluation metric interpretation: ① Spearman r≈0.44 (p<1e-150) indicates moderate positive correlation between fusion embedding cosine similarity and RM-IDEAL topology scores, embeddings indeed capture cross-section niche structure information; ② P@K close to 0 indicates large head differences between two rankings (embedding similarity and topology structure measure different dimensions of information), but overall ranking trend consistent; ③ Same-label rate low indicates low proportion of same labels in top-K, possibly because embedding spaces across sections not fully aligned; ④ 151676→151673 direction r=0.63 significantly better than reverse r=0.39, indicating 151673’s embedding quality is better
  • AI conversation logs long-term value: through gadget/summarize tool batch retrospective generation of 13 historical reports, systematically preserved complete technical evolution chain across robot simulation, spatial transcriptomics, AI assistant three directions over two weeks (key bug root causes, architecture decisions, human-AI collaboration patterns), this structured knowledge can still be used for retrospection, knowledge transfer, and paper writing months later
  • NSVisualEffectView supports two blending modes: .behindWindow (samples desktop/wallpaper) vs .withinWindow (samples same window’s content); SwiftUI .material uses .behindWindow and cannot sample video content
  • DLPFC dataset file organization and scanpy loading correct method: ① filenames have section prefix (151673_filtered_feature_bc_matrix.h5); ② spatial/ data is in Dataset/DLPFC/{section}/spatial/, not in {section}/ direct subdirectory; ③ load_dlpfc_data() uses read_10x_h5, missing uns[‘spatial’], cannot use for sc.pl.spatial(); ④ must use sc.read_visium("/path/to/Dataset/DLPFC/151673") to load, auto-handles spatial/ subdirectory and metadata; ⑤ annotation files are in /hpc/group/yizhanglab/shared/10x_human_brain_20250515/DLPFC_annotations/{section_id}_truth.txt, format is barcode\tlabel
  • STAIG fusion model collapse detection and auto-recompute: Phase 2 saved STAIG fusion embeddings may collapse to all zeros due to training instability (e.g., insufficient GPU compute, abnormal data distribution). Need proactive detection and auto-recompute in benchmark script: ① detect embeddings all zero; ② print warning; ③ load gene_emb and vision_emb from cache; ④ call apply_fusion(“staig_fusion”, …) to retrain 300 epochs; ⑤ save new embeddings. Avoid manual intervention, ensure benchmark can auto-recover
  • Batch historical data processing key practices: ① tool design should consider batch processing mode (date-range parameter, progress recovery, parallel processing); ② multi-segment merge needs explicit consolidation instructions (‘consolidate similar tasks into one entry’) to avoid verbose output; ③ long JSON generation scenarios need truncation detection + auto-repair retry mechanism; ④ batch task progress visualization (e.g., ‘completed 8 of 13 dates’) can significantly reduce manual monitoring cost
  • CALayer properties (contents, frame, etc.) have implicit ~0.25s animations by default; use CATransaction.setDisableActions(true) + commit() for instant updates in mask layers
  • Robomimic’s TensorUtils.to_tensor() requires dict values to be numpy arrays; numpy scalars (float64, int64) from robosuite obs dict must be filtered before passing to robomimic policies
  • Slurm NODE_FAIL diagnosis workflow: sacct shows job end state, scontrol show node reveals BootTime/SlurmdStartTime, but root cause requires syslog/dmesg access. Common causes: OOM, GPU driver crash, thermal issues.
