Problems and Solutions#

Key Problems#

1. STEGO’s training loss is NaN throughout, causing the model not to converge#

Solution: Two-step fix: ① Perform L2 normalization on input image_emb to prevent excessive amplitude; ② Replace InfoNCE with a numerically stable version (subtract the maximum value first and then perform logsumexp), increase temperature to 0.1.

Key insight: The exponentiation operation after dividing a 3639×3639 dense similarity matrix by temperature=0.07 overflows at float32 precision. Log-sum-exp is a complete solution, and this form must be used for large-scale comparative loss calculations.

2. MILWRM was incorrectly classified as an image-only method, and when AI summarized the results for the first time, multi-modal methods were mixed into the image-only results, resulting in significant revisions to the first version of the research conclusions.#

Solution: Confirmed through reading the full text of PMC that MILWRM is actually based on gene expression using WebFetch. It was removed from the list of image-only methods. Considering the user’s two clear boundary constraints, re-focused on pure image scenarios and specifically mined image-only data points from ablation experiments.

Key insight: Literature summaries often contain misleading information. It is necessary to read the full method section to confirm the input modality. Most multi-modal methods’ ablation experiments never test image-only methods, so it is necessary to specifically retrieve them from a few articles such as SpaConTDS.

General Problems#

3. Paid academic journals (Nature Medicine, Elsevier, etc.) returned 303/403 papers, and numerical values embedded in paper figures could not be extracted from web text.#

Solution: Switch to using PMC full-text mirrors, arXiv HTML versions, HuggingFace model cards, and GitHub README as alternative sources. When it is impossible to obtain numerical values embedded in figures, use textual qualitative conclusions instead and clearly label the data source and confidence level.

Key insight: PMC and arXiv HTML are effective ways to access paid journals. The key model performance values are usually fully listed in GitHub README, so they should be retrieved first. When precise values cannot be obtained, qualitative conclusions with source labels are better than speculation.

In eval_scan_fusion.py, the spatial coordinate dimension was abnormal (became (1,2)), and multiple fusion methods (basic_contrastive/qformer/staig_fusion) reported errors.#

Solution: Abandon calling load_spatial_coordinates() (barcode matching failed), and directly read coordinates from adata.obsm[‘spatial’]. Also fixed the issue of incorrect unpacking of return values in load_dlpfc_data (the function returned a single value instead of a tuple).

Key insight: Tool functions that rely on precise barcode matching are prone to failure when used across data sources. Directly accessing AnnData’s native fields is more reliable, and before calling project internal functions, the function signature should be verified in real time.

5. AI repeatedly triggered ExitPlanMode in document tasks (rejected twice), and model verification was performed using CPU by default, resulting in unnecessary interaction friction and efficiency loss.#

Solution: After explicit user instructions, pure document writing tasks directly called the Write tool. All model verifications were changed to run on GPUs, and three methods were executed in parallel in the background, significantly reducing the total time.

Key insight: In a HPC environment, GPU is the default computing device. Testing with CPU will掩盖 real performance problems. Pure document tasks do not require the “plan→exit plan” process.

6. When regenerating visualizations, it was necessary to avoid re-running UNI2 inference (which took hours), and section 151510 lacked tissue_lowres_image.png.#

Solution: It was found that pred_labels and gt_labels were already saved in .npz cache files. After directly loading them, the modified visualization function was called. All 11 sections were successful. Section 151510 was fixed by creating hires→lowres symlinks.

Key insight: MIHD’s caching design (embedding + labels saved together) completely decoupled visualization updates from inference. sc.read_visium defaulted to searching for lowres images, and using soft links for hires was the minimum change solution.

Today’s Gains#

Core Gains- The ARI for pure Image-Only in DLPFC fine-grained classification tasks is only 0.11–0.16 (multi-modal reaches 0.45–0.64). This indicates that the morphological differences between brain tissue layers are minimal, and the training domain does not match the foundation model. This is a significant research gap, as multi-modal methods almost never test image-only scenarios independently (gene expression is considered core).#

  • There are five main reasons why the foundation model fails in spatial domain recognition: ① Training with cancer tissues (domain gap); ② The pre-training tasks (patch classification/reconstruction) do not match the inter-layer gradient recognition; ③ The morphological differences between brain layers are very small; ④ Image features and gene expression are highly redundant; ⑤ Single patches lack spatial context for independent encoding. UNI2’s brown repeated patch phenomenon is a direct manifestation of reasons ① and ③.
  • SCAN achieves the best ARI in image-only spatial transcriptomics scenarios (0.303 vs baseline 0.251, +20.6%). Its core advantage lies in offline feature k-NN mining, which decouples embedding learning from clustering, avoiding numerical instability in STEGO. Its 256-dimensional optimized embedding fused with gene features has a high complementary effect (mean fusion +0.065, llava_mlp +0.018 ARI).
  • STAIG uses BYOL for unsupervised domain adaptation on target datasets of H&E patches (discarding the projector/predictor after training, retaining encoder features). This is a direct precedent to introducing unsupervised domain adaptation into spatial transcriptomics. BYOL does not require negative samples, making it suitable for small batch ST scenarios (thousands of patches per section), and it is robust against H&E staining variations.
  • When calculating InfoNCE comparative loss for large-scale dense similarity matrices (n>3000), it is necessary to use a numerically stable log-sum-exp function (minus the row maximum). Otherwise, exponential operations with temperature=0.07 under float32 precision will cause overflow and lead to NaN values. This is a key engineering constraint in implementing large-scale comparative learning using HPC.
  • The CV community has developed a four-level consensus framework for dealing with “domain gap + fine-grained tasks + unlabeled” scenarios: Level 1 uses pre-trained features for clustering; Level 2 involves refining STEGO/SCAN features; Level 3 requires re-training SSL within the domain (BYOL/MAE); Level 4 involves using dedicated foundation models. The appropriate level should be chosen based on computational resources constraints. GPFM/CHIEF are the best PFM for spatial domain recognition ARI; UNI2 is the best for spot retrieval; 224×224 is the industry-standard patch size.
  • In the original UNI paper, 34 supervised evaluation tasks (linear probe) and MIHD unsupervised clustering (KMeans) have similar tasks but different evaluation methods. The high accuracy of UNI cannot directly infer the performance of unsupervised clustering. HEST-1k shows that the size of PFM is logarithmically related to spatial gene expression prediction performance (R=0.81). Pathology-specific PFM is about 8.2% higher than ResNet50 in Pearson r.
  • The literature review on spEMO reveals that GPFM/CHIEF are the best for spatial domain recognition clustering ARI; UNI2 has the best correlation in spot retrieval rankings; 224×224 is the industry-standard patch size, consistent with MIHD. This provides solid literature support for selecting the encoder for MIHD.
