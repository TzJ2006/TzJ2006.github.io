Problems & Solutions#

Critical Issues#

1. How to prove Error Recovery Benchmark is useful beyond ‘can generate scenes’ - need evidence that data actually improves models for thesis contribution#

Solution: Design G4 (dataset usefulness proof) as 4-step experiment: (1) convert 2000+ scenes to LeRobot format with state+image observations, (2) LoRA fine-tune π0.5 on error scenes for 2-4 hours on A800, (3) evaluate on held-out test split with stratified sampling, (4) measure improvement (expect 15-30% SR gain, 10-20% RP gain, statistical significance via bootstrap CI and Wilcoxon test)

Key Insight: Benchmark usefulness requires closed-loop proof: error data → model training → measurable performance improvement on unseen errors. This is the core thesis contribution beyond scene generation infrastructure - the ‘4-pillar validation’ that distinguishes publishable benchmarks from toy datasets.

2. UNI2 foundation model features achieve only ARI 0.11-0.16 on DLPFC spatial domain clustering (vs 0.5+ for gene+image multimodal), with brown repeated patches appearing uniformly across clustering results#

Solution: Identified 5 root causes: (1) training data domain gap (cancer-heavy, brain-rare), (2) pretraining task mismatch (coarse tissue classification vs fine-grained layer detection), (3) DLPFC layer differences are subtle (cell density/size, not color/texture), (4) feature redundancy with gene expression, (5) lack of spatial context. Analysis revealed UNI2 clusters by morphological similarity (blood vessels across all layers) rather than biological domains. Proposed 6 adaptation methods (STEGO, SCAN, BYOL, DINOv2-GAT, TTT-MAE, SFDA) following solution hierarchy: domain-specific pretraining > test-time adaptation > feature refinement > task-specific fine-tuning. Added H&E reference panel to visualization to confirm hypothesis.

Key Insight: Pure vision encoders cluster by morphological features not biological function - this is a classic fine-grained recognition problem seen across CV domains (bird subspecies, rock types, cell morphology). Low ARI reflects structural grouping (what looks similar) not functional grouping (what is biologically similar). General foundation models learn coarse semantics; domain-specific fine-tuning or feature refinement is necessary for fine-grained tasks.

3. Error Recovery Benchmark dataset phase imbalance - 90% scenes in pre_reach/reach, transport/place completely empty (0 scenes)#

Solution: Design strategy using BC-RNN (60-70% SR) as key to reaching later phases, lower detector min_step thresholds from 100→5-50 to capture lift/transport failures, use MimicGen to generate 200+ demo variants for increased trajectory diversity and later-phase trigger opportunities

Key Insight: Phase coverage requires policies with sufficient success rate to reach late phases (BC-RNN sweet spot) combined with adjusted detector timing - random policy fails too early (lift never reached), Pi0 has 0% SR (grasp never reached), only BC-RNN reaches transport/place with failures. This is a ‘reachability problem’ determined by success rate distribution.

4. How to rigorously validate that 24-type error taxonomy is principled, complete, and non-redundant for academic publication#

Solution: Three-pillar validation: (1) map 8 categories to FMEA failure modes from ISO 15066/10218/IEC 61508 industrial safety standards (provides theoretical grounding), (2) prove completeness via 6 phase × 3 family = 18-cell matrix showing no blind spots, (3) statistical validation using Cohen’s/Fleiss’ kappa (target: within-category κ>0.6, cross-category κ<0.3) to verify types are distinct

Key Insight: Academic taxonomy validation requires both theoretical grounding (industry standards provide FMEA-based rationale) and empirical evidence (statistical independence tests prove non-redundancy) - cannot rely on intuition alone for 24 types

5. M14 baseline evaluation (5448 episodes, 5-15h runtime) risks losing all progress if VLA server crashes mid-run#

Solution: Implemented resume support in collector.py: on startup, reads existing episodes.jsonl, builds a set of completed (scene_id, policy_name, seed) tuples, skips them in the triple loop, and rebuilds all_episodes dict to preserve summary correctness. Added –resume flag to 3_collect_data.py. Now crash-safe with automatic resume.

Key Insight: For long-running data collection (>1h), append-only JSONL logging + resume logic is essential. The key is idempotency: reading back the JSONL and rebuilding the summary dict ensures final results are identical whether run completes in one shot or across multiple resume attempts.

6. User didn’t understand ‘recovery training data’ concept in Phase II plan, blocking plan approval#

Key Insight: Technical jargon must be grounded in examples before implementation discussion; user prioritized data quality and real recovery demonstrations over automated proxies, willing to invest manual effort for authentic data

Learnings#

Key Learnings#

  • Pure vision encoders cluster by morphological similarity not biological domains: UNI2 groups blood vessels across different cortical layers because they look similar, explaining why vision-only spatial domain ARI is limited (0.11-0.23 vs 0.5+ multimodal). HEST-1k shows pathology foundation models significantly outperform ImageNet baseline (UNI2 Pearson r=0.414 vs ResNet50 r=0.326), but comprehensive vision-only spatial domain clustering benchmarks largely don’t exist - most papers only evaluate multi-modal fusion, not isolated vision encoder performance. Need gene expression to distinguish functional regions with similar visual appearance.
  • Error recovery benchmarks require proving usefulness through 4 dimensions (4-pillar validation): (1) large-scale diverse data (2000+ scenes), (2) theoretically-grounded taxonomy (FMEA/ISO standards), (3) baseline performance characterization (4+ policies), (4) demonstration of training improvement on foundation models (15-30% SR gain). This distinguishes publishable benchmarks from toy datasets - the core thesis contribution is closed-loop proof: error data → model training → measurable improvement.
  • Cross-domain literature search is extremely valuable for identifying solution patterns. The image-only clustering problem in spatial transcriptomics is nearly identical to problems in: (1) fine-grained bird classification (subspecies differ only in beak shape/feather pattern), (2) remote sensing (ImageNet features fail on satellite images), (3) microscopy (material microstructures), (4) geology (rock thin sections). All domains converged on the same 4-tier solution hierarchy: domain-specific pretraining > test-time adaptation > feature refinement > task-specific fine-tuning. This suggests deep learning failure modes are universal across modalities.
  • Phase imbalance in error scene generation stems from success rate distribution creating a ‘reachability problem’ - need policies with 60-70% SR (BC-RNN sweet spot) to reach late phases while still producing failures. Random policy fails too early (lift never reached), Pi0 has 0% SR (grasp never reached), only BC-RNN reaches transport/place with failures. Solution requires BC-RNN + adjusted detector timing (min_step thresholds 100→5-50) + MimicGen trajectory diversity (10→200 demos).
  • When user says ‘I need to know what/why/how’, they want understanding-first workflow: explain concept → user decides direction → then build detailed plan. Not: build plan → explain afterward. Strategic planning questions (data collection method, task scope) should be asked before building detailed implementation plan, not after - saves rework. Documentation-driven development for complex multi-milestone projects ensures shared understanding and alignment before execution, critical for research with advisor feedback loops.
  • Evaluation paradigm fundamentally changes what is measured: same task (tissue type identification) evaluated as supervised classification (with labels, tests linear separability via KNN) vs unsupervised clustering (no labels, tests natural cluster structure via ARI) measures different aspects of embedding quality - high KNN accuracy doesn’t guarantee good clustering ARI. This explains why CRC-100K 9-class classification success doesn’t predict DLPFC spatial domain clustering performance.
  • BYOL (Bootstrap Your Own Latent) solves SimCLR’s key weakness: need for large batches and negative samples. BYOL uses teacher-student framework with EMA updates + predictor head to prevent collapse, making it robust to small batches (256 vs 4096+), augmentation choices, and domain shift. This explains why STAIG chose BYOL over SimCLR for histology patches (where similar-looking patches from adjacent layers should NOT be pushed apart as negatives). The key architectural insight: predictor head asymmetry (online has it, target doesn’t) + EMA target prevents trivial solutions.
  • FMEA (Failure Mode and Effects Analysis) from industrial safety standards (ISO 15066 collaborative robots, ISO 10218 robot safety, IEC 61508 functional safety) provides rigorous theoretical grounding for robot error taxonomies - maps 8 categories to standardized failure modes like ’loss of grip’, ‘uncontrolled release’, ‘unexpected contact’. Academic taxonomy validation requires both theoretical grounding (industry standards) and empirical evidence (Cohen’s/Fleiss’ kappa: within-category κ>0.6, cross-category κ<0.3).
  • VLA model landscape for error recovery: 8 models claim recovery capabilities but only 2 are open-source (RoboFAC, VLAC). FailSafe and VLA-SCT are most practical for integration (training-free, plug-in architecture, already validated on π0/OpenVLA). Recovery methods fall into 3 categories: dual-system (fast/slow), proactive (subtask backtracking), reactive (supervisor-guided correction). MimicGen can amplify 10 demos to 200+ variants for trajectory diversity, requires has_offscreen_renderer=True for VLA training.
  • Resume logic for long-running data collection must rebuild summary state from JSONL, not just skip episodes. The naive approach (only skip completed episodes) breaks final summary statistics. The correct approach: (1) read JSONL on startup, (2) rebuild all_episodes dict, (3) add completed tuples to skip set, (4) continue collection. This ensures idempotency: running A→crash→resume→B produces identical results to running A+B in one shot.

Practical Learnings#

  • Large file reading strategy: use limit parameter for initial exploration, then targeted offset ranges or Grep for specific content; avoid reading entire file unless necessary. For files >30k tokens, chunked reading with 400-line segments works well. Documentation refactoring with ‘complete preservation’ requirement needs verification: count all checklist items before and after to prove nothing lost.
  • Windows OpenSSH does not support Unix domain sockets (ControlMaster/ControlPath), causing silent failures with cryptic error messages (‘Bad file descriptor’, ’nonexistent pipe’). The solution is not to switch SSH clients, but to disable connection multiplexing options. This is a reminder that cross-platform compatibility issues often manifest as low-level errors that look like bugs rather than feature unavailability.
