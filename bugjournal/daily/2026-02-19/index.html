Problems and Solutions#

Key Issues#

1. How to prove that error recovery benchmark testing is useful beyond “generating scenarios” – Need evidence to show that the data truly improves the model, so it can contribute to papers#

Solution: Designed G4 experiment (dataset usefulness proof), divided into 4 steps: (1) Convert over 2000 scenarios to LeRobot format, including state information and image observations; (2) Fine-tune π0.5 on error scenarios for 2-4 hours using LoRA on A800; (3) Evaluate using stratified sampling within retained test samples; (4) Measure improvement effects (expected 15-30% SR gain, 10-20% RP gain, statistical significance analysis using bootstrap confidence intervals and Wilcoxon tests) Key Insight: Beneficialness of benchmark testing requires closed-loop verification: error data → model training → measurable performance improvement on unseen errors. This is a core paper contribution beyond scenario generation infrastructure—the “four-pronged validation,” which distinguishes publishable benchmarks from simple datasets.

2. UNI2 basic model accuracy in DLPFC spatial domain clustering is only 0.11-0.16 (compared to 0.5+ for gene+image multimodality), and brown repeating areas are evenly distributed in clustering results#

Solution: Identified 5 root causes: (1) Differences in training data domains (more cancer data, less brain data), (2) Incompatible pre-training tasks (rough tissue classification vs fine-grained layer detection), (3) Subtle differences in DLPFC layers (cell density/size, not color/texture), (4) Redundancy between features and gene expression, (5) Lack of spatial context. Analysis showed that UNI2 clustering is based on morphological similarity (blood vessels across layers), not biological categorization. Proposed 6 adaptation methods (STEGO, SCAN, BYOL, DINOv2-GAT, TTT-MAE, SFDA), in order of solution hierarchy: domain-specific pre-training → adaptive tuning during testing → feature optimization → task-specific fine-tuning. Added H&E reference panel in visualization to confirm hypothesis. Key Insight: Pure visual encoders cluster based on morphological features, not biological functions—a typical fine-grained recognition problem, common in computer vision (bird subspecies, rock types, cell morphology). Lower accuracy reflects structural grouping (similar things), not functional grouping (biologically similar things). General basic models learn coarse semantic information; for fine-grained tasks, domain-specific fine-tuning or feature optimization is needed.

3. Imbalance in error recovery benchmark dataset—90% scenarios belong to pre_reach/reach, transport/place, while transport/place is completely empty (0 scenarios)#

Solution: Used BC-RNN as strategy to achieve subsequent phase goals, reducing detector minimum step threshold from 100 to 5-50 to capture lift/transport failures. Used MimicGen to generate 200+ demonstration variants to increase trajectory diversity and trigger opportunities for subsequent phases. Key Insight: Phase coverage requires a strategy with sufficient success rate to proceed to later phases (optimal timing for BC-RNN), while adjusting detector timing—random strategy fails prematurely (never reaches mastery level), Pi0’s SR is 0% (never reaches mastery level), only BC-RNN can achieve transport/placement when failing. This is a “reachability problem” determined by success rate distribution.

4. How to rigorously verify that 24 error classification methods are principled correct, complete, and non-redundant for academic publication#

Solution: Three-pronged validation: (1) Mapping 8 categories to ISO 15066/10218/IEC 61508 industrial safety standards’ FMEA failure modes (providing theoretical basis), (2) Verifying completeness through 6 phases × 3 families = 18 cell matrices to ensure no blind spots, (3) Using Cohen’s/Fleiss’ kappa for statistical validation (goal: κ>0.6 within same category, κ<0.3 between categories) to verify type distinction Key Insight: Verification of academic classification methods requires theoretical basis (industry standards provide FMEA rationale) and empirical evidence (statistical independence tests proving no redundancy)—cannot rely solely on intuition for handling 24 types

5. M14 baseline evaluation (5448 cases, runtime 5-15 hours) may lose all progress if VLA server crashes during operation#

Solution: Implemented recovery function in collector.py: read existing episodes.jsonl files at startup, build a set of completed tuples (scene ID, strategy name, seed), skip these tuples in a triple loop, and rebuild all_episodes dictionary to maintain consistency of summary. Added –resume flag in 3_collect_data.py. Now with automatic recovery, ensures continued operation even if server crashes. Key Insight: For long-running data collection (over 1 hour), appending to JSONL logs + recovery logic is essential. Key is immutability: reading back to JSONL and rebuilding summary dictionary ensures consistent final results, whether running once or multiple recoveries

6. Users do not understand the concept of “recovery training data” in phase II plan, hindering approval of the plan#

Solution: Provided specific examples, comparing normal demonstrations (reset→success) with recovery trajectories (error state→adjustment→success); explained how it teaches the model “how to correct errors,” not just “how to operate correctly from scratch.” Users ultimately chose manual remote operation to obtain high-quality data, although this increases time cost. Key Insight: Before implementation discussions, technical terms must be explained with concrete examples; users prioritize data quality and real recovery demonstrations over automated agents, willing to invest manual work for real data

7. Errors like “client_loop: send disconnect: Bad file descriptor” and “tried to write to a nonexistent pipe” occurred when connecting VS Code Remote-SSH to HPC cluster#

Solution: Windows OpenSSH does not support ControlMaster/ControlPath/ControlPersist used by Unix domain sockets. Commented out these options in ~/.ssh/config file, and removed expired socket files from ~/.ssh/sockets/ directory. Connection restored immediately. Key Insight: Multiplexing of SSH connections is a Unix-only feature; when these options exist, Windows OpenSSH silently fails, resulting in unintelligible low-level errors, seemingly faults rather than functional unavailability

9. When running MimicGen with CUDA_VISIBLE_DEVICES=3, MUJOCO_EGL_DEVICE_ID assertion failed#

Solution: EGL device ID must match GPU index in CUDA_VISIBLE_DEVICES, not physical GPU ID. When using CUDA_VISIBLE_DEVICES=3, set MUJOCO_EGL_DEVICE_ID to 3 (or 0 if using single GPU) Key Insight: EGL device ID is relative to visible devices, not absolute GPU hardware index—robosuite’s EGL assertion is designed to ensure consistency of rendering targets
