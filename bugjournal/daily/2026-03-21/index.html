Problems & Solutions#

Critical Issues#

1. npx @ccusage/codex@latest hung indefinitely when called from Python subprocess#

Solution: Added –yes flag to npx calls to skip the ‘Ok to proceed? (y)’ interactive prompt that subprocess.run(capture_output=True) couldn’t answer due to no stdin

Key Insight: subprocess.run with capture_output=True has no stdin, so any interactive prompts will hang forever; always use –yes or -y flags for package managers in automated scripts

2. Monthly summary didn’t aggregate or display Codex token usage despite daily exports collecting it#

Solution: Added separate aggregation path for codex_token_usage in aggregate_token_usage(), created combine_usage_summaries() for merged overview, updated format_monthly_markdown() to render Codex section

Key Insight: When adding new data sources to aggregation pipelines, need to update not just input parsing but also aggregation logic, LLM input sanitization (strip mechanical fields), and output formatting

General Issues#

3. Windows Path().parent produces backslashes breaking rclone remote paths (e.g., ‘gdrive:gadget/website\personal’)#

Solution: Used Path.as_posix() to convert Windows paths to forward slashes for rclone remote path construction

Key Insight: rclone remote paths must use forward slashes even on Windows; Path operations need explicit .as_posix() conversion when building remote URIs

4. PowerShell execution policy blocked .ps1 wrappers for npx and codex commands#

Solution: Recommended using .cmd wrappers (npx.cmd, codex.cmd) as immediate workaround and Set-ExecutionPolicy -Scope CurrentUser RemoteSigned for permanent fix

Key Insight: On Windows, npm installs create both .cmd and .ps1 wrappers; when PowerShell execution policy is restricted, .cmd wrappers work universally while .ps1 requires policy changes

5. Git merge conflicts when pulling from remote after local changes#

Solution: Used git stash, git pull, then resolved remaining conflicts by accepting upstream (–theirs) and dropping stash

Key Insight: When syncing with remote after local cleanup, accepting remote version wholesale with –theirs is cleaner than manual conflict resolution if local changes are meant to be discarded

Learnings#

Key Learnings#

  • subprocess.run with capture_output=True has no stdin - any interactive prompts will hang indefinitely; always use –yes/–non-interactive flags for package managers and CLI tools in automation
  • When adding new data sources to aggregation pipelines, must update: (1) input collection, (2) aggregation logic, (3) LLM input sanitization (strip new fields from prompts), (4) output rendering - missing any step creates silent data loss
  • Windows Path objects produce backslashes; when building URIs or remote paths that require forward slashes (rclone, URLs, Unix paths), must explicitly use Path.as_posix() for cross-platform compatibility
  • Everything Claude Code (ECC) plugin architecture: 5 layers forming a closed loop - Skills (passive knowledge) → Agents (specialized reasoning) → Commands (user-facing) → Hooks (automated triggers) → Continuous Learning (feedback capture) → Skills

Practical Learnings#

  • On Windows with restrictive PowerShell execution policies, npm-installed commands have both .ps1 (blocked) and .cmd (works) wrappers; .cmd is the universal fallback
  • ECC effortLevel has 4 tiers: low/medium/high/max; ‘max’ is new as of recent updates and enables maximum thinking tokens (31999) and deepest reasoning
