Problems and Solutions#

Key Problems#

1. Infinite suspension when calling npx @ccusage/codex@latest from Python subprocess#

Solution: Add the –yes flag to npx calls to skip the interactive “Continue?” prompt, as capture_output=True in subprocess execution prevents stdin input.

Key Insight: Subprocess with capture_output=True has no stdin input, so any interactive prompt will hang indefinitely; always use –yes or -y flags for package managers in automated scripts.

2. Monthly summary still cannot summarize or display Codex token usage despite daily data exports#

Solution: Add a separate codex_token_usage summary path in aggregate_token_usage(), create combine_usage_summaries() for combined summaries, update format_monthly_markdown to include Codex section, and update all documents (README.md, summarize/README.md, AGENTS.md) to mention Codex support.

Key Insight: When adding new data sources to the summary process, not only need to update input parsing but also summarize logic, LLM input cleaning (remove mechanical fields), and output formatting.

General Problems#

3. Windows Path().parent generates backslashes, causing rclone remote path issues (e.g., ‘gdrive:gadget/website\personal’)#

Solution: Use Path.as_posix() to convert Windows paths to forward slashes for rclone remote paths.

Key Insight: Even on Windows, rclone remote paths must use forward slashes; explicit.as_posix() conversion is required for path operations when building remote URIs.

4. PowerShell execution policy blocks npx and codex command.ps1 wrappers#

Solution: Suggest using cmd wrappers (npx.cmd, codex.cmd) as a temporary solution, and Set-ExecutionPolicy -Scope CurrentUser RemoteSigned for a permanent fix.

5. Git merge conflicts occur after pulling changes from remote#

Solution: Use git stash, then git pull, and finally accept the upstream version and discard the stash to resolve remaining conflicts.

Key Insight: Synchronizing to remote after local cleanup is more straightforward using –theirs to accept remote versions, especially if local changes are intended to be discarded.

Learning Outcomes#

Key Learnings#

  • When using subprocess.run with capture_output=True, no standard input is automatically received—any interactive prompt will hang indefinitely; always use –yes/–non-interactive flags for package managers and other CLI tools in automated processes.
  • When adding new data sources to the aggregation pipeline, you must update: (1) input collection, (2) aggregation logic, (3) LLM input cleaning (remove new fields from prompts), (4) output rendering—missing any of these steps will result in data loss.
  • Windows path objects generate backslashes; when building URIs or remote paths that require forward slashes (e.g., rclone, URLs, Unix paths), Path.as_posix() must be used to avoid cross-platform compatibility issues.
  • The Claude Code plugin architecture consists of 5 layers forming a closed loop: skills (passive knowledge) → agents (specialized reasoning) → commands (user-facing) → hooks (automated triggers) → continuous learning (feedback capture) → skills.

Practical Learnings#

  • On Windows systems, due to PowerShell execution policies, npm installed commands have both.ps1 (blocked) and .cmd (runnable) wrappers; .cmd is a general alternative.
  • ECC’s effortLevel has 4 levels: low/medium/high/maximum; “maximum” level is the newly added feature, allowing up to 31,999 thinking tokens and deepest reasoning capabilities.
