Problems and Solutions#

Key Issues#

1. MIHD’s embedding standardization inconsistency - UNI2/scGPT uses L2 norm, while HIPT/ResNet50/MLP/GCN do not, and no standardization after fusion.#

Solution: Designed a three-point standardization system (after encoding, before fusion, after fusion) and can configure different methods (L2, standard scaling, none) through the new utils/normalization.py module. Key Insight: Inconsistent standardization between encoders can affect downstream clustering results; the three-point method provides fine-grained control.

2. QueST uses adversarial training for batch corrections, but its effectiveness in spatial transcriptomics is unclear.#

Solution: Designed a plug-in BatchCorrection module, following QueST’s discriminator/generator pattern, and set evaluation metrics to verify its effectiveness. Key Insight: Porting batch corrections to the MIHD environment requires empirical validation, rather than assuming its applicability.

General Issues#

3. User idea 3 is not clear enough - “Q-Former similar to LLaVA”, but LLaVA uses simple MLP, while Q-Former comes from BLIP-2.#

Solution: Proposed clarifying questions; users chose to implement both (Q-Former with cross-attention mechanism and 2-layer MLP of LLaVA) to compare lightweight and heavyweight fusion effects. Key Insight: Clarifying architectural ambiguity in advance can avoid wasting implementation effort on the wrong direction.

Lessons Learned#

Key Lessons#

  • For complex multi-project plans, scope and architecture choices should be clarified before deep exploration — ask “both or one,” “understanding or implementation,” and “partial process or complete process” in advance.
  • Dependency analysis is crucial for multi-stage plans — standardization must be done before fusion experiments, the configuration base allows smooth progress, and QueST’s batch corrections require standardized fusion outputs.
  • Document improvement is important: MIHD’s CLAUDE.md was reduced by 67%, while adding missing content by removing redundancy, discovering information, and recording implementation history.
