Problems and Solutions#

Key Problems#

1. Inconsistency in MIHD embedding standardization: UNI2/scGPT uses L2 normalization, while HIPT/MLP/GCN do not. No standardization was performed after fusion, affecting the quality of downstream clustering.#

Solution: A three-point normalization architecture was designed (after encoding, before fusion, after fusion). This was achieved by creating utils/normalization.py and integrating CLI parameters from run_benchmark.py.

Key Insight: L2 normalization is idempotent for normalized vectors. Therefore, additional L2 normalization for UNI2/scGPT is safe and harmless, ensuring that all encoder outputs are consistent in space.

General Problems#

2. AI starts executing tasks directly after users explicitly request a plan document, causing process interruption.#

Solution: After the user’s explicit request, AI writes the complete plan into docs/ENHANCEMENT_PLAN.md, and then begins the phased implementation. Key Insight: For large multi-stage tasks, it is better to save the plan in a file for review before granting execution rights.

3. AI tries to start execution using ExitPlanMode, but the user rejects it, causing stage 1 to be halted.#

Solution: When the user rejects it, the execution stops and waits for further instructions. Key Insight: Using ExitPlanMode requires explicit user authorization. It should not be assumed that accepting a plan means granting execution rights.

Human Thinking vs AI Thinking#

Strategic Level#

Writing plans in documents before implementation#

Role Thinking Style
Human The user actively requests to write plans in documents before implementation, establishing a review-authorization workflow.
AI AI tends to create TaskCreate tasks and execute them directly, ignoring the documentation process.

Difference Analysis: Humans are more concerned with reviewability and version control; AI prefers immediate action, ignoring the value of keeping intermediate results.

Hyperparameter Configuration#

Role Thinking Style
Human The user requests a complete configuration, meaning all hyperparameters are controlled by config.
AI AI预设了几个可能的参数范围（仅融合层或仅关键参数），没有建议最大覆盖范围。

Difference Analysis: Humans have higher requirements for system reproducibility and adjustability, preferring to use complete configurations.

Choice between Q-Former and LaVA MLP#

Role Thinking Style
Human The user explicitly requested both implementations, indicating that QueST functionality should be implemented in MIHD, not just analyzed.
AI AI provides two options for the user to choose from, suggesting a trade-off rather than superposition.

Difference Analysis: Humans have a stronger “complete implementation” mindset, not constrained by resource assumptions; AI tends to provide simplified options.

Limitations of AI#

General Limitations#

  • AI often acts immediately in large multi-stage tasks, without suggesting that plans should be saved in documents for review before execution. User prompts are needed to adjust the workflow.
  • When calling ExitPlanMode, AI did not anticipate that users might want to manually control the execution speed. This led to user interruptions when rejecting it, with a lack of awareness between “plan approval” and “execution authorization”.
  • When providing options (Q-Former vs LaVA MLP), AI defaulted to mutually exclusive options, without suggesting a solution for “two implementation methods”. This requires user correction.

Today’s Summary#

Key Summary - Inconsistency in MIHD fusion process: UNI2/scGPT underwent L2 normalization, while HIPT/GCN/MLP did not. ConcatFusion only performed pre-connection normalization, without post-fusion normalization. The three-point normalization architecture (after encoder, before fusion, after fusion) is a key architectural decision that helps improve downstream clustering results.#

  • QueST achieved batch correction through BatchDiscriminator (MLP discriminator) + GIN layer adversarial training. This method can be used as a plug-and-play module for MIHD post-fusion. Its unique query mechanism utilizes k-hop subgraph pooling and cosine similarity for cross-sample retrieval. Both mechanisms are worth replicating.
  • Q-Former (BLIP-2) uses learnable queries and cross-attention to extract visual feature information. In practice, LaVA MLP used a simpler two-layer MLP projector instead of Q-Former. Their roles differ: Q-Former is suitable for scenarios requiring sparse queries, while LaVA MLP is better suited for efficient alignment tasks.

Practical Summary#

  • For large multi-stage implementations, the correct workflow is: explore code library → write plans in documents → user review → user authorization → phased implementation. Skipping any step may lead to user interference and decreased efficiency.
