Problems & Solutions#

Critical Issues#

1. MIHD has inconsistent embedding normalization - UNI2/scGPT are L2-normed but HIPT/ResNet50/MLP/GCN are not, and no post-fusion normalization exists#

Solution: Designed 3-point normalization system (post-encoder, pre-fusion, post-fusion) with configurable methods (L2, standard scaling, none) via new utils/normalization.py module

Key Insight: Normalization inconsistency across encoders hurts downstream clustering; systematic 3-point approach provides fine-grained control

2. QueST uses batch correction via adversarial training but unclear if effective for spatial transcriptomics#

Solution: Designed plug-in BatchCorrection module following QueST’s discriminator/generator pattern with evaluation metrics to assess effectiveness

Key Insight: Porting batch correction requires empirical validation in MIHD’s context rather than assuming transferability

General Issues#

3. User’s Idea 3 was ambiguous - ‘Q-Former like LLaVA’ but LLaVA uses simple MLP while Q-Former is from BLIP-2#

Solution: Asked clarifying question; user chose to implement both (Q-Former with cross-attention + LLaVA’s 2-layer MLP) to compare lightweight vs heavy fusion

Key Insight: Clarifying architectural ambiguities upfront prevents wasted implementation effort on wrong approach
