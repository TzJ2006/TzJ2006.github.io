Problems and Solutions#

Key Issues#

1. RTX 2080 Ti + PyTorch 2.9.0+cu129 triggers cuBLAS CUBLAS_STATUS_EXECUTION_FAILED for high-dimensional large tensors (3D tensors with N>3500), and conflicts with PyG scatter’s CUDA deterministic mode, leading to CUDA illegal memory access#

Solution: Dual architectural modifications: (1) For full embeddings, first project them to hidden_dim(128) using Linear, then index neighbors using idx_tensor to avoid high-dimensional large tensors entering cuBLAS; (2) Replace full InfoNCE with mini-batch contrast loss (batch_size=512) to avoid N×N matrix backward triggering bugs

Key Insight: “Projecting to low dimension first and then aggregating neighbors” is not only a GPU compatibility workaround but also a general best practice for high-dimensional embedding cross-modal fusion. Mini-batch contrast loss is the standard approach for large-scale contrast learning. It should be fixed to verified version combinations (PyTorch 2.1-2.4).

2. The paper on scGPT-spatial claims it outperforms the baseline, but its actual value is difficult to evaluate#

Solution: System literature research revealed that its baseline was weak methods from 2021-2022 (SpaGCN/stLearn, ARI≈0.30-0.40). SOTA methods (GraphST, ARI≈0.55-0.63) were not compared, and it was not included in any independent third-party benchmark

Key Insight: The paper avoids direct competition by choosing a weak baseline. Evaluating new methods requires verifying whether their baseline is the current SOTA. The significance of “outperforming the baseline” depends entirely on the level of the baseline.

3. Natural capture of severe overcapacity in online quota generation: Some error types (premature_release 7,233 items) are extremely frequent, while 7 types have completely zero occurrences#

Solution: Users proposed an offline architecture injection—let the strategy run a complete trajectory, detect injectable points and create indexes offline, then selectively inject according to quotas. Skip types that have reached their limit

Key Insight: Online natural capture is highly influenced by the distribution of strategy behavior. It’s impossible to control type balance. The offline architecture decouples “exploring injectability” from “executing injection,” making it the correct approach for constructing balanced error scenario datasets.

4. MCP Server tools default only write files and return paths, so Claude Code cannot directly read the content#

Solution: Reorganized the tool to bypass the cmd_* wrapper and call underlying functions directly, returning complete markdown/JSON through a save parameter. This allows the file writing to be controlled via a save parameter

Key Insight: The primary purpose of MCP tools is to deliver content to AI consumers. File writing is an optional side effect rather than a main function—when designing tools, one should consider who the consumer will be.

5. CALVIN evaluation script has multiple hard-coded issues (CALVIN_ROOT path, missing window_size field, import typo), and the 1.3GB dataset transfer cost is high#

Solution: Identified and fixed these issues one by one. Key finding: CALVIN evaluation is purely online simulation, without reading any episode data. It only requires validation/.hydra/merged_config.yaml. The 1.3GB dataset can be compressed to a 600KB eval-only version

Key Insight: Evaluation scripts often have surprising storage requirements: Pure online simulation doesn’t read historical data, significantly reducing storage and transmission costs.

6. When the LLM returns incomplete JSON, the motivation/innovation_point static fields are empty. No logs are available for debuggingSolution: After Stage1/2 evaluation, add a warning for the count of missing fields. Add try-except+logger.error in _eval_with_anthropic, and record the original response as DEBUG log in _try_repair_result.#

Key Insights: Silencing.get() fallback掩盖LLM response quality issues. Structured logs are crucial for troubleshooting LLM integration failures, and log埋点 should be designed from the beginning of integration.

7. For STAIG fusion embedding with ID 151676, all values were zero, resulting in model collapse. This cannot be used for cross-slice queries. GPU retraining failed due to compatibility issues between PyTorch 2.9.0 and CUDA 13.1 drivers.#

Solution: Attempted GPU retraining but was blocked due to environmental compatibility issues, causing the task to pause. The fundamental solution needs to address GPU environment version issues.

Key Insights: The statistical characteristics of cached embeddings should be verified before use (norm range). Zero values are a clear sign of model collapse, and health checks should be added during caching write operations. Older PyTorch versions may introduce regression issues ahead of driver versions.

8. In the injection scenario, the validator names used (drop/tip_over/stuck) instead of the 24 taxonomy types led to inconsistencies in the label system. This caused errors in distribution statistics and quota calculations.#

Solution: Add _map_to_taxonomy_type() in generates_labels(), mapping from (validator, task_phase) to taxonomy types. database._classify_scene() retains existing valid labels without overwriting them.

Key Insights: The label system must be correct from the moment it is generated. Dependence on post-mapping will掩盖 real type distribution issues.

Difference Analysis: Humans have a stronger intuition of ’not giving up easily’ and require understanding the problem before changing strategies. AI tends to be conservative and prefer quick fallback.

AI Limitations#

Major Limitations#

  • System design phase failed to anticipate data distribution issues: When designing an online quota generation system, it was not predicted that uneven distribution of strategic behavior would lead to severe skewness in natural capture. When observing 21,001 scenarios and excessive data during runtime, no proactive alerts were issued, and direction changes required user intervention. A better offline injection architecture did not propose this solution proactively.
  • MCP tool design did not consider the perspective of consumers: It did not take into account that the tool consumer is AI itself. The initial design followed the traditional CLI tool pattern of file writing and path return, which only needed user explicit indication to be corrected. This reflects a lack of proactive reasoning ability regarding ‘who consumes the tools’.
  • Too quick fallback when encountering hardware compatibility issues: After encountering CUDA errors, it tended to directly switch to CPU instead of systematically diagnosing the root cause of PyTorch version compatibility issues. GPU debugging experienced multiple attempts with inaccurate directions. The initial failure to quickly identify the mini-batch was the core strategy for solving N×N matrix problems.

General Limitations#

  • Not verifying signatures before using APIs/libraries: When using FastMCP, it was not checked whether the version parameter existed first (only discovered during runtime crashes). When using the conda –no-banner parameter, the local version was not detected (only discovered when the program failed to run). When handling LeRobot output paths, the Python module-level import timing issue was not anticipated. It should be inspected with inspect.signature before using unfamiliar APIs.
  • Some literature is not accessible in full (bioRxiv PDF returns 403), and related data relies on abstracts and secondary information. The complete GPU pipeline evaluation of CrossModalEnhancer has not been completed, and the actual effects of the modules have not yet been verified.

Today’s Achievements#

Core Achievements#

  • scGPT zero-shot clustering was less effective than PCA/scVI in independent benchmarks (Genome Biology 2025). scGPT-spatial only compared with a weak baseline and had no independent third-party verification. The value of using scGPT as a zero-shot gene encoder for the MIHD project is questionable, and the strategy for choosing a gene encoder needs to be re-evaluated.
  • The offline injection architecture (first collecting complete trajectories → detecting injectable points and building indexes → injecting in batches according to quotas) is more suitable for creating balanced error scenario datasets than the online quota system. It decouples ’exploring injectability’ from ’executing injections’, allowing precise control of the number of each type of error. The online natural capture based on the BC-RNN strategy will strongly favor high-frequency errors, and some types are almost impossible to trigger naturally.
  • CALVIN evaluation is a completely online simulation that does not read episode .npz frame data at all. Only validation/.hydra/merged_config.yaml is needed to initialize the simulation environment. The 1.3GB dataset can be compressed into a 600KB eval-only version, significantly reducing storage and transmission costs.
  • In MCP server, the capture_stdout() context manager is a key security design issue: All legacy code that depends on print() and sys.exit() must execute within this context. Otherwise, any print output will contaminate the JSON-RPC stdio transmission, leading to protocol errors. MCP tools should return content rather than file paths. File writing is an optional side effect.
  • RTX 2080 Ti + PyTorch 2.9.0+cu129 has a cuBLAS large tensor bug (N>3500 high-dimensional operations) and a CUDA deterministic compatibility issue with PyG scatter. General strategy: first project to low-dimensional hidden_dim, then aggregate neighbors. Use mini-batch instead of the full N×N matrix for contrast loss. It should be fixed to verified version combinations (PyTorch 2.1-2.4).
  • Pi0 flow matching time convention: t=1 corresponds to pure noise, t=0 corresponds to target action (opposite of some literature directions). The Beta(1.5,1) distribution has a higher weight for noise terms, making training more stable. Pi0.5 uses adaRMS normalization for injecting time conditions, with stronger expression ability than simple concatenation.
  • UniVLA’s three-module joint training: LAM online encoding (initial frames, target frames) → VQ-VAE discrete coding as a supervisory signal; VLA main branch predicts latent action tokens; ActionDecoder decodes continuous actions from the VLA hidden state. Each step requires 12 consecutive frames and depends on online LAM inference, with significant computational overhead.
  • PubMed esearch→efetch two-step E-utilities API can freely index subscription journals like Nature/Cell/Science. It is the best free alternative to obtain metadata for these journals. The bioRxiv API(api.biorxiv.org/details/biorxiv) is also open, requiring no new dependencies (urllib.request).
  • uvx creates a temporary environment that is not suitable for MCP servers that rely on local data directories. pip install -e .(editable install)+console entry point should be used to make the server run within the repo directory to maintain stable data paths. The standard way to call scripts across conda environments is: conda run -n python script.py, and setting cwd to the script working directory is key.
  • DLPFC cross-sample RM-IDEAL results: Layer_1 (ρ=0.62) and Layer_4 (ρ=0.66) have unique structures with clear boundaries, and the fusion embedding performs well. Layer_3 (ρ=-0.21) has high internal heterogeneity, making it a major challenge for cross-sample retrieval.
  • Error labels should be correct from the moment they are generated (use taxonomy type names instead of validator names). Dependence on post-mapping will掩盖 real type distribution issues. The cached embedding should have its statistical characteristics verified before writing (norm range). Zero values are a clear sign of model collapse.
