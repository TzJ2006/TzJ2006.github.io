Issues and Solutions#

Key Issues#

1. RTX 2080 Ti vulnerability in PyTorch 2.9/CUDA 12.9 cuBLAS caused crashes during backpropagation for large matrix operations (comparative loss required 3639×3639 logits). Multiple architectural rewrites were attempted before finding the root cause.#

Solution: Redesigned the comparison loss function, using mini-batch sampling (512 samples per batch) to avoid full N×N matrix calculations. Added gradient clipping and LayerNorm for stability. For cross-sample visualization tasks, when GPU incompatibility was confirmed, switched to CPU training as an alternative solution (PyTorch 2.9.0+cu129 vs CUDA 13.1 driver).

Key Insight: Model architecture alone cannot solve GPU hardware/driver compatibility issues—algorithmic solutions are required at the loss function level. Incompatibility between PyTorch CUDA version and driver version can lead to “illegal memory access” errors, which can be detected through basic tests but fail in complex training loops. For medium-scale training (3460 sample points), using CPU as an alternative is a valid strategy.

2. MCP server stdout corruption: gadget tools heavily use print(), but MCP uses stdout as a JSON-RPC transport method#

Solution: Created capture_stdout() context manager to redirect print() output, wrapped all tool calls in safe_call(), captured stdout/stderr and converted SystemExit to RuntimeError, returning the captured logs in the _logs field of the JSON response.

Key Insight: Isolating stdout is crucial when integrating existing CLI tools into protocol-based systems. The context manager provides a clean, recoverable way to redirect output without modifying the original code

3. research_scout hard-codes arXiv_id throughout the codebase, hindering multi-source expansion#

Solution: Introduced paper_id as a standard ID with source field, retained arXiv_id as an backward-compatible alias, created _paper_id()/_paper_url() helper functions, updated prompts to use paper_id, and made the merging logic check both fields when matching LLM responses.

Key Insight: Introducing new abstractions from single-source to multi-source expansions while keeping old fields as aliases. Helper functions centralize ID/URL logic, making future data source additions simple

4. In cross-sample queries for 151673-151508, Layer_3 has low correlation (ρ=-0.21/-0.36), while Layer_1/4/5 show high positive correlations (ρ>0.4)#

Solution: Analysis showed that Layer_3 is the largest DLPFC layer with high internal heterogeneity, making embeddings unable to capture unified spatial structures. Smaller and well-defined layers (Layer_1: ρ=0.62, Layer_4: ρ=0.66) perform best.

Key Insight: The scale and internal heterogeneity of organizational layers are crucial for capturing spatial structure in embeddings. Independent layers with clear boundaries perform better cross-sample matching than large heterogeneous regions

5. When each sample has only one gene vector and one image vector, single-label cross-attention degrades to linear projection#

Solution: Used spatial k-NN neighbors to construct K/V sequences, allowing cross-attention to operate in neighborhood contexts. Reused QFormerFusion’s batch neighbor construction infrastructure.

Key Insight: Spatial transcriptomics data provides natural sequential context through neighborhood structures. Utilizing spatial adjacency allows meaningful attention to be achieved without time or batch dimensions

6. RoboTwin demo_clean data (14-DOF, based on directories) is incompatible with existing LeRobot conversion script (16-DOF, based on zip format)Solution: A new script, convert_robotwin_democlean_to_lerobot.py, was created. This script reads data directly from the data/commands directory, changes ACTION_DIM/STATE_DIM to 14, uses the LeRobotDataset.create() function, and sets the root parameter as an environment variable instead of using the HF_LEROBOT_HOME variable.#

Key Insight: Changes in the robot data format (number of DOF, storage format) require custom conversion scripts. The directory structure and HDF5 key names are project-specific.

7. Due to JSON parsing failures by the LLM, empty motivation/innovation fields appeared in the research_scout filtering stage.#

Solution: Empty field detection was added after _screen_papers() and _screen_papers_chunked(). Warnings with counts and DEBUG levels were recorded for each paper. The _try_repair_result() function was enhanced to record the original response preview in case of failures during repairs.

Key Insight: Unintentional.get() backtracking masked data quality issues. Clear verification and logging at the aggregation point (after batch processing) could provide actionable debugging information without complicating the logic for each project.

8. The initial CrossModalEnhancer design output original modal dimensions (image is 1536d), which caused cuBLAS GEMM errors during the 128→1536 projection.#

Solution: The architecture was redesigned to hide the dimension space output (128d) instead of keeping the large matrix projection problem. While maintaining a compact core output, projection heads were used for contrast loss calculations.

Key Insight: Engineering limitations (GPU failures) may force architectural decisions that differ from ideal semantic designs. The actual system must find a balance between theoretical elegance and implementability.

General Issues#

9. The FastMCP constructor rejected the ‘version’ parameter in the user’s conda environment.#

Solution: Inspect.signature() was used to check the actual constructor. The version parameter was removed, and only ’name’ and ‘instructions’ were passed, allowing the mcp library to handle version management internally.

Key Insight: The SDK API is constantly evolving. When integration issues arise, it’s important to check the actual code signature rather than relying on documentation or assumptions. The FastMCP API of the mcp library has changed over time.

10. The CLAUDE.md documentation is outdated—missing author search functionality, init –from-overview function, and incorrect line count statistics.#

Solution: The entire codebase was scanned using Grep to find undocumented functions like search_arxiv_author, create_project_from_overview. The line count was verified using wc -l. Six targeted modifications were made to the documentation section to update it with instructions, code structure, key implementation details, and file layout.

Key Insight: In tools that are actively developed, outdated documentation is inevitable. Systematic code scanning (such as grep –author, –from-overview, etc.) is more reliable than manual review.

11. DNS resolution failed in the Kubernetes Pod, preventing pip from accessing PyPI images.#

Solution: The issue was diagnosed to be the domain name server 10.96.0.10, with a proxy local host of 9997. However, this did not solve the problem—it might be a cluster-level network configuration issue.

Key Insight: The DNS and outbound proxy configurations of the Kubernetes cluster can disrupt the standard pip workflow. It may be necessary for the cluster administrator to intervene or use cached packages.

12. A “Parameter Not Recognized” error occurred when using the –no-banner flag during conda runtime on the user’s conda version.#

Solution: The –no-banner flag was completely removed. Using conda run without this flag worked normally, and the banner output was sent to stderr (without damaging stdout). Testing showed that using conda run -n calendarpro python -c ‘print(test)’ was feasible.

Key Insight: If certain optional flags improve the clarity of the output but are not necessary in terms of functionality, they should be removed. Bannering in stderr does not affect the protocol based on stdout.

Lessons Learned#

Key Lesson - RM-IDEAL uses WWL (Wasserstein Weisfeiler-Lehman) graphs to calculate spatial niche matching: By iterating neighbor tag hashing → node feature distribution → optimal transmission between subgraphs (Wasserstein distance), it provides a topological benchmark for evaluating embedding quality.### Cross-Attention for Single Tags#

Cross-attention on individual tags degenerates into linear projection. Spatial transcriptomics offers a natural solution: k-NN neighbors form a K/V sequence, encoding neighborhood context. This pattern (using spatial neighbors as attention context) applies to various spatial genomics architectures.

PyTorch 2.9 + CUDA 12.9 Issues with cuBLAS on RTX 2080 Ti#

There are errors in the cuBLAS library for PyTorch 2.9 + CUDA 12.9 on RTX 2080 Ti, leading to illegal memory access issues during large matrix operations. For example, gemm operations of shape 128×1536 fail, and reverse operations with N×N greater than 3500 also fail. Solution: Use small batch processing to keep the matrix size small. The CUDA build version of PyTorch (e.g., cu129) must match the GPU driver’s CUDA version.

scGPT Performs Poor in Zero-Sample Clustering Benchmarks#

scGPT performs poorly in zero-sample clustering benchmarks (Genome Biology 2025). The AvgBIO score on the Pancreas/PBMC dataset shows that scGPT has a performance range of 0.52-0.60, while HVG baseline scores are 0.60-0.77. scGPT-spatial requires fine-tuning and can only be compared with weaker baseline models (SpaGCN, stLearn), not with the latest models (GraphST, STAGATE). Independent benchmarks are much more reliable than claimed in the original paper.

Conditional Flow Matching for Robot Motion Generation#

The vector field u_t is learned by linear interpolation of the path x_t = t·noise + (1-t)·actions, from noise to action. A time sampling method with Beta(1.5,1) ensures stability near noise endpoints, and an Euler integration of 10 steps is used for inference. The key insight is that temporal conditioning is achieved through sine embeddings and MLP (or adaRMS for Pi0.5).

Design Principles for MCP Tools#

Claude Code tools should default to returning content (Markdown reports, JSON data) and optionally performing file persistence. This allows contextual use without traversing the file system. Summaries/research are “enhancing skills,” not just file generators.

stdout/stderr Isolation as a First-Level Design Principle in Packaging CLI Tools for Protocol-Based Systems#

Isolating stdout/stderr is a first-level design principle, not something considered after the fact. Using context managers allows for clear, reversible redirection without modifying the original code.

Introducing New Abstract Layers (paper_id) and Maintaining Old Field Names (arxiv_id) When Generalizing Single-Source Systems to Multi-Source Systems#

Auxiliary functions (_paper_id, _paper_url) centralize cross-source logic, making future extensions simpler.

Characteristics of DLPFC Tissue Level Affecting Embedding Sample Matching#

Layer_1 (sparse, thin) and Layer_4 (granular, densely cellular) show Spearman ρ>0.6 correlations; the most heterogeneous layer, Layer_3, shows ρ<-0.2 correlations. The quality of embeddings is related to the uniqueness of structures and boundary clarity, not just the size of the layers.

RoboTwin Data Format#

A 14-degree-of-freedom motion space (6 degrees of freedom for left arm + 1 degree of freedom for gripper + 6 degrees of freedom for right arm + 1 degree of freedom for gripper), stored in HDF5 format, with joint_action/vector fields saved and observation values stored as compressed image formats. The Demo_clean file structure includes data/episodeX.hdf5 and instructions/episodeX.json, containing information about seen/unseen language variants.

STAIG Fusion Training for Spatial Genomics#

UNI2 (1536 dimensions of visual information) + PCA (50 dimensions of genetic information) → STAIG fusion encoder (64 dimensions). Key points: use tau=26 for alignment mode, softmax_log boundary weights, 300 hypothetical clusters, 550 training cycles. If training fails, the model may collapse (all embedding values become zero).

UniVLA Training Process Divided into Three Stages#

(LAM Phase 1/2, VLA pre-training, fine-tuning for each task). CALVIN training: DiskCalvinDataset loads auto_lang_ann.npy for episode language mapping, reads episode_.npz windows (12-13 frames), extracts rgb_static → LAM → latent_action_idx → VLA input → ActionDecoder → relactions.

Conditionally Running Tools (conda run -n )#

Provides clear isolation for sibling projects with dependency relationships. Setting the working directory as the root level of the tool project is important because many Python scripts will use relative imports and sys.path operations.

Production-Level Logging Requires Two Processors#

StreamHandler (INFO level) for displaying visible progress information, RotatingFileHandler (DEBUG level) for logging diagnostic details. The print() method is retained in interactive UI commands (lists, displays), but all operation/diagnostic messages are transferred to logger.

CALVIN Evaluation Dataset Structure#

Only validation/.hydra/merged_config.yaml is needed for environment initialization. During evaluation, episode .npz files are not read (online simulation generates observations). Data extraction reduces the dataset size from 1.3GB to 600KB.

Zero-依赖性API Client (urllib.request + xml.etree) Feasible for Production Tools#

bioRxiv and PubMed use standard libraries to achieve this, maintaining minimal dependencies.

Python Package Distribution for MCP Server#

Standard mode provides stateless tools via PyPI and uvx. However, when the server needs to rely on project data directories, pip install -e . (editable installation method) is required. In pyproject.toml, specify [project.scripts], then use Claude mcp add gadget –gadget-mcp to add tools.

Actual Learning Outcomes#

  • When integrating new fusion strategies into existing processes: 1) Register in the apply_fusion() scheduler, 2) Add needs_coords tuple in runner.py and evaluation_planner.py, 3) Add config dictionary items in runner.py, 4) Define experiments in pipeline_config.yaml. This four-step integration approach is standard for all trainable fusion methods.
  • When updating documentation, use grep to search for CLI symbols and function names, allowing systematic identification of omissions rather than relying on manual code review.
