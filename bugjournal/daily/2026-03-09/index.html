Issues and Solutions#

Key Issues#

1. Bug in PyTorch 2.9/CUDA 12.9 cuBLAS on RTX 2080 Ti causes crashes during backpropagation of large matrices (comparative loss requires 3639×3639 logical vectors). Multiple architectural rewrites were attempted before finding the root cause.#

Solution: Redesigned comparative loss function, using small batch sampling (512 samples per batch) to avoid full N×N matrix calculations. Added gradient clipping and LayerNorm to improve stability. For cross-sample visualization tasks, when GPU incompatibility is confirmed, switch to CPU training as an alternative (PyTorch 2.9.0+cu129 compatible with CUDA 13.1 drivers).

Key Insight: Model architecture alone cannot solve GPU hardware/driver compatibility issues—algorithmic solutions are needed at the loss function level. Incompatibility between PyTorch CUDA version and driver version can lead to “illegal memory access” errors, which are tolerated in basic tests but fail in complex training loops. For medium-sized training tasks, using CPU as an alternative is a viable strategy.

2. MCP server stdout corruption: Gadget tools heavily use print(), but MCP uses stdout as JSON-RPC transmission.#

Solution: Created capture_stdout() context manager to redirect print() output, wrapped all tool calls in safe_call(), captured stdout/stderr and converted SystemExit to RuntimeError, returning captured logs as the _logs field in JSON responses.

Key Insight: Isolating stdout is crucial when integrating existing CLI tools into protocol-based systems. Context managers provide clean, recoverable redirection without modifying the original code.

3. research_scout hard-codes paper_id throughout the codebase, hindering multi-source data expansion.#

Solution: Introduced paper_id as the standard ID, set source fields, retained paper_id as an alias for backward compatibility, created _paper_id()/_paper_url() helper functions, updated prompts to use paper_id, checking both fields when matching LLM responses.

Key Insight: Introducing new abstraction layers while maintaining old fields as aliases allows easy addition of new data sources. Helper functions centralize ID/URL logic, making future data source additions simple.

4. In 151673-151508 cross-sample queries, Layer_3 has low correlation (ρ=-0.21/-0.36), while Layer_1/4/5 show high positive correlations (ρ>0.4).#

Solution: Analysis showed that Layer_3 is the largest DLPFC layer with high internal heterogeneity, making it difficult for embeddings to capture unified spatial structures. Smaller and more well-defined layers (Layer_1: ρ=0.62, Layer_4: ρ=0.66) perform best.

Key Insight: The size and internal heterogeneity of organizational layers significantly affect the quality of spatial structure embeddings. Layers with clear boundaries perform better in cross-sample matching than larger heterogeneous regions.

5. When there is only one gene vector and one image vector for each position, single-label cross-attention degrades to linear projection.#

Solution: Used spatial k-NN neighbors to construct K/V sequences, allowing cross-attention to operate within neighborhood contexts. Reused QFormerFusion’s batch neighbor construction infrastructure.

Key Insight: Spatial transcriptomics data provides natural sequential context through neighborhood structures. Utilizing spatial adjacency enables meaningful attention mechanisms even without time or batch dimensions.

6. RoboTwin demonstration data (14-DOF, directory-based structure) is incompatible with existing LeRobot conversion script (16-DOF, zip-based structure)Solution: A new convert_robotwin_democlean_to_lerobot.py script was created: it reads data directly from the data/directives directory, changes ACTION_DIM/STATE_DIM to 14, uses the LeRobotDataset.create() method, and specifies the root parameter instead of using the HF_LEROBOT_HOME environment variable for modifications.#

Key Insight: Changes in the robot data format (number of DOFs, storage format) require a dedicated conversion script. The directory structure and HDF5 keynames are project-specific.

7. Due to JSON parsing failures by the LLM, the motivation/innovation_point field is empty during the research_scout filtering phase.#

Solution: Empty field detection was added after _screen_papers() and _screen_papers_chunked(). Warnings with detailed counts and DEBUG level information were recorded. The _try_repair_result() function was enhanced to record a preview of the original response when repairs fail.

Key Insight: The silent.get() backtracking mechanism masks data quality issues. Explicit verification and logging at the aggregation point (after batch processing) can provide actionable debugging information without complicating the logic of each project.

8. The initial CrossModalEnhancer design triggered cuBLAS GEMM errors during the projection process when outputting the original modality dimensions (image is 1536d).#

Solution: The architecture was redesigned to hide the output from the dimension space (128d), avoiding problems with large matrix projections. The projection head was used in the contrast loss while maintaining the compactness of the core output.

Key Insight: Engineering limitations (GPU errors) may force architectural decisions to differ from ideal semantic designs. The actual system must balance theoretical elegance and implementation robustness.

Lessons Learned#

Key Lessons—RM-IDEAL uses WWL (Wasserstein Weisfeiler-Lehman) graph kernel to calculate spatial niche matching: processes data by iterating neighbor tag hashing, then performs optimal transmission between subgraphs based on node feature distributions (using Wasserstein distance). This method provides a topological benchmark for evaluating embedding quality.### Practice Notes#

  • Cross-attention processing on individual markers degenerates to linear projection. Spatial transcriptomics provides a natural solution: k-NN neighbors form a K/V sequence, encoding neighborhood context. This approach works for various spatial omics architectures.
  • PyTorch 2.9 + CUDA 12.9 has errors in cuBLAS on RTX 2080 Ti, leading to illegal memory access issues during large-scale matrix operations. For example, matrix operations fail when the shape changes from 128 to 1536, or when N exceeds 3500 in N×N mode. The solution is to reduce the matrix size for batch processing. The CUDA build version of PyTorch (e.g., cu129) must match the GPU driver’s CUDA version.
  • The scGPT base model performs worse than traditional methods (such as PCA, scVI, Harmony) in zero-shot clustering benchmarks (Genome Biology 2025). In the Pancreas/PBMC dataset, scGPT scored between 0.52-0.60, while the HVG baseline model scored between 0.60-0.77. scGPT-spatial requires fine-tuning and can only be compared with weaker baseline models (such as SpaGCN, stLearn), not state-of-the-art models (such as GraphST, STAIGATE). Independent benchmark results are more reliable than those described in the original paper.
  • Conditional flow matching for robot action generation: Learn the vector field u_t by linearly interpolating the path x_t = t·noise + (1-t)·actions. Use Beta(1.5,1) time sampling to ensure stability near noise endpoints. Use Euler integration for inference with a step size of 10. The key idea is to implement time conditioning through sine embeddings and MLP (or adaRMS for Pi0.5).
  • Design principles for MCP tools: Claude Code tools should default to returning content (e.g., Markdown reports, JSON data) and allow selective saving of files. This allows direct use of these tools without involving the file system. Summarization/research is about “enhancing skills,” not just generating files.
  • When wrapping existing CLI tools for protocol-based systems (e.g., MCP, JSON-RPC), separating stdout/stderr is an important architectural consideration, not something that should be done later. Using a context manager allows clean, reversible redirection without modifying the original code.
  • When extending a single-source system to multi-source systems, introduce new abstractions (e.g., paper_id) while retaining old fields (e.g., arxiv_id) as aliases. Auxiliary functions (e.g., _paper_id, _paper_url) can handle cross-source logic, making future extensions easier.
  • The characteristics of the DLPFC organizational layer directly affect cross-sample matching of embedded data: Layer_1 (superficial, thin layer) and Layer_4 (granular, densely packed) show Spearman ρ>0.6 correlation; while Layer_3 (maximum, heterogeneous layer) shows ρ<-0.2 correlation. The quality of embedded data is related to the uniqueness of its structure and the clarity of its boundaries, rather than the size of the layers.
  • RoboTwin data format: Contains a 14-degree-of-freedom action space (6 degrees of freedom for left arm, 1 degree of freedom for gripper, 6 degrees of freedom for right arm, 1 degree of freedom for gripper), stored in HDF5 format. The joint_action/vector fields are included in the data, while the observation data is stored in compressed image format. The Demo_clean directory includes data/episodeX.hdf5 and instructions/episodeX.json, which contain observation data for different language versions.
  • STAIG fusion training for spatial omics: UNI2 (1536-dimensional visual data) + PCA (50-dimensional gene data) → STAIG fusion encoder (64-dimensional). Key findings include using tau=26, softmax_log edge weights, 300 pseudo-clusters, and 550 training cycles. If training fails, the model may crash (all embeddings become zero).
  • The UniVLA training process consists of three stages: (1) LAM phase 1/2, (2) VLA pre-training, and (3) fine-tuning for each benchmark test. During CALVIN training, DiskCalvinDataset loads auto_lang_ann.npy to obtain the mapping between episodes and languages, then reads 12-13 frames of data from episode_.npz files, extracts rgb_static data, processes LAM, and finally obtains latent_action_idx, which is then input into VLA to obtain the actual actions via ActionDecoder.
  • Cross-environment tool calls (conda run -n ) provide a clean isolated environment for sibling projects with conflicting dependencies. It’s important to set the current working directory as the root directory of the tool project, as many Python scripts use relative imports and sys.path operations.
  • Production-level logging requires two handlers: StreamHandler (INFO level) for user-readable progress information, and RotatingFileHandler (DEBUG level) for recording diagnostic details. You can use print() in interactive UI commands, but all operational and diagnostic messages should be logged through logger.
  • The structure of the CALVIN evaluation dataset requires only verifying/.hydra/merged_config.yaml file for environment initialization. During evaluation, the episode_.npz file isn’t read (online simulations generate observation data). Data extraction reduces the dataset size from 1.3GB to 600KB.
  • Zero-dependency API clients (e.g., urllib.request + xml.etree) are ideal for production tools, as they avoid introducing new dependencies. bioRxiv and PubMed use only standard libraries to create these clients, minimizing the number of dependencies.
  • The standard Python package distribution pattern for MCP servers uses PyPI and uvx to handle stateless tools. However, when the server relies on the repository’s data directory, pip install -e . should be used (an editable installation method).

Practice Insights#

  • When integrating new fusion strategies into existing processes: First, register them in the apply_fusion() scheduler. Second, add the needs_coords tuple in runner.py and evaluation_planner.py. Third, add configuration dictionary entries in runner.py. Fourth, define experiments in pipeline_config.yaml. This four-step integration approach is standard for all trainable fusion methods.
  • When updating documentation, use grep to find CLI flags and function names, so you can systematically identify missing content instead of relying on manual code reviews.
