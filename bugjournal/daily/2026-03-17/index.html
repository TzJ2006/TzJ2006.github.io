Issues and Solutions#

Key Issues#

1. QCVLA: Only improved by 1 percentage point (60%→61% SR) in the second round of Dagger, indicating saturation. Bridgers can only see their own natural error distributions, which are limited in diversity.#

Solution: Proposed perturbation and bridging data augmentation methods: During Dagger propagation, randomly inject short action sequences followed by reverse recovery actions. This allows Bridgers to capture larger deviations that would never occur naturally, thereby expanding the training distribution. At the same time, train the Bridger (for feature prediction of perturbation states) and action heads (recovery action sequences).

Key Insight: When the model only sees its own error patterns, self-supervised training can become saturated. Injecting synthetic but real deviations (inspired by FLARE’s perturbation strategy) can expand the distribution range. Compared to Gaussian noise per frame, coherent action perturbations are more effective in robot control.

2. VLA Inference: Infinite loops occurred because current_prompt was not reset after task completion, keeping chunk_count non-zero and the model continuously outputting high scores.#

Solution: Set current_prompt = "" after task completion and call publish_state(“idle”) to prevent immediate triggering in the next round; move None checks before task_completed.item() to avoid crashes in pi05 models.

Key Insight: The state machine needs to explicitly reset all related variables at each termination node, otherwise unpredictable loops can occur. Multi-device compatibility (pi05 and non-pi05) requires handling boundary conditions in advance.

3. MIHD: NEEDS_COORDS_FUSIONS tuple is duplicated in pipeline/runner.py and pipeline/evaluation_planner.py, and both lack adaln_attention (potential error), making it difficult to maintain consistency when adding new fusion strategies.#

Solution: Centralize NEEDS_COORDS_FUSIONS as a constant in models/Fusion.py, with both files importing this constant. Add the missing adaln_attention to the tuple, solving the problem of fusion strategies not being able to handle spatial coordinates.

Key Insight: Duplication of constants between modules leads to inconsistencies, requiring a single source to ensure no potential errors and simplify maintenance work.

4. MIHD: refine_labels_spatial_majority in utils/clustering.py uses scipy.spatial.distance.cdist, which has an O(N²) memory complexity, causing OOM issues on Visium HD data with 17K+ points.#

Solution: Replace the cdist algorithm with scipy.spatial.cKDTree.query, which has an O(N*k) complexity suitable for k-nearest neighbor searches. Switch from calculating complete distance matrices to querying radius neighbors as needed.

Key Insight: For large point clouds, using tree-based structures (such as KDTree, BallTree) can avoid memory explosions. For HD spatial data (10K+ points), the algorithm choice is more important than implementation details.

5. Combining combined_action_loss and task_loss into one value and logging it to wandb prevents monitoring each value separately. Solution: Modify compute_loss return value to (tuple)(combined_loss, {“action_loss”: …, “task_loss”: …}), and use nxn.value_and_grad(…, has_aux=True) in the training loop to parse auxiliary metrics and add them to the info dictionary.#

Key Insight: JAX/Flax’s has_AUX=True mechanism allows for returning additional monitoring metrics without affecting gradient calculation; all compute_loss implementations (pi0 and pi0_fast) need to be synchronized to maintain consistency of interfaces.

6. In error benchmark testing, thread tasks can only inject collision_empty type errors, other types cannot be generated.#

Solution: Analysis revealed that all clean trajectory phase_labels for threads were ‘pre_reach’, missing grasp/lift/transport stages, resulting in errors dependent on these stages (such as drop_in_transit) not being injected; the get_task_phase logic or thread task definition needs to be checked.

Key Insight: Error injection depends on the division of task phases; incomplete phase identification can lead to certain types of errors being overlooked; it’s necessary to verify all tasks cover expected phases when generating opportunity graphs.

7. gpumon.py counts bash/ffmpeg/claude processes, which have inherited CUDA_VISIBLE_DEVICES as GPU processes, and displays DDP working processes separately from parent processes.#

Solution: Modified get_proc_gpus() function to require both CUDA_VISIBLE_DEVICES environment variables and open/dev/nvidia* file descriptors; added parent chain traversal functionality in get_gpu_procs() to merge child processes with the same GPU set with their ancestors.

Key Insight: Detecting process use of GPUs requires checking environmental variables for inheritance and actual device file descriptors to avoid false positives.

8. All 317 RoboCasa tasks encountered errors downloading MimicGen data, unclear whether it’s a code issue or data availability problem.#

Solution: System analysis showed that 257 tasks had no mg_path registered (yellow warning), while 60 tasks had mg_path but no MimicGen download links in box_links_ds.json (red error). This confirmed an upstream data publishing issue rather than a code defect.

Key Insight: When system-wide download failures occur, it’s necessary to check the availability of data before attempting to fix the code. Using grep on box_links_ds.json to search for ‘/mg/’ yielded no matches, clearly indicating a lack of download links.

9. CALVIN RLDS dataset conversion failed with KeyError: ‘image’, the actual field names should be ‘rgb_static’ and ‘rgb_gripper’.#

Solution: Read features.json to confirm the actual schema, modified the keys for frame extraction, and removed missing environment_state and relative_action field definitions from the data set.

Key Insight: Before converting dataset formats, it’s necessary to verify the schema of the source data; RLDS features.json is an authoritative source for determining the actual field names.

10. How to implement interactive keyboard navigation in monitoring tools that require periodic updates while avoiding blocking input.#
