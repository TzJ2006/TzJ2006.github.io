AI Limitations#

Major Limitations#

  • Threading phase labeling is all pre_reach problems: AI can diagnose symptoms (checking NPZ files and get_task_completion_stages code), but it cannot run the robosuite environment to actually verify sensor readings. It can only infer the root cause and provide directions for troubleshooting.

General Limitations#

  • AI tends to check external documentation/remote resources rather than prioritizing local information: During LIBERO suite analysis, multiple Explore subagents were started consecutively, while git status already contained the answer. When diagnosing CALVIN dataset issues, it first tries to fetch external GitHub URLs instead of reading user-specified local code files. It should prioritize reading diff/status and local files before deciding whether to delve deeper.
  • Git remote authentication cannot handle this on its own: SSH keys, GitHub tokens, and VS Code credential helpers are all on the client side. AI can only provide command solutions but cannot execute pushes directly. In server environments, when credential helpers are unavailable, user intervention is required.
  • CALVIN conversion script requires multiple iterations (name errors → key name errors → directory existence errors) to be resolved. Each fix exposes the next problem, and it failed to foresee all issues through a一次性 review of features.json and the entire code.

Today’s Takeaways#

Key Takeaways#

  • Task state machine design principle: Completing an event must involve two actions simultaneously - resetting trigger conditions (clearing current_prompt) and broadcasting state changes (_publish_state(‘idle’)). Doing either alone will result in hidden bugs that lead to infinite loops or failure of upstream perception. Both are essential.
  • JAX has_aux mechanism: nnx.value_and_grad supports has_aux=True parameter, compute_loss can return(loss_array, aux_dict), gradients are calculated only for loss_array, and aux_dict transparently carries monitoring indicators. This is the standard way to carry auxiliary information in JAX functional design.
  • Spatial data nearest neighbor query: For large-scale point clouds (>10K points), radius nearest neighbor queries should prioritize KD trees (O(N log N) preprocessing), rather than cdist (O(N²) memory). In HD spatial transcriptomics scenarios with 17K+ cells, the difference between the two can result in OOM vs normal operation.
  • Reliable method for detecting GPU processes: In a Kubernetes PID namespace isolation environment, nvidia-smi cannot display process information. Scanning/proc//fd for.dev/nvidia* FDs is a more reliable way to detect processes, but FD evidence must be provided to avoid false reporting due to inherited environment variables.
  • Single Source of Truth principle: Constant collections that need to reference multiple files (such as NEEDS_COORDS_FUSIONS) must be imported from the only definable source. Manually maintaining multiple copies inevitably leads to inconsistencies, which often occur in edge cases (such as missing adln_attention).
  • Error messages for dataset download tools may come from multiple levels (registry not registered vs download link not exist). It’s necessary to distinguish them to correctly determine the root cause. Directly checking configuration files (such as box_links_ds.json) is more efficient than analyzing error messages, sometimes allowing us to discover that the root cause is an unpublished upstream data rather than a local code bug.
  • Benchmark data quality: Uneven distribution of data (some error types with D0<10) is a systematic issue. Distribution monitoring and balancing mechanisms should be incorporated into the pipeline design phase. General framework adaptation for specific tasks (such as threading needle insertion) requires clear task-level testing and verification.

Practical Takeaways#

  • The field names in TFRecord datasets in RLDS format vary depending on the dataset. It’s necessary to confirm the actual schema through features.json, rather than relying on experience from other datasets or default keys in code templates.
  • On shared servers, the priority of the WANDB_API_KEY environment variable is higher than ~/.netrc (location for wandb login storage). This can serve as a lightweight solution that overrides global accounts without requiring modifications to wandb configuration files.
