Problems & Solutions#

Critical Issues#

1. macOS non-sandbox applications cannot persist file access permissions. Old videos in history records report Operation not permitted errors.#

Solution: Enabled App Sandbox permissions (com.apple.security.app-sandbox and files.user-selected.read-write). Fixed ensureFileAccess to add non-scoped bookmark fallback parsing. Added NSOpenPanel reauthorization mechanism for showVideo/showImage.

Key Insight: macOS TCC requires the App Sandbox to create security-scoped bookmarks. Non-scoped bookmarks can only parse URLs without granting access rights, resulting in inability to open old files. A combination of sandbox and downgrade paths is needed.

2. Torch 2.7.1 and torchvision 0.21.0 are incompatible, causing torchvision::nms operator registration failures.#

Solution: Upgraded torchvision to 0.22.1+cu126 to match torch 2.7.1.

Key Insight: Custom operators in torchvision link to the C++ API of torch during compilation. Incompatible versions cause operator registration failures. torchvision 0.21.0 is compatible with torch 2.6.x, not 2.7.x.

3. The pre-compiled curobo.so file is incompatible with the current PyTorch ABI. JIT compilation cannot find cuda_runtime.h.#

Solution: Deleted the old.so file, installed the ninja compiler, set CUDA_HOME to /path/to/conda.env and CPATH to /path/to/targets/x86_64-linux/include/. Recompiled using pip install -e .

Key Insight: C++ extensions need to be recompiled after PyTorch version upgrades. Nvidia packages installed via pip in the conda environment place CUDA headers in non-standard paths (targets/x86_64-linux/include/), requiring correct environment variables to allow JIT compilation to find them.

General Issues#

4. The GPU monitoring tool’s output is too wide and processes are displayed repeatedly, making it difficult to read.#

Solution: Limited the output to 80 columns, aggregated GPU lists by process, and filtered out noise from multiprocessing workers and other sub-processes.

Key Insight: The original implementation grouped GPUs, causing a multi-gpu process to appear multiple times on multiple GPUs. It’s necessary to aggregate by process and display all used card numbers in the GPU column.

Learnings#

Key Learnings#

  • macOS TCC permission mechanism: Non-sandbox applications cannot create security-scoped bookmarks. They can only create non-scoped bookmarks (which only parse URLs without granting access rights), causing historical file access to fail after app restart. App Sandbox must be enabled to persist file access permissions.
  • PyTorch C++ extension ABI compatibility: Custom operators in torchvision (such as torchvision::nms) link to the C++ API of torch during compilation. Versions must be exactly matched (torchvision 0.21 → torch 2.6, torchvision 0.22 → torch 2.7). Incompatible versions cause operator registration failures.
  • Condna environment CUDA toolchain configuration: Nvidia packages installed via pip place CUDA headers in targets/x86_64-linux/include/ instead of the standard include/ directory. JIT compilation of C++ extensions requires setting CUDA_HOME to point to the conda env and CPATH to include the targets path.
  • Defensive programming principles: The combination of sandbox and reauthorization is more robust than a single solution. Sandbox addresses the root cause (persistent permissions), while NSOpenPanel provides a fallback path (temporary authorization). Together, they ensure users can always access files.
