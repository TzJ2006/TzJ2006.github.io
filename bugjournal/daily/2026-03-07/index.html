Lessons Learned#

Key Lessons#

  • Zero-shot baseless models perform significantly worse than PCA in clustering tasks: FM is designed to optimize generalization across tasks (pre-trained goal is language modeling/contrastive learning), not to maximize variance. PCA performs task-related dimensionality reduction, and maximizing variance means maximizing cluster separation. This conclusion is confirmed by Microsoft’s research and several papers, as well as MIHD’s own data (PCA’s ARI is 0.261, while scGPT’s is 0.170, a difference of 54%).
  • Evaluation metric bias often results from multiple errors rather than a single mistake: In the BC-RNN demo_len=0, trigger_step=0, and transport always False scenario, the successful episode RP dropped to the expected 0.5 due to four errors, but actually only reached 0.003. End-to-end validation is needed, not isolated unit tests.
  • The macOS window layer not only controls visual Z order but also event routing permissions: The.documentsWindow layer (located below desktop icons) doesn’t receive any user input events. True interactive “browsing mode” requires using the higher-level.normal layer. This is a system-level constraint, not an API configuration issue.
  • In a GPU cluster environment, it’s better to prioritize searching for existing model copies and performing format conversions (e.g., from HF format to original LLaVA format), rather than downloading through proxies. This can save weeks of time. Successful conversions from HF format to original LLaVA format (including vocabulary truncation, from 32064 to 32000) prove this.
  • When designing automated paper management tools, it’s important to follow human researchers’ cognitive processes: quick filtering (relevance + novelty + source authority) → in-depth analysis (core insights + problem definition + experimental design + limitations). Don’t try to evaluate everything at once.
  • h5py file handling cannot be shared between forked subprocesses (this is a limitation of PyTorch DataLoader). The only safe solution is to load the main process into numpy before forking. NFS random reads are much faster than sequential reads. Loading everything at once is 10 times faster than loading on demand. Using uint8 can reduce memory usage by 4 times.
  • Before starting large-scale training, it’s necessary to verify the actual dataset size, step time, and total number of steps to avoid wasting resources. In the Phoenix MPM training case, an incorrect estimate of dataset size increased it from 160,000 to 1 million, resulting in a 13.5-day delay in training time. This proves the importance of early verification.
  • Context-aware intent classification needs to combine session state (session_id, last_activity_at) and conversation history. Simple keyword matching can lead to ambiguity (e.g., “What time is it?” could refer to querying the time or asking about a schedule). CalendarPro solves this problem with context-dependent keyword classification and quick confirmation.
  • The research_scout.py architecture consists of a single 1871-line file, divided into 0-10 numbered sections. It has a two-stage LLM process (filtering → in-depth analysis) and three backends (claude_cli, anthropic, openai). The parameter parsing chain is CLI > project.json > config.json > default. Cache validation is done using keyword hashing.
  • Command line tool parameter configuration should use a hierarchical parsing strategy: CLI flags > specific project configurations > user-level global configurations > hard-coded defaults. This provides flexibility while maintaining reasonable default values.
  • Batch document generation tools need a two-stage architecture: the export stage parses raw data locally on each device, and the merge stage aggregates data across devices. When there are a large number of daily sessions, a “range merging” step is needed to remove duplicates and consolidate multiple outputs from the same day.
  • The solution to conflicts in conda environment import priorities for multi-user sharing is to set PYTHONPATH=/path/to/user/src:$PYTHONPATH during script execution. This prioritizes loading user’s own source files without affecting other users, and no separate environment needs to be created.
  • Language configuration for LLM processes needs to be injected at the prompt level, not later in the process. Different parts of the output require different rules (e.g., JSON keys should remain in English, and content should be in the target language).
  • Extracting structured data from documents along with an LLM works well. OVERVIEW_EXTRACT_PROMPT can reliably parse overview.md, extracting titles, keywords, categories, and questions.
  • Pipeline regular expression patterns must match exactly. “## 3. Current Technical Approach” won’t match “##\s*Current Method and Progress”. The template-driven architecture needs to strictly follow the format.
  • arXiv search results are returned in reverse chronological order. A continuous known paper counter is an effective early stopping heuristic because it indicates that the processed time range has been reached.

Practical Experience#

  • Key adjustments for cross-platform script migration: Windows uses Get-Item .LastWriteTime instead of find -newer, Start-Job instead of & parallel, Push-Location/Pop-Location instead of cd, [System.IO.File]::WriteAllText ensures UTF-8 without BOM, -replace instead of sed/perl. PowerShell and bash differ greatly in terms of process control and error handling mechanisms.
