Problems & Solutions#

Critical Issues#

1. BC-RNN RP(0.003) abnormally lower than Random RP(0.020), evaluation results completely unreasonable#

Solution: Code review found 4 bugs jointly causing metric distortion: demo_len=0 broke SPL, trigger_step=0 further broke SPL, transport always False capped RP at 0.5, successful episode RP not 1.0 caused baseline deviation. Fixed collector.py:487 field name demo_length→demo_len, collector.py:488 changed to read from scene.dataset for trigger_step, env_wrapper.py added _get_target_pos() runtime dynamic target position retrieval, collector.py added if success: rp=1.0

Key Insight: Evaluation metric distortion is often not a single bug but cascading chain of multiple bugs. RP ceiling of 0.5 revealed transport stage judgment failure, demo_len=0/trigger_step=0 revealed field propagation errors

2. Browse mode WKWebView cannot receive mouse/keyboard events, even with ignoresMouseEvents=false ineffective; .desktopWindow layer causes window non-interactive#

Solution: Set window.level=.normal in enterBrowseMode (not on top but interactive), restore CGWindowLevelForKey(.desktopWindow) in exitBrowseMode

Key Insight: macOS .desktopWindow layer is below Finder desktop icons layer, windows at this layer don’t receive any input events at system level, ignoresMouseEvents=false ineffective for this layer. .normal(level=0) is lowest layer that’s both interactive and doesn’t cover other apps

3. LLaVA base model download blocked by proxy bandwidth bottleneck (~50KB/s, downloading 14GB model would take weeks); GPU node no DNS cannot direct connect; xet CDN through proxy timeout#

Solution: Discovered cluster had nyw’s HF format llava-1.5-7b-hf (safetensors), wrote _convert_hf_to_llava.py to convert to original LLaVA pytorch_model.bin format (including vocab truncation fix 32064→32000), entire process completed locally without network

Key Insight: Don’t assume must download—first search for existing model copies on cluster, even different formats can be converted. HF model format conversion isn’t just renaming keys, involves semantic differences (like vocab size)

4. Phoenix diffusion policy training GPU utilization only 5-6%: DataLoader num_workers=0 caused GPU to wait for CPU random reads from NFS HDF5 files#

Solution: Preload all HDF5 data to memory once in init (uint8 saves 4× storage), close file handles, then all getitem slices from numpy arrays; changed DataLoader to num_workers=8, pin_memory=True

Key Insight: h5py file handles cannot be shared across fork child processes (known PyTorch DataLoader limitation), only safe solution is complete loading in main process before fork. NFS random reads far slower than sequential reads, one-time loading 10× faster than on-demand

5. Phoenix MPM training after launch discovered severe training duration estimation error (actual 13.5 days vs expected hours), dataset scale estimation error (160K→1M)#

Solution: User halted training, requires replanning and optimization (possibly including reducing training steps, increasing GPU count, optimizing data loading, etc.)

Key Insight: Before launching large-scale training, must first verify actual dataset size, per-step time and total steps to avoid resource waste. Dataset scale estimation error can lead to weeks of wasted compute resources

6. CalendarPro intent classification ambiguity due to general_keywords pre-check interception: ‘你安排在几点了呀’ → ‘几点了’ matches general_keywords → forced GENERAL → error#

Solution: Split general_keywords into always_general (always force GENERAL, like ‘天气’) and context_dependent (force GENERAL only without active session, like ‘几点了’); added confirmation_words (‘ok’,‘好的’,‘是的’ etc.) with _is_confirmation_input() fast path; rewrote _process_input() as 6-step flow

Key Insight: Context-aware intent classification needs to combine session state (session_id, last_activity_at) and conversation history, pure keyword matching causes ambiguity; confirmation phrases (‘ok’) should route directly when active proposal exists

7. Current research scout paper evaluation processes all 50 papers at once, wasting tokens and lacking depth#

Solution: Designed two-stage pipeline: Stage 1 fast screening of all papers (only title+abstract first 1000 chars, classify high/low relevance), Stage 2 deep analysis of high-relevance papers (max 20 papers, extract 3 highlights + detailed scoring + direction suggestions)

Key Insight: Mimicking human researcher paper reading workflow (fast screening→deep reading high-value papers) can dramatically improve LLM evaluation efficiency and quality, while also generating literature reading records for low-relevance papers for future reference

8. Pipeline couldn’t read overview.md due to project.json ID (‘robot-manipulation’) not matching directory name (’error-recovery-benchmark’)#

Solution: Updated project.json id field to match directory name; pipeline uses project[‘id’] to construct overview.md path

Key Insight: Directory name and project.json id must be identical; _extract_current_methods() uses project[‘id’] to locate overview.md

9. Stage 2 evaluation received ‘(暂无记录)’ for current methods because overview.md section header ‘## 3. 当前技术路线’ didn’t match pipeline’s regex pattern ‘##\s*当前方法与进展’#

Solution: Renamed overview.md section headers to match OVERVIEW_TEMPLATE format; removed numbered prefixes and section dividers

Key Insight: Pipeline uses strict regex matching for section extraction; template format must be exactly followed

10. FLARE norm stats calculation failed: norm stats script import openpi loaded zhaoganlong’s installed version, couldn’t find tangzijia’s new config name#

Solution: Set PYTHONPATH=/HOME/…/tangzijia/shared_deps/openpi/src:$PYTHONPATH in script to prioritize loading tangzijia’s openpi source copy, bypassing old version in conda env

Key Insight: When multiple users share same conda env, their source modifications create import priority conflicts. PYTHONPATH is minimal invasive solution, doesn’t affect other users

11. WKWebView scrollbar track transparent (CSS injection setting background-color ineffective) + sandbox app cannot load any network resources#

Solution: Removed webView.setValue(false, forKey: “drawsBackground”) to restore default opaque background making scrollbar track visible; added com.apple.security.network.client=true to .entitlements

Key Insight: drawsBackground=false makes entire WKWebView rendering layer transparent, CSS layering semi-transparent styles on transparent layer still transparent, fix must be at rendering layer not CSS layer. App Sandbox blocks all outbound network by default

12. LLM output contained too much English when user expected Chinese; no way to configure output language#

Solution: Implemented 3-tier language system: _DEFAULT_LANGUAGE constant, _language_instruction() generator, {language_instruction} in all prompts; config resolution chain CLI > project.json > config.json > default

13. No way to create project from existing overview.md without manually extracting keywords/categories#

Learnings#

Key Learnings#

  • Zero-shot foundation model embedding fundamentally underperforms PCA in clustering tasks: FM optimizes for cross-tissue generalization capability (pretraining objective is language modeling/contrastive learning), not variance maximization; PCA does task-aligned dimensionality reduction, maximizing variance≈maximizing cluster separability. This conclusion validated by Microsoft and 3+ papers + MIHD own data (PCA ARI=0.261 vs scGPT=0.170, 54% gap)
  • Evaluation metric distortion often not single bug but cascading chain of multiple bugs: BC-RNN’s demo_len=0, trigger_step=0, transport always False, successful episode RP not 1.0 four bugs jointly caused RP to drop from expected 0.5 to 0.003. Need end-to-end validation not isolated unit tests
  • macOS window layer controls not only visual Z-order but event routing permissions: .desktopWindow layer (below desktop icons) fundamentally doesn’t receive user input events, must use .normal or higher layer to implement truly interactive ‘browse mode’; this is system-level constraint not API configuration issue
  • In GPU cluster environment, prioritize searching existing model copies and performing format conversion (like HF format→original LLaVA format) rather than downloading through proxy, can save weeks; LLaVA base model conversion from HF format to original format (including vocab truncation fix 32064→32000) successful case proves this
  • When designing automated paper management tool, should mimic human researcher’s actual cognitive flow: fast screening (relevance+novelty+source authority)→deep analysis (core insight+problem definition+experimental design+limitations), not one-time full evaluation
  • h5py file handles cannot be shared across fork child processes (known PyTorch DataLoader limitation), only safe solution is main process preload to numpy then fork. NFS random reads far slower than sequential reads, one-time loading 10× faster than on-demand, can use uint8 to save 4× memory
  • Must verify actual dataset size, per-step time and total steps before launching large-scale training to avoid resource waste; Phoenix MPM training case where dataset scale estimation error (160K→1M) caused 13.5-day training duration severe mistake, proving importance of upfront validation
  • Context-aware intent classification needs to combine session state (session_id, last_activity_at) and conversation history, pure keyword matching causes ambiguity (like ‘几点了’ could be query time or ask scheduled time); CalendarPro resolved this through context_dependent keyword classification and confirmation fast path
  • research_scout.py architecture: 1871-line single file organized into 0-10 numbered sections; two-stage LLM pipeline (screening → deep analysis); three backends (claude_cli, anthropic, openai); parameter resolution chain CLI > project.json > config.json > default; cache validation by keywords hash
  • Command-line tool parameter configuration should use layered parsing strategy: CLI flag > project-specific config > user-level global config > hardcoded default, providing flexibility while maintaining reasonable defaults
  • Batch document generation tool design needs two-stage architecture: export stage locally parses raw data on each device, merge stage aggregates cross-device; when single-day session count especially large, also need ‘intra-segment merge’ step to deduplicate and consolidate multiple outputs from same day
  • Multi-user shared conda environment import priority conflict minimal invasive solution: set PYTHONPATH=/path/to/user/src:$PYTHONPATH at script execution, prioritize loading user’s own source copy, doesn’t affect other users and doesn’t require creating separate env
  • Language configuration for LLM pipelines requires injection at prompt level, not post-processing; different parts of output need different rules (e.g. JSON keys stay English, content in target language)
  • Reverse-engineering structured data from documentation works well with LLM; OVERVIEW_EXTRACT_PROMPT can reliably parse overview.md to extract title/keywords/categories/questions
  • Pipeline regex patterns must match exactly; ‘## 3. 当前技术路线’ won’t match ‘##\s*当前方法与进展’; template-driven architectures require strict format adherence
  • arXiv search returns results in reverse chronological order; consecutive-known-paper counter is effective heuristic for early-stop since it indicates reaching already-processed timeframe

Practical Learnings#

  • Cross-platform script porting key adaptations: Windows use Get-Item .LastWriteTime instead of find -newer, Start-Job instead of & parallel, Push-Location/Pop-Location instead of cd, [System.IO.File]::WriteAllText ensure BOM-free UTF-8, -replace instead of sed/perl. PowerShell and bash differ significantly in flow control and error handling mechanisms
