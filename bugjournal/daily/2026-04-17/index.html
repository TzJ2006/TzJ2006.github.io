Problems & Solutions#

Critical Issues#

1. Website menu layout completely broken (vertical stacking instead of horizontal nav bar) after config.yml modifications#

Solution: Discovered custom header.html used class=‘nav’ while PaperMod theme CSS (after update) expects class=‘header-nav’ — changed template to match theme

Key Insight: Custom theme overrides must stay synchronized with theme updates — CSS selector mismatches cause silent layout failures without build errors

2. faster-whisper running on CPU only on macOS (no CUDA), causing slow inference#

Solution: Migrated to mlx-whisper which uses Apple Silicon’s Metal GPU, changing from WhisperModel API to mlx_whisper.transcribe() with warmup step

Key Insight: CTranslate2-based models require NVIDIA CUDA; on Apple Silicon, mlx-based alternatives provide native Metal acceleration

3. Whisper producing repetitive hallucinations (‘Meh Meh Meh…’) when keyboard typing sounds captured#

Solution: Adjusted silence_threshold from 0.008 to 0.02, added compression_ratio_threshold=2.4 and no_speech_threshold=0.6 to filter non-speech audio

Key Insight: Low silence thresholds cause ASR hallucinations on non-speech sounds; VAD pre-filtering or stricter thresholds essential for noisy environments

4. Single-language ASR parameter insufficient for mixed-language meetings (Chinese+English)#

Solution: Designed –language with three modes: zh (force Chinese), en (force English), zh+en (Whisper uses initial_prompt for mixed-language, Qwen3-ASR auto-detects)

Key Insight: Mixed-language support requires backend-specific strategies: initial_prompt for Whisper, language=None for Qwen3-ASR, native bilingual for Sherpa

5. MeetingHelper ASR accuracy suffered in noisy environments without pre-processing#

Solution: Implemented NoiseSuppressor class using noisereduce library with spectral gating (stationary=True, prop_decrease=0.8), automatic noise profile calibration during first 0.5s of capture, integrated into sd.InputStream pipeline before ASR

Key Insight: Real-time audio processing requires denoising between capture and ASR, not after; RMS-based silence detection should happen post-denoising for accurate thresholding

6. Hugo build failures could produce empty or partial public/ directory, but update.sh would still attempt git push deployment#

Solution: Added if [ ! -f “$PUBLIC_DIR/index.html” ]; then exit 1; fi check after hugo command

Key Insight: Critical build steps need sanity checks before irreversible actions like git push — exit codes alone are insufficient

7. compress_video.py used module-level code that created temp files without cleanup if HandBrakeCLI failed, making it unsafe to import#

Solution: Refactored into compress_video() function with try/finally for temp file cleanup, moved CLI logic to main()

Key Insight: Python scripts intended for both CLI and import use must separate side effects into functions and use if name == ‘main’ guard

General Issues#

8. update.sh run_translation_phase() used shift to discard first argument but call sites passed description strings, creating confusing parameter handling#

Solution: Removed shift from function and removed description string arguments from all call sites

Key Insight: Shell functions don’t need to explicitly consume description parameters — let callers pass only what the function actually uses

9. stop.sh failed to reliably terminate subtitle window processes, leaving orphaned Python processes#

Solution: Implemented two-stage termination: send SIGTERM, wait 3 seconds, escalate to SIGKILL if alive, plus pgrep-based cleanup for orphans

Key Insight: PyObjC’s app.run() may not respond to basic signals; robust process management requires timeout + escalation + process scanning

Learnings#

Key Learnings#

  • Custom Hugo theme overrides must be audited against upstream theme changes — CSS class renames in themes silently break custom templates with no build errors, only runtime layout failures
  • Expensive background operations (large downloads, project scans) should be explicit user actions with progress indicators, not hidden automatic fallbacks
  • Apple Silicon ASR optimization: CTranslate2 (faster-whisper) requires CUDA, use mlx-based alternatives for Metal GPU acceleration on macOS
  • Audio pipeline denoising must occur post-capture, pre-ASR: inserting NoiseSuppressor after data.flatten() but before stream.accept_waveform() maximizes effectiveness. Noise fingerprint calibration using initial silent period (0.5s) enables stationary noise reduction without pre-trained models.
  • Browser-based visual debugging (DevTools inspect → check applied styles → trace selector mismatch) is more effective for CSS layout issues than server-side file verification
  • Mixed-language ASR requires per-backend strategies: Whisper needs initial_prompt guidance, Qwen3-ASR auto-detects with language=None, Sherpa has native bilingual models
  • ASR hallucination (repetitive output) often stems from low silence_threshold capturing non-speech sounds; VAD pre-filtering or stricter thresholds solve this
  • .claude/settings.json is team-shareable (should be committed), while .claude/settings.local.json is personal (should be gitignored)
  • Bash set -e does not propagate exit codes from background jobs — must explicitly capture exit codes with wait $pid || fail_flag=1 pattern for parallel task error handling
  • Claude Code hook system can only execute shell commands, not skills directly; use JSON output to prompt Claude for skill execution after hook triggers

Practical Learnings#

  • Process termination robustness requires timeout + signal escalation (SIGTERM → wait → SIGKILL) + cleanup scanning (pgrep)
  • ANSI terminal escape codes can pollute configuration files when values are copy-pasted from colorized CLI output, breaking parsers that expect clean strings
  • Python scripts intended for both CLI and import must use if name == ‘main’ guard and wrap side effects in functions — module-level code with resource allocation creates cleanup hazards
  • Hugo’s public/ directory is often a separate git repository for deployment (GitHub Pages) — rm -rf public/ during cache cleanup destroys deployment history
