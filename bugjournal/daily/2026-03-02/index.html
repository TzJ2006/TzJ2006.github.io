Problems & Solutions#

Critical Issues#

1. Desktop Video: 100% failure rate restoring videos/images after app restart with Code 257 ‘Operation not permitted’#

Solution: 4-part fix: 1) Add com.apple.security.files.bookmarks.app-scope to entitlements (was empty dict). 2) Add fallback in restoreFromBookmark: if .withSecurityScope fails, try options:[] to recover old non-scoped bookmarks, then re-save as scoped. 3) Track active URLs in activeSecurityScopedURLs dict, release in clear(). 4) ensureFileAccess() auto-resolves bookmarks before file I/O

Key Insight: macOS sandbox bookmark system requires BOTH compile-time entitlement declaration AND runtime URL.startAccessingSecurityScopedResource() lifecycle management. Empty entitlements → bookmark creation silently falls back to non-scoped → restore with .withSecurityScope fails → Code 257. The 879-line errorLog() was never writing errors due to concurrent FileHandle race condition, masking the root cause for months

2. MIHD: STAIG two-stage fusion ignores gene encoder embeddings, making scGPT+UNI2+STAIG equivalent to PCA+UNI2+STAIG#

Solution: Traced through models/Fusion.py and pipeline/runner.py: staig_fusion uses staig_gene_feat (raw HVG matrix from cache) instead of gene_emb parameter. This is architectural, not a bug. Documented in weekly report as key finding that affects interpretation of all STAIG results

Key Insight: Gene encoder (PCA/scGPT) outputs are only utilized by non-STAIG fusion methods. For graph-based fusion like STAIG, the pretrained spatial embeddings don’t enter the GCN node features. This explains why scGPT underperforms: it’s optimized for cross-tissue generalization, not within-dataset variance maximization like PCA

3. VLA Evaluation: Low GPU utilization (~10%) in Pi0.5 evaluation pipeline despite 100% utilization during training, causing slow evaluation throughput#

Solution: Root cause identified: Action chunking architecture where Pi0.5 outputs 50-step action sequences reduces inference calls to only ~8 per 400-step trial (8 × 2.5s = 20s GPU work / 200s total = 10% utilization). Solution planning interrupted but optimization strategies requested via web research and batched evaluation implementation

Key Insight: VLA evaluation bottleneck is not GPU compute capacity but pipeline serialization: action chunking minimizes GPU calls, and each inference cycle includes CPU preprocessing, WebSocket I/O, and simulation steps that dominate total time. Batched inference with timer-based request collection could improve throughput

4. SSH access denied on GPU nodes an49/an53 due to PAM slurm_adopt rejecting connections with ’no active jobs on this node’ error#

Solution: Bypassed SSH restriction using Slurm’s srun command with –jobid flag to execute commands directly in allocated job context: srun --jobid=45628 bash -c 'command'

Key Insight: Slurm’s srun with –jobid provides a way to run commands on allocated nodes even when PAM blocks direct SSH, useful for cluster environments with strict access control

General Issues#

5. Background task timeouts when using srun on heavily loaded GPU node (an49), with commands taking 60+ seconds to respond#

Solution: Added –overlap flag and reduced timeout expectations, falling back to simpler nvidia-smi when nvitop unavailable or too slow. Accepted that heavily loaded nodes have higher command latency

Key Insight: When GPU nodes are running 100% utilization workloads, administrative commands via srun can experience significant delays; design monitoring scripts to handle timeouts gracefully and use lightweight fallback commands

6. MIHD: mclust clustering hung indefinitely on scGPT (512d) and UNI2 (1536d) embeddings during visualization#

Solution: Switched from default mclust to kmeans via –cluster_method flag in visualize_from_cache.py. High-dimensional data causes mclust’s EM algorithm to become computationally intractable

Key Insight: Gaussian mixture model clustering (mclust) scales poorly beyond ~100 dimensions. For foundation model embeddings (512d-1536d), kmeans or spectral methods are necessary for practical runtime

7. Desktop Video log file filled with 1550+ spam entries per session (updateClockLabels × 879, startScreensaverTimer × 671)#

Solution: Removed dlog from updateClockLabels (called every second during screensaver). Moved dlog in startScreensaverTimer to after all early-exit guards (only logs when actually scheduling timer). Removed dlog from idle timer callback

Key Insight: High-frequency dlog calls in timer callbacks create massive log files that obscure real errors. Logging should be gated on state transitions, not periodic ticks

Learnings#

Key Learnings#

  • Foundation model embeddings (scGPT 512d, UNI2 1536d) are NOT drop-in replacements for dimensionality reduction (PCA 50d) in clustering tasks. scGPT optimizes for cross-dataset transfer, not within-dataset variance. Architecture choice (gene-only vs multimodal fusion) matters more than embedding quality: QFormer achieved 2× ARI improvement over scGPT-alone by learning gene-vision alignment. STAIG graph-based fusion completely ignores gene encoder outputs, using raw HVG matrix instead
  • Action chunking in VLA models fundamentally changes the GPU utilization profile: Pi0.5 outputs 50-step action sequences, so a 400-step evaluation trial requires only ~8 inference calls (~20s GPU work) vs 200s total time, resulting in ~10% GPU utilization. The bottleneck shifts from GPU compute to pipeline serialization (CPU preprocessing, WebSocket I/O, simulation steps). Batched evaluation with timer-based request collection could improve throughput
  • macOS sandbox security-scoped bookmarks require both compile-time entitlement declaration (in .entitlements file) AND runtime lifecycle management (startAccessingSecurityScopedResource/stop). Empty entitlements → silent fallback to non-scoped bookmarks → 100% restore failure. Bookmark system has no visible error channel when entitlements are missing
  • Log archaeology (reading full log files) can reveal patterns invisible in code: Desktop Video’s ‘saved non-scoped bookmark’ repeated 20× across sessions exposed the missing entitlements. Concurrent write bugs (errorLog FileHandle race) silently destroy diagnostic information. High-frequency timer logs (879 updateClockLabels calls) obscure real errors → need state-based logging, not tick-based
  • Slurm’s srun with –jobid flag can bypass SSH PAM restrictions (like slurm_adopt) by executing commands directly in the allocated job context, essential for cluster environments with strict node access control. When GPU nodes are running 100% utilization workloads, administrative commands via srun can experience significant latency (60+ seconds) → robust monitoring scripts should handle timeouts gracefully and use lightweight fallback commands (nvidia-smi over nvitop)
  • External dependency documentation should serve operational purposes (structured inventory: paths, sizes, purposes, references) distinct from conceptual documentation (explanatory guides). Clear file inventories with exact paths enable reproducibility and dependency auditing. For VLA benchmark project, comprehensive inventory includes 14 categories: config directories, logs, binaries, environment variables, Python packages, rclone paths, Git repos, Ollama models, conda envs, datasets, checkpoints, bootstrap procedures, and troubleshooting guides

Practical Learnings#

  • SwiftUI .mask() + .ultraThinMaterial enables ‘frosted glass text’ effect on macOS 12.4+ by masking the material blur to the text shape. The material automatically samples content beneath it in the window’s view hierarchy (video player → material → masked text). Dual-layer approach (material mask + gradient overlay) simulates physical glass with refraction + specular highlights
  • Gaussian mixture model clustering (mclust) scales poorly beyond ~100 dimensions. For foundation model embeddings (512d-1536d), kmeans or spectral methods are necessary for practical runtime. mclust’s EM algorithm becomes computationally intractable on high-dimensional data
