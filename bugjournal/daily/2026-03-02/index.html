Problems and Solutions#

Key Issues#

1. Desktop Video: 100% failure rate of video/image recovery after application restart, resulting in code 257 “Operation Not Allowed” error#

Solution: Four-step fix: 1) Added com.apple.security.files.bookmarks.app-scope to the permission list (originally an empty dictionary). 2) Added a fallback option in restoreFromBookmark: If withSecurityScope fails, try using options[] to recover old non-security-grade bookmarks, then save as security-grade bookmarks again. 3) Track active URLs via activeSecurityScopedURLs dictionary and release them in clear(). 4) EnsureFileAccess() automatically resolves bookmark issues before file I/O. Key Insight: macOS sandbox bookmark system requires compile-time permission declarations and runtime url.startAccessingSecurityScopedResource() lifecycle management. An empty permission list caused bookmark creation failures, leading to a return to non-security-grade bookmarks, ultimately resulting in withSecurityScope recovery failure and code 257 error. Due to concurrent FileHandle race conditions, the 879-line errorLog() never recorded errors, masking the root cause for months.

2. MIHD: STAIG two-stage fusion ignored gene encoder embeddings, making scGPT+UNI2+STAIG equivalent to PCA+UNI2+STAIG#

Solution: Checked models/Fusion.py and pipeline/runner.py: staig_fusion used staig_gene_feat (from cached raw HVG matrix), not the gene_emb parameter. This was an architectural issue, not a bug. This key discovery was recorded in the weekly report, affecting the interpretation of all STAIG results. Key Insight: Gene encoder (PCA/scGPT) output was only used by non-STAIG fusion methods. For graph-based fusion, like STAIG, pre-trained spatial embeddings did not enter the GCN node features. This explains why scGPT’s performance was poor: it was optimized for cross-organizational generalization, not for maximizing internal dataset variability like PCA.

3. VLA Evaluation: Although GPU utilization reached 100% during training, it was lower during Pi0.5 evaluation (about 10%), resulting in slower evaluation#

Solution: Identified the root cause: action segmentation architecture reduced inference calls to 8 per 400-step experiment (8 × 2.5 seconds = 20 seconds GPU work/total 200 seconds = 10% utilization). Although the solution plan was interrupted, optimization strategies were requested through network research and batch evaluation implementation. Key Insight: The bottleneck in VLA evaluation wasn’t GPU computing power, but rather stream serialization: action segmentation reduced GPU calls, while each inference cycle included CPU preprocessing, WebSocket I/O, and simulation steps, which accounted for the total time. Using timer-based request collection could improve throughput.

4. Due to PAM slurm_adopt rejecting connections, SSH access was denied on GPU nodes an49/an53, with the error message “No active tasks on this node”#

Solution: Bypass SSH restrictions using Slurm’s srun command, executing commands directly in the assigned task context: srun --jobid=45628 bash -c 'command' Key Insight: Slurm’s srun with –jobid provided a way to execute commands on allocated nodes, even if PAM blocked direct SSH connections. This was very useful for cluster environments with strict access controls.

General Issues#

5. When running background tasks using srun on heavily loaded GPU nodes (an49), timeout occurred, with command response times exceeding 60 seconds#

Solution: Added –overlap flag and reduced the timeout expectation value. When nvitop was unavailable or too slow, switched to the simpler nvidia-smi. Accepted the fact that commands could be delayed on heavily loaded nodes. Key Insight: When GPU nodes ran workloads with 100% utilization, management commands executed through srun might experience significant delays; designed monitoring scripts should handle timeout issues gracefully and use lightweight fallback commands.

6. MIHD: During visualization, mclust clustering hung indefinitely on scGPT (512d) and UNI2 (1536d) embeddings#

Solution: In visualize_from_cache.py, replaced the default mclust with kmeans using the –cluster_method flag. High-dimensional data made mclust’s EM algorithm computationally difficult to handle.

Key Insight: Gaussian mixture model clustering (mclust) performed poorly when there were more than about 100 dimensions. For base model embeddings (512d-1536d), kmeans or spectral methods needed to be used for practical operation.

7. Desktop Video log files contained over 1550 spam entries per session (updateClockLabels × 879, startScreensaverTimer × 671)#

Solution: Removed dlog from updateClockLabels (called once per screen saver period). Moved startScreensaverTimer’s dlog after all early exit protection mechanisms (only logging actual scheduling timer). Also removed dlog in idle timer callbacks.

Key Insight: High-frequency dlog calls in timer callbacks generated huge log files, masking real errors. Logging should be based on state transitions rather than periodic ticks.

Lessons Learned#

Key Lessons - Basic Model Embeddings (scGPT 512d, UNI2 1536d) Can’t Replace Dimensionality Reduction in Clustering Tasks (PCA 50d). ScGPT Focuses on Cross-Dataset Transfer, Not Internal Differences within Datasets. Architecture Choice (Gene-only or Multimodal Fusion) Is More Important Than Embedding Quality: QFormer Achieves Higher ARI Improvements Than Using scGPT Alone by Learning Gene-Visual Alignment Methods. STAIG’s Graph-Based Fusion Completely Ignores Gene Encoder Output, Instead Directly Using the Original HVG Matrix.#

  • The action chunking behavior in VLA models fundamentally changed GPU utilization: Pi0.5 generated 50 steps of action sequences, so an evaluation experiment with 400 steps only required about 8 inference calls (about 20 seconds of GPU time), giving a total of 200 seconds, meaning GPU utilization was around 10%. The bottleneck shifted from GPU computation to pipeline serialization (CPU preprocessing, WebSocket I/O, simulation steps). Using timer-based request collection could improve throughput.
  • macOS Sandbox Security-Level Bookmarks require compile-time permission declarations (in.entitlements file) and runtime lifecycle management (startAccessingSecurityScopedResource/stop). Empty permission declarations silently revert to non-security-level bookmarks, leading to 100% recovery failures. When permissions are missing, the bookmark system has no visible error channels.
  • Log archeology (reading complete log files) can reveal hidden patterns in code: In Desktop Video, “saved non-security-level bookmarks” appeared 20 times across multiple sessions, revealing missing permission information. Concurrent write errors (errorLog FileHandle concurrency) silently corrupted diagnostic information. High-frequency timer logs (879 updateClockLabels calls) masked the real error—therefore, state-based logging was needed instead of counting-based logging.
  • Slurm’s srun with –jobid flag can bypass SSH PAM restrictions by executing commands directly in the allocated task environment (e.g., slurm_adopt), which is important for cluster environments with strict node access controls. When GPU nodes are running workloads with 100% utilization, management commands executed through srun may encounter significant delays (over 60 seconds)—therefore, robust monitoring scripts should be able to handle timeout issues gracefully and use lightweight alternative commands (e.g., nvidia-smi instead of nvitop).
  • External dependency documentation should serve operational purposes (structured lists: path, size, purpose, reference), which are separate from conceptual documentation (explanatory guides). A clear list with exact paths can achieve reproducibility and dependency auditing. For VLA benchmark projects, a comprehensive list includes 14 categories: configuration directories, logs, binary files, environment variables, Python packages, rclone paths, Git repositories, Ollama models, conda environments, datasets, checkpoints, launch programs, and troubleshooting guides.

Practical Learnings#

  • SwiftUI’s .mask() + .ultraThinMaterial feature can achieve “sandglass text” effects on macOS 12.4 and later versions by using a blur effect for the masked material. This material automatically samples its content below it in the window view hierarchy (video player → material → masked text). A two-layer method (material mask + gradient overlay) simulates the physical glass effect with refraction and mirror highlights.
  • Gaussian Mixture Model clustering performs poorly when more than 100 dimensions are involved. For basic model embeddings (512d-1536d), kmeans or spectral methods are needed for actual operation. mclust’s EM algorithm becomes computationally infeasible in high-dimensional data.
