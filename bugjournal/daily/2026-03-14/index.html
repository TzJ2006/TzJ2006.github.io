Problems and Solutions#

Key Issues#

1. Training failure for pi05: Missing fields like manip_progress_time, manip_progress_distance_left/right, target_endpose, and target_joint in the dataset.#

Solution: Need to modify ~/HDD_POOL/mozihao/VLA/convert_robotwin_democlean_to_lerobot.py to add these fields before converting the dataset again. Key insight: Before designing the training code, it’s important to ensure that the data pipeline supports all required fields to avoid discovering missing data after the code is written.

General Issues#

2. os.get_terminal_size() cannot read COLUMNS/LINES in pipeline/subprocess environments, resulting in tables not being fully expanded during 120-column tests.#

Solution: Modify _get_term_size() to prioritize reading COLUMNS/LINES from the environment variables. If this fails, revert to os.get_terminal_size(). Key insight: Terminal width detection needs to handle both real tty and environment variable scenarios.

AI Limitations#

Major Limitations#

  • Before implementing the pi05 training code, no verification was done to ensure that the data pipeline was complete (fields like manip_progress were written into the lerobot dataset). This led to discovering missing data during training after the code was implemented, wasting engineering time.

General Limitations#

  • gpumon.py’s initial adaptive implementation overlooked an issue where os.get_terminal_size() couldn’t read COLUMNS in child processes. This issue was discovered and fixed through user testing.

Today’s Achievements#

Core Achievements#

  • Using stop_gradient to isolate the gradients of the main task and auxiliary tasks in VLA is a safe starting point—ensuring that action prediction isn’t interfered with first. If the results aren’t good, then compare the gradients.
  • When adding auxiliary task headers in JAX/Flax NNX, use GT values for teacher forcing during training and predict values for injection during inference. Both paths need to be implemented separately in compute_loss and sample_actions while maintaining consistent interfaces.

Practical Achievements#

  • The terminal buffer (\033[?1049h to enter, \033[?1049l to exit) combined with signal.SIGINT can achieve a full-screen UI similar to nvitop. It automatically restores the original terminal content after exiting.
