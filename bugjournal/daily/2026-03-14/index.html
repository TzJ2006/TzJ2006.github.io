Solution: Try reading COLUMNS/LINES environment variables and converting them to integers, using os.get_terminal_size() if failed

Key Insight: Environment variables are more reliable than system calls, suitable for cross-process terminal communication

Human and Artificial Intelligence Approaches#

Strategic Level#

Gradient strategy selection in pi05 experiment design#

Role Method
Human Directly choose option A (all stop_gradient), considered the safest approach as it does not affect action prediction. Adjust if results are poor — reflects conservative startup strategy in iterative experiments
AI AI proposed 3 options (A completely blocking/B completely returning/C hybrid method) and analyzed trade-offs, tending to choose option A but emphasizing specificity of special marking

Difference Analysis: Humans make quick decisions based on engineering experience, while AI tends to list options and trade-offs, but ultimately human’s concise decision-making strategy (first conservative handling, then iteration) is reasonable

Experiment configuration naming and organization#

Role Method
Human Require “configuration switch, easy to start experiments” reflects emphasis on experiment efficiency
AI AI designed a unified ProgressConfig class and 4 independent named configurations (exp1~4), allowing direct selection through tyro CLI

Difference Analysis: Humans focus on practicality and convenience, while AI translates it into configuration classes and CLI interfaces. Both agree on the goal of “easy to switch experiments”

Limitations of Artificial Intelligence#

Key Constraints#

  • AI did not actively detect data pipeline issues in the end — should immediately check if data loading after model modification supports new fields, rather than waiting for user errors to be discovered
  • AI did not run actual training smoke tests when modifying pi05, only tested import of configuration classes, resulting in missing dependencies in data conversion script

Lessons Learned#

Key Lessons#

  • For complex model experiments, integrity checks should cover model code, configuration, data pipeline, and training scripts, not just verification of model code
  • Using stop_gradient to implement auxiliary tasks is a common multi-task learning pattern that can utilize features while avoiding interfering with the main task
  • Special marking mechanism requires 3 steps: tokenizer registration of marker IDs, model embedding layer extension, and concatenation during data loading prompts
