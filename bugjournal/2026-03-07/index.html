日报 — 2026-03-07#

用户识别两个并发缺陷：意图分类忽略上下文（关键词预检位置错误导致’ok’被误判为问候、‘几点了’等歧义词触发错误分类）、对话历史无限增长。借鉴 OpenClaw 四层 context window 压缩策略，设计并实现 A-D 四阶段方案：Phase A 新增 token_utils.py 及 6 项 token 预算配置；Phase B 完全重写 conversation_history.py，引入 session_id / started_at / last_activity_at 追踪、消息超阈值时 AI 自动压缩生成摘要、会话重置归档至 conversation_summaries.jsonl、长消息软截断（>2000字符）、has_active_session() 接口；Phase C 将 general_keywords 拆分为 always_general / context_dependent 两类，重构意图分类为六步决策树（确认快捷路径 → 始终通用 → 上下文相关 → 短问候 → 双重验证 → 安全检查），有活跃会话时跳过 context_dependent 预检；Phase D 重写 _format_conversation_context()，按 token 预算从最新消息反向填充，并在上一条助手消息含提案标记时自动注入 [AWAITING USER CONFIRMATION]，同步更新 prompts.py 以支持 session summaries 注入。新建 tests/test_context_aware_intent.py 含 41 个专项测试，与原有 319 项共 360 个全部通过，彻底解决歧义词误分类与对话历史无限增长两个根本问题。
