问题与解决方案#

关键问题#

1. Desktop Video 历史记录旧视频无法打开，报错 Operation not permitted (Code=257)#

解决方案: 根本原因是 App 缺少 com.apple.security.app-sandbox，security-scoped bookmark 无法创建；修复：entitlements 启用 Sandbox + user-selected.read-write，同时修复 ensureFileAccess 中 non-scoped bookmark 回退解析逻辑，并在文件访问失败时触发 NSOpenPanel 重新授权

关键洞察: macOS TCC 机制下，非沙盒应用通过 NSOpenPanel 获得的文件访问权限在 app 重启后不持久化；只有 security-scoped bookmark 配合 App Sandbox 才能实现跨启动的文件访问权限持久化

一般问题#

2. curobo 预编译 .so 文件与新版 torch ABI 不兼容，JIT 重编译又因找不到 cuda_runtime.h 失败#

解决方案: 删除旧 .so 文件，设置 CPATH=/HOME/…/envs/RefineVLA/targets/x86_64-linux/include，从源码重新 pip install -e . 编译 curobo

关键洞察: conda 安装的 CUDA toolkit 将头文件放在非标准路径 targets/x86_64-linux/include/ 而非 include/，需手动通过 CPATH 告知编译器

3. torchvision::nms 算子注册失败，导致整个 torchvision 无法 import#

解决方案: 将 torchvision 从 0.21.0 升级至 0.22.1+cu126，与 torch 2.7.1+cu126 版本匹配

关键洞察: torchvision 的 C++ 扩展与 torch 紧密耦合，小版本不兼容即会导致算子注册失败，需严格按 torch 版本对照表匹配

4. RoboTwin eval 脚本指定 checkpoint_id=5000 但目录中不存在该步数#

解决方案: 需将 deploy_policy.yml 中 checkpoint_id 改为实际存在的值（15000/25000/29999）

关键洞察: eval 脚本参数通过 YAML 配置文件传入，且路径拼接在 pi_model.py 中硬编码，调用前须确认 checkpoint 步数与磁盘一致

人类思路 vs AI 思路#

战略层面#

Desktop Video 历史记录 bug 根因定位#

角色 思路
人类 用户直接描述现象：「历史记录有时因太久了导致无法打开视频」，没有提供日志或技术细节
AI AI 通过阅读日志和代码，完整还原了 Sandbox 缺失 → security-scoped bookmark 创建失败 → non-scoped bookmark 保存但解析逻辑有缺陷 → TCC 拒绝 的四步错误链

差异分析: 用户只知道现象，AI 完整还原了技术根因；但修复方案中「两者结合（Sandbox + NSOpenPanel）」是用户在选项中主动选择的，而非 AI 主动提出的最优解

实现层面#

gpumon.py 进程去重逻辑#

角色 思路
人类 用户指出「一个进程可能跑在多个 GPU 上，应该说每个进程在一个 GPU 上只显示一次才对」，即去重粒度应为 (pid, gpu) 而非 pid
AI AI 将去重描述为「每个进程只出现一次」，逻辑错误：一个多卡训练进程会被合并成一行，但实际上在不同 GPU 上的状态（显存、利用率）不同

差异分析: 人类对多卡进程的业务语义理解更准确；AI 过度简化了去重逻辑，需要按 (pid, gpu_id) 为粒度而非 pid

AI 局限性#

一般局限#

  • 在 gpumon.py 进程去重逻辑上描述不准确：声称「每个进程只出现一次」，但忽略了同一进程可能跑在多个 GPU 上、每个 GPU 对应不同状态的情况，需用户纠正
  • CUDA 头文件路径排查需要多次尝试：先后尝试了系统 /usr/local/cuda、pip nvidia 包、conda env include 目录，才最终定位到正确的 targets/x86_64-linux/include 路径，过程低效
  • eval.sh 最后遇到 checkpoint_id=5000 不存在的问题时，AI 开始读 eval_policy.py 追踪参数映射，但实际正确入口应是直接检查 deploy_policy.yml 配置文件，方向不够精准

今日收获#

核心收获#

  • macOS TCC + App Sandbox 的权限模型：非沙盒应用的 security-scoped bookmark 选项无效，NSOpenPanel 授权不持久化；只有启用 Sandbox 后才能通过 bookmarkData(options: .withSecurityScope) 创建可持久化的访问凭据

实践收获#

  • conda 环境中的 CUDA toolkit 头文件路径为非标准布局（targets/x86_64-linux/include/），编译需要 CUDA 扩展的 Python 包时需手动设置 CPATH 或 CUDA_HOME
  • torch 与 torchvision 版本必须严格对应，torch 2.7.x 需要 torchvision 0.22.x；torchvision ABI 与 torch 小版本不兼容会导致自定义算子注册在 import 阶段就失败
