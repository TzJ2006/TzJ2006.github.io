问题与解决方案#

关键问题#

1. 奖项数据（spotlight/highlight/best paper）无法从 ArXiv 或 S2 API 获取，传统 API 方案失效#

解决方案: 将论文标题+年份+venue 发给 LLM，利用其训练数据中的奖项知识进行识别，返回结构化 JSON

关键洞察: LLM 的世界知识可弥补 API 数据缺口，将知识库检索转化为 LLM 推理是一种有效的数据获取新范式，适用于训练数据覆盖范围内的事实查询

2. RoboTwin：新变量（target_endpose、target_joint 等）需要「未来知识」（move 结束时的末端状态），而数据是逐帧收集的#

解决方案: 采用后处理方案：先正常收集帧，move() 结束后再 patch pkl 文件填入所有新字段；pkl2hdf5.py 使用泛型递归字典转换，无需修改该工具

关键洞察: 当变量计算依赖未来状态时，后处理 patch 方案优于实时收集，这种设计解耦了收集与计算，并兼容通用字典转换工具

3. ArXiv 仅返回最近100篇论文，以其为主数据源导致历史高引论文大量丢失，无法体现完整学术脉络#

解决方案: 以 Semantic Scholar 为主数据源（可获取500篇带完整历史的论文），ArXiv 仅用于丰富 arxiv_id/pdf_url 等元数据，两者按标题匹配合并

关键洞察: 学术脉络分析要求历史完整性，主数据源选择应优先覆盖范围而非近期性；S2 的历史深度与 ArXiv 的最新性是互补关系

4. JSON 解析失败时原始实现仅用 Haiku 修复一次后丢失响应，且存在递归调用风险（修复后输出仍无法解析会触发无限递归）#

解决方案: 将解析和修复解耦：_try_parse_json 只做纯解析不触发 LLM，逐级循环 haiku→sonnet→opus，全部失败则原子写入日志文件

关键洞察: 容错管线中「解析」与「修复」必须分离以防递归；逐级升级平衡成本与成功率；失败响应落盘支持离线调试

5. RoboTwin：删除 get_critical_region_label() 子类 override 后，HDF5 文件中 critical_region 字段仍然存在#

解决方案: 在 move() 的 pickle 后处理阶段显式调用 pkl_data.pop(‘critical_region’, None) 删除该 key

关键洞察: 基类 _base_task.py 的 get_obs()（line 510）无条件写入 critical_region，仅删除子类 override 不够，必须在后处理阶段主动删除；修改子类前必须检查基类隐式行为

一般问题#

6. RoboTwin：manip_progress_distance 出现负值#

解决方案: 用 np.clip(…, 0.0, 1.0) 将左右手距离进度强制限制在 [0,1] 范围

关键洞察: 机器人末端执行器路径为曲线而非直线，中间帧到终点的距离可能大于起点到终点的距离，导致负进度值；基于距离的进度指标在曲线路径下必须 clamp

AI 局限性#

重要局限#

  • 在需要创新方案时（奖项识别数据源、ArXiv HTML 全文提取），AI 只能穷举已知选项等待用户决策，无法主动想到将 LLM 训练知识作为检索源或利用平台最新特性的创新路径，需要用户直接提出才能实现。
  • 在复杂代码库中初次实现时，易遗漏基类的隐式行为（如 get_obs() 无条件写入 critical_region）以及容错管线中的递归风险（parse_json_response 的 LLM 修复递归）；这两类问题都需要人类通过实验发现后才修复，反映了 AI 在「理解继承体系的隐式副作用」上的系统性盲点。

一般局限#

  • LLM 在长篇复杂中文内容的 JSON 格式化上不稳定（Pieter Abbeel 分析经历4次修复），且在设计阶段未能预见边界条件（曲线路径导致距离进度越界），说明 AI 对运行时异常情况的预判能力仍不足。

今日收获#

核心收获#

  • LLM 的训练数据知识可作为结构化 API 数据的补充来源：奖项信息、关系推断等无法从 API 获取的历史知识，可通过向 LLM 提问获取。这是「将知识库查询转化为 LLM 推理」的新范式，适用于训练数据覆盖范围内的事实查询。
  • 需要「未来知识」的变量设计应采用后处理 patch 方案：先正常收集帧，操作结束后统一 patch pkl 文件。这种设计解耦了收集与计算，且兼容泛型字典转换工具（无需修改 pkl2hdf5.py）。
  • 容错管线的分层设计原则：「解析」和「修复」必须分离（用不同函数）以防递归；逐级升级（小模型→大模型）平衡成本与成功率；失败响应落盘支持离线调试。这三点构成 LLM 输出解析的鲁棒性设计模式。
  • 学术分析工具的数据源策略：历史完整性（S2 500篇带引用数据）比近期性（ArXiv 100篇）更重要；两者配合最佳——S2 提供历史深度和引用数据，ArXiv 提供最新 preprint 和 HTML/PDF 链接。单一数据源无法满足脉络分析需求。
  • 修改子类前必须检查基类是否有无条件写入该字段的代码，否则仅删除 override 不足以去除字段；基于距离的进度指标在曲线路径下会越界，需 clamp 到 [0,1]。两者都体现了「充分理解基类行为和运行时边界」的重要性。
  • 顶级学者（Levine/Abbeel）成功的核心在于「数学严谨性+工程实用性+范式转换时机」的三重结合；Sergey Levine 的轨迹揭示 VLA 范式在 2023 年前后的关键转折：以 GPT-4/CLIP 成熟为契机，机器人控制被重新表述为「接地的语言-视觉-动作生成」问题，这是当前 Robotics 领域最重要的技术趋势。
