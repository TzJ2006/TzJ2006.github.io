问题与解决方案#

关键问题#

1. SameLabel@K 指标全部为 0，评估结果异常#

解决方案: 修复为：同标签检索率在全体有效 spot（含 niche 成员）上计算，而非将 niche 成员排除后计算

关键洞察: Precision@K（与 RM-Ideal 秩序对比）和 SameLabel@K（cell type 检索率）是两个不同问题，前者需排除 niche 自身以避免自匹配偏差，后者不应排除

2. query 和 reference 子图节点 ID 可能冲突（共享 label dict 时 key 碰撞）#

解决方案: 为 reference 子图节点添加偏移量 offset = adata_q.n_obs，保证两组节点 ID 不重叠

关键洞察: WWL 算法中 query 和 reference 节点需共享哈希表，因此节点 ID 必须全局唯一

一般问题#

3. POT（最优传输库）在当前 conda 环境中未安装#

解决方案: 实现 POT→scipy.optimize.linprog 的回退策略，无需新增依赖

关键洞察: 在 HPC 环境中 scipy 已普遍安装，以 linprog 为回退保证了在任何环境下的可用性

4. AI 在 MIHD 目录下找不到 QueST 的 src/utils.py，初次搜索方向错误#

解决方案: 通过 Glob 全局搜索定位到正确路径 /hpc/group/yizhanglab/zt81/QueST/

关键洞察: 两个项目（MIHD 和 QueST）路径不同，用户请求的实现目标在 QueST 目录中，而非当前工作目录 MIHD

今日收获#

核心收获#

  • RM-Ideal（WWL 图核 + Wasserstein 距离）是衡量空间 niche 结构相似度的 ground truth 指标，与 embedding 余弦相似度相关但不等价（Spearman r~0.42），说明 embedding 仅中等程度地捕获了 niche 精细结构
  • 同标签检索率（SameLabel@K）与精度@K（Precision@K）是两个独立评估维度，前者衡量 cell type 类型级检索，后者衡量与 RM-Ideal 排序的秩相关，不能用同一种 mask 逻辑处理
  • WWL 算法要求 query 和 reference 子图共享哈希表并使用无碰撞的全局节点 ID，在实现时需引入节点偏移量；Hamming 距离需按维度归一化（除以 K+1）才能保证 W ∈ [0,1]

实践收获#

  • 在 HPC 多环境下，优先用已有依赖（scipy linprog）作为回退，而非强制安装新包（POT），可提升代码在不同 conda 环境中的可移植性
