Key Achievements This Month#

  1. Identified root cause of MIHD cross-slice retrieval failure and found a solution (2026-03-23, MIHD) — Confirmed root cause is that PCA/STAIG processes each slice independently, producing incomparable embedding spaces. scGPT, due to its shared coordinate system, is the optimal encoder for cross-slice retrieval (average SL@50=0.416, 100% hit rate), revealing the architectural principle that single-slice clustering and cross-slice retrieval require different encoders.
  2. Discovered and fixed 4 critical bugs in extract_teddy() (2026-03-29, MIHD) — Systematic source code review revealed: incorrect Tokenizer class name, wrong model forward parameter name (input_ids→gene_ids), wrong output key (last_hidden_state→outputs[‘cell_emb’]), and missing gene symbol to Ensembl ID mapping. After fixes, the first slice (151508, 4384 spots) was successfully validated.
  3. Verified STAIG framework’s high robustness to gene encoders (2026-03-30, MIHD) — Teddy 1024d (ARI=0.495) and PCA 50d (ARI=0.497) performed nearly identically under the STAIG framework, with a gap of only 0.002, demonstrating that STAIG’s spatial graph contrastive learning mechanism dominates clustering performance and the impact of gene embedding quality is greatly diluted.
  4. Paper Fig 1–6 framework planning completed (2026-03-31, MIHD) — Referencing four competing papers — MISO (Nat Methods), STAIG (Nat Comms), QueST, and Yuan (Nat Methods) — established a complete paper structure: Fig1=framework diagram, Fig2=DLPFC benchmark, Fig3=ablation, Fig4=HD application, Fig5=cross-slice Query, Fig6=Query biological discovery. Written into ECL YAML and plans.md, marking the project’s official transition to the writing phase.
  5. TEDDY-G 400M full 11-slice embedding extraction completed (2026-03-30, MIHD) — Completed 1024d gene embedding extraction for all 11 DLPFC slices (151508–151676) in a dedicated conda teddy environment, totaling 185MB, providing a complete data foundation for multi-encoder comparison benchmarks.
  6. STHD Ground Truth integration (100% match rate) (2026-03-31, MIHD) — Added _load_sthd_annotations() function, achieving 17502/17502 spots 100% match via r_big/c_big coordinates; CE+Tumor biological merging (Neoplasm + Non-neoplastic Epithelium → Epithelium) yielded 3 meaningful clusters, providing quantitative Ground Truth for HD evaluation.
  7. HD full fusion strategy evaluation revealed naive fusion information dilution (2026-03-31, MIHD) — Completed 10/13 HD fusion strategy experiments; found that simple fusion strategies (concat/mean/element_wise_sum) produced ARI (0.19–0.20) lower than unimodal PCA (0.351), while attention fusion approached PCA (ARI=0.342). UNI2 vision-only achieved the highest NMI (0.387) but lower ARI than PCA, revealing that NMI and ARI can be inconsistent across different datasets.

Systematic analysis confirmed the root cause: Foundation Models are optimized for disease discriminability (not variance maximization), high-dimensional spaces (1024d) degrade KMeans performance, and training data (dissociated scRNA-seq) differs in distribution from spatial transcriptomics.

Recurring Issues#

1. Systematic failure on a specific slice (151676) (occurred 3 times)#

Dates: 2026-03-23, 2026-03-30, 2026-03-31 Root Cause: Slice 151676 has data-specific issues: zero-variance embeddings cause mclust to fail and fall back to KMeans (dragging down the global mean), and model collapse under the STAIG framework cannot be recovered. The root cause remains unidentified, possibly related to anomalous gene expression distribution or spatial structure in this slice. Status: 🔄 Ongoing

2. Pipeline hardcoded configuration inconsistent with slice-specific parameters (occurred 2 times)#

Dates: 2026-03-30, 2026-03-31 Root Cause: Pipeline global configuration (e.g., top_genes=3000, dataset defaults, image path structure) has implicit divergences from slice-specific config (151676 uses 5000 genes) or dataset-specific structure (HD vs DLPFC image paths), causing cache check false positives (sys.exit) and visual extraction failures. Status: 🔧 Workaround available

3. HPC environment configuration obstacles (conda activation, missing config files) (occurred 2 times)#

Dates: 2026-03-23, 2026-03-31 Root Cause: Config files do not automatically migrate to new machines (sync.json must be manually reconstructed), and conda initialization on SLURM compute nodes differs from login nodes (requires eval $(conda shell.bash hook) rather than source activate). Both issues stem from a lack of experience with environment migration. Status: ✅ Resolved

4. Cross-session context loss requiring repeated background provision (occurred 2 times)#

Dates: 2026-03-23, 2026-03-29 Root Cause: Claude’s session isolation mechanism means key research progress (e.g., Teddy test results) is not persisted to the memory system, requiring users to re-provide arXiv links or background information in subsequent sessions. Status: 🔄 Ongoing

Selected Insights This Month#

Architecture#

  • STAIG framework is highly robust to gene encoders: Teddy 1024d vs PCA 50d under STAIG shows only 0.002 ARI difference, demonstrating that spatial graph contrastive learning dominates clustering performance and the impact of gene embedding quality is greatly diluted. Therefore, the bottleneck for optimizing MIHD clustering performance does not lie in the choice of gene encoder. (Source: 2026-03-30)
  • The optimal encoder for single-slice clustering (STAIG/PCA) differs from the optimal encoder for cross-slice retrieval (scGPT): the comparability of embedding spaces (shared coordinate system) is more critical than local quality. Foundation Models naturally suit cross-slice retrieval due to their fixed representation space output, while PCA processes slices independently, producing incomparable coordinate systems. (Source: 2026-03-23)
  • Foundation Models exhibit systematic transfer loss on cross-domain downstream tasks: TEDDY achieves SOTA on disease classification (ACC=0.72) but underperforms PCA on spatial transcriptomics clustering (ARI 0.186 vs 0.255). Root cause is optimization objective mismatch (discriminative representation vs variance maximization), and high-dimensional space (1024d) is unfavorable for KMeans. (Source: 2026-03-30)

Domain Knowledge#

  • HVG1500 (direct concatenation of highly variable genes without dimensionality reduction) achieves ARI=0.330 on single slices, outperforming all Foundation Models; Leiden clustering improves ARI by ~15% over KMeans on DLPFC, but improvement over STAIG is limited — STAIG’s advantage comes from spatial graph structure and is relatively independent of the clustering algorithm. (Source: 2026-03-30, 2026-03-31)
  • NMI and ARI show inconsistency on HD data: UNI2 vision-only achieves the highest NMI (0.387) but lower ARI than PCA (0.344 vs 0.351); naive fusion (concat/mean) on HD yields ARI (0.19–0.20) lower than unimodal PCA (0.351), indicating information dilution from simple fusion. Both metrics should be reported together. (Source: 2026-03-31)

Tools#

  • Single-cell foundation model forward interfaces in the HuggingFace ecosystem vary significantly and standard interfaces cannot be assumed: TEDDY uses gene_ids (not input_ids), GeneTokenizer (not Tokenizer), outputs[‘cell_emb’] (not last_hidden_state), and its vocab uses Ensembl IDs (43808 tokens) rather than gene symbols. Source code must be read to confirm. (Source: 2026-03-29)
  • DCC HPC cluster practical notes: SLURM compute nodes require eval $(conda shell.bash hook) && conda activate to initialize conda (source activate does not work); scavenger-h200 provides H200×8 preemptible nodes; account TRES limits are cpu=100/mem=500G; Glob tends to time out on large HPC filesystems — use exact path search instead. (Source: 2026-03-23, 2026-03-31)

Debugging#

  • Implicit divergence between slice-specific configuration (e.g., staig num_gene) and pipeline global defaults is a high-risk bug type: cache check logic should dynamically resolve actual configuration rather than hardcoding defaults; multi-dataset pipeline image paths should be explicitly differentiated rather than using a unified data_root/section_id logic. (Source: 2026-03-30, 2026-03-31)
