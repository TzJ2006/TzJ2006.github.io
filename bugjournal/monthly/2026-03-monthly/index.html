Recurring Problems#

1. Training/Model configuration issues causing silent failures, severe performance degradation, or dependency fragility (10 occurrences)#

Dates: 2026-03-01, 2026-03-05, 2026-03-06, 2026-03-07, 2026-03-09, 2026-03-10, 2026-03-11, 2026-03-12 Root Cause: Template-based config generation without per-task validation, missing observation keys (object state), normalization bugs (apply_tree strict=False silently skipping), checkpoint misconfigurations. PyTorch C++ extensions and CUDA libraries have strict ABI compatibility requirements—version mismatches manifest as operator registration failures, cuBLAS bugs on specific hardware, ABI breaks requiring recompilation, performance regressions. Zero-padding for architectural compatibility severely dilutes gradient signals (56% in RoboTwin case) if included in loss computation. JAX/XLA minor version upgrades (0.5.0→0.5.3) can deliver 20-30% training speedups through compiler optimizations. Status: ongoing - systematic fixes applied but validation gaps remain

2. Documentation-code mismatch causing confusion and blocking progress (11 occurrences)#

Dates: 2026-03-02, 2026-03-04, 2026-03-05, 2026-03-07, 2026-03-08, 2026-03-09, 2026-03-10, 2026-03-15, 2026-03-17, 2026-03-19, 2026-03-20 Root Cause: Documentation drift inevitable in active projects—outdated statistics, missing implementation details, incorrect pipeline diagrams, project.json ID mismatches. 3. LLM output truncation, JSON parsing failures, and format instability in batch generation (16 occurrences)#

Dates: 2026-03-01, 2026-03-03, 2026-03-07, 2026-03-16, 2026-03-17, 2026-03-19 Root Cause: Complex multi-day reports exceed 32K output token limits. Chinese-language nested JSON at edge of Claude’s format stability. Special character escaping unstable (Chinese quotation marks, technical term citations). Long-text fields exhibit truncation (especially final fields), unclosed strings, missing closing brackets in nested structures >3 levels deep. LLM generates incomplete/malformed JSON (quote escaping errors, missing closing brackets, mixed non-JSON text) especially in ultra-long structured output. Failure modes vary across sessions despite identical prompts. Status: workaround - layered fault tolerance (direct parsing→code block→brace matching→LLM repair with model escalation), manual JSON repair for complex cases, need automatic validation and retry

4. Spatial transcriptomics data alignment failures due to implicit index assumptions (6 occurrences)#

Dates: 2026-03-09, 2026-03-10, 2026-03-11 Root Cause: Different loading functions (sc.read_10x_h5 vs sc.read_visium) and scanpy operations reorder barcodes unpredictably. Position-based array indexing silently maps to wrong spots when cache creation order differs from runtime reload order. Coordinate systems also vary between data sources. Spatial transcriptomics workflows involve multiple file I/O boundaries where index preservation is not guaranteed, but implicit position-based lookups are common anti-pattern. Status: resolved - explicit barcode-based alignment throughout pipeline

5. GPU resource management issues: idle processes, memory exhaustion, utilization bottlenecks (6 occurrences)#

Dates: 2026-03-01, 2026-03-02, 2026-03-03, 2026-03-05, 2026-03-07 Root Cause: Idle model servers monopolizing GPU memory (Phoenix server 407GB across 7 GPUs at 0% utilization), action chunking reducing inference calls causing 10% GPU utilization, DataLoader num_workers=0 causing GPU waiting for CPU I/O, training duration estimation errors (13.5 days vs expected hours). High memory ≠ high utilization—needs multi-dimensional validation. Status: ongoing - case-by-case fixes but systemic monitoring lacking

6. Academic database name disambiguation failures causing merged profiles with incoherent paper lists (8 occurrences)#

Dates: 2026-03-15, 2026-03-16, 2026-03-19 Root Cause: Common Chinese names like ‘Ming Yu’, ‘Xiaolong Wang’ match multiple researchers across disparate fields. Semantic Scholar merges based on name alone without institutional/domain verification. Single data source insufficient—lacks co-author clustering, topic consistency checks, temporal continuity validation. Affects ~30% of analyzed researchers. 7. Cross-platform compatibility gaps in Tauri/Windows development (8 occurrences)#

Platform abstractions (tauri_plugin_positioner) sacrifice accuracy for portability. Win32+WebView2 architecture has inherent 1-frame delay between native window operations and web re-layout. Monthly Learnings Digest#

Architecture (architecture)#

  • VLA temporal context dependency requires offline trajectory-based error injection, not online physics perturbation. Solution: replay N-1 frames to build context window, inject error at frame N, let VLA infer freely. This shifts from ‘initial state perturbation’ to ’trajectory interruption’ matching real-world failure modes. Key insight: VLA models need behavioral history to make contextually appropriate recovery decisions. (Source: 2026-03-10)
  • Foundation Models’ key advantage for cross-sample tasks: provide shared embedding space without joint training. Zero-shot cross-section retrieval fundamentally requires this—no per-sample optimization can substitute. Batch correction (Harmony) and semantic alignment are orthogonal properties: linear correction cannot handle non-linear dimension meaning misalignment from independent training. Single-slice clustering and cross-slice tasks require different encoders. (Source: 2026-03-16, 2026-03-19, 2026-03-23)
  • ECL (Evolving Constraint Language) methodology with adversarial review catches critical issues pre-implementation: 12+ findings across 4 projects including CRITICAL-level bugs. Phase 7 Red-Blue confrontation exposes design flaws at far lower cost than post-implementation refactoring. Persistent ECL documents + Feature Guard Protocol prevent cross-session regression via disk-persistent guards that survive context compaction. (Source: 2026-03-24, 2026-03-25, 2026-03-26, 2026-03-28)
  • Ultra-large-scale text analysis paradigm: ①automatic segmentation (150K chars/chunk ~15-20 papers), ②semantic merge (LLM-driven not concatenation), ③dynamic timeout (300s+120s/50K chars), ④caching mechanism. All four essential for deep-wide mode usability. Breaks single-LLM capacity limit enabling arbitrary-scale full-text analysis. Pattern applies broadly to chunked LLM processing workflows. (Source: 2026-03-19)
  • Single-intent classification is architectural constraint unfixable with keyword hacks. CalendarPro’s 7-layer keyword override system treated symptoms of broken abstraction. Multi-intent LLM analysis eliminates need for all heuristics. Keyword hacks (always-general, context-dependent) were symptoms, not solutions. EventBus + BackgroundCoordinator + ContextAssembler pattern enables modular AI assistants with pub-sub decoupling. (Source: 2026-03-17, 2026-03-19)
  • Git vs rclone responsibility boundary: git manages reproducible public code/config (requirements.txt, scripts, CLAUDE.md), rclone manages personal data/runtime outputs (logs, reports, cache, results, content). Mixing causes repo bloat (94MB), privacy risks, collaboration obstacles. Clear separation establishes data privacy boundary and enables independent versioning strategies. (Source: 2026-03-20)
  • Hub+spoke design pattern for multi-variant systems: shared general framework (hub) + on-demand specific configurations (spokes) prevents both duplication and prompt dilution. Applied successfully to optimize skill (4 languages) and cross-platform code (conditional compilation). Single-language focus directive is key mechanism preventing dilution when processing one variant. (Source: 2026-03-25, 2026-03-26)
  • Cross-attention on single tokens degenerates to linear projection. Spatial transcriptomics provides natural solution: k-NN neighbors form K/V sequences encoding neighborhood context. This pattern (spatial neighbors as attention context) is broadly applicable to spatial omics architectures. Avoids single-token degradation while preserving biological spatial structure. (Source: 2026-03-09)
  • MCP tool design principle: tools for Claude Code should return content by default (markdown reports, JSON data) with optional file persistence, enabling in-context use without filesystem round-trips. When wrapping CLI tools for protocol-based systems, stdout/stderr isolation is first-class architectural concern requiring context managers for clean, reversible redirection. (Source: 2026-03-09)
  • Multi-source data architectures (demo injection, policy rollout injection, natural failure capture) should be explicit in documentation as first-class citizens. Two-phase data collection (seed discovery + deterministic replay) separates stochasticity from data quality. Single source of truth pattern: Centralize auto-generated content in staging directory with link-or-copy sync to final locations—prevents dual-write inconsistencies. (Source: 2026-03-07, 2026-03-22)

Debugging (debugging)#

  • Evaluation metric distortion often cascades from multiple bugs: BC-RNN’s demo_len=0 + trigger_step=0 + transport always False + successful RP≠1.0 jointly caused RP drop from 0.5 to 0.003. Training-vs-evaluation mismatch systematic checklist: observation keys, environment initialization, action space bounds, data normalization, camera setup. Silent failures in data preprocessing (apply_tree strict=False) allow scale mismatches to propagate through entire training. (Source: 2026-03-01, 2026-03-05, 2026-03-06, 2026-03-07)
  • Name disambiguation #1 data quality issue in automated researcher profiling—affects ~30% analyzed. Requires multi-signal approach: co-author clustering, topic clustering, temporal continuity, institutional affiliation consistency. Single data source (Semantic Scholar) insufficient. Faculty homepages more authoritative than co-authorship inference. LLMs can serve as knowledge bases for unstructured metadata (conference awards, paper prestige) when APIs don’t provide it. (Source: 2026-03-15, 2026-03-16, 2026-03-19)
  • MimicGen augmentation fundamentally about coordinate transformation, not noise injection. pos_delta (source→target offset) must be applied to action trajectories. Without this, 10→1000 augmentation collapses due to <5% success rate. Coherent action sequences more effective than per-frame noise for robotic manipulation. Coordinate transformation bugs can be masked by state replay (set_sim_state_flat), surfacing only during fresh initialization. (Source: 2026-03-16, 2026-03-17, 2026-03-19)
  • PyTorch 2.9 + CUDA 12.9 on RTX 2080 Ti has cuBLAS bug causing illegal memory access on large matrix operations. Workaround: mini-batch processing.
