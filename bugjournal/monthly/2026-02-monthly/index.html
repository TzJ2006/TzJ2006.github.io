  1. scGPT checkpoint loading bug fix improving spatial clustering by 44% (2026-02-28, MIHD Spatial Transcriptomics) — Discovered missing self.use_fast_transformer attribute causing 17.7% of attention layer parameters (9.4M weights) to use random initialization instead of pretrained weights. Fix improved average ARI from 0.118 to 0.170 (+44%), NMI from 0.208 to 0.277 (+33%) across all 11 DLPFC sections, enabling valid research results.
  2. Pi0.5 evaluation revealing fundamental VLA training failure (2026-02-28, Error Recovery Benchmark) — Comprehensive 9-task evaluation showed Pi0.5 Phoenix model achieves only 0-24% success rate on its own training tasks despite 100k-step fine-tuning on 4500 episodes. Combined with M14 evaluation showing <1% success on error scenarios, this indicates fundamental architecture limitations or training data quality issues requiring investigation.
  3. Diagnosed and fixed MIHD hyperparameter misalignment causing 9x performance drop (2026-02-05, MIHD Spatial Transcriptomics) — Discovered MIHD used learning_rate=0.001 (vs 0.0005), num_layers=2 (vs 1), num_epochs=550 (vs 300) compared to original STAIG, causing ARI drop from 0.452 to 0.05. Two-phase diagnosis: algorithmic alignment (4 fixes) + configuration alignment restored performance.
  4. Implemented end-to-end STAIG fusion with flexible preprocessing architecture (2026-02-05, MIHD Spatial Transcriptomics) — Fixed NameError and hardcoded preprocessing bugs, added gene_preprocessing parameter (auto/standard/clip/none) supporting both raw expression and pre-encoded embeddings, enabled multi-encoder compatibility (PCA/MLP/GCN/none). Eliminated major architectural inflexibility.
  5. Fixed MIHD spatial coordinate bug enabling correct vision embedding extraction (2026-02-18, MIHD Spatial Transcriptomics) — Corrected tissue_positions_list.csv column mapping where pxl_row (Y) and pxl_col (X) were swapped, causing vision embeddings to be extracted from transposed image locations. UNI2 clustering performance improved 86% (ARI 0.065→0.121), validating multimodal fusion approach reaching ARI 0.275.
  6. Discovered fundamental vision encoder limitation via clustering collapse analysis (2026-02-16, MIHD Spatial Transcriptomics) — Identified that STAIG fusion’s 64-dim embedding bottleneck combined with radius-15 spatial majority voting causes 91% cluster collapse. Root cause: vision encoders cluster by morphological similarity (blood vessels across layers) not biological domains—a universal fine-grained recognition problem requiring domain-specific adaptation.
  7. VLA natural error capture system generating 150 realistic failure scenarios (2026-02-16, Error Recovery Benchmark) — Completed VLA integration infrastructure (v4.3-v4.5) enabling natural policy failure capture. 50 rollouts with Pi0 model generated 150 error scenarios (~3 per rollout vs 0.3 for synthetic injection), expanding total dataset from 121→271 scenarios across 8 error types. Dominant modes: joint_limit_approach (47%), grasp_wrong_pose (30%).
  8. MIHD pipeline optimization achieving 100-500x speedup (2026-02-23, MIHD Spatial Transcriptomics) — Systematic vectorization with scipy/NumPy primitives (cdist, softmax) replaced O(n²) Python loops, GPU-native dropout eliminated CPU transfers, and batch processing for QFormer reduced training time from 3.5 hours with 5% GPU utilization to efficient parallel execution.   9. Aligned STAIG implementation with reference code through 4 critical fixes (2026-02-05, MIHD Spatial Transcriptomics) — Added adaptive edge dropout (drop high-weight edges), gene feature clipping, corrected edge weight direction (softmax_neglog), and tau decay mechanism by comparing line-by-line with reference implementation. Recurring Problems#

1. Environment and configuration reproducibility failures - clustering methods (mclust vs kmeans), conda environments (General vs base/AI), library versions, and hyperparameters cause dramatic performance differences despite correct code logic (8 occurrences)#

Dates: 2026-02-01, 2026-02-05, 2026-02-20, 2026-02-23, 2026-02-26, 2026-02-28 Root Cause: Holistic reproducibility requires alignment across multiple layers: code + hyperparameters + runtime environment + downstream tools + library versions. Papers and documentation omit critical environmental dependencies. Version control tracks code but not execution context. Minor version differences cause API incompatibilities invalidating results. Status: 🔄 Ongoing

2. PyTorch checkpoint loading failures with silent partial initialization - missing instance attributes or incorrect configuration propagation causes 10-20% of parameters to use random initialization with no error messages when using strict=False (4 occurrences)#

Dates: 2026-02-20, 2026-02-28 Root Cause: Using strict=False allows mismatched keys to be skipped silently. Missing instance attributes preventing key remapping (scGPT’s use_fast_transformer) or alignment config issues causes substantial parameters to remain uninitialized, manifesting as poor performance rather than obvious failures. Status: ✅ Resolved

3. GPU resource management issues - memory exhaustion from parallel experiments, lack of resource awareness (checking nvidia-smi, reusing processes), HPC access control confusion (Slurm vs SSH) (9 occurrences)#

Dates: 2026-02-09, 2026-02-16, 2026-02-21, 2026-02-22, 2026-02-23, 2026-02-26, 2026-02-27, 2026-02-28 Root Cause: Q-Former and STAIG fusion training require ~22GB GPU memory each. Running 2+ experiments simultaneously causes CUDA OOM. Lack of understanding of shared cluster etiquette (50% memory threshold), pam_slurm_adopt constraints, and proxy environment variables interfering with localhost connections. Status: 🔧 Workaround

4. Task/model distribution mismatches causing catastrophic failure - models trained on one distribution evaluated on different tasks or error-injected scenarios without considering distribution shift (5 occurrences)#

Dates: 2026-02-20, 2026-02-26, 2026-02-27, 2026-02-28 Root Cause: Models trained on successful demonstrations fail completely on error recovery scenarios (<1% success). Distribution shift between training and evaluation (different tasks, object counts, error injection) invalidates results. Root issue is insufficient validation of training-evaluation alignment and assumption that task success transfers to error recovery. Status: 🔄 Ongoing

5. Documentation drift from codebase causing factual errors - manual documentation not updated when code evolves, cross-session context loss requiring manual plan preservation (7 occurrences)#

Dates: 2026-02-02, 2026-02-03, 2026-02-15, 2026-02-16, 2026-02-19 Root Cause: Documentation (CLAUDE.md, docstrings) not updated with code changes. Manual documentation workflows create sync issues. AI cannot bridge session boundaries without explicit artifacts. Docstrings contained incorrect counts; ground truth is registered code, not docs. Status: ✅ Resolved

6. Implementation divergence from reference code - missing algorithmic details (adaptive dropout, feature clipping, edge weight direction, tau decay) cause significant performance drops even when high-level logic matches paper descriptions (2 occurrences)#

Dates: 2026-02-05 Root Cause: Research papers omit implementation nuances critical for reproduction. Line-by-line code comparison necessary but often skipped. Subtle differences accumulate to cause large performance gaps (0.05 vs 0.45 ARI). Status: ✅ Resolved

7. Missing optimization infrastructure causing severe performance bottlenecks - lack of caching, Python loops instead of vectorization, CPU operations instead of GPU batching (3 occurrences)#

Dates: 2026-02-20, 2026-02-23 Root Cause: Incremental development without performance-first thinking. Embedding re-computation, O(n²) nested loops, sequential QFormer forward passes, and GPU↔CPU data transfers in training loops cause 100-1000x slowdowns. Status: ✅ Resolved

8. Coordinate system ambiguity and spatial data errors - generic naming and lack of validation causing transposed patch extraction (2 occurrences)#

Dates: 2026-02-09, 2026-02-18 Root Cause: Generic pixel_x/pixel_y naming was ambiguous. Columns pxl_row_in_fullres (Y) and pxl_col_in_fullres (X) incorrectly mapped, causing transposed image patch extraction. No validation caught the swap until performance analysis. Status: ✅ Resolved

9. MIHD data integrity issues - missing coordinates, corrupted images, NaN from model collapse (4 occurrences)#

Dates: 2026-02-09, 2026-02-10 Root Cause: Sections 151510, 151672 missing spatial coordinates. Section 151674 has corrupted H&E image. Section 151676 causes zero-variance embeddings indicating fundamental data quality issue. Status: 🔄 Ongoing

Monthly Learnings Digest#

Debugging (debugging)#

  • Environment configuration (conda env, clustering libraries, library versions, system dependencies) can dominate code-level fixes in ML reproducibility. Always verify execution environment matches historical success before debugging code. Git history comparison with last known good version often faster than forward-fixing. PyTorch checkpoint loading with strict=False silently skips mismatched keys; missing instance attributes can cause 10-20% parameters to use random initialization with no error messages. (Source: 2026-02-01, 2026-02-05, 2026-02-20, 2026-02-28)
  • When user questions redundant work or assumptions, they’re identifying missing infrastructure or invalid premises to validate before proceeding. Root cause investigation (architectural analysis, dependency tracing) prevents wasted implementation. When implementing published methods, compare line-by-line with reference code BEFORE claiming alignment—papers omit critical implementation details. Training completion ≠ evaluation done—always verify evaluation artifacts exist. Checkpoint recovery mechanisms validate design when crashes occur; resume logic must rebuild state from logs for idempotency. (Source: 2026-02-05, 2026-02-15, 2026-02-19, 2026-02-20, 2026-02-22, 2026-02-26, 2026-02-28)
  • Spatial coordinate systems in medical imaging require explicit semantic naming (pxl_row=Y, pxl_col=X) and validation via physical units (pixels × scalefactor = microns). Vision foundation model integration critically depends on spatial alignment—incorrect patch extraction can halve performance. In large codebases with modular utilities, always grep for all occurrences before modifying—local copies can shadow imports. Documentation verification requires checking actual code registries/enums, not docstrings. Stub functions with TODO comments can completely break features while appearing to work. (Source: 2026-02-09, 2026-02-16, 2026-02-18, 2026-02-19, 2026-02-23, 2026-02-28)

Architecture (architecture)#

  • Deep learning reproduction requires complete alignment: correct code logic + exact hyperparameters + matching environment. Subtle parameter differences (learning_rate 0.0005→0.001) cause cliff-like performance drops (ARI 0.452→0.05). Distribution shift between training (successful demonstrations) and evaluation (error-injected scenarios) causes catastrophic failure (<1% success)—models require explicit training on recovery trajectories. Embedding dimension bottlenecks cause model collapse: STAIG’s 64-dim output compressing 1536-dim vision features by 96% produces nearly indistinguishable embeddings. (Source: 2026-02-05, 2026-02-16, 2026-02-19, 2026-02-26, 2026-02-27, 2026-02-28)
  • Flexible software architecture requires strategy pattern at key nodes: gene_preprocessing’s auto/standard/clip/none design demonstrates supporting multiple input types without breaking interfaces. API flexibility matters more than narrow optimization. OSC position controllers (kp=150) generate torques far exceeding external forces, actively compensating disturbances—neutral actions disable tracking allowing force accumulation. Error injection pipeline is policy-agnostic at action level; natural error capture more efficient (3 scenarios/rollout vs 0.3). (Source: 2026-02-05, 2026-02-15, 2026-02-16, 2026-02-19)
  • VLA end-to-end integration requirements: ensure observations include camera images via explicit include_images=True, handle multiple dict formats with unified preprocessing, use high-level library APIs encapsulating norm_stats/transforms, cross-conda-env communication via TCP+pickle. Dimensionality reduction’s value depends on downstream model capacity: helps simple methods (+0.05-0.09 ARI) but hurts sophisticated learnable models (-0.05 ARI) that exploit high-dimensional information. Two-stage fusion creates new experimental dimension but requires optimization objective alignment. (Source: 2026-02-16, 2026-02-20, 2026-02-22, 2026-02-23)
  • Strategic planning workflow: humans prefer understanding-first (explain concept → decide direction → detailed plan) over plan-first delivery. Strategic questions should be asked before building implementation plan—saves rework. Documentation-driven development for complex projects ensures alignment before execution. ‘Tests passing’ ≠ ‘system operational’—integration testing with real dependencies essential. Evaluation paradigm fundamentally changes what is measured: supervised classification vs unsupervised clustering measure different embedding aspects. (Source: 2026-02-16, 2026-02-19)

Domain Knowledge (domain)#

  • Vision foundation models cluster by morphological similarity not biological domains: UNI2 groups blood vessels across different cortical layers because they look similar, explaining vision-only spatial domain ARI limitation (0.11-0.16 vs 0.5+ multimodal). This is universal fine-grained recognition problem—general foundation models learn coarse semantics; domain-specific fine-tuning or feature refinement necessary. Need gene expression to distinguish functional regions with similar visual appearance. Input dimensionality mismatches (50D PCA vs 3000D raw features) cause catastrophic GNN performance degradation. (Source: 2026-02-01, 2026-02-19)
  • Benchmark usefulness requires closed-loop proof via 4-pillar validation: (1) large-scale diverse data, (2) theoretically-grounded taxonomy (FMEA/ISO standards), (3) baseline performance characterization, (4) demonstration of training improvement (15-30% SR gain). Cross-domain literature search identifies universal solution patterns—image-only clustering problem identical across fine-grained bird classification, remote sensing, microscopy, geology. RM-IDEAL WWL graph kernel for spatial niche evaluation captures spatial structure independent of embeddings. (Source: 2026-02-08, 2026-02-19, 2026-02-28)
  • Spatial transcriptomics: RM-IDEAL Spearman correlation reveals STAIG fusion (r=0.396) significantly outperforms simple strategies (concat/mean ≈0.25). Leiden community detection provides robust auto-clustering via k-NN graphs. Visium HD (8µm bins) requires dynamic visualization scaling. Pre-processed external datasets need explicit transformation history tracking to prevent corruption (double log-transform). Crowdsourced data pipelines require three-tier architecture with validation/audit trails. Privacy sanitization needs precise boundary conditions to avoid false positives. (Source: 2026-02-06, 2026-02-28)

Tools (tools)#

  • Vectorization with scipy/NumPy (cdist, softmax, advanced indexing) delivers 100-1000x speedups over Python loops. GPU batch processing critical for transformers—Python loops causing serial GPU calls produce severe underutilization (5% GPU). Atomic write pattern prevents corruption: write to temp file, flush, os.replace for atomic rename. Multi-device collaboration: local generation → cloud subdirectories → one-command download+merge. Smart-skip logic implements idempotency through persistent state. Cloud LLM structured output (Anthropic tool_use, OpenAI json_object) significantly reduces parsing failures. (Source: 2026-02-09, 2026-02-15, 2026-02-23, 2026-02-27, 2026-02-28)
  • HPC clusters enforce job-based access control (pam_slurm_adopt) requiring salloc/sbatch before SSH. Resource awareness patterns: check nvidia-smi before launching models, use 50% memory threshold for shared GPU allocation. Robust error handling at iteration boundaries prevents cascade failures while preserving partial results. Unbuffered output (PYTHONUNBUFFERED=1 + stdbuf -oL) essential for monitoring long training. Always profile before optimizing—I/O often dominates. Slurm job attachment more stable than direct SSH. (Source: 2026-02-21, 2026-02-22, 2026-02-23, 2026-02-27, 2026-02-28)
  • GPU batch processing: MuJoCo EGL uses relative GPU indexing within CUDA_VISIBLE_DEVICES—set MUJOCO_EGL_DEVICE_ID=0 regardless of physical GPU. Deep learning benchmarks on spatial data require exclusive GPU access—parallelize only CPU-bound tasks. Q-Former training requires ~22GB; on 32GB GPU, parallel experiments cause OOM. Experiment management requires explicit caching strategy (–override, –skip_cached). External tool output formats may change—defensive programming with type checking. (Source: 2026-02-09, 2026-02-15, 2026-02-16)
