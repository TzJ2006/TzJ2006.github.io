Recurring Problems#

1. Implementation-paper discrepancies causing silent performance degradation (5 occurrences)#

Dates: 2026-02-01, 2026-02-05, 2026-02-07, 2026-02-16 Root Cause: Code reproduction from papers involves subtle architectural choices (normalization presence/absence, dimension sizes, activation placement, coordinate system mappings) that dramatically impact results but are rarely documented explicitly. Default assumptions (e.g., ‘standard ViT preprocessing = ImageNet normalize’) often contradict actual paper implementations. Requires line-by-line comparison across code/config/data preprocessing. Status: ✅ Resolved

2. Environment dependency mismatches and cross-platform compatibility issues (14 occurrences)#

Dates: 2026-02-01, 2026-02-05, 2026-02-14, 2026-02-17, 2026-02-20, 2026-02-22, 2026-02-23, 2026-02-26, 2026-02-28 Root Cause: Multi-conda-env projects with non-overlapping dependencies; Windows-specific subprocess behavior; rpy2 lifecycle management in repeated calls; robosuite/robomimic/MimicGen evolve APIs independently with incompatible signatures; datasets>=4.0 breaks LeRobot 0.1.0 column access; MimicGen environments require explicit import mimicgen side-effect for registration; robosuite adds ‘-state’ suffix to modality keys at runtime but HDF5 stores raw keys. Status: 🔧 Workaround

3. GPU cluster resource management and configuration issues (12 occurrences)#

Dates: 2026-02-17, 2026-02-19, 2026-02-21, 2026-02-22, 2026-02-23, 2026-02-27, 2026-02-28 Root Cause: Multiple independent configuration dimensions (SSH stability, SLURM partition permissions, GPU memory visibility, EGL device binding, residual process VRAM occupation, proxy env vars blocking localhost) with non-obvious interactions. pam_slurm_adopt policy prevents direct SSH to compute nodes. MUJOCO_EGL_DEVICE_ID must match physical GPU ID not relative index. Status: 🔄 Ongoing

4. Documentation-code divergence with stale metrics and missing updates (4 occurrences)#

Dates: 2026-02-02, 2026-02-03, 2026-02-16 Root Cause: Human-readable logs (experiments_log.md, CLAUDE.md) manually updated separately from code registries and CSV data. No automated sync between ground truth (DETECTOR_REGISTRY, ErrorType enums, CSV files) and narrative documentation. Generated visualizations not part of standard pipeline execution. Status: 🔄 Ongoing

5. Cascading failures from shared global state in plugin architectures (2 occurrences)#

Dates: 2026-02-14 Root Cause: Multi-detector systems using global cooldown counters and distance gating create hidden cross-detector dependencies. One detector’s activation silently blocks others; stub implementations in one code path mask working implementations elsewhere. Error injection framework has orthogonal ‘when to trigger’ (detector) vs ‘which injector type’ (injector) dimensions—framework must independently enumerate injector alternatives. Status: ✅ Resolved

6. Third-party API usage assumptions contradicting actual library behavior (3 occurrences)#

Dates: 2026-02-14, 2026-02-16 Root Cause: Using low-level APIs manually instead of high-level wrappers (openpi’s create_trained_policy() vs manual norm_stats loading); assuming fixed output formats (ccusage dict vs list); insufficient inspect.signature() verification before integration. Documentation often incomplete/outdated. PyTorch strict=False model loading silently skips mismatched keys, leaving parameters randomly initialized. Status: ✅ Resolved

7. AI defaulting to symptom mitigation over root cause analysis (6 occurrences)#

Dates: 2026-02-19, 2026-02-23, 2026-02-26, 2026-02-28 Root Cause: AI prioritizes fastest path to green tests: disabling failing components (Coffee rollout), skipping unmatched checkpoint keys (strict=False), bypassing detector logic. Lacks architectural preemption—discovers structural defects (friction/gripper_bias producing 0 scenes, vision encoder complete failure ARI≈0) only after runtime failure. Requires explicit human directive to pursue deeper fixes. Status: 🔄 Ongoing

8. Overly optimistic ‘completion’ criteria missing integration validation (3 occurrences)#

Dates: 2026-02-16 Root Cause: Equating ’tests passing’ with ‘system operational’ without end-to-end validation using real dependencies (model checkpoints, external APIs, hardware). Synthetic unit tests validate code correctness but miss integration issues like camera image passing, observation format mismatches, GPU OOM. Status: 🔄 Ongoing

Monthly Learnings Digest#

Architecture (architecture)#

  • Embedding dimension bottlenecks cause catastrophic collapse: STAIG’s 64-dim output compressing 1536-dim vision features by 96% (std=0.12) combined with spatial majority voting eliminates minority clusters. Other methods using 256-512 dims avoid this. Cascading failures compound: poor embeddings → refinement amplifies rather than fixes errors. SCAN-refined embeddings show divergent effects: beneficial for simple fusion (mean +5.5%, attention +8.6%), detrimental for learnable fusion (qformer -5.4%)—learned fusion handles high-dim noise better, prefers original 1536d over 256d refined. (Source: 2026-02-14, 2026-02-16, 2026-02-19, 2026-02-20, 2026-02-23)
  • Spatial coordinate systems in Visium data have three mapping layers (full-res → hires×scale_factor≈0.15 → lowres). tissue_positions_list.csv columns are counter-intuitive (col4=pxl_row/Y, col5=pxl_col/X). Vision encoder fixes proved patch-level coordinate validation is critical—wrong coordinates caused 100% background sampling. Statistical validation: use known array_row/col physical step lengths vs. pixel change amounts to unambiguously determine CSV column meanings. Embedding quality validation must include: unique row count, per-feature std >0.01, pairwise cosine similarity mean. (Source: 2026-02-17, 2026-02-19)
  • Large-scale benchmark design pattern: cache at minimal reuse granularity (encoder×section), not full experiment combinations. Phase 1 extracts embeddings once per encoder-section pair; Phase 2 all fusion experiments load from cache. Reduces redundant computation 80%+. Training-type fusions (contrastive, Q-Former) need separate timeout/epoch configs from inference-type fusions. Error injection framework design principle: ‘when to trigger’ (detector) and ‘which injector type’ (injector) are orthogonal dimensions—framework must independently enumerate injector alternatives. (Source: 2026-02-14, 2026-02-17, 2026-02-19, 2026-02-22)
  • Error classification system architecture: two-layer design with rule-based classifiers (primary, state-threshold-based, fast deterministic) covering observable state space + VLM layer (auxiliary, renders video, natural language descriptions, cross-validation) for high-level semantics and ambiguous boundaries. Plugin architecture with base ABC + registry enables extensibility. Multi-detector systems require per-detector lifecycle management (independent cooldown, distance gating) to prevent silent cross-detector blocking. (Source: 2026-02-14, 2026-02-16)
  • Idempotency design patterns: embed processed-item tracking in data structures (_merged_devices list in export JSON, _source_device field at end for tail readability). Python 3.7+ dict insertion order preservation is reliable. LLM-in-the-loop requires complete fallback chains: LLM parsing → regex extraction from _raw_input → user clarification. Never assume data field populated by LLM in SR-only paths. (Source: 2026-02-14)

Debugging (debugging)#

  • Implementation-paper alignment requires line-by-line comparison, not high-level matching: image normalization presence/absence, activation on final GCN layer, patch sizes, edge weight directions, coordinate system mappings. Default assumptions (‘standard ViT preprocessing’) often contradict papers. Root cause analysis workflow: systematic comparison (code + config + data preprocessing) across three sources (original implementation, new implementation, paper description) prevents missing subtle differences. PyTorch strict=False model loading is double-edged: allows partial weight matching but silently skips mismatched keys. scGPT bug (self.use_fast_transformer not saved) caused 17.7% parameters (9.45M) to stay random, boosting ARI +44.4% after one-line fix. (Source: 2026-02-01, 2026-02-05, 2026-02-07, 2026-02-28)
  • Environment compatibility debugging: Windows npm commands are .cmd files invisible to subprocess list mode (need shell=True); Claude CLI detects nesting via CLAUDECODE/CLAUDE_CODE_ENTRY env vars (clear in subprocess); rpy2 activate()/deactivate() must pair in try/finally blocks to prevent converter stack corruption in loops; CUBLAS_WORKSPACE_CONFIG must set before all imports for deterministic CUDA. (Source: 2026-02-05, 2026-02-07, 2026-02-14)
  • Integration testing discipline: ’tests passing’ ≠ ‘system operational’. Synthetic unit tests validate code correctness but miss real dependency issues (camera image passing, model checkpoint loading, GPU OOM). Run end-to-end with actual hardware/APIs/checkpoints before claiming completion. For multi-step tasks, persist plans to files (markdown with nested goal structure) for cross-session continuity and user review. File existence + row count checks are insufficient—validate embedding unique row count, per-feature std, pairwise cosine similarity. (Source: 2026-02-12, 2026-02-16, 2026-02-17, 2026-02-19)

Domain Knowledge (domain)#

  • Image-only ST spatial clustering has fundamental ceiling: DLPFC ARI 0.11-0.16 (vs. multimodal 0.45-0.64) due to foundation model cancer-tissue training bias, brain cortex layer morphology subtlety, patch-level encoding lacking spatial context. SCAN self-supervised clustering optimal for image-only (ARI 0.303, +20.6%). Spatial transcriptomics evaluation beyond clustering metrics: RM-Ideal (WWL graph kernel + Wasserstein distance) captures niche similarity at spatial structure level. Implementation requires: shared hash tables across query/reference subgraphs, node ID offset to prevent collisions, k-hop subgraph caching (8min→0s). (Source: 2026-02-08, 2026-02-19, 2026-02-20, 2026-02-23)
  • VLA task-specificity and cross-task generalization limits: Pi0.5 0% success on PickPlace was task misalignment (trained on MimicGen Coffee/Stack/Threading, evaluated on PickPlace not in training set). Pi0.5 Phoenix on MimicGen shows task-dependent performance: Stack_D0=24%, Stack_D1=12%, Threading_D0=2%, Coffee/StackThree≈0%—simple stacking learned, contact-intensive complex tasks fail. Natural error capture (natural_capture mode) is 10x more efficient than synthetic injection (3 scenarios per rollout vs 0.3). Robot benchmark validity requires four-layer proof: dataset scale + taxonomy meaningfulness + baseline performance + data improves foundation models. M14 results quantify necessity: all learned policies fail worse than random (BC-RNN 0.28%, Pi0/Pi0.5 0%, Random 0.0199). (Source: 2026-02-16, 2026-02-18, 2026-02-19, 2026-02-22, 2026-02-23, 2026-02-26, 2026-02-27, 2026-02-28)
  • OSC controller dynamics fundamentally change force injection behavior: kp=150 stiffness causes controller to recompute compensation forces in each physics substep, potentially negating external forces within single env.step(). Visualization requires neutral actions (zero vector) to prevent trajectory tracking. MuJoCo xfrc_applied clears every step—continuous force requires per-step reapplication. EEF body name is ‘gripper0_eef’ not in common lists; fallback substring search risks false matches. (Source: 2026-02-14, 2026-02-16)

Tools (tools)#

  • Third-party ML library integration best practices: always use high-level API wrappers (e.g., create_trained_policy()) rather than manual low-level assembly; verify actual signatures with inspect.signature(); read example code in source repos as documentation is often incomplete. Observation preprocessing must handle multiple input formats with robust key lookup and fallbacks. Dependency version pins: LeRobot 0.1.0 incompatible with datasets>=4.0 (must pin datasets<4.0). MimicGen envs register via import mimicgen side-effect; stripping _D0 suffix loses extra observables. robosuite adds ‘-state’ suffix to modality keys at runtime but HDF5 stores raw keys. (Source: 2026-02-16, 2026-02-20, 2026-02-22, 2026-02-23, 2026-02-26, 2026-02-28)
  • JAX FSDP limitation: init_train_state() passes partial_params with replicated_sharding, so every GPU needs full model params during initialization regardless of final sharding strategy—this causes OOM even with FSDP. Pi0.5_base full fine-tuning needs >62GB/GPU; LoRA reduces trainable params 90% (VRAM ~22.5GB), making 80GB A800 viable. JAX multi-GPU auto data-parallel via CUDA_VISIBLE_DEVICES, no code change needed. (Source: 2026-02-22, 2026-02-23)
  • Multi-device toolchain design pattern: config resolution chain (CLI flag > env var > config.json > default), per-device export tracking for idempotency (_merged_devices list), atomic writes (temp file + os.replace), cloud sync via rclone (headless-friendly, 70+ services, single binary). Claude CLI backend enables API-free operation on servers. Defense against format variations (ccusage dict vs list). Crowdsourced data pipeline security model: three-tier architecture (untrusted submit → trusted gateway → protected storage) with validation/audit trails. (Source: 2026-02-06, 2026-02-14)
  • GPU cluster deployment checklist: (1) check existing allocations/residual processes via nvidia-smi, (2) unset proxy env vars (http_proxy blocks localhost WebSocket), (3) MUJOCO_EGL_DEVICE_ID = physical GPU ID (not CUDA_VISIBLE_DEVICES relative index), (4) SLURM: use srun –jobid –overlap to attach to existing job when sbatch partition permissions unavailable, (5) long SLURM jobs: PYTHONUNBUFFERED=1 + stdbuf -oL + set -eo pipefail + ERR trap for monitorability. A800 80GB can run 9 BC-RNN training concurrently (~2.2GB each). Lustre distributed filesystem du scanning extremely slow—use parallel background tasks + –max-depth=1. (Source: 2026-02-21, 2026-02-22, 2026-02-23, 2026-02-27, 2026-02-28)
  • Performance optimization patterns: O(n²) vectorization via scipy cdist (nested loops 2-5s → 10ms, 100-500x speedup). GPU-based adaptive_dropout_adj eliminates epoch-level CUDA sync overhead. QFormer batched forward: convert 2700 sequential spot calls to 1 batched GPU parallel (20-50x speedup). Dense contrastive loss (InfoNCE) with large batch (n_spots~3600) + small temperature (0.07) prone to float32 overflow—need clamp(-10,10) + log_softmax + L2 norm input. Patch extraction: move np.array(pil_img) for 20000×20000 image outside loop (1x vs 3500x conversion). (Source: 2026-02-07, 2026-02-23)
  • Documentation verification workflow: code registries/enums (DETECTOR_REGISTRY, ErrorType) are ground truth, not docstrings or prior docs. After edits, grep all old values to ensure complete cleanup. Visualization must be explicit pipeline step, not assumed side effect of data generation. Automated sync between CSVs and markdown logs prevents divergence. Matplotlib + scanpy integration: use ax= parameter with explicit fig, axes = plt.subplots() instead of save= parameter to retain figure control. (Source: 2026-02-03, 2026-02-06, 2026-02-09, 2026-02-16)
