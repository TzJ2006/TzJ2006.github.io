Linux 服务器下的指令#

Docker 的安装和调试#

Docker相当于一台虚拟机。安装之后就可以在这台虚拟机上跑代码了。

Docker 的安装#

Docker 的安装方式：#

首先上 Dockerhub 挑选一个心仪的 docker, 下面以 nvidia/cuda:12.0.0-cudnn8-devel-ubuntu22.04 为例：

然后运行以下代码：

docker run -it -rm\
  --name <your-instance-name> \
  --network host \
  nvidia/cuda:12.0.0-cudnn8-devel-ubuntu22.04

注：这里的 -it 指的是打开一个可交互界面，-rm 指的是用后即焚

这时候就会自动下载 docker 并打开一个 bash 来用。

现在你会发现这个虚拟机里面什么都没有，所以就需要 apt-get install

另外，如果你的宿主机器的根目录比较小，想要挂载一个硬盘的话，就在 docker run 中间加上：

-v /path/to/large/storage:/somepath \

这样就可以在 somepath 下挂载这个硬盘了。

注：不能挂载在根目录下，必须挂载在一个文件夹下

如果要用 GPU 语法和 CUDA_VISIBLE_DEVICES类似:

--gpus all

# 全部gpu

--gpus 0,1,2,3

# 只用gpu 0,1,2,3

如果要加入系统变量, 比如 proxy

-e HTTPS_PROXY="http://127.0.0.1:10808"

注意：image name (比如就是 nvidia/cuda:12.0.0-cudnn8-devel-ubuntu22.04) 要放在最后一行

这里在测试的时候建议加上 -rm,这样不会产生很多个休眠中的 docker 但是在要频繁使用的时候不建议使用 -rm, 而是就让 docker休眠就好。

不知道我以前在这里写的啥 应该是说，反正休眠的 docker 可以通过 docker start 唤醒，如果不是必要就别删了呗

docker 的调试#

常用的 docker 指令：#
# 启动 docker
docker start <容器ID或名字>

# 关闭 docker
docker stop <容器ID或名字>

# 重启 docker
docker restart <容器ID或名字>

# 删除容器
docker rm <容器ID或名字>

# 进入容器
docker exec -it <容器ID或名字> bash

# 查看正在运行的容器
docker ps

# 查看所有容器（包括停止的）
docker ps -a

# 列出本地镜像
docker images

# 删除镜像
docker rmi <镜像ID或名字>

# 挂载目录
docker run -v /host/path:/container/path

# 增加环境变量
docker run -e HTTP_PROXY=http://localhost:10086

argparse#

这个的作用就是给 python 代码传参数的 写脚本的时候这个非常方便和有用

首先呢，这个是写得非常详细的基础教程

这里摘抄一些比较重要的部分

import argparse

parser = argparse.ArgumentParser()

parser.add_argument('-m','--model', type=list, default=['llama'],help="which model to use", required=True, metavar='FILE', nargs='+')

cfg = parser.parse_args()

用法解释

  • -m 是缩写
  • --model 是全称
  • type 是类型
  • default 是 默认值
  • help 是辅助信息(在 -help 的时候会显示)
  • required 当这个值是 True 的时候就必须在命令中出现
  • metavar 是在帮助文档中显示的名称
  • nargs 是一种输入方式，可以输入很多个信息

使用方法：

python main.py -m llama Gemma

jupyter notebook 配置#

首先安装 jupyter notebook

pip install jupyter

然后为坠落的内核命名

python -m ipykernel install --user --name your-kernel-name --display-name "Custom Name (Python)"

–name：用于ipython识别 (建议用英文或下划线) –display-name：Jupyter 显示在界面上的名字 (甚至可以有中文)

如果要删除这个内核，则：

jupyter kernelspec uninstall your-kernel-name

理论上添加内核不需要重启 jupyter notebook server

Visual Studio Code#

VS code 连接不上但是 SSH 能连接上：#

原因： 原因分析

结论：

  1. VS code 远端服务器版本和本地版本不一致。
  2. 网络环境导致无法同步。
  3. 断点续传机制导致无法通过重启解决问题。

解决方案：

  1. 在 VS Code 中打开命令面板 快捷键：(Ctrl/Cmd + Shift + P) , 输入 Remote-SSH: Kill VS Code Server on Host… ，选择对应主机，强制终止并清除旧实例
  2. ssh 登录远程服务器，输入 rm -rf ~/.vscode-server 清除远程服务器缓存

Hugging face 数据下载#

方法1：Git LFS#

但是GIT LFS有时候会丢失一部分数据，不知道为什么

比如我现在正在下载libero dataset, 但是他的 Git LFS clone 就只会下载 Libero 10, Libero Spatial, Libero Object, 和 Libero Goal, 并不会下载 Libero 100

我认为是仓库的设置有点问题，因为 Git LFS clone 下来的数据格式和hugging face上下下来的格式有点不同。

但是 Git LFS真的是最简单轻松的下载方式了

只需要点击这里然后跟着操作做就完事了。

下载的速度也比较快，基本跑满了这个服务器的代理网络。

听说Github不能断点续传，但是实测下来几乎没有断点，真的是最稳定的那个服务了。

方法2： hugging_face CLI#

huggingface-cli 是 Hugging Face 官方提供的命令行工具，自带完善的下载功能。

但是实际用起来体验很差，经常莫名其妙就 Internet Error

方法4: hf-mirror 镜像站下载#

hf-mirror镜像站 推出了 hfd, 一个 huggingface 专用下载工具，基于成熟工具 aria2，可以做到稳定高速下载不断线。

这是 hf-mirror 网站给出的 tutorial, 方法清晰简单: 1. 下载hfd

wget https://hf-mirror.com/hfd/hfd.sh
chmod a+x hfd.sh

2. 设置环境变量

# Linux
export HF_ENDPOINT=https://hf-mirror.com

or

# Windows Powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"

3.1 下载模型

./hfd.sh gpt2

3.2 下载数据集

./hfd.sh wikitext --dataset

根据实测，速度也不赖

就是他的这个 Copy 有点问题，要一行一行的 Copy 才可以正常运行

或者直接从我这里copy也行

Hugo#

Hugo如何置顶一篇文章#

添加 weight 参数 e.g.:

---
title: "My Post"
date: 2020-09-15T11:30:03+00:00
weight: 1
---

这里 weight 越小置顶优先级越高

除了文章的 weight, 还可以设置 tag weight 在同一个 tag 下排序： tags_weight: 10

Hugo 如何打开侧边目录#

在页面 yaml 处添加：

---
ShowToc: true
TocOpen: true
---

ShowToc 为 True 表示展示目录 TocOpen 为 True 代表展开所有

Mujoco 默认只会调用主 GPU, 也就是 GPU 0

软链接#

有时候，我们会因为一些原因把一份文件复制得到处都是

但是大部分时候，这种文件都非常的大

那复制到很多个地方就会占用掉很多的磁盘空间

在这种情况下呢，软连接就起作用了

软连接可以理解为桌面快捷方式，其实就是一个指向真正文件的指针

那要怎么做呢？

很简单，只要：

ln -s

即可。

具体用法：

ln -s /original_file /target_file

这样的话就可以把original_file 链接到 target_file那里去

这样就可以在target_file 中访问 original_file 中的内容辣。

GPU 环境配置 bug#
nvidia-driver版本冲突#

通常发生在重装 nvidia-driver 之后

无法通过 apt --fix-broken install 解决

这时候可以：

方法 1：

sudo dpkg --purge --force-all \
  nvidia-driver-525 \
  nvidia-dkms-525 \
  xserver-xorg-video-nvidia-525 \
  libnvidia-decode-525 \
  libnvidia-encode-525

sudo apt-get clean
sudo apt-get update
sudo dpkg --configure -a
sudo apt-get -f install
sudo apt-get autoremove --purge -y

sudo apt-get install aptitude
sudo aptitude purge '~i nvidia-*'
sudo aptitude safe-upgrade

方法 2：

sudo apt-mark unhold libnvidia-* nvidia-*
# 先解除 hold

sudo apt-get remove --purge \
     libnvidia-* nvidia-* \
     --allow-change-held-packages
# 然后完成删除整条依赖链

sudo dpkg --configure -a
sudo apt --fix-broken install
sudo apt autoremove -y
sudo apt clean
# 最后让 apt 来收尾

方法3 (强行复写 apt package, 不建议在物理机上使用):

# 先让 dpkg 走“强覆写”把损坏状态收尾
sudo apt-get  -o DPkg::Options::="--force-overwrite" \
              --fix-broken install

# 如果还有半配置包，再跑一次
sudo dpkg --configure -a
