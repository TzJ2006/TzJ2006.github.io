问题与解决方案#

关键问题#

1. Pi0.5 评估 success rate 为 0%#

解决方案: 根因是 task mismatch：Pi0.5 训练在 5 个 MimicGen task，评估在 PickPlace（不在训练集中）。解决方案：在已训练的 5 个 task 上重新评估

关键洞察: 用户主动质疑「是否把 9 个 task 全部混训了？」直接触发深入根因分析；train/eval task alignment 是机器人学习实验的基础前提，显著失败往往源于配置不对齐而非模型本身

2. BC-RNN 评估报 RuntimeError: input.size(-1) must be equal to input_size. Expected 65, got 37#

解决方案: 根因：create_env() 剥离 _D0 后缀导致使用 base robosuite env 而非 MimicGen env，obs 维度不完整。修复：在所有 4 个 scripts 中 import mimicgen 注册 _D0 环境

关键洞察: MimicGen _D0 环境注册了额外 observables（完整物体状态），base robosuite 环境只返回主物体状态，是两套不同的 observation schema

3. BC-RNN 训练配置为 low_dim 模式，与 MimicGen 论文主结果不一致#

解决方案: 全面重写 5 个配置文件：obs 移除 object key、加入双摄像头、加入 ResNet18Conv encoder + SpatialSoftmax + CropRandomizer、调整 batch_size/hidden_dim/actor_layer_dims

关键洞察: image 模式和 low_dim 模式不只是 obs 不同，整个网络架构也不同；hdf5_cache_mode 必须从 all 改为 low_dim，否则 80GB VRAM 也会 OOM

4. SLURM sbatch 分区无权限且登录节点无 GPU，无法直接提交训练或评测任务#

解决方案: BC-RNN 训练通过 srun –jobid=45628 –overlap 附加到已运行的 interactive job 绕开权限问题；Pi0.5 评测需正式提交 Slurm GPU 作业（#SBATCH –gres=gpu:4）到 ai 分区

关键洞察: SLURM 集群可能有多套工具链（系统级 vs 集群专用），需 source set-XY-I.sh 使用正确版本；集群登录节点工作需时刻意识到 GPU 不可用约束

一般问题#

5. Pi0.5 checkpoints 与评测代码分属不同用户目录，路径关系不明确#

解决方案: 通过并行代理任务分别探索两个用户目录，确认 phoenix_comparison 检查点位于 zhaoganlong 目录，评测脚本位于 tangzijia 的 error_recovery_benchmark，需使用绝对路径引用

关键洞察: 跨用户协作项目需在脚本中明确约定绝对路径，避免相对路径引起的跨用户访问问题

AI 局限性#

重要局限#

  • 诊断显著失败（如 0% success rate）时倾向列举多个复杂假设，而非优先核查最基础的配置一致性（task alignment、obs schema），需要用户点拨才聚焦根本原因
  • 规划完成后未能给出用户可直接执行的下一步操作（如草拟 Slurm 提交脚本），对话在规划阶段结束而非推进到可执行方案
  • 工程决策上缺乏主动权衡提示：默认选择实现简单的方案（low_dim 模式），未主动询问用户在简便性与论文一致性之间的取舍

一般局限#

  • 代码分析阶段出现事实错误（将 5 tasks 描述为 9 tasks）；Write 工具需先 Read 才能覆写已有文件，导致部分创建流程略微低效

今日收获#

核心收获#

  • 机器人学习实验中 train/eval task alignment 是基础前提：必须在评估前确认两侧使用同一套任务定义和 observation schema。Pi0.5/BC-RNN 0% 这类「显著失败」根因往往是配置不对齐，而非模型本身问题
  • MimicGen 官方不提供预训练 BC-RNN checkpoint，需自行训练。官方 image 模式关键参数：ResNet18Conv encoder、SpatialSoftmax 32 keypoints、CropRandomizer 76x76、rnn hidden_dim=1000、actor_layer_dims=[]、batch_size=16、hdf5_cache_mode=low_dim
  • MimicGen _D0 环境通过 import mimicgen 注册到 robosuite，提供比 base 环境更完整的 observables；剥离 _D0 后缀会导致 obs 维度不完整，是 BC-RNN 维度不匹配的根因
  • Pi0.5 MimicGen 评测需要双进程架构：VLA 策略服务器（openpi05环境+GPU）通过 WebSocket 向评测客户端（mimicgen_env）提供动作预测，两者解耦但必须在 GPU 节点上运行

实践收获#

  • SLURM 集群可能有多套工具链，需 source 特定初始化脚本（如 set-XY-I.sh）使用正确版本；当 sbatch 分区无权限时可通过 srun –jobid –overlap 附加到已运行的 interactive job；登录节点工作需时刻意识到 GPU 不可用约束
