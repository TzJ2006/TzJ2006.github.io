Citation#

Content mainly from here: 具身智能基础技术路线.

Part1: 场景理解#

这一部分主要是机器人对环境输入的理解

希望让机器人识别出环境中比较重要的部分， 比如：要抓起的物件

检测分割#

检测分割的算法如：SAM, Open-VOC Detection 可以比较好地检测和分割输入中需要的部分和其他部分

多模态分割#

多模态大模型可以理解更加复杂的语言，比如“汽车旁的穿蓝色衣服的男人”， 并且可以做到像素级别。

prompt 的多样化可以让分割更有针对性，模型可以更好地理解和定位

Part2: 数据收集和引导#

视频学习#

优点：不需要遥控，数据量更广

缺点：需要让机器学习视频中的动作，训练难度更高

VR遥控#

优点：可以让人手模拟，并且可以“手把手”教

缺点：数据量少，需要每一种新情况都需要手动模拟

环境模拟#

优点：环境搭建完成之后就有无限的数据集

缺点：环境搭建复杂，并且无法模拟特殊情况

动作执行#

生成式模仿学习#

把每一个时间点的信息丢进去学习

Diffusion policy#

在每一个时间点之间做 diffusion

Affordance#

输入每一个 region可以被如何操作：比如输入一个瓶子可以被夹起来

这样的话就不再是low level joint, 而是控制器实现的目标

Q&A from LLM#

将 prompt 拆解，并且可以在文本中提取出机器人最后要做的事情

Language Correction#

在人类观察的时候用语言/语音帮助机器人更好地完成任务

世界模型#

预测下一步会发生什么

我对具身智能的理解#

实际上是一个 Agent, 输入是用户的指令以及整个环境。

输出是对环境做出一些改变，比如让机器人举起一个杯子。

所以有这些需要做的：识别输入，做出反应，强化学习

Random Thoughts#

新的LLM生成方式：token -> 大纲 -> 句子 -> 文章 pyramid
