研究轨迹#

Linghao Song是一位专注于计算机体系结构领域的研究者，研究轨迹以存储器计算（in/near-memory computing）为核心线索贯穿始终。2017年以ReRAM图处理加速器GraphR斩获Best Paper奖，奠定了其在非易失性存储器计算领域的早期声誉。2020-2022年进入最高产期，系统性地围绕稀疏线性代数加速（SpMV、SpMM、共轭梯度求解器）与FPGA高层次综合框架（TAPA）构建了完整的研究生态。2023年以NDSEARCH登上计算机体系结构顶会ISCA，将近数据处理范式移植到向量数据库与LLM检索增强生成场景，完成了从传统HPC到AI系统的关键转型。

其成长轨迹的关键特征是’工具+应用’双线并进：TAPA/RapidStream-IR系列提供FPGA编程基础设施，而Serpens/Sextans/NDSEARCH系列则是这套基础设施驱动的高影响力应用。这种工具链自研的策略使其研究具有较强的复现性和社区影响力。值得注意的是，2024年出现的两篇宇宙学/中微子论文（Seesaw机制相关）在研究方向上与其他论文完全割裂，极有可能属于同名不同人的研究者，不纳入本次轨迹分析。

突破性工作#

1. GraphR: Accelerating Graph Processing Using ReRAM (2017)#

描述: 提出了首个基于ReRAM阻变存储器的图处理加速器，将图的邻接矩阵直接映射到ReRAM交叉阵列上，利用模拟电路执行大规模并行矩阵-向量乘法，从而绕过传统冯·诺依曼架构中内存带宽瓶颈，实现了极低能耗的图计算。

为什么之前做不出来: ReRAM器件在2015年前工艺不够成熟，器件间变异性（variability）过高，难以精确控制电导值。更关键的insight缺失在于：此前研究者将ReRAM视为数字存储替代品，而GraphR首次系统论证了’图计算对精度容忍性高、稀疏矩阵天然适合模拟域并行’这一核心匹配性，使得模拟误差可接受。

影响: 开创了’ReRAM作为计算基底’的研究方向，Best Paper奖使其成为近存计算领域标志性工作之一，直接催生了大量后续ReRAM加速器研究；方法论（将稀疏矩阵映射到模拟阵列）被广泛引用和扩展。

2. Refloat: Low-Cost Floating-Point Processing in ReRAM for Accelerating Iterative Linear Solvers (2020)#

描述: 解决了ReRAM模拟计算中最核心的工程难题：浮点数支持。提出将浮点操作分解为可在ReRAM中低成本执行的形式，并应用于迭代线性求解器（科学计算的核心算子），发表于超算领域顶会SC。

为什么之前做不出来: 浮点运算需要高精度（IEEE 754 单精度需23位尾数），而ReRAM模拟计算天然存在噪声，此前普遍认为’ReRAM仅适合低精度整数/定点运算’。关键insight缺失在于：科学计算的迭代求解器具有自收敛特性，允许中间步骤使用混合精度；Refloat找到了精度误差与收敛速度之间的可控权衡点，是认知突破而非单纯工程改进。

影响: 将ReRAM加速的适用范围从图/神经网络扩展到科学计算领域，打开了HPC+ReRAM的新研究空间；11次引用在SC论文中属于稳健表现。

3. TAPA: A Scalable Task-Parallel Dataflow Programming Framework for Modern FPGAs with Co-Optimization of HLS and Physical Design (2022)#

描述: 提出了TAPA端到端编译框架，允许用户用C++任务并行数据流程序描述计算，自动编译为高频率FPGA加速器，关键创新是同时协同优化高层次综合（HLS）与物理布局，解决了大规模FPGA设计频率崩塌的顽疾。

为什么之前做不出来: Xilinx Vitis HLS等主流工具将HLS与物理实现视为分离阶段，导致HLS生成的逻辑在布局布线后频率严重下降。核心insight缺失在于：任务并行粒度天然与FPGA的SLR（Super Logic Region）物理分区对齐，只要在HLS阶段就感知物理约束，就能同时实现高生产力和高性能。这需要对FPGA架构与HLS内部机制都有深入理解，是跨层抽象能力的体现。

影响: 成为学术界FPGA加速器研究的重要基础设施工具，后续TAPA-CS进一步扩展到分布式多FPGA场景；降低了高性能FPGA加速器的开发门槛。

4. NDSEARCH: Accelerating Graph-Traversal-Based Approximate Nearest Neighbor Search through Near Data Processing (2023)#

描述: 针对向量数据库中基于图遍历的近似最近邻搜索（ANNS，如HNSW/NSG算法）提出近数据处理加速方案，发表于计算机体系结构顶会ISCA，直接瞄准LLM检索增强生成（RAG）的底层检索引核。

为什么之前做不出来: ANNS的图遍历算法在2022年前主要被视为软件/算法问题；2022-2023年LLM+RAG的爆发式增长才使ANNS成为数据中心关键负载，产生了硬件加速的强烈动机。算力层面，此前缺乏足够的实测数据证明ANNS是内存带宽瓶颈而非计算瓶颈；NDSEARCH通过profiling给出了关键证据，使近数据处理方案具有说服力。

影响: 22次引用显示其是迄今引用最高的工作，恰好踩在LLM基础设施热潮的时间节点；将体系结构界对近存计算的关注引向向量数据库赛道，推动了后续大量ANNS加速器研究。

研究方向#

  • 近存/存内计算与硬件加速器设计（ReRAM、HBM、NDP）
  • 稀疏线性代数与图计算加速（SpMV、SpMM、图遍历）
  • FPGA高层次综合与编程框架（TAPA、RapidStream）
  • 面向AI/LLM基础设施的系统加速（ANNS、向量数据库、RAG）

方法论演进#

早期（2017-2019）以模拟计算范式为主，利用ReRAM物理特性执行矩阵运算，方法论核心是’精度换能效’的trade-off分析。2020-2021年转向数字域的稀疏加速器设计，方法论升级为’数据流分析+内存访问模式建模’，重点解决稀疏计算中的负载不均衡与随机访存问题，Pyxis数据集的发布也反映了对系统评测方法论的重视。2022年引入’跨层协同优化’方法论，TAPA同时跨越编程模型、HLS编译器和物理设计三层，标志着研究视野从单一加速器扩展到完整工具链。2023年方法论进一步向’工作负载驱动的架构设计’演进，NDSEARCH从真实LLM应用场景出发做性能瓶颈分析，再反推硬件架构决策，体现了以应用需求为锚点的系统研究范式成熟。

整体而言，方法论演进路径是：模拟近存计算 → 数字稀疏加速 → 编译器与硬件协同 → AI应用驱动的体系结构。贯穿始终的是对内存带宽瓶颈的持续关注，只是解决手段从模拟计算演变为HBM利用再演变为近数据处理。

领域影响#

Linghao Song在计算机体系结构领域的贡献体现在两个维度：一是方向性贡献，GraphR（2017 Best Paper）和NDSEARCH（ISCA 2023）分别在ReRAM图加速和向量数据库加速两个时间节点上发出了具有导向性的研究信号，前者启发了一代ReRAM加速器工作，后者恰逢LLM基础设施爆发期引发广泛跟进；二是基础设施贡献，TAPA框架作为开源工具被实际使用，降低了FPGA加速器研究的进入门槛，具有乘数效应。从引用分布看，NDSEARCH的22次引用集中于2023年发表后短短一两年内，显示其在AI系统与体系结构交叉领域有较强的近期影响力，与当前LLM基础设施研究热点高度契合。
