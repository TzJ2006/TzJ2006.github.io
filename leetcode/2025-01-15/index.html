2429. Minimize XOR Problem Solution
January 15, 2025 · 2 min

Today’s Question#

https://leetcode.com/problems/minimize-xor/description/

XOR Operation#

The following diagram shows the truth table of $XOR$: From this table, we can see that: if A and B have the same number of 1s, then $A \space XOR \space B = 0$; otherwise, $A \space XOR \space B = 1$.

Solution Approach#

  • The first condition of the problem, “the same number of 1s”, means that the number of 1s in each bit of the numbers must be equal.

  • This is because a number can have an infinite number of 0s, while the number of 1s is limited.

  • Therefore, we need to count the number of 1s in number2.

  • According to the XOR truth table, for a certain bit to become 0 after XORing, that bit in the result must also be set to 1, thereby canceling out the 1 in the corresponding bit of number1.

  • To minimize the result, we should eliminate the 1s in number1 from the highest bits first.

  • If number2 has more 1s than number1, we need to add additional 1s to the result.

  • To minimize the result, these additional 1s should be added from the lowest bits to the highest bits.

Algorithm Steps#

  1. Use the $bit_count()$ function to count the number of 1s in num2.
  2. Iterate from the highest bit to the lowest bit, eliminating the 1s in number1 first.
  3. Iterate from the lowest bit to the highest bit, adding the remaining 1s as needed.

Complexity Analysis#

  • Time complexity: $O(log_2(N))$, where N is the size of the input numbers.
  • Space complexity: $O(1)$, as only a few variables are used.

Code#

class Solution:
    def minimizeXor(self, num1: int, num2: int) -> int:
        cnt,ans = num2.bit_count(), 0
        for i in range(31, -1, -1):
            if cnt > 0 and (num1 & (1 << i)) > 0:
                ans += (1 << i)
                cnt -= 1
        
        for i in range(31):
            if cnt > 0 and (ans & (1 << i)) == 0:
                ans += (1 << i)
                cnt -= 1
        
        return ans
