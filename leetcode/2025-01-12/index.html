LeetCode Daily 2025-01-12

January 11, 2025 · 3 min

Today’s Problem#

https://leetcode.com/problems/check-if-a-parentheses-string-can-be-valid/

Solution Approach#

To construct a valid parentheses string, we need to find an ‘(’ for each ‘)’ on its left side, and an ‘)’ for each ‘(’ on its right side. When traversing from left to right, we can satisfy the first condition first.

When encountering a ‘)’, there are three ways to match it with an ‘(’: one is using an existing ‘:’, another is finding an unlocked ‘)’, and thirdly, if the ‘)’ itself is unlocked, it can be converted into an ‘:’.

Assuming we have found matches for all ‘)’, the next step is to find an ‘)’ for the remaining ‘:’s. The only way to do this is to find an unlocked bracket, as all ‘)’s have been matched.

Solution#

First, if the string length is odd, it is impossible to form a valid parentheses string. In this case, directly use return False.

Maintain two arrays: $anyBracket$ to store the indices of all unlocked brackets, and $openBracket$ to store the indices of all ‘:’s.

As we traverse the entire string, we will encounter the following situations:

  1. If the current character is an unlocked bracket, push its index onto the $anyBracket$ stack.
  2. If the current character is ‘:’, push its index onto the $openBracket$ stack.
  3. If the current character is ‘)’, we need to find an ‘:’ to match it.
    • First, pop an ‘:’ from the $openBracket$ stack, which helps us find an ‘)’ for the ‘:’.
    • If the $openBracket$ stack is empty, take an unlocked bracket from the $anyBracket$ stack (to act as an ‘:’).
    • If both stacks are empty, then return False, because no ‘:’ can be found for the current ‘)’.

After traversing, there may still be unmatched ‘:’s in the $openBracket$ stack.

Now, iterate through each unmatched ‘:’ in the $openBracket$ stack, and check if there is a bracket in the $anyBracket$ stack with a larger index (i.e., an unlocked bracket on its right side).

  • If there is such a bracket, it means we have successfully found an ‘)’ for the ‘:)’, and the match is successful!
  • Otherwise, it is not possible to match, and return False.

If the number of elements remaining in the $anyBracket$ stack is even (since we already checked the parity earlier, this must be true), then return True, and we are done!

Important Techniques#

  • Why use stacks for both $openBracket$ and $anyBracket$?
  • Because ‘)’ always tries to match with the closest ‘:’ on its left side. Therefore, we need a FIFO data structure to retrieve the “nearest element”.

Complexity Analysis#

  • Time complexity: $O(N)$, where N is the length of the string.

  • Space complexity: $O(N)$, as we store the indices of all brackets.
