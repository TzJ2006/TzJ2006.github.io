Today’s Problem#

https://leetcode.com/problems/construct-k-palindrome-strings/description/

Idea#

In a palindrome string, at most one character can have an odd number of occurrences. Therefore, to construct $k$ palindromes, the number of characters that have an odd number of occurrences can be at most $k$.

Additionally, since there are only 26 different characters, if $26 \leq k$, the result will definitely be true. However, if the length of $s$ is less than $k$, the result will be false.

Let’s prove this. Let’s denote the number of characters with an odd number of occurrences as $cntO$. The total number of occurrences for all other characters is $2 \times cntE$.

If all characters have an even number of occurrences, then as long as $k \leq N$ (where $N$ is the total length of $s$), it is possible to construct a palindrome. This is because we can place one character at the beginning of the palindrome and the corresponding repeated character at the end, thus maintaining the palindrome structure.

Since $k \leq N$, we have $k \leq cntO + 2 \times cntE$, which implies $k - cntO \leq 2 \times cntE$. This means that all characters with an odd number of occurrences can form a palindrome, totaling $cntO$ such palindromes.

We have already proven that when all characters have an even number of occurrences, as long as $k \leq N$, it is possible to construct a palindrome. Therefore, if $cntO \le k$, the result is true; otherwise, it is false.

In summary, the problem boils down to determining whether $k \leq N$ when all characters have an even number of occurrences. This condition is always true.

Tips#

Since there are only 26 different characters, if $26 \leq k$, the result will definitely be true.

Solution Method#

Just count the number of occurrences for each character, and then check if the number of characters with an odd number of occurrences does not exceed $k$.

Complexity Analysis#

  • Time complexity: $O(N)$, where $N$ is the length of the string.
  • Space complexity: $O(1)$, as we only need to store the counts of 26 characters.

Code#

class Solution:
    def canConstruct(self, s: str, k: int) -> bool:
        cnt = [0] * 26
        if len(s) < k:
            return False
        if k > 25:
            return True
        # The code that makes the code run very fast.
        for ch in s:
            cnt[ord(ch) - ord('a')] += 1
        x = 0
        for i in range(26):
            if cnt[i] % 2 == 1:
                x += 1
        return x <= k
