2375. 如何从DI字符串构造出最小数字的解题思路
May 21, 2025 · 4 min

Today’s Problem#

https://leetcode.com/problems/construct-smallest-number-from-di-string/

Approach#

This problem requires us to find a number sequence under certain constraints.

First, it can be observed that the length of the sequence N is very small. Therefore, we can use Depth-First Search (DFS) to enumerate all possible cases and obtain the result.

Another approach is to use a stack. When the sequence increases, the optimal strategy to construct a valid sequence is to fill in the smallest available number at each step. When the sequence decreases, since we still want the overall sequence to be as small as possible, we need to fill in the smallest available number, which must be greater than the number at the subsequent position. Thus, the smallest number that can be filled at the current position is equal to the current index plus the number of consecutive ‘D’s that follow. Based on this, we can use a stack to temporarily store the numbers being processed. Whenever we encounter an ‘I’ or reach the end of the sequence, we can append the elements in the stack in reverse order to the result array.

Method#

For Solution 1 (DFS), we only need to enumerate all possible solutions and return the first one that meets the conditions.

For Solution 2 (stack), when we encounter an ‘I’, we add the element from idx + 1 to the result array. Then, pop all elements from the stack in reverse order and add them to the result array. Finally, empty the stack. When we encounter a ‘D’, push the element from idx + 1 onto the stack for later processing.

Tips#

  • DFS: Since the problem requires finding the smallest valid sequence, the first non-None result is the answer. This means that once a valid result is found, we can return it immediately without continuing the search.
  • Stack: Here, I specifically added a ‘D’ at the end of the sequence. Intuitively, the last element of the sequence is the largest number in the entire sequence. To include it in the result array, we need to perform an ‘D’ operation. In this way, if the last character of the original sequence is ‘I’, we can directly add the largest number to the end of the result array, which is equivalent to performing an ‘D’ operation. Because the ‘I’ operation will empty the stack, leaving only one element in the stack, appending it in either order results in the same outcome. If the last character of the original sequence is ‘D’, then the largest number should be placed at the beginning of the consecutive ‘D’s. In this case, we need to perform an ‘D’ operation to place it in the correct position. This tip makes the code more concise and elegant, but it sacrifices readability. It is not recommended for actual use.

Complexity#

  • DFS solution: Time complexity is $O(N!)$, where N is the length of the sequence.

  • Stack solution: Time complexity is $O(N)$, where N is the length of the sequence.

  • DFS solution: Space complexity is $O(N)$, where N is the length of the sequence.

  • Stack solution: Space complexity is $O(N)$, where N is the length of the sequence.

Code#

class Solution:
    def smallestNumber(self, pattern: str) -> str:
        arr = []
        n = len(pattern) + 2
        def dfs(arr):
            if len(arr) == n - 1:
                return arr
            now = len(arr) - 1
            res = None
            if pattern[now] == 'I':
                for i in range(arr[now] + 1, n):
                    if i not in arr:
                        res = dfs(arr + [i])
                        if res is not None:
                            return res
            else:
                for i in range(1, arr[now]):
                    if i not in arr:
                        res = dfs(arr + [i])
                        if res is not None:
                            return res
            
            return res


        for i in range(1, n):
            ans = dfs([i])
            if ans is not None:
                return ''.join(map(str, ans))
                # return ans
        
        return None
