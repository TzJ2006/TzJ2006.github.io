经验教训#

关键经验 - 在跨样本任务中，基模型的关键优势在于它们能够在不进行联合训练的情况下提供共享嵌入空间；预训练的权重确保了任何输入的可比性。零样本交叉采样检索本质上依赖于这一特性——对每个样本的优化无法替代它。#

  • 名称歧义是自动化特征分析中的主要数据质量问题——影响了大约30%的分析对象。需要多种信号方法：h指数和领域一致性、引用时间分布、机构元数据、合作者网络分析。单一数据源是不够的。
  • 恢复数据增强的三层策略：场景配置（使用不同的错误情景作为初始状态）、交叉程度（将扰动从D0旋转到D1/D2）、交叉类型（在RBG内部进行交叉增强）。结合MimicGen轨迹分割技术，数据需求可以减少81%（从1,740个样本减少到329个样本）。
  • 复合模型系统中的坐标变换错误可以通过状态重放机制来掩盖——该机制仅覆盖XML中定义的位置，只有在新的初始化时才变得明显。这解释了在演示过程中通过目视检查难以发现的长期存在的错误。
  • 为什么之前没有这样做：“Before”反事实推理框架揭示了技术前提条件（GPU/预训练模型）、数据可用性（大规模数据集）和理论基础（新概念）之间的时间依赖性。学术成就高度依赖于“技术栈成熟度”和“时间窗口”的匹配——而不仅仅是个人才能。
  • 用户更喜欢使用Claude Cli后端进行本地操作（claude –print），而不是使用带有密钥的云API。在将来调用小工具/研究时，必须记住这一偏好。重复的身份验证失败应引发关于后端假设的问题。
  • 基于LLM的奖项认可在没有实时API集成的情况下具有基本的召回限制——训练数据无法跟上会议论文的步伐。需要采用混合方法：（1）筛选LLM候选者；（2）使用DBLP/OpenReview API进行验证；（3）手动解决冲突。
  • 2016-2023年间机器人领域的范式转变：深度强化学习（2016-2019）→大规模数据集用于跨视角性能评估（2020-2022）→基于视觉语言模型的模型（自2023年起）。每个时期仅持续了2-3年；在这些转型期间工作的研究人员产生了不成比例的显著影响。
  • 热路径优化的黄金法则：读取路径假设资源存在，写入路径创建这些资源。DiskCache mkdir已从读取操作（通常称为path_for）移动到写入操作（很少称为put()），从而提高了10倍的效率。这一规则适用于所有持久层，如文件系统、数据库和缓存。
  • Claude代码中的后台Bash任务不继承conda init；必须使用’conda run -n ENV’或绝对路径（${CONDA_PREFIX}/bin/python）。在非交互式子shell中，直接调用conda activate会导致CondaError。
  • 结构化输出验证应与生成过程分离——两阶段的“先生成再修复”模式更适合复杂的嵌套模式，而不是单一完美的输出；这使得可以有一个不可知的验证层。
  • 恢复行为群体抽象：根据所需的运动原语对29种错误进行分类（重新抓取、检索、收缩、重定向、重新排列），而不是按类型分类。这允许在同一群体内共享数据——这比按错误类型单独收集数据更为实用。
