问题与解决方案#

关键问题#

1. Leiden clustering 需要与 KMeans 公平对比，但 Leiden 默认以 resolution 参数控制 cluster 数，无法精确匹配 n_clusters#

解决方案: 在 cluster_with_leiden() 中实现二分搜索：在 [0.01, 10] 区间迭代调整 resolution 直到 cluster 数匹配，最多 20 次迭代

关键洞察: Leiden 的 resolution 与 cluster 数是单调关系，可以用二分搜索高效匹配目标 cluster 数，实现与 KMeans 的公平对比

一般问题#

2. HD pipeline Vision Extraction 失败：VisionExtractor 用 data_root/section_id 路径查找图像，但 HD 数据在 crop_dir 中，导致 11/13 实验失败#

解决方案: VisionExtractor.init 增加 crop_dir 参数；extract() 传递 image_dir_override=crop_dir；_load_image_and_coords() 优先使用 image_dir_override；runner.py 创建 VisionExtractor 时传入 crop_dir

关键洞察: DLPFC 和 HD 数据的图像存储结构完全不同，pipeline 需要显式区分两个数据集的图像路径

3. SLURM 脚本使用 source activate General 导致 conda 环境激活失败，Leiden 实验全部在无 GPU 的 login node 运行#

解决方案: 将 SLURM 脚本中 source activate 替换为 eval $(conda shell.bash hook) && conda activate，重新提交 job

关键洞察: DCC 集群的 SLURM 节点需要显式初始化 conda shell hook，不能直接 source activate

4. run_pipeline.py 中 dataset 默认逻辑错误：hd_global 配置段存在时自动切换为 HD 数据集，导致 Leiden DLPFC 实验跑成了 HD 模式#

解决方案: 将 dataset 默认值改为 DLPFC，只在 –dataset HD 显式指定时切换

关键洞察: 配置文件中 hd_global 段的存在不应影响默认数据集选择

角色 思路

角色 思路

AI 局限性#

一般局限#

  • 未考虑 DLPFC 和 HD 数据集图像路径的结构差异，VisionExtractor 实现中硬编码了 data_root/section_id 的查找逻辑，需要人类发现失败后才修复
  • 对 HPC 环境的 conda 激活方式判断错误，SLURM 脚本使用了在计算节点不生效的 source activate，导致首次提交的两个 job 均存在环境问题
  • 无法主动判断 17502 spots 的 HD 数据集是否需要 GPU/SLURM，过度依赖集群资源，需要人类纠正
  • 对 web search 启动了 background agent 但等待时间过长，用户体验差；缺乏对异步任务耗时的预期管理

今日收获#

核心收获#

  • Leiden clustering 在 HD 数据上表现与 KMeans 相当（pca_leiden ARI≈0.333 vs pca ARI≈0.288 on DLPFC），对简单 gene-only 方法提升明显，但对 STAIG 这类图对比学习方法收益递减——STAIG 的优势来自空间图结构，与聚类算法相对独立
  • HD 上简单融合策略（concat/mean/element_wise_sum）ARI 低于单模态 PCA（0.19-0.20 vs 0.351），表明 naive 融合在 HD 大规模数据上可能有信息稀释问题；attention 和 staig_fusion 是例外（attention ARI=0.342）
  • HD 数据（STHD Ground Truth，3 clusters：Epithelium/Connective Tissue/Smooth Muscle）上，vision-only UNI2 的 NMI 最高（0.387），但 ARI 低于 PCA（0.344 vs 0.351）——NMI 和 ARI 在不同数据上可能不一致，需要两个指标共同报告
  • MISO(Nat Methods) 的 HD 部分用 Visium HD colorectal cancer 展示了 SPP1（tumor-associated macrophages）和 CEACAM6 的空间表达，可以直接作为 MIHD Fig 4 marker gene 验证的参考框架

实践收获#
