问题与解决方案#

关键问题#

1. 在Linux系统中，FloatBall窗口始终保持400x400的大小，Tauri的set_size()功能无效#

**解决方案：**GTK自下而上的大小协商方式会取max(window_requested, widget_natural_size)，而WebKitGTK的自然大小约为200x200，这会导致窗口锁定。修复方法是在GTK主线程中通过with_webview()调用widget.set_size_request()，移除set_resizable开关（with_webview是异步的，set_resizable(false）会在回调执行前排队并锁定旧大小），并使Linux窗口能够永久调整大小。

**关键洞察：**Tauri的set_size/set_min_size/set_max_size操作遵循GTK窗口级别的提示，而GTK自下而上的协商方式忽略了这些提示，以符合小部件的自然大小。with_webview()的回调尚未执行，因此后续的Tauri API调用仍然处于FIFO异步队列中。

2. 在Linux系统中折叠窗口时出现视觉闪烁现象：双阶段序列暴露了中间状态，而WM限制的旧宽度导致了系统性的坐标偏移解决方案： 两阶段处理：macOS具有250毫秒的动画缓冲时间，而Linux则禁用过渡效果，使得中间状态非常明显。在Linux系统中，首先使用Rust原子操作将窗口大小缩小到56像素，然后设置CSS的展开状态为false（此时100%的窗口大小相当于56像素，因此变化是看不见的）。窗口定位问题：GTK重力功能在某些窗口管理系统中无法正常工作（相反，它会导致系统性的偏移），尝试进行修正后，仍然会出现双重帧闪烁现象。根本的解决方案是使用固定大小的窗口，并通过内部CSS来实现展开和折叠功能。#

关键洞察： 禁用CSS过渡效果需要同步JS/Rust的时序逻辑；并非所有窗口管理系统都支持GTK重力功能；事后补救措施本身就会导致可见的闪烁现象，根本的解决方案是让窗口管理系统一次性接受正确的坐标。

3. Life-Copilot MCP服务器出现了三层故障：虽然存在日历令牌，但代理仍然报告连接失败#

解决方案： 三个地方同时出现故障：FastMCP被实例化但尚未调用run()，.claude/mcp_servers.json文件缺失，Butler命令（包括MCP工具列表）未能传递到Claude CLI子进程。我们创建了__main__.py文件来启动stdio传输，重新生成mcp_servers.json文件，并在ClaudeAdapter中添加了–append-system-prompt参数。至此，所有19种MCP工具都能正常工作了。

关键洞察： “编写代码”和“调用代码”是两回事；使用工厂模式创建对象并不等同于启动服务；外部程序需要配置文件来发现服务；应用程序验证层和CLI工具上下文注入是两个不同的层次；令牌验证通过并不意味着CLI工具收到了正确的上下文信息。

4. 用于Linux屏幕时间跟踪的初始解决方案使用了xdotool，但该工具不支持Wayland（用户机器的操作系统是GNOME 50 Wayland）#

解决方案： 进行了可行性检查以确认是否可以使用Wayland环境，将主路径替换为GNOME Introspect D-Bus（org.gnome.Shell.Introspect.GetWindows()），同时使用xprop作为XWayland的备用方案，最后降级为PID→cmdline。所有子进程调用都有5秒的超时时间，并使用proc.kill()来终止进程。

关键洞察： GNOME 41及以上版本提供了标准的Introspect D-Bus接口，这是获取Wayland下活动窗口的正确方式。这个问题是在评审过程中由Critic子代理主动发现的。如果忽视了这个问题，那么该功能在用户机器上将完全无法使用。

5. AionUI无法通过URL或其他方法（如SSH）连接到远程无头服务器上的Claude Code CLI#

解决方案： AionUI远程代理架构假设代理会暴露一个HTTP端口，而CLI进程则没有HTTP端点。这两种方式根本是不兼容的。建议暂时使用tmux/screen来管理多个会话，并向AionUI团队提交功能请求。

关键洞察： AionUI数据库中已经有一个remote_agents表（包含allow_insecure字段），这表明开发团队正在构建远程代理功能。直接通过SSH连接可能是未来的发展方向。

6. X11窗口管理系统拒绝放置部分超出屏幕边界的窗口，导致吸附和折叠过程中出现漂移#

解决方案： X11窗口管理系统如Mutter/GNOME会将未装饰的窗口放置在可见区域内，拒绝x < 0或x > screen_width - window_width的位置。Linux使用edge_offset=0，而macOS/Windows则使用edge_offset=ball_size/2。

关键洞察： macOS和Windows允许未装饰的窗口部分超出屏幕边界，而X11窗口管理系统通常不允许这种情况。两种平台都需要不同的边缘偏移量。

7. ClaudeCLIAdapter.is_available()函数缺少异步功能，导致Claude默默降级到Codex并超时#

解决方案： 普通的await方法不会引发编译错误，但运行时TypeError会被except Exception捕获。因此，Claude被判定为不可用。添加async关键字后，Claude的可用性恢复为True。

关键洞察： 在Python中，等待同步方法会引发TypeError。如果整个except Exception捕获到了这个错误，可能会导致错误的False结果，从而导致功能降级，而不是错误消息。这个问题很难排查。

8. 在多轮对话中，机器人无法理解跨轮参考关系（DualVerifier参数连接中断）#
