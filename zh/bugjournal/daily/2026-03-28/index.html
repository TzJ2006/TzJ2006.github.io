问题与解决方案#

关键问题#

1. cchypothesis缺乏运行时调试能力 — 所有假设仅通过静态代码分析进行验证，无法检测动态状态/时间线/数据流问题。#

**解决方案：**实现了第三阶段的智能分类：静态假设保持并行只读模式；运行时假设使用带有[DEBUG Hx]标记日志和Git安全检查点的顺序调试模式，并通过git restore自动清理。

**关键洞察：**双轨调查在需要时保持速度（并行静态处理），同时增加深度（调试模式检测）——避免减慢所有调试会话。

2. 持续以1-2Hz频率出现的垂直控制台抖动是由一个三层正反馈循环引起的：数据刷新 → UI重新渲染 → 窗口大小调整 → 数据刷新。#

**解决方案：**多层中断机制：(1) 窗口：16毫秒到100毫秒的定影延迟 + 500毫秒/3次操作的有限速度；(2) 数据：浅层PayloadEqual，跳过不必要的.set()函数；(3) 解析器：2分钟的活动超时，以防止five_hour_cost振荡。

**关键洞察：**自振荡错误需要在所有反馈层同时进行干预；单独修复一个层只会加重问题，而不会消除循环。

3. Git存储堆栈冲突：ccplan提案在现有的第四阶段存储之上增加了第三阶段的存储，导致弹出顺序混乱，可能会丢失未提交的用户工作。#

**解决方案：**移除了第四阶段的Git存储，并使用非破坏性的git restore进行清理；现在只有第三阶段使用存储（单一存储模型）。

**关键洞察：**自动化工作流程中的多个存储层使得状态管理变得脆弱——更倾向于使用非状态的清理（git restore）而不是状态的撤销（存储）。

4. TokenMonitor窗口出现在屏幕中央而不是系统托盘附近；大小调整导致窗口向下延伸进入任务栏，而不是保持底部边缘。#

**解决方案：**实现了直接控制托盘位置的Win32 API（FindWindowW → TrayNotifyWnd）；将前端大小调整从webviewWindow.setSize()改为后端set_window_size_and-align IPC，以实现原子级别的大小和位置更新。

**关键洞察：**tauri的跨平台定位插件抽象了平台差异，但精度降低了；Windows系统托盘的位置需要特定于平台的Win32 API来实现精确对齐。

5. 42个Rust编译错误：command.rs包含了删除模块的错误导入（change_stats、integrations、parser），以及新命令/子模块的重复代码。#

**解决方案：**将command.rs重写为一个简单的80行模块结构，声明了6个常见模块，仅保留AppState/UsageDebugReport和4个共享辅助函数，修复了可见性和类型问题。

**关键洞察：**不完整的重构（同时使用旧代码和新代码）比单一单元代码更糟糕；要么完成过渡，要么撤销重构。

6. 窗口锚点检测不可靠：最初为visible:false，存在position_near_tray和GetWindowRect之间的竞争，连续的大小调整会导致锚点翻转。#

**解决方案：**完全移除了动态检测功能；托盘弹出窗口始终使用position_near_tray来工作.bottom，因此底部锚点的行为被硬编码了。

**关键洞察：**对于已知类型的窗口，静态配置比动态检测更可靠；运行时检测增加了复杂性和故障模式，但没有带来额外的好处。

学习成果#

主要结论#

  • 多阶段规划工作流程（ccplan）在处理具有冲突问题的复杂集成时非常有效——对抗性评审阶段发现了可能导致数据丢失的关键git暂存冲突
  • 级联反馈循环需要多层干预：仅修复窗口大小调整问题（16ms→100ms）是不够的；还需要解决数据层（负载均衡）和计算层（超时）的问题
  • 指示的调试需要仔细的生命周期管理：每个假设都需要独立的指标→执行→分析→清理循环，以避免交叉污染和残留的调试代码
  • “不完整的模块重构会产生隐藏的依赖地狱：commands.rs仍然导入crate::parser，而新的commands/calendar.rs导入crate::usage::parser——这两条路径无法共存”
  • 动态运行时检测增加了失败模式，而在静态配置足够的情况下没有好处：Tray窗口始终使用position_near_tray，因此底部锚点应该硬编码
  • Tauri中的跨平台窗口定位：插件抽象牺牲了准确性以换取可移植性；完美的像素定位（尤其是系统托盘对齐）需要特定于平台的原生API

实际学习成果#

  • 全面的测试套件可以捕捉到重构问题：218个Rust测试和191个前端测试验证修复没有破坏现有行为
  • 自动化清理工作流程必须区分被跟踪和未被跟踪的文件：Git恢复处理被跟踪文件；未跟踪文件（日志、临时目录）需要明确的rm/rmdir操作
