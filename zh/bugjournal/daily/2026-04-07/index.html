问题与解决方案#

严重问题#

1. Hugo未安装，脚本在Step 5以’hugo: command not found’（exit 127）终止#

解决方案： 用户通过系统包管理器安装Hugo extended v0.154.5。

**关键洞察：**set-e会将任何命令失败直接转化为脚本终止，Hugo缺失使得整个7步流程无法到达关键构建阶段。

2. themes/PaperMod/和public/目录均不存在，Hugo构建和部署无法运行#

解决方案： 用户克隆PaperMod主题和tzj2006.github.io部署仓库。

**关键洞察：**Hugo站点部署依赖三个独立组件（hugo二进制、主题仓库、部署git仓库），任何一个缺失都会静默或终止式失败，应在文档中明确前置条件。

一般问题#

3. static/videos目录不存在，find命令返回错误，set-e导致Step 4意外终止#

**解决方案：**在find之前添加目录存在性检查（[ -d “$DIR” ]），目录不存在时跳过该步骤。

**关键洞察：**对可选/空目录的find操作必须在set-e脚本中进行防护，否则正常的“目录为空”场景会触发非预期的脚本终止。

4. HandBrakeCLI内联回退分支的PID被重复加入pids数组，且压缩后未替换原文件（输出为.pdf）#

**解决方案：**将pids+=($!)和echo语句移入各自的if/else分支；在HandBrakeCLI命令后加&&mv替换原文件。

**关键洞察：**并行后台任务的PID追踪逻辑需要与if/else控制流精确对齐，将公共代码放在分支外部容易引入重复执行和静默数据丢失的错误。

5. git push因GitHub认证未配置失败（could not readUsernamefor’[[[HYMTPH_0]]’）#

**解决方案：**用户配置了SSHkey或ghauth后重新运行，脚本完全通过。

**关键洞察：**HTTPS远程在非交互shell中需要预配置凭据，部署仓库应改用SSHremote以避免环境差异。

学习心得#

关键学习点#

  • Hugo站点部署有三个必须同时完成的独立依赖（hugo二进制、主题仓库、部署git仓库），任何一个缺失都会导致不同阶段失败，应在项目文档中明确列出这些前置条件

实用学习点#

  • bashset-e脚本中对可选目录的find操作必须加目录存在性检查，否则正常的“目录尚未创建”场景会触发非预期的脚本终止
  • 并行后台任务（& + pids追踪）的代码必须和if/else控制流精确对齐，将公共代码放在分支外部容易引入重复PID追踪和静默的文件替换遗漏错误
