问题与解决方案#

关键问题#

1. Linux WM无法原子性地处理透明无边框窗口的大小调整，导致系统性的位置偏移+305px/+113px，闪烁的探测/校正机制，以及WebView和原生窗口之间的尺寸不一致。#

**解决方案：**将窗口大小设置为展开状态（152×56），并且不会进行任何调整。使用GDK input_shape_combineRegion来控制点击区域：仅在折叠时显示球区（56×56），在展开或拖动时显示整个窗口。 **关键洞察：**这是一个平台架构问题，而不是计算错误。Linux WM对相同的rect请求有不同的解释。需要在大小调整层面解决问题；局部修复只能暂时解决症状。

2. Life-Copilot Discord机器人继续报告“需要OAuth认证”的日历查询。根本原因有两个重叠的问题：（1）Claude -p不会加载项目级别的MCP配置；（2）在非交互模式下使用MCP工具时出现权限拒绝问题。#

**解决方案：**在claude_adapter.py命令中添加–mcp-config和–permission-mode bypassPermissions参数。直接运行Claude -p揭示了权限拒绝的根本原因。由于与MCP服务器初始化冲突，–allowedTools通配符在非交互模式下不可靠，因此使用bypassPermissions是后端服务调用的正确方法。

3. 实施固定窗口布局后，出现了三个平台特定的问题：球的显示变形（窗口构建器仍然使用56×56大小）、无法扩展以及拖动失败（鼠标离开球区后输入形状被阻塞）。#

解决方案：（1）窗口构建器应使用展开大小并基于GTK geometry_hints锁定min=max。（2）CSS justify-content应始终根据data-direction生效。（3）添加了一个新的set_float_ball_dragging命令，用于在拖动过程中临时扩展输入形状，并在过程完成后恢复。

**关键洞察：**WebKitGTK有默认的最小尺寸限制。X11层在setPointerCapture之前就拦截了输入形状事件，因此需要在Rust中同步扩大区域。

一般问题#

4. 使用structlog包装stdlib日志记录时，额外的{}字段会从JSON输出中悄悄消失。#

**解决方案：**将ExtraAdder()处理器添加到shared_processors条目中，确保额外字段被合并到结构化输出中。

**关键洞察：**当使用structlog包装stdlib时，额外的参数不会自动传递。必须显式添加该参数，这样字段就会悄无声息地消失，而不会报告任何错误。

5. 当用户请求回退“本次会话的更改”时，人工智能错误地使用了git stash，可能导致历史工作丢失。#

**解决方案：**使用git stash pop后，手动识别并纠正本次会话期间所做的具体更改。

**关键洞察：**Git stash会影响所有未提交的变化，因此不适合只需要纠正“本次会话的更改”的情况。最好在单独纠正之前使用git diff来明确变化的范围。

今日成就#

核心成就#

  • Linux X11 WM并不保证透明无边框窗口在调整大小时的原子性。不同的WM对相同坐标的解释不同，导致系统性的偏移问题。唯一可靠的解决方案是消除调整大小的功能（固定窗口+GDK输入形状）。本地补丁只能暂时解决症状。
  • Claude CLI -p模式需要两个步骤来完成MCP配置：（1）显式传递–mcp-config以加载项目级工具；（2）添加–permission-mode bypassPermissions以绕过非交互模式下的权限确认。在这种情况下，–allowedTools通配符是不可靠的，因为它与MCP服务器初始化发生冲突。bypassPermissions是进行后端服务调用的正确方法。
  • GDK输入形状在X11层拦截事件。setPointerCapture无法覆盖它。在拖动场景中，需要使用Rust中的set_float_ball_dragging命令临时将输入形状扩展到整个窗口。使用cairo-rs 0.18和gdk 0.18时，可以直接在Tauri with_webview回调中使用input_shape_combineRegion，从而不需要额外的复杂依赖项。
  • 在调试嵌套进程CLI工具（主进程→Claude CLI→MCP服务器）时，首先记录实际的命令参数，然后隔离并验证每个参数的效果更为高效。
  • ccplan方法通过探索替代方案（6个计划）、经过反评审过程以及测试可行性，有效地将复杂的平台架构问题转化为可执行的计划。ECL文档提供了跨会话的上下文持久性，降低了重复分析的成本。

实际成就#

  • 当使用structlog包装stdlib日志记录时，必须显式添加extra={}参数作为ExtraAdder()处理器，以便出现在输出字段中。否则，该字段会无声地消失，而不会报告任何错误。
