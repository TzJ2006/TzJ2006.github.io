问题与解决方案#

关键问题#

1. Linux WM 无法原子处理透明无边框窗口的动态 resize，导致系统性 +305px/+113px 位置偏移、probe/corrective 机制闪烁、WebView 与 native window 尺寸不一致裁切#

解决方案: 固定窗口尺寸为 expanded（152×56）永不 resize；用 GDK input_shape_combine_region 控制点击穿透：collapsed 时仅球区域（56×56），expanded 及拖拽时全窗口接受事件

关键洞察: 这是平台架构问题而非计算 bug：Linux WM 用不同语义解释同一 rect 请求；消除 resize 才是根治，局部补丁只会在症状间转移

2. Life-Copilot Discord Bot 对日程查询持续回复"需要 OAuth 认证"，根本原因为两个叠加问题：(1) claude -p 不加载项目级 MCP 配置；(2) 非交互模式下 MCP tool 调用权限被静默拒绝#

解决方案: 在 claude_adapter.py 命令中依次添加 –mcp-config .claude/mcp_servers.json 和 –permission-mode bypassPermissions；通过直接运行 claude -p 命令复现 permission_denials 字段定位根因

关键洞察: Claude CLI session-less 模式与 interactive 模式 MCP 加载行为不同；–allowedTools 通配符在非交互模式下因与 MCP server 初始化存在竞争条件而不可靠；bypassPermissions 是后端服务调用的正确模式

3. 固定窗口方案首次实现后出现三类平台细节问题：球显示压扁（window builder 仍用 56×56 创建）、无法展开（CSS 未按 data-direction 应用）、拖拽失效（input shape 阻断鼠标移出球区域后的 X11 事件）#

解决方案: (1) window builder 改用 expanded 尺寸并用 GTK geometry_hints 锁定 min=max；(2) CSS justify-content 根据 data-direction 始终生效；(3) 新增 set_float_ball_dragging 命令，拖拽期间将 input shape 扩展到全窗口，结束后恢复

关键洞察: WebKitGTK 有默认最小自然尺寸约束；X11 层 input shape 事件拦截早于 setPointerCapture，需在 Rust 侧同步扩大区域

一般问题#

4. structlog 包装 stdlib logging 时，extra={} 字段在 JSON 输出中静默丢失#

解决方案: 在 shared_processors 中加入 structlog.stdlib.ExtraAdder()，使 extra 字段合并到结构化输出

关键洞察: structlog 包装 stdlib 时 extra 参数不会自动传递，需显式添加 ExtraAdder，无报错但字段静默丢失

5. 用户要求回退"本会话改动"时 AI 误用 git stash，险些造成历史工作丢失#

解决方案: git stash pop 恢复后，手动识别本会话三处具体改动并精确撤销

关键洞察: git stash 影响所有未提交改动，不适合"只撤销当前会话改动"的场景；应先 git diff 理清改动范围再逐一撤销

今日收获#

核心收获#

  • Linux X11 WM 对透明无边框窗口 resize 请求无原子性保证，不同 WM 用不同语义解释同一坐标，导致系统性偏移；彻底消除 resize（固定窗口 + GDK input shape）是唯一可靠解法，局部补丁只会在症状间转移
  • Claude CLI -p 模式的 MCP 完整配置需两步：(1) 显式传 –mcp-config 加载项目级工具；(2) 添加 –permission-mode bypassPermissions 绕过非交互模式下无法完成的权限确认；–allowedTools 通配符在此模式下因与 MCP server 初始化存在竞争条件而不可靠，bypassPermissions 是后端服务调用的正确模式
  • GDK input shape 在 X11 层拦截事件，setPointerCapture 不能越过它；拖拽场景需在 Rust 侧用 set_float_ball_dragging 命令临时扩展 input shape 到全窗口；cairo-rs 0.18 + gdk 0.18 下可在 Tauri with_webview 回调中直接使用 input_shape_combine_region，无需新增复杂依赖
  • 调试嵌套进程 CLI 工具（主进程 → Claude CLI → MCP server）时，先记录实际传递的命令行参数再逐一隔离验证参数效果，比直接猜测参数行为更高效
  • ccplan 技能的发散探索（6 方案）+ 对抗审查 + 可行性探测流程能有效将复杂平台架构问题转化为可执行计划，ECL 文档提供跨会话上下文持久化，减少重复分析成本

实践收获#

  • structlog 包装 stdlib logging 时，stdlib extra={} 参数需显式添加 ExtraAdder() processor 才能出现在输出字段中；否则字段静默丢失且无报错
