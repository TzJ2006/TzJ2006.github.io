本月关键成就#

  1. 识别 MIHD 跨切片检索失败根因并找到解决方案 (2026-03-23, MIHD) — 确认根因是 PCA/STAIG 对每片切片独立处理产生不可比嵌入空间，scGPT 因共享坐标系成为跨切片检索最优 encoder（平均 SL@50=0.416，命中率100%），揭示了单切片聚类与跨切片检索需使用不同 encoder 的架构原则。
  2. 发现并修复 extract_teddy() 4 个关键 bug (2026-03-29, MIHD) — 通过系统性源码审查发现：Tokenizer 类名错误、模型 forward 参数名错误（input_ids→gene_ids）、输出 key 错误（last_hidden_state→outputs[‘cell_emb’]）、缺少 gene symbol 到 Ensembl ID 映射。修复后首个切片（151508，4384 spots）提取成功验证。
  3. STAIG 框架对 gene encoder 高度鲁棒性验证 (2026-03-30, MIHD) — Teddy 1024d（ARI=0.495）与 PCA 50d（ARI=0.497）在 STAIG 框架下性能几乎相同，差距仅 0.002，证明 STAIG 的空间图对比学习机制主导聚类性能，gene embedding 质量的影响被大幅稀释。
  4. 论文 Fig 1-6 框架规划完成 (2026-03-31, MIHD) — 参考 MISO（Nat Methods）、STAIG（Nat Comms）、QueST、Yuan（Nat Methods）四篇竞争论文，确立 Fig1=框架图、Fig2=DLPFC benchmark、Fig3=消融、Fig4=HD应用、Fig5=跨切片Query、Fig6=Query生物发现的完整论文结构，写入 ECL YAML 和 plans.md，标志着项目正式进入写作阶段。
  5. TEDDY-G 400M 全量 11 切片 embedding 提取完成 (2026-03-30, MIHD) — 在专用 conda teddy 环境中完成 DLPFC 全部 11 个切片（151508–151676）的 1024d gene embedding 提取，总计 185MB，为多编码器对比 benchmark 提供完整数据基础。
  6. STHD Ground Truth 集成（100% 匹配率） (2026-03-31, MIHD) — 新增 _load_sthd_annotations() 函数，通过 r_big/c_big 坐标实现 17502/17502 spots 100% 匹配；CE+Tumor 生物学合并（Neoplasm + Non-neoplastic Epithelium → Epithelium）得到 3 个有意义 cluster，为 HD 评估提供量化 Ground Truth。
  7. HD 全融合策略评估揭示 naive 融合信息稀释问题 (2026-03-31, MIHD) — 完成 10/13 HD 融合策略实验，发现 concat/mean/element_wise_sum 等简单融合 ARI（0.19-0.20）低于单模态 PCA（0.351），attention 融合接近 PCA（ARI=0.342）；UNI2 vision-only NMI 最高（0.387）但 ARI 低于 PCA，揭示 NMI 和 ARI 在不同数据上可能不一致。
  8. Foundation Model 聚类迁移损失规律总结 (2026-03-30, MIHD) — Teddy gene-only ARI（0.186）不如 PCA（0.255），系统性分析确认根因：Foundation Model 以疾病区分性为优化目标（非方差最大化），高维空间（1024d）使 KMeans 性能下降，且训练数据（dissociated scRNA-seq）与空间转录组分布不同。
  9. Leiden 聚类二分搜索集成（+15% ARI） (2026-03-31, MIHD) — 在 clustering.py 中实现二分搜索匹配 target n_clusters（[0.01, 10] 区间，最多 20 次迭代），实现 Leiden 与 KMeans 的公平对比。DLPFC 上 Leiden 比 KMeans 提升约 15% ARI，但对 STAIG 提升有限。
  10. MIHD 项目历史问题首次结构化文档化 (2026-03-23, MIHD) — 通过整合 plans.md、RESULTS.md 和诊断报告，生成 docs/PROBLEM_SUMMARY.md，首次集中记录最佳 ARI 0.546、跨切片检索失败根因、已修复 bug 清单和 4 阶段消融实验计划，为后续研究提供结构化入口。

反复出现的问题#

1. 特定切片（151676）系统性失败 (出现 3 次)#

日期: 2026-03-23, 2026-03-30, 2026-03-31 根本原因: 151676 切片存在数据特异性问题：embedding 零方差导致 mclust 失败并回退 KMeans（拖低全局均值），且在 STAIG 框架下 model collapse 无法恢复。根本原因尚未查明，可能与该切片的基因表达分布或空间结构异常有关。 状态: 🔄 持续中

2. Pipeline 硬编码配置与切片特异性参数不一致 (出现 2 次)#

日期: 2026-03-30, 2026-03-31 根本原因: Pipeline 全局配置（如 top_genes=3000、数据集默认值、图像路径结构）与切片特异性配置（151676 使用 5000 基因）或数据集特异性结构（HD vs DLPFC 图像路径）存在隐式分叉，导致缓存检查误报（sys.exit）和视觉提取失败。 状态: 🔧 有变通方案

3. HPC 环境配置障碍（conda 激活、配置文件缺失） (出现 2 次)#

日期: 2026-03-23, 2026-03-31 根本原因: 新设备上配置文件不自动迁移（sync.json 需手动重建），且 SLURM 计算节点的 conda 初始化机制与 login node 不同（需 eval $(conda shell.bash hook) 而非 source activate），两类问题均源于环境迁移经验的缺失。 状态: ✅ 已解决

4. 跨会话上下文丢失导致重复提供背景信息 (出现 2 次)#

日期: 2026-03-23, 2026-03-29 根本原因: Claude 的会话隔离机制导致关键研究进展（如 Teddy 测试结果）未被持久化到 memory 系统，下次会话需用户重新提供 arXiv 链接或背景信息。 状态: 🔄 持续中

本月收获精选#

架构 (architecture)#

  • STAIG 框架对 gene encoder 具有高度鲁棒性：Teddy 1024d 与 PCA 50d 在 STAIG 下 ARI 差距仅 0.002，说明空间图对比学习机制主导聚类性能，gene embedding 质量的影响被大幅稀释。因此优化 MIHD 聚类性能的瓶颈不在 gene encoder 的选择上。 (来源: 2026-03-30)
  • 单切片聚类最优 encoder（STAIG/PCA）与跨切片检索最优 encoder（scGPT）不同：embedding 空间的可比性（shared coordinate system）比局部质量更关键。Foundation Model 因输出固定表示空间天然适合跨切片检索，而 PCA 独立处理导致坐标系不可比。 (来源: 2026-03-23)
  • Foundation Model 在跨域下游任务上存在系统性迁移损失：TEDDY 在疾病分类上 SOTA（ACC=0.72），但在空间转录组聚类上不如 PCA（ARI 0.186 vs 0.255）。根本原因是优化目标不匹配（区分性表示 vs 方差最大化），且高维空间（1024d）不利于 KMeans。 (来源: 2026-03-30)

领域知识 (domain)#

  • HVG1500（高变基因直接拼接，无降维）在单切片上 ARI=0.330，优于所有 Foundation Model；Leiden clustering 相比 KMeans 在 DLPFC 上提升约 15% ARI，但对 STAIG 提升有限——STAIG 优势来自空间图结构，与聚类算法相对独立。 (来源: 2026-03-30, 2026-03-31)
  • NMI 和 ARI 在 HD 数据上出现不一致：UNI2 vision-only NMI 最高（0.387）但 ARI 低于 PCA（0.344 vs 0.351），HD 上 naive 融合（concat/mean）ARI（0.19-0.20）低于单模态 PCA（0.351），表明简单融合存在信息稀释问题。两个指标需共同报告。 (来源: 2026-03-31)

工具 (tools)#

  • HuggingFace 生态的单细胞基础模型 forward 接口差异显著，不能假设标准接口：TEDDY 使用 gene_ids（非 input_ids）、GeneTokenizer（非 Tokenizer）、outputs[‘cell_emb’]（非 last_hidden_state），且 vocab 使用 Ensembl ID（43808 tokens）而非 gene symbol。必须阅读源码确认。 (来源: 2026-03-29)
  • DCC HPC 集群实用信息：SLURM 计算节点需 eval $(conda shell.bash hook) && conda activate 初始化 conda（source activate 无效）；scavenger-h200 提供 H200×8 抢占式节点；账户 TRES 上限 cpu=100/mem=500G；Glob 在大型 HPC 文件系统上易超时，需使用精确路径搜索。 (来源: 2026-03-23, 2026-03-31)

调试 (debugging)#

  • 切片特异性配置（如 staig num_gene）与 pipeline 全局默认值之间的隐式分叉是高风险 bug 类型：缓存检查逻辑应动态解析实际配置而非硬编码默认值；多数据集 pipeline 的图像路径应显式区分而非用统一的 data_root/section_id 逻辑。 (来源: 2026-03-30, 2026-03-31)
