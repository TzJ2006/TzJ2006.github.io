学习收获#

领域知识 (domain)#

  • 空间转录组跨切片任务中，embedding 空间的可比性（shared coordinate system）比 embedding 局部质量更关键。Foundation Model（scGPT/TEDDY）因输出固定表示空间，天然适合跨切片检索；PCA/STAIG 独立处理导致坐标系不可比，不适用于跨切片任务。
  • 单切片聚类最优方法（STAIG）与跨切片检索最优方法（scGPT）是不同的，需针对不同任务目标选择不同 encoder，不应追求统一最优解。
  • TEDDY-G（rank-value encoding）优于 TEDDY-X（binned expression）；linear probe 远优于 fine-tuning（RA: 0.56→0.93），说明 embedding 质量高但 fine-tuning 易过拟合；在 held-out donor 多分类任务上比 Nicheformer 高8%。

工具 (tools)#

  • HuggingFace 生态的单细胞基础模型（TEDDY、scGPT 等）forward 接口差异显著，不能假设标准 HuggingFace 接口。必须阅读源码确认 gene_ids 参数名和输出 dict 结构，TEDDY 的 vocab 使用 Ensembl ID（43808 tokens）而非 gene symbol。

架构 (architecture)#

  • TEDDY 使用 rank-value encoding 保留 top 2048 个基因作为 token 序列，序列长度比 scGPT 等模型更长，显存需求更高，推理时需要更小的 batch_size。

AI 使用备注#

有效模式:

  • ✓ 系统性代码审查（读源码→验证API→对比实现），主动发现用户未知的隐藏 bug，节省调试时间
  • ✓ 在需求模糊时选择确认而非猜测（私人文件同步需求），避免错误执行
  • ✓ 将口头分析结果持久化为结构化文档（PROBLEM_SUMMARY.md），在用户主动推动下完成知识外化

局限性:

  • ✗ 跨会话记忆未持久化：无法回忆上次 TEDDY 测试的讨论，需用户重新提供背景信息
  • ✗ PDF 解析失败：直接获取 arXiv PDF 时无法处理二进制内容，需回退到 HTML 版本
  • ✗ 依赖已有文档进行项目总结，无法推断文档未记录的隐性进度或自主判断消融实验优先级
